```markdown
# 监控 Docker 容器（cpu， 内存等）产生的数据放哪里呢的？
# 产生的数据就放在 influxDB
# 放在 influxDB 的数据就使用 Grafana 来进行展示
# 当 Docker 容器的资源达到我们设置的预警规则的时候，就通知
# 也就是说 Grafana 负责读取 infulxDB 数据库的数据


#一句话就是： 
# cadvisor 负责往 influxDB 写数据， Grafana 负责往外读数据，并做预警



# 预警： 当Garafana 达到了一个某个值的时候，他会自动的出发一个预警
 然后就去触发 Rancher里面的一个 WebHook地址（钩子地址），这样就可以实现
 Docker 内容器的自动扩容或者缩容（也就是常说的弹性扩容缩容）

#---------------------------------------------------------
docker --> 容器 （开发的运用，中间件，数据库等）--> 操作容器的 Rancher
--> 搜集性能数据的cadvisor --> 性能数据存储 influxDB --> 监控预警
 Garafana --> Rancher 收到监控预警平台的调用 WebHook 钩子地址--->
Rancher 自动实现容器的扩容缩容 --> 资源的合理分配 
# -------------------------------------------------------


# ------------------------------------------
Jenkins 通过调用 DockerMavne 插件 --> 实现持续的部署Resitry私服
--> 快速的迭代 --> 生成Docker的运用镜像--> 并被部署生成容器 
--> Rancher操作运用容器 --> 根据服务的请求情况，做到负载均衡 
# ------------------------------------------------


这些内容息息相关
```
