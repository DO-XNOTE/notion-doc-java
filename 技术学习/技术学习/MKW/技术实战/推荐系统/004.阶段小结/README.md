# 004.阶段小结

那最后我们来总结一下我们对应的整个的这个项目设计的一个课程，在课程当中我们了解了整个的项目的一个推进过程，是需要首先有一个较为清晰的业务需求。根据这个业务需求我们可以将其翻译为对应的用户故事，并且最终落地为研发可以执行的闭环的一个产品 PRD 的方案。那我们最终研发可以通过这个产品 PRD 对应的一些业务实体的实体以及行为去定义我们对应的业务模块的设计，来完成我们对应的模型与模型之间关系的一个设计。最终我们需要结合整个的一个技术选型产出系统模块的设计的一个过程。那在这节课程当中也教会了大家我们在做技术选型当中既要考虑点，也要考虑面。我们既要考虑每个不同的模块所需要选用哪些技术，也要通盘的去考虑我们对应的一个设计的过程，比如说在做推荐的一个数据存放的过程当中，我们采用了 MySQL 做对应的离线计算后的推荐数据的一个存放。那这个 MySQL 我们采用了跟业务数据库共库的方法节绳接入成本，那这种情况也就是我们需要从一个细节点上抽出来通盘的考虑我们整体的一个技术选型的过程。
