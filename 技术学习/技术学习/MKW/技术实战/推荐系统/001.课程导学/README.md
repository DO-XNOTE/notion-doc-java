同学们大家好，我是龙虾三少，欢迎来到我的课程。本次课程的主题是 Elastic search 加Spark，构建高匹配度的搜索服务与千人千面的推荐系统。在如今的时代当中，无论是线下的 o to o 的程序，比如说美团点评，还是线上的电商，比如说淘宝和京东商品服务，以及门店的数据呈指数级别的一个增长，内容庞大且特色各异。那对于一个合格的面向 c 端的系统来说，搜索与推荐的能力很显然是必不可少的。那如何快速的根据用户输入的关键词做一个搜索满意度结果的一个输出，并且可以结合用户的一些历史行为、兴趣偏好等推荐用户想要的一个服务及商品，以提升浏览点击率和交易转化率，那是很多公司至始至终做的一件事情。


对于一个合格的搜索服务来说，它必须要具备哪些能力呢？首先它必须得要有一个完整且丰富的爱搜索的数据源。那这些数据源质量的好坏从某种意义上来说取决了搜索服务的一个能力的好坏，有了待搜索的数据源之后，我们还需要有一个高效且稳定的搜索引擎来帮助用户搜索对应的一个内容。


那最后搜索引擎虽然说是一个存储加关键词级别的一个引擎，但是我们更需要的是一个相关性语义理解的智能化的一个引擎。

也就是说能够理解识别用户的语义，以满足高匹配度、高相关性这样的一个搜索的智能理解。那对于一个推荐系统来说，它又需要具备哪些能力呢？首先我们的推荐系统是需要有一个完整且丰富的训练及测试数据集，这些训练及测试数据集的好坏就代表了推荐系统所产生的机器学习的一个模型的推荐的能力。数据集越好，质量越高、内容越丰富，就越可能训练出非常贴合实际情况的一个机器学习的推荐模型。
