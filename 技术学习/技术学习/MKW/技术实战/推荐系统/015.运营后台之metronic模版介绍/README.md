# 015.运营后台之metronic模版介绍

好，同学们，那我们继续进入下面的课程，之前的课程我跟大家讲解了用户服务的一些接入方式，以及将对应的前端的登录注册页面做了一次完美的接入。那接下来我们要转入点评、搜索推荐的运营后台的一个管理跟创建上面，那我们首先来看一下我们要利用运营后台实现哪些功能？首先会给大家介绍我们搭建运营后台所需要用到的一些模板，因为我们对应的一个运营后台不像是一个前端的页面，是需要很强的关注一些交互的设计以及用户的体验。


那往往在企业级的应用当中，对于运营后台的一些 UI 其实并没有一些很高的要求，甚至于许多企业对运营后台的一些研发团队来说，都没有 UI 交互，设计师完全就是靠在网上或者自己在历史经验过程当中得到总结的一些运营后台的模板来完成对应的一个交互的设计以及代码的实现，因此我们做运营后台的同学往往会采用一些比较好看的，或者说操作上面比较符合企业特点的一些后台模板去完成后台 UI 界面的一个搭建。因此我将在这边向大家介绍一款运营后台模板。然后我们需要去实现运营后台的一个登录授权，也就是说我们对应的一个运营后台从某种意义上来说掌控了整个产品的一个业务系统的中枢的后台。因此它必须要有管理员的账号权限去做对应的一些登录，并且需要去控制对应的一些菜单的访问权限，因此需要做登录及授权的功能。那最后会教大家使用切面去做统一的一个接入权限的管理。


好，那我们首先来跟大家介绍一下运营后台的模板。那在这边我已经为大家准备了课程的一个辅助资料。我将给大家介绍 Metronic version 3.0 对应的一个模板，那这款模板在网上去购买的话，当时是花了将近 100 元左右的人民币去买到的。那在这边首先可以在这个模板上面看到有这个 documentation 文档的一些界面，会跟我们介绍对应的一些文档的一个内容。那我们点开可以看到这边都是全英文的一些模板对应的一个介绍，那其实我们暂时可以先不关注这个东西，我们要看的是这边的一个metronic，直接可以在这边看到它的一些效果，那 assets 对应的就是我们所要掌握的一些使用到它内部的一些资源，它内部会有包含CSS、image、plugin、 script 这种样子已经封装好的一些组件可以供我们直接拿来使用。


那更多的我们可以直接通过这个templates，那我们点开这边，我们可以看到有两个选项，一个是admin，一个是 front head，那我们可以点开admin，找到对应的 index 点html，我们可以到根节点上去看，这个就是 Metronic 提供给我们后台管理模板的标配。


那我们可以看到一个后台管理的页面往往是分为这么几个元素，首先是一个 navigation bug，这个 navigation bug 除了它自己的一个角标之外，还会有对应的一些 message books 相关的一些提醒，还有当前登录的这个用户的一个头像，然后会有对应的一个footer，这个 footer 的话会标注对应的一个企业的一些信息。
最关键的就在于这边的一个边栏，也就是一个 Tab bar，我们在这边的话 Metronic 的一个示例的模板 Tab bug 的组成形式主要是为我们介绍它其中内部的一些组件，比如说 dashboard 会有这些栏目的一些组件 UI 的feature，我们随便打开看一个 button and icon 会给我们列出它内部所有的一些 button 的样式，以及我们如何去配置这些button，那我们怎么去看对应的这个效果呢？其实也比较简单，我们只需要将对应的这个东西直接在我们的某一些编辑器当中去打开，就可以看到对应的一个 h 天猫的效果，以及我们如何去运用这些东西。


说白了就是我们可以通过这个 h 天猫文件直接将对应的内容即 CSS GS copy 到工程内，实现我们想要的效果。那同学们如果说有兴趣的话，可以直接下载课程当中所选用的 Metronic 的一些资料，去研究它里边的一些组件。其实比如说像这边的Triviu，我们可以看到去实现了 tree 的这样的一个功能。


那其实若是让我们后端研发人员直接去写这些组件，那其实就已经是会花费大量的精力，那更何况我们做的仅仅只不过主谓语音提供一个操作快捷简便的一个操作方式，那在这边我就带大家一起将这个 mechanic 模板做了一下处理，我们可以看到在我们的课程辅助资料里有一个处理前和处理后的，那处理后的模板，我将对应的一些没有用的，比如说这边的一些RTL，或者是这种resources，包括 documentation 内部的一些文档全部都删除，组成了一般在工程当中所能够用到的一些样式。比如说 CSS 内部包含了对应的 mechonic 的所有的 layout modules 对应的一些样式。那节约词也包含了 metronic 相关的，比如说这边的一些 plugin common 里边包含了所有的 metronic 相关的用到的一些解词。那 image 就是我们所对应的一些图片， phones 对应的一些字体，然后这边的 assets 就对应了我们 mature 那个内部所实现的这些 layout 跟 pages 相关的一些组件，那我们可以直接将对应的这些文件 copy 到我们的工程目录下，将我们对应的工程目录打开我们对应的这个工程目录的 may resources 目录下之前有个static，那我已经将这些包全部都 copy 到了我们的 static 目录下，并且跟我们 static 目录下原本的一些 CSS 跟 JS 已经做了一个融合，那同学们也可以这么操作一下。
