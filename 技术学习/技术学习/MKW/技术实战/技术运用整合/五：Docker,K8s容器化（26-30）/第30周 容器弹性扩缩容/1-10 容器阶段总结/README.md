大家回过头去想一想是不是还是集装箱理论。而在这个集装箱理论里面，还有一个关键的名词叫做你要练武功，除了要有一把好剑以外，不管你是 wrong C 还是 Docker 这些容器都是一把好剑。除了好件以外，一定要有一套好的功法，一套好的门派。那到底三大平台是什么样的门派？然后孰优孰劣如何选择呢？我们如果说以 cloud foundry 为例，它是一个整体的一体化的这个武功平台的。不管你是这个我们叫它 pass 还叫它 CAS pass 就是平台级服务 CS 容器级服务还是一套整体化的武功平台，你入了此门你就练会了什么？基本上所有的功法，这是一套很独立的，比如像少林寺一样，就是边念经边练功的这样一套平台。


梅瑟斯马拉松是什么样？套平台是一套集武功各种门派的能力为一体的一个集达成者。有点像什么华山的气派，华山气派强调什么，要掌握天下武功的各种精华，然后反复的什么把它锤炼一切的功夫全部以一套华盛的气为总纲。那 masses 就是这套器的总纲，它以所有的这个资源整合为总体。


然后在上面可以运行华山派的减法，可以运行松山派的减法，有一个气为整体的运行各种各样的键盘，把各种武功集打成者，那就是 mess 马拉松。那同样还有一个派系，就是什么华山派的一个比较另类的分支，不强调什么功力很深，但强调什么剑法很独到。不管你是长势还是刀士还是剑士还是弓箭士，他都有专门的方法来克你，也就是你应用所需要的，你的套路是什么，你的难点是什么，我都可以有一招来制胜。我不需要练很多很多很复杂的气功，也不需要像少林那种武功一样一体化的对吧。你要练功术还要念经，念经还要练功，它是什么？完全应用为导向。


那如果大家不喜欢武侠，喜欢游戏的话，我们换个方法来说，cloud foundry 就像手游一击在手是吧，万事我有那种感觉，就是非常方便灵活小型化 car fungi 你不需要管理的 eureka 服务了，eureka服务摆脱出去，让外面的什么服务提供方来进行提供，让 cloud 方式的供应方来进行供应，你完全专注你的应用的小模块，然后你路由也不用管，服务也不用管全，只要绑定上去就可以了，这是一种手游的感觉。


那 mess 马拉松很像主机游戏，也就是我们有什么 PS four 有 switchxbox 有几大分支。在 mess 统一的平台上面，我们可以跑马拉松常应用，也可以跑 Spark 大数据作业，甚至于可以跑一些这种特殊的 job 这种短应用的形式，但它统一起来，要用 mess 这个资源总池来进行管理。


所有的资源我们统称叫主机游戏对吧，其实又分好几大分支，那这就是有点像 mess 跟马拉松一样，它的特点是什么？是你要玩得非常纯粹，玩得非常高级，你可以玩得很溜。对吧？你的游戏可以玩得非常之爽，那爽跟手游跟网游相比，主机游戏它的画面它的质量可以它更精确。
