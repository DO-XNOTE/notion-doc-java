那除此以外，还有一个很精彩的这叫什么叫**集群底层扩缩容**，大家看右边的图也能感觉出来这是什么？一个 node 就是一个物理节点，一个物理节点里面包含四个容器，但它不是一个物理节点，它是根据压力不同，它把物理节点也在不停的扩张。这个通常都是在云平台提供，当你在自己的机房里，你还是比较难什么，很快的从外面立马去买一个物理机过来摆在这里面。我们这个可没法做到自动化，但是在云平台很方便。一个语音平台，通常一个机房里面有几十万台机器，对不对？有上千个这个机柜，这么多的机器怎么处理呢？很简单，它是租给不同的用户的。所以你租的时候可能比如说只租了这里两台服务器，但是当你压力很大的时候，我可以要求什么？我要租 20 台服务器，只为了支撑今天晚上这个抢购或者秒杀。但这用完以后，我又回到两台服务器这个状态。所以这种 auto scale 的方式也是什么，也是语音平台里面最主推的。
