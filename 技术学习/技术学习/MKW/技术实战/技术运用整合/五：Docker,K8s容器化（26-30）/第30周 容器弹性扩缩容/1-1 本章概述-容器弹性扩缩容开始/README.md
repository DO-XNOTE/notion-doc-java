第一个点就是先了解一下我们对抢购，我们对秒杀这些业务来说，什么样的需求是最迫切的，什么样的技术是最核心的？


而这其中哪些技术是可以通过容器技术，通过一些容器的高级的功能的延伸来实现。那然后我们会重点看一下，就是从企业架构来看，不光是应用架构，从整个企业架构来看就是包含我们应用，包含我们数据，包含我们技术整体的衡量，整个电商系统怎么样的一些改造，处理方法，调整以后容易能更好的支撑抢购秒杀这样的业务。那最后就是实实在在的一个实际的扩缩容的展现了，我们会结合几个不同的技术点来看看怎么样从资源的使用情况，怎么样从业务的压力的情况以及从业务表现等情况进行监测进行决策。最后的实现我们容器、我们负载、我们路由，我们网络的扩缩容，从而真正的实现很弹性来支撑抢购秒杀系统。那整个过程当中，我们会尝试有一个闭环，就是从监控到编排到最后什么，我们的容器实现路由实现了一个编排。

那之后我们会从三个点，就刚刚说的 cloud foundry 主要在于什么？资源根据这个资源的实际使用情况，怎么样进行合理的决策，进行弹性的扩缩容？那在 mess 码的时候，我们更强调于负载，根据比如说我们一个 API call 进来以后，它的这个每秒钟的这个 query 的数量或者每秒钟触发的这个数量，怎么样合理的调整后台的容器的资源以及它的网络情况，实现动态的扩缩容。
