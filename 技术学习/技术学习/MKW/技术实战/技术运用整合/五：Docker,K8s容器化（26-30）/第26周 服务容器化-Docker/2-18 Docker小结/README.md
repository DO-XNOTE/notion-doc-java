作业
好记性不如烂笔头，让我们运用学到的技术来处理一个实际的架构师问题吧。
问题：微服务之间的异步通信过程中，如何实现消息的稳定传输是应用架构师和系统架构师关心的重中之重。请挑选一个消息队列产品，并将其容器化，同时确保以下几点非功能性需求得到满足：
1.高可用：服务器故障不会导致消息队列可用性降低
2.业务连续性：如果数据中心断电恢复后，原始消息队列未消费数据仍然可重发，而不会丢失
3.敏捷发布：消息中间件版本升级后，可以分钟级并行发布到所有节点
提示：
1. 可优先考虑站在巨人的肩膀上，从公共仓库选择现有的消息队列产品
2. 可配置数据持久化层，防止数据丢失.可配置多节点网络通信，实现消息中间件集群的高可用
4. 可配置私有仓库，实现快速软件发布
