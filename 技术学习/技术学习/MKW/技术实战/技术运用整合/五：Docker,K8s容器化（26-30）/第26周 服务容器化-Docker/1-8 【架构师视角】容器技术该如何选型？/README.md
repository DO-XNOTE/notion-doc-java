那 Doc 整个这个环境当中可谓是一个第一号生态圈的支持者。以前有句话叫做文武第一，武武第二。其实我们的这个容器化战斗就相当于是一个武斗大平台，第一出现了之后，第二第三很可能就会逐步的没落衰退。所以在整个生态圈里，我们要 PK 就 PK 谁是第一大生菜圈主导者？那这个非 Doc 莫属。


Doc 具有最多的编排支持者，最多的 pass 平台者，也具有最多的兼容性支持者。甚至于我们拿 cloud 方瑞生态圈为例，很多 cloud 方瑞生态圈的主流产品都会提一句，我也兼容 Doc 我也支持 Doc 为什么会这样看呢？是因为当前环境来看都是有这样的共同的认识，就是生态圈第一者是 Doc 我们必须向 Doc 靠拢，所以这是 Doc 最大的一个优势。


那除此以外，环境准备优势也是一个很明显的特点。 Doc 不像其他的这种容器化技术，它的环境准备非常方便。你只要下载了 Doc 的这种引擎下载 Doc 的客户端，你就很方便的可以从网上直接以一些基础镜像，比如像 hello world 镜像、 Redis 镜像为例，很快的下载一两个镜像，跑起来运行起来感受起来。所以 Doc 的生态圈以及它 Doc 的环境准备是它的优势。


那同时我们有大量的镜像胚胎，我们有公共的仓库，我们也有一些著名的第三方的这个镜像加速，比如像阿里的镜像加速云。那这种镜像加速帮助我们在国内很快的下载到最新的国外的新发布的镜像。同时我们也有很多的国内的云平台，它也搭建自己的镜像产品，发布了大量不同的中间件微服务基础的依赖的模块的镜像产品。通过这些镜像可能很方便的以容器的方式运行起来，只要做少量的改动就可以支撑我们的业务平台，所以这些都是它的一些优势。


除此以外，业界的技术支持也是一个很大的点。那 Doc 有支持很多的不同技术，也有很多不同技术已经落在 Doc 的这样容器圈上。所以你很容易的对需要的功能模块，比如像中间件模块、网关模块，比如像是我们的数据库模块，来找到相关的 Doc 的这个镜像来支撑。那它有一些劣势，比如像业务 DIY 就是一个劣势，比如你这个业务需要有一定的密钥管理系统，需要一定的用户认证对接。


对于这一块，我们 Doc 语言生态的不支持，它更相当于一个独立的小方块。这方块里你自己开发自己业务，自己去完善它的一些，比如像用户认证，比如像是密钥管理，比如像数据加密这样一些业务模块要靠自身，所以它的这种天生的业务对接是不具备的。除此以外，有一个比较大的劣势就是它的网络和存储。它的解决方案非常非常多，每个解决方案也有它的优势和劣势，所以你很难挑选一个很合适解决方案。一览，完成你所有容器的部署。在很多过程当中，你要用某些容器采用这个方案，另外一些容器采用另外一个方案，你还要考虑它们之间的兼容性。就相对来说，网络和存储的部署是 Doc 略有复杂的地方。通常我们现在当前就通过一些编排技术或者通过一些平台技术来解决这些问题。通过这些统一化的编排和统一化的平台，逐步适应我们各种不同的容器的需求。除此以外，对于非功能性， Doc 是相对比较弱的，比如安全性， Doc 里面并没有强调我们所有容器都做一定的这个安全增强，也没有强调我们 Doc 对外提供的所有服务都要一定的安全加密。除了这个安全性以外，我们的高可用，我们的并发性、我们的性能这些很多的点 Doc 技术都没有做特殊的处理，所以它是比较一个开放的平台，让我们来自发的实现我们的 eo 自发实现这样的一些非工人新需求。那从未来角度考虑，有威胁和机会这两个层次。那机会我们很可能会看到，就是未来 5 到 10 年 Doc 江山一统，所有的这个容器生产权全部以 Doc 为核心进行扩展。


也有可能我们会看到更多的 pass 平台以 Doc 为基础，因为毕竟 Doc 是一个开源的广泛支持的一个平台，我们有更多的这种大的企业，公有云的企业会加入到我们的 Doc 成为我们的黄金会员、铂金会员来支撑我们 Doc 平台的开发和发展，那也有可能会出现一些危险。


作为一个架构，是我们通常都要以客观的角度来讲，我们的 it 的风口是一直在变化。去看我们的这个 got net 我们的业务眼镜图，我们任何技术都是有波峰和波谷的。如果过了 5 到 10 年的波峰， Doc 进入了波谷的时候，会不会有一个新的公有的 PaaS 平台来代替它或者有些新的技术来取代它，这是有可能的。所以这是我们横向来看 Doc 对应的我们克尔方群也会做一个优势劣势机会和威胁的比较。那它的优势就是它天生支撑多语言，像 notijspythongoal Java 里面还包含很多，比如像我们 spring boot 解 boss 的支持，那也可以支持我们 spring boot 里面某些分支。比如我们要支持这种交互式 React native 的 erective 的那种比如像 web flux 这样一些基本的功能的支持。


除此以外，我们还支持网络，我们还支持安全天生，它就有用户认证天生它就具有数据传输，安全天生它就可以支撑我们的这个负载均衡、蓝绿发布等等功能，同时也有高可用、可扩展、可弹性伸缩这样一些基本功能，在 callfoundry 的这个基础的框架里，天生得到了考虑。
