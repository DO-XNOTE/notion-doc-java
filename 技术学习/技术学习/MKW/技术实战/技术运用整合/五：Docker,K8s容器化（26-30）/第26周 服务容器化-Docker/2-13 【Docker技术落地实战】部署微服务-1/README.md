好，那我们这一节先来看一下就是微服务改造里的平台改造部分是怎么样来实现的。切回我们的一个云平台，我这里用了一个阿里的云，因为整个服务要运行起来，在我们的笔记本上通常是 8g 内存是不够的。所以我这里申请了一台机器，它有一个公网地址，叫 101 开头的这样公网地址，还有一个什么 172 这样的一个内网地址。那它的整个资源是 4 核，就是 4 克 cpu16g 内存，基本上能勉强刨清过前面的微服改造完以后的这样一堆的这个基础架构，还有一大量改造完的这个具体的微服务应用。那这样一个运行环境，如果大家是在腾讯云、百度云或者其他的有比较强的服务器，只要超过 16g 内存基本上都能勉强运行。那如果能到 24g 内存或者更高会更加好。


那我们登录到这个节点上去看一下，我们实际情况怎么样啊？那这是一台这个我们的实际的服务器。那在这个服务器里面，我们建了一个目录叫福迪 cloud 然后把我们之前什么所有就是我们微服改造完以后的这些软件都下载下来了。那在下载下来以后，其实我们也可以什么下载到本地，然后对代码进行一个简单的调整，再推送上去也是可以的，有好几种具体的部署方法。那我们先讲一下哪些地方是需要改变的，然后再逐步的进行部署。
