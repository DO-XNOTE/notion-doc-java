首先他会验你的资质，看看你这个人是属于阳刚型的还是阴柔型。如果是个男生，他又很阴柔，他有可能就分配你一把辟邪剑，你用这个键谱去列可能很适合你。那同样道理，我们的这个容器，如果它里面跑的应用是一种 CPU 负载型的，它就会分配你一个强 CPU 的操作系统。下面这个物理的服务器或者是虚拟机，如果你是个 IO 型的，它就会分配你一个比如是全部用闪存盘或者用 SSD 盘搭起来的这样一个服务器。



在这上面，跑你的容器，根据你的合适的兼容性来选择合适的摆放位置，同时根据你的资源的需求来进行资源的分配，这就是容器编排的第一步验资质。那第二步就是你资质有了以后，就会给你这个键谱对吧，辟邪荐谱，你就要开始练起来了。它这里面其实一共没几招是吧，你要练得滚瓜烂熟才行，每一招都要有反复的变化，都要很熟练。那怎么样算熟练呢？就是你能够很快的把这个容器按要求部署，就是你要起多少个实例，你要占多少 CPU 内存，根据你验的资质，你要按要求部署数据。同时如果出错，你能很快的回滚。那这就是学剑招，你把这个剑招列得很熟很熟。甚至于这个剑招怎么样稍有点小变化。比如像蓝绿发布、金丝雀发布，这些小变化你都练熟了是吧。就是学建招，建招完了以后，你需要串成一个套路，光这么一招出去，人家熟了以后就知道你就只会这一招是没用的。就算你会右边这九招其实也未必有用。独孤九剑不是说一剑都能搞定天下对应对方不同的这个剑招，你要拆招变招，你要组合起来有。所以练套路就是怎么样通过服务发现应用依赖等方式把你的容器串联起来。



那我们其实在我们的容器编排工具里，天生就具有很强的串联性，它可以把很多依赖直接加进去，这样你就可以通过容器编排来进行很强的什么小的这种建造的组合，形成一套完整的大建造。这时候再通过你的 API 网关等等方式组合成我们的业务系统。好建招学完以后，我们进入下一个环节，就是物变通，有了一个强大的建招，其实真正在实战过程当中，别人见招也会千变万化，所以你也要千变万化。那比较经典的例子，

就是我们时时刻要检查我们的容器的状态，如果发现容器所在的物理机或者虚拟机坏了，那个时候你要让容器合适的飘起来。
那如果发现我们的容器的这个 CPU 内存资源狂涨的时候，比如说我们外面压力很大，开始有些促销活动的时候，得让我们的容器进行很弹性的渗透，很快的涨资源。当我们的容器很闲的时候，我们尽量把资源释放出去，让其他的应用能够来抢占这些CPU 、内存、网络等等资源，这些是物变通。那除此以外，因为我们是容器是支持一个小业务的。而这个小业务有可能是一个微服务，有可能是一个比微服务更小的。我们俗称的什么叫 aggregate 它是个小聚合，那它上面会形成大的幅，然后就形成房地的 context 领域。


在领域上面才是一个大的 domain 它有很多层次。那这些层次我们在容器里面是以这样的形式来部署的。就是我们先把我们很多的小的这个容器编排成一个应用服务。然后我们可以把这些应用服务向上以软件包的形式进行统一打包。我们特别是在 kubernetes 章节，我们会有一个功能叫 help 来进行软件统一打包。等大家熟悉了这个 helm 打包以后，会发现好多好多应用软件的部署，其实就是一键安装整个应用软件，它的后台的数据库到中间件，它的前台的应用，甚至于它的图形化界面一键安装，所以是软件包的部署。
那在这种软件包上面，再形成我们强大的业务集群，这就是道就是业务对吧法就是软件包数据，是那些小应用服务最后落地到我们的实际的工具，就是那把剑好到最后一个阶段，就是最关键的一个阶段，真正一套好的剑法，或者说是一个好的这种什么一个剑派，他不会只能教你一个，他可以教很多很多人。那一样道理。
