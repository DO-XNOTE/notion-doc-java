面我们介绍了什么，那个 kubernetes 容器怎么样来实现什么全链路的高可用。那从这个章节开始，我们重点先介绍一下容器编排怎么和我们的 CICD 工具相结合的。


不知道是不是对 CICD 这个概念非常熟悉呢？那 CICD 其实什么？是一个偏这个部署运营和开发测试流程的一套概念，它通常和 DevOps 这样一个概念相结合。就是我们通常我们开发人员完成了所有代码的设计开发测试过程。而运维团队完成什么代码的部署，上线监控和回馈出现问题以后的一个回环。这样两个回环逐步在我们的这个自动化的环境当中，在我们上云的过程当中，在我们微服务改造的过程当中，越来越多的什么融合起来，就让我们的 developer 和我们的运维团队、 OPS 团队成为一个整体。那这种情况底下我们会建议什么？采用一些 CICD 的自动化工具来实现全自动的集成，全自动的部署。那 CI 的全称其实就是什么是 continuous integration 它重点强调几个方面。

那一个方面就是说我们本来的代码如果是采用瀑布模型的话，你会有一个什么会有个规划是吧？


在规划的某一个时间点，我们开始进行代码的开发。代码开发过几周以后或者几个月以后，我们会什么有一个统一的点进行什么代码的融合？很多个 branch 的融合，最后以什么以一个 release 的 branch 或者是一个 master branch 的来统一对外的话提供一个发布。那这样的过程会有几个问题，一个是什么？这个 team 里面如果团队人员比较多的情况下，很可能在最后融合的时候会产生冲突，这个时会延缓我们的发布。


第二个整个过程不是够灵活，不够挨着。因为我们现在什么互联网环境或者说是我们的这种微服务的情况底下要求什么？服务的变更是吧。越来越快，我们的服务的变化越来越少，可能就改一两行代码，就要重新发布一次，来适应某一个新的需求或者是一个需求的更新。那这个时候怎么样能很快的很方便的全自动的或者说连续的在进行持续的这种代码的改变呢？通常我们会建议什么？我们会建议我们的所有的 developer 在自己的 branch 上进行很方便的开发。但是一旦开发完成，经过自己简单的测试，就是在本地的台式机或者笔记本上，经过一个简单的 unit test 以后单元测试以后就自动什么提交代码到我们的什么 release the branch 或者是 master 的 branch 里面来尝试进行融合和发布。


那这种时候它的融合发布感应什么是由什么？我们的这个相当于是代码库，如果你是 github 那就是通过 github 来感应的。那如果你用 GitLab 就用 GitLab 来进行感应，一旦感应好以后，它会自动的触发一些审批流程。比如我们需要我们的这个开发人员的组长或者是我们同级的其他成员进行什么 code review 代码审查。那审查完了以后，会触发一个全自动的什么这个代码的单元测试代码的这个打包编译的过程。打包编译单元测试完以后，我们甚至会在一个开发运行环境进行部署，进行简单的集成测试。那等等这样一系列的流程，就在我们的 continuous integration 里面得到实现。


你当测试完成以后，我们认为什么这个构建包其实是相对比较完整，我们就会把它打包成那个包，不管你是架包还是踏包还是袜包，反正是一个这种压缩包是吧或者是一堆的这个文件，把这一堆文件以一种特殊的形式上传到我们的一个仓库里面。那在我们的传统的环影当中，我们有哪些仓库呢？我们会有一些比如像接 frog artifactory 这种就是专门做什么做代码仓库管理的。那我们也可能类似于用 nexus 这种就是都是一种收费或者开源的一些代码仓库管理方式。那后面我会提到在我们容器过程当中，代码仓库会略有不同，但也是一样的，也是一种仓库，他把这个实际的已经打包完成，可以运行的构建放在这个仓库里面。


那当放完仓库以后，下一步就进入了 continuous delivery 就是从仓库里面拿出这个构建去部署到我们所有的环境。比如你用第一级开发功能性测试环境、第二级什么集成测试 integration 环境、第三级我们准生产环境以及第四级生产环境。


在生产环境之前所有的处理通常我们叫 continuous delivery 对生产环境的部署，我们原来可能只有什么运维团队的人员才有资格，但是现在通过一套 CD 的工具 continues deployment 的工具它会拥有一些什么？ service account 有一些账号，而类似于模仿我们运维人员才有的账号，或者说是系统管理员或者应用管理员或者说是运用运维人员才有的功能，能把权限把我们代码实际的发布到生产，同时实现什么蓝绿发布金丝雀发布，并且做一些什么生产的冒烟测试。这时候同时把我们的健康检查的回环反馈给我们的什么，我们前面的 CI 系统来告知我们说当前这个版本发布是不是完全正常，这个我们的代码是不是需要回滚这样整完整的一套 CICD 的流程。
