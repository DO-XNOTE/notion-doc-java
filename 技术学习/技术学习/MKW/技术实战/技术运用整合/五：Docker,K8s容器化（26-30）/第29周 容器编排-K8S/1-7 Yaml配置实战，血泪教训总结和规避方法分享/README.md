从 K8S 的 gcrdrio 通常就是我们普尼 tis 的配置里面默认的一个下载源。那这种情况其实主要原因就是因为什么我们的网络连谷歌中间出现了一些问题。那常见的解决方法其实有好几种，一种是我们去申请一个比如海外的这种运行或测试环境，可以直接访问谷歌的 DNS 的，很方便的去从谷歌的这个 CCR 的这个 registry 里面去下载它的一些容器，这样你就可以很方便地去部署古尼 tis 所依赖的那些绕克容器了。


那另外一个方法就是我们通过我们国内的一些镜像服务器，通过这个镜像服务器它内置的那些 registry 去逐一的下载需要下载到的这些我们的 image 还有就是如果我们可以临时找一个代理服务去尝试去把整个我们库内的装一下的话，然后把它所有的那些 image export 出来之后，你就可以 import 到我们各自的什么笔记本或者是我们国内的这些服务器上面，然后 import 进去，这样也可以用来搭建我们的整个这个集群。所以有好几种方法。


那通常来说，最主要的点就是我们要把网络打通，或者就是把能打通网络的服务器上的那些镜像提前下载好，然后传输到我们的这个本地的服务器上。当然还有一些通过什么类似于 github 或者类似于一些其他的方法进行额外的一种拖拉方式。那大家可以查网上的一些攻略来实现。那最主要就是要实现能连接到谷歌的这个 registry 镜像，那这之后就可以解决那个 image 破的问题了。
