通常我们回过头来看一下一个应用如果从架构师的角度来看，怎么样对整套应用或者整套系统来进行监控呢？通常需要从三个维度。第一个维度是它的系统的整体性能，包含系统的所有的什么资源的使用，比如像CPU 、内存，也包含系统的里面的的应用特性。比如说我们的一个 API 的响应时间，一个浏览器渲染我们整个界面的这个响应时间，同时也包含我们什么出错率，有多少时间回的是 200 的这个 status code 有多少时间，回的是 400 或者是 300 的跳转或者是 500 的资源不可达等等。那这些不同的这个返回的结果，不同的响应时间，不同的 CPU 内存的这个占用不同的我们什么我们的这个就是资源回收，我们 garbage collection 的这些能力都体现了我们应用和系统架构的整体特性。所以通常我们需要有一个应用的整体性能的监控工具的 metrics 来监控我们的整套应用。
