hello，小伙伴们，大家好，那么从今天开始，我们就正式的去进入这个消息中间件的这个课程的一个第一大块内容，也就是说关于 rap MQ 的实战的学习，当然第二大块就是卡夫卡相关的内容了。那之前在课程最开始的导航老师也说过，我们整个课程就围绕着两个消息证件，一个就是 Rabbit MQ，第二个就是我们的卡夫卡。那 red MQ 我们要做什么呢？就是把它一些基础，包括可靠性投递都全面的去讲解，包括跟 Sponbot 整合，然后最终写了一个我们对应的，想要自己去封装一个 RABMQ 的基础组件，让大家提升一下编码能力，这是核心增的核心。


首先我们看一下课程的这个目录，对于 red MQ 它里边有一些基础的概念，这些基础的概念如果小伙伴们学过的话，你就去复习一下，如果没学过，你就要提前预习，比如说 AMQP 的核心概念，像 x 圣址交换机，然后 Queen 队列以及绑定，包括 routine t 等等。

这一些概念包括维持host，虚拟主机等等，这些你都要自己去学习，如果已经有基础的，那你复习一下就可以了，当然老师会给你发一些相关的资料文档，采用这种纹图的这种学习的方式，让小伙伴快速的去预习去学习。因为这些都是比较基础的，包括你看后面 rap MQ 的一些极速安装跟入门，只需要你在 Linux 7 上安装一个 rap MQ 对应的版本，然后写一个简单的 hello word，能够收发消息就可以了。


那这些内容我们肯定不会说占用小伙伴们非常非常多的这种视频的时间，因为本身我们是一个高端的这种架构课程，我希望小伙伴们能够真正的在视频课程中能够学到一些干货，而不是说一些基础的 hello world，包括 v m q 的核心API，这些老师都会提供一些文图的这个资料，不需要担心以及 f m q 的一些高级特性。


比如说我们的这个 t r 消息，死刑队列，包括像 k fail 模式， return 模式以及这个 o QS 限流等等，这些都是 RAM MQ 的一些高级特性，那我们都是采用纹图的方式，再往下看接下来这个就比较关键了，就是 RAM MQ 的集群架构操作。


那本身这一块呢？老师开始的时候本打算着是想着手把手带着小伙伴们从头到尾的去把 rain MQ 的集群挨着牌搭建一遍，但是后来我觉得这个东西其实也没有太大的必要去利用视频课程去讲，所以说我也把它改成文图课程了，

直接发给小伙伴们，小伙伴们按照老师的这个操作，一步一步操作就可以了，这样的话也是锻炼你的一个集群搭建的一个怎么说？硬性的一个能力，OK，后面如果对于集群搭建有什么疑问，都可以在群里或者是跟老师直接沟通，也算是一个小的作业。


就是对于 red MQ 集群搭建，那我们在这里肯定是对镜像集群搭建，那再往下看这个就比较关键了，就是与 spring boot 怎么去整合实战，
