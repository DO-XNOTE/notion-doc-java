我们来看一下那这个镜像队列。我们来看一下上边是我们的这个整个应用服务，就是说 spring application，当然这个你的 application 不一定是 spring boot 或者其他的都可以。然后我们从下面去看，我们从下面一个红色的这个虚线，这个框，它其实表示的就是我们的镜像队列的一个这个集群组成。然后里边看这里边都是我们的这个应用服务，黄色的就是我们具体的应用服务器，我现在是三个节点，然后其中里边红色的部分就是代表的我们的 f m q broker，然后在里边就是minerking，是不是？也就是说这里边的数据跟第二个服务器的数据，跟第三个服务器的这个队列，它们三者之间数据是要一个同步的过程，就是三者的数据完全相等。


当我们外部发送过了一条消息，那肯定首先落到我们的主服务器上，然后主服务器把数据进行一个copy，同步到另外的两个节点上，这就是一个镜像队列，它能够保障我们数据百分之百的不丢失。为什么呢？因为我主节点挂了的话，我重节点还有两个节点能够提供一个高可靠，能够把数据保存下来，然后在我们的前端看我们的前端由于说我们要做负载均衡，是不是所以说我的这个 application 应用的这个连接可能是连接第一个服务，也可能连接第二个这个服务器，或者是第三个服务器。


所以说我们前端要做一个 h a proxy 的一个什么呢？一个负载均衡，然后为什么用到 keep l 的就是要做多个的 h a proxy？比如说我们这个 h CROC 节点，它如果挂掉了蓝色的这块，那我虚拟的这个 VIP 是同一个，它就可以漂移到另外一个服务节点继续的去提供一个服务。那也就是说稍后我们会讲什么呢？我们会讲一个这个完整的这种镜像集群的一个构建。OK，那么这节课主要就是来跟咱们同学讲一讲我们的集群镜像队列模式。好，谢谢咱们同学收看。
