那么这节课我们来说一说什么是 a m q p 高级消息队列协议。首先我们来看一下这个单词，那这个就是说 advice message cleaning protocol，那它的这个全称翻译过来就是说高级消息队列协议。OK，那这个概念我们来聊一聊。首先它是具有一个现代特性的二进制协议，也就是说它其实是一个基于二进制去做的这么一个协议。然后是一个提供了统一消息服务应用层标准的高级消息队列协议。那通过这块咱们同学就知道它就比较类似于我们的这个 Java 的这个GMS， Java message service，是不是它其实就是一个比较上层的这么一个规范，那我们下面呢？基于这个规范可以开发出各种各样的这个消息中间件，是不是比如说 AMQP 协议，它是一个规范，那它的下层可能有 rap m q，那以后可能有其他的 m q，那我们的 java g m s 也是一样的，是不是它这个规范定出来可能遵循这个规范有这个 active m q，那如果大家有了解过 active m q 的话，应应该知道这个消息中间件也是非常主流的一种消息。

那我们看这个图左边这里边有个叫做 publish application，也就是说我们的生产者应用服务。然后这里边有个箭头，就是说他要把具体的这个生产出来的这个消息扔到我们的这个 server 端，那这个 server 指的其实就是我们的 RabbitMQ  的节点了哈，OK。


然后这个节点里边具体有一些内容，比如说像我们里边这块有一个叫做 virtual host 这么一个东西，那这个叫做虚拟主机的概念，那这个我们后续再去说，它其实就是一个比较上层的这么一个路由，并且它也是一个逻辑的概念，这个我们后续详细去讲就好了。然后他在里边有一个叫做 exchange 的东西，那这个是我们 RabbitMQ，也就是不仅仅是 RabbitMQ 了，更上一级别的就是说 amqp 协议的这么一个核心，就是 exchange 叫做交换机。


然后我们的生产者其实把消息直接投递到这个 server 里边的什么位置，就是这个 exchange 的这么一个位置。当然他要经历这三个过程。

第一个过程就是说我把消息投递到哪个服务器，那也就是说你得去建立连接，然后去设置一些地址端口号，然后第二层是什么呢？

第二层你要投递到哪个 virtual host 上进行一个定义，

然后接下来你要把你的这个消息投递到具体的哪一个 exchange 上，

所以说它这个图画的层次感非常的强，让人一目了，然后我们的生产者就不用管其他事情了， 

MQ 本身它的这个作用就是用来解耦的，

所以说我们下面这块有一个叫做 consumer application，就是我们消费者的这个应用服务端。然后他只需要做什么事情，他只需要去监听 message queue，也就是说只需要监听消息队列，这个队列里有一条消息，那我就拿出来进行消费就好了。


那那么其实核心的点就在于什么呢？就是说这个 exchange 跟这个 message queuq 之间有没有什么关系能够关联上，如果说你关联不上的话，生产者投递到 exchange 上，消费者却从 message queue 上去取数据，那这个很显然不太合理，对吧？所以说 exchange 跟 Queen 其实是有一个绑定的这么一个关系的，那这个在我们后续的课程中再跟大家去详细的去介绍，那我们先初步着有这么一个图形化的概念，OK，那么这节课呢？就到这儿，谢谢咱们同学收看。
