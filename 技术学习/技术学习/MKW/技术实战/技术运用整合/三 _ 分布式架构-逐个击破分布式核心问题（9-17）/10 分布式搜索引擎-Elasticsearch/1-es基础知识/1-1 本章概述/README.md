上一个阶段我们是接触了用户的分布式会话以及是单点登录，那么这个其实我们是基于 Redis 去做的一个实现。那么在这章节，那么我们会继续的进入到搜索这一块内容。那么其实也就是分布式搜索引擎，先来看一下我们会接触到的一些内容。对于我们搜索引擎来讲的话，那么其实会涉及到一些相关的技术血型，比方说我们会采用 nice search 但是在早期其实可以去使用 losing 或者说是 sola 都可以。所以对于技术选型来讲的话，其实我们也是需要去考虑的。随后我们会来讲一下什么是 ST search 它的用场景有哪些，如何去使用 ES 其实就是 ST search 的缩写，我们可以称之为是 ES 那么我们可以去安装一个可视化的插件叫做 header 插件，那么我们是可以去做一些相应的操作的。那么其次我们会有一个 IK 分词器以及是词库的概念，那么在我们后续都会去讲。那么此外我们会快速的进入 ES 的入门，因为会涉及到一些基本的术语，一些文档结构以及是一些基于 rest for 接口的操作等等，其实这些都是由 ES 提供的，那么我们都可以去做。


那么在接触完毕一些基础内容之后，肯定的我们要去安装一下 ES 那么我们也是会在咱们的 3 头 s7 在七点叉这个版本去安装 ES 那么当然如果说你没有使用三投 s7 使用的是三投 s6 点叉这个版本的话，那么其实 ES 的话其实也是可以去装的。那么随后我们基于 ES 其实还会有 DSL 的一个搜索，相关的我们都会去讲。


另外的话就是 ES 它也是可以去做一个水平扩展，那么其实就是一个集群，所以我们也会去安装一下 ES 的一个集群。那么通过一个集群不仅是可以达到一个水平的扩展，它也是可以做到一个扩容的。因为它本身单个节点的话，它的一个存储容量，它的一个数据量是有限的，以水平扩展为集群以后，那么你的数据量是可以成倍的提升的。


那么接下来我们是还会有一个数据采集，也就是我们 ES 和数据库的一个数据同步，因为我们的一个搜索。那么现阶段我们是基于数据库的，所以我们要把数据放入到 ES 库中去进行搜索，所以我们会有一个同步的操作。那么其实这也是一个数据采集。那么当我们再把一些数据准备好了以后，那么我们是需要去和咱们的一个项目做一个整合的，也就是 ES 和 spring mode 去整合，并且我们要实现咱们的一个搜索。那么在最后的话我们还是会提到一个植优坐标，因为 ES 的话它也提供了植优坐标的一个搜索。比方说像我们在使用一些附近的人进行一些搜索的时候，我要摇一摇，或者我要左滑右滑去查询一些异性的朋友的话，那么其实也是可以把你的一个范围可以缩短或者说是扩大，我们可以定制在 1 公里两公里 5 公里。那么这个其实我们也都可以通过 us 去做的，所以 ES 的话它本身它是非常强大的一个搜索引擎。


那么我们目前的话其实我们的搜索其实是有一定的弊端的，我们可以来看一下。比方说第一个是空格的支持，那么这个是什么意思呢？我们可以来看一下。比方说现在我们在咱们的一个网站上，我们去搜索一个好吃，点击一下，然后我们的内容是可以全部都搜索出来的。那么咱们的一个关键字其实就是好吃这两个字。如果说这个时候我想搜索一下网红空格好吃，那么也就是说其实我们是有两个关键字，那么在这种情况之下，我点击搜索的话它其实是查询不了的。那么对于搜索引擎的话，它其实是可以支持这样的一个功能，我们中间是可以去加空格的。那么其实这个是一种分词，就说分析以后我们是可以去做一个查询都是没有问题。只不过我们现在基于数据库的话，你中间加了一个空格，它肯定是不能够支持的。所以说下一个是一个拆词查询。


那么拆词查询是什么意思呢？就是指它可以把我们的一个词汇进行拆分，拆分之后再去做一个相应的查询，那么也是可以的。那么是什么意思比方说，那么还是一样，我还是要搜索一个好吃搜索好吃的话像这个好吃好吃这些关键词其实都有。那么如果说我某一个商品里面它是有好和吃这两个是分开来的，它分开来比方说是有一个非常好的吃的东西，那么这个时候好的吃的其实他会帮我们做一个拆分，好吃拆分为好和吃了以后他也能够帮我们去把相应的一些商品搜索出来的。那么这样子其实对于用户来讲的话，其实可以提高用户的一种容错率，因为有时候用户他可能会输错一些词，或说他是会漏写一些词话，那么通过这种方式可以更有效的把用户想要的可能的一些结果值给搜索出来，那么这也是非常不错的。那么对于搜索引擎来讲的话，它也是提供有这样一种方式的，那么这个其实也就是一种分子的功能。


随后我们再来看一下我们的一个搜索内容的话，目前它是不能够有一个高亮的。那么什么叫高亮？就是我们的搜索的结果它是有一个红色的标识。比方说你在使用慕课网或者说是使用百度或者淘宝京东去进行搜索的时候，搜索结果集里面会有你的一个关键字，你的关键字会以一个红色的形式去展示的。比方说我们在进行搜索的时候，我们现在搜索出来是好吃，那么好吃的话其实在我们现在目前所有的一些展示的效果里面来看的话叫好吃。两个字其实全部都是黑色的。


如果说你能够把这两个字给突出的话，那么其实更能够便于用户的一个观看，也就是说用户的一个视觉体验会更好。因为你搜索出来的内容就是用户想要的，它能够更加精准，更加快速的去查看用户想要的那些结果。此外的话还有是一个基于海量数据的一个搜索。目前来说如果说我们数据库某一张表的数据量比较少，可能只有几万十几万的话，你要去查询其实问题不大。但是如果说我们的一个数据量达到百万级别，达到六七百万八九百万的时候，那么这个时候你再去进行一个全表搜索的时候可能会比较的慢了，它的性能就会出现一定的影响，因为它的一个海量数据就可能是达到它的一个瓶颈。


另外如果说这个时候是一种并发的搜索，因为本身对于一些我们互联网的网站来讲的话，那么查询量是远远的，要比更新插入的操作要多得多，所以搜索的并发肯定会有很多。所以这个时候如果说是一个并发的搜索的话，那么你的数据库就有可能会导致崩溃。所以我们就需要使用搜索引擎来为我们的一个搜索提供这样的一种搜索服务。


所以说作为搜索服务的话，那么搜索引擎其实它是可以从海量的数据中快速的能够搜索出我们用户想要的那些数据，并且反馈给用户，而且它是可以去提升存储量的。因为分布式的一个搜索引擎，因为从字面上我们就能够看得出来它是一个分布式的，它是可以部署在多个节点之上的。所以每一个节点我们都是可以有一些数据的存储，它是可以分散开来去进行一些收纳，去进行一些存储的。另外一定要注意就是说对于我们数据库查表来讲的话，我们现在是一个全表扫描，全表去查的。所以这一点的话其实是在数据量非常庞大的时候是非常恐怖的。所以为了优化咱们的搜索的话，我们肯定会使用搜索。
