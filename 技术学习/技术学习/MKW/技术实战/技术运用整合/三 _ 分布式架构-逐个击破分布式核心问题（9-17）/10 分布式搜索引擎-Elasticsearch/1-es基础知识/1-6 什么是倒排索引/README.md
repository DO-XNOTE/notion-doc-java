那么既然是倒排索引，肯定还有一种叫做正排索引。那么我们先来看什么是正排，这个其实就是一种最简单的正排索引，就相当于是在我们数据库中的一个存储，就以这些词以这些语句为例作为是我们的一些记录。那么这些记录的话称之为是文档的内容，每一条记录的话它都会有一个对应的 ID 那么这个 ID 就相当于是一个一个的组件的值，它是按照顺序去排列的。所以它是一种正排索引，而且它是根据文档的一个 ID 的值，根据 123 可以去获取我其中文档的一个内容。


什么是倒排索引呢？那么倒排索引的话其实是这样子，我们来看一下，我们会把这些语句里面的内容进行一个分词。分了以后可以看到我们像慕课网值得学习视占视频这些内容其实全部都是从我们这些语句中切出来的。我们就拿第一个来看，第一行是慕课网值得学习，这个其实就被切分为慕课网值得学习总共是有四项。下面一个就是慕课网是网站，然后这个其实是被切分了五项，因为网和站的话它还是能够被切的。


另外最后一个是慕课王学习视频，总共是四项。然后由于他的一个词被分了以后，这个时候他就可以把他的词汇他的单词对应的出现的一个位置，就可以在这里进行一个展示。这个其实也就是一个文档 ids 或者说这个就是称之为是一个倒排列表。在这里面比方说慕课，那么慕课的话它是出现在123，这三个文档全部都会出现。所以在这里写上一二三网的话，那么其实也是一样，一二三都会出现职责的话是在我们的第一个文档，所以在这里记录是一学习的话在一和三都有，所以一三去记录。


那么下面其实也是一个道理，这个其实就是根据我们的一个词去查找这个词是在哪些文档中出现的，会把这些文档的一些记录值直接给反馈出来。如果说现在我们进行一个搜索，我直接搜一个慕课，他会发现这个单词里面包含了慕课，所以他会直接把这个文档就 123 这三条的记录值直接给反馈出来。那么这个时候用户就直接可以搜索到相应的一个技术值了，而且它的一个搜索的性能是肯定要比你正排索引要比你从数据库里面去查来的更快，性能也会更好。


OK ，这个的话是正牌索引，是根据我们的 ID 去查的，也就是根据 key 去查 value 的值。其实它本身它的工作原理是非常简单的，但是涉及到如果说你的数据量非常庞大的时候，你的一个检索的效率，检索的性能会十分的低下，会严重的影响用户的体验。


倒排索引的话它是相反，它是根据它的 value 根据它的属性的值去查询的。根据单词去搜索包含这个单词的一些所有文档，它的搜索性能要更好。Ok 。所以它的这个整个这一块内容其实就是它的一个倒排索引。另外文档 IBS 这一块是它的一个倒排列表，也就是说我们这种普通的搜索它是正牌的。那么这种逆向的倒着去排的它是一个倒排索引。
OK ，索引的话其实这对于我们来讲，整个索引的系统就是不仅是可以去记录信息，它的属性记忆，它还是能够去记录我们这个单词出现的一个频率以及是它的一个位置，也就是说它是出现重复的一个次数。什么意思呢？我们来看一下这边有一个磁屏，TF还有是一个位置， POS 其实是一个 position 那么磁频是什么意思？比方说慕课，现在在我们第一行它出现的时的一个频率，也就是重复出现的一个次数。那么它是出现中间的这个，也就是一次。前面的这个 1 代表它是这里面的文档的 ID 随后我们再来看第二个。那么在第二行里面它出现的次数也是一次，在第三行第三个里面出现也是一次，所以在这里记录的是 2 冒号1。三冒号一。我们随后来看一下这个网。这个网的话很明显在我们的第二行里面是出现了两次。所以在这里可以很明显的看到 2 冒号 2 中间就是代表食品出现的一个次数。


OK ，那么随后还会有一个位置 position 这个位置 position 是指我们把这个词如果说分割了以后它出现的一个位置。那么像慕课的话，当我们切分了以后，其实很明显，在我们的每一条记录里面它都是1，所以这个监控号里面设置的全部都是1。像这个网的话很明显，在第二个记录里面，网的话被切分为慕课网是网站。那么这个网的话它出现的位置是 2 和4，所以在这个里面会记录奥逗号 4 这个就是它所出现的一个位置。


OK ，为什么要去使用这种磁频和位置 position 这种东西呢？主要是因为这其实是计算的一个因子，是为了更加方便的去排序，相当于是一种打分。就是说它出现的一个频率越高，这个文档和用户搜索的词条单词的一个相关性就越大，它的一个计算的分值分数也就越高了，这样子也就更能够被用户搜索到，这也是可以去提高用户搜索体验的一种手段。那么对于普通的一种数据库去查询的话，这种搜索的体验肯定没有这样子好，而且它也达不到这样的一个效果。


OK ，这个其实就是它的一个倒排索引。然后我们再来看一下它的一个概念，这个是从百度上抄过来的。就是说我们的倒排索引它其实起源于实际应用中，是需要去根据我们属性的值来查找它的记录。根据属性的值也就是我们这里的单词根据这个单词去查找它所出现的一些记录有哪些，把它给拉取出来。


这种索引表中的每一项其实都包括了一个属性，这个就是单词它的属性和包含该属性值的各个记录地址，也就是文档 IBS 另外还有是磁频贴以及是这个位置 position 那么由于它不是根据记录来确定属性的，这个是不是指根据这个文档的记录 123 去确定它的一些具体的位置。它不是根据这种情况确定它的属性的值的，而是根据属性根据单词来确定它记录的位置。所以称之为倒排索引。Ok 。
