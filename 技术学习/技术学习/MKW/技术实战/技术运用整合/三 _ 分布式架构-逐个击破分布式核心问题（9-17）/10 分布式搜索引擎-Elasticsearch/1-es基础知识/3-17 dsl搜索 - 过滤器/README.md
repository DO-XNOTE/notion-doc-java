那么前面几节课我们都是讲的一些查询和搜索。那么我们先来看一下，现在我们使用 match 来进行一个检索，解锁慕课网游戏。然后现在我们查询出来总共是有六条数据，现在可能会有一个需求，就是说当我们解锁数据出来以后，我们能不能够对于我们解锁出来之后的一些数据和内容做一个过滤比方说我们现在以这个 money 为基准，就是说我们的每一个这个用户她的一个账户的金额余额可能会有不同的一些金额。所以我们可以做一些过滤，我们可以取一个区间，比方说这里有 88 对吧55。那么在这样子我们就可以去取一个在 60 块钱到后面有一个 1000 块，只要是在 60 到 1000 块之间的，我们是可以把这些用户给过滤出来，给筛选出来。我们可以使用这种方式去做。虽然我们现在只有六条数据，但是也可以去做，因为我们可能在搜索的时候可能会有 100 盘数据。那么对于用户在网页上去看的话可能会太多。如果说我们加上一个过滤的话，那么这样子对用户来讲的话，搜索的体验可能会更好。然后用户也能够更快速的去筛选出一部分的数据，也是比较的人性化。


那么这个其实就是叫做一个 filter 那么在我们这里面是可以去使用的，是可以和这个 query 结合在一起。那么这个 query 是一种用于去做查询的，当我们要去使用过滤器了，它的关键字在这里。那么它们两个是处于是同级的顶级的一种元素，我们称之为叫做 post filter 好OK ，那么它也是一个 JSON 的对象，在这里面我们可以去输入一些内容。


那么现在的话我们要去做一个范围，我们是针对这个 money 去做一个其实就是一个大于等于和小于等于这样的一种区间。那么这个范围的话其实称之为叫做 range 叫做这个关键字，写上它也是一个折损对象。那么随后我们这个 range 是要针对某一个 field 去做一个对应的范围的过滤。所以你要把它的一个 field 名称给写过来，叫做 money 随后我们在这里面就可以去为它做一层过滤的。


那么这个过滤的话它会包含有一个叫做 gtltgtelte 那么这些是什么意思呢？这些其实我觉得大家应该能够猜得出来， GT 其实就是大于等于， LT 其实就是小于等于，这个 E 就是代表一个 equal 就是相等的意思。那么也就是说 gte 就是大于等于， lte 就是小于等于。OK ，那么这些就是它的一个关键字。


好，我们可以来这里写一下。大雨的话，比方说我们就可以来设置为一个 60 块钱，小雨的话我们设置为 1000 块钱。这里的逗号不要忘记。那么目前我们如果说仅仅只是搜慕课网游戏的话是有六条数据，然后我们现在使用这个过滤，过滤之后，然后我们来看一下，当我们过滤之后，你会发现我们的这个取值获得的这个命中的一个数据就只有 4 条了。那么他是帮我们做了一个筛选，因为一些没有用的数据我们直接给过滤掉了，那么这样子其实会比较好。


那么像我们在使用电商平台去搜索一些商品的时候，那么我们有很大的可能性会去选择某一个金额的区间去查询我们所要去看的那些商品。可能有些商品太便宜了，也有可能是几块钱十几块钱。那么这种商品有可能是假冒伪劣的，那么你就直接给筛选给去掉了。我们看一些金额比较高的超过 50 或者 50 到 100 之间的一个价格区间的一些商品，我们会针对这些商品再去做一个筛选，所以这种功能其实也是对于用户来说是比较不错的。


好最后我们再来看一个，就说我们现在是 GT 和LT ，我们可以再加一个 E 也就是相等。比方说我们来可以筛选一个88，比如说88.8，那么在这里我们加上一个 gte 那么在这边我们再可以加上一个 ltelte 的话，我们可以这样子我们搜一个使用这个，1558点支线的。然后目前我们查询出来的数据就只有两条数据，那么这两条数据是符合我们这个区间的，一个是88.8，一个是155.8。那么这个区间是只有两条数据，并且都是处在于它的一个区间的两端。那么这样子我们可以把它设置为50，再去搜一个，我们可以去取一个 55.8 对吧，再点击搜索。那么这样子的话我们现在还是可以命中 4 条，应该是 55.8 在这里加上点击线的。那么这样子其实我们的这个数据是可以被我们包含进去的，你把这个 E 去掉。那么这个 55.8 是不会被包含，也就是减少一条变成了三条。


OK ，那么这个其实就是我们的 post filter 那么这两者有什么区别呢？那么 query 的话其实是一种解锁，是查询，它是使用我们相应的一些搜索的内容，一些词语到我们的索引库里面去做整体的检索，去做查询的。那么 filter 它其实是一个过滤器，那么过滤器的话它是不会去进行检索的，它是不会去计算我们的一个文档整体的一个相关性的一个分数。相关的性的分数其实就是这个 scope 对吧，它是不会去计算的，那么它是针对于我们的数据检索出来以后再去做的一个过滤。那么这样子的话它的一个整体的性能会比较高，所以这两者是区分开的。


OK ，query的使用场景是用于去检索到我们的 ES 库里面去检索的。 filter 是针对于我们检索出来的数据去做的一个筛选。那么 post filter 那么它和我们的 query 都是在同一级的顶层的元素一定要注意，它只会对我们的搜索结果去过滤，不会去计算数据相关度的匹配的分值的。那么这个其实就是我们的一个过滤器。
