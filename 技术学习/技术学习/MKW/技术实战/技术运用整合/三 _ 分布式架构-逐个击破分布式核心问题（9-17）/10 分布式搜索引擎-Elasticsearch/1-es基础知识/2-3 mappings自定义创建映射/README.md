那么当我们有了一些相应的锁以后，那么我们就可以为它去设计一些相应的数据结构，其实就是它的一个 mappings 是什么东西呢？我们可以来看一下，就是我们在某一个节点下会有相应的索引吗？点击索引信息，其中就会包含有一个叫做 mapping 这个东西。


那么这个我们这节课来一起讲一下什么叫做映射。映射的话其实就是我们数据的一些类型的设置，其实就是一些数据结构，或者说其实也就类似于我们在数据库中去设计表的时候，比方说某一张表的某一个字段，它是怎样的，数据类型是 int 还是 ver 叉，或者说它是否是一个组件等等其实就是类似于这样的一个概念，这个我们可以通过在 ES 中的一个 mapping 去做一个对应的设置。


好。随后我们现在可以通过这个 postman 可以先来创建一个新的索引，我们是可以在创建一个新的索引的同时可以替他去创建一些相应的映射。那么首先创建索引，这里使用 put 随后我们的 URL 写过来，然后在后面我们取一个名字叫做 index 下划线 mapping 那么这个就是作为我们的一个索引的名称。


随后你要去创建的时候，在这里我们要使用它的一个 JSON 那么在我们的这个 body 里面，首先我们要去写上一个大括号，在这里面我们要去指明属性，我们现在的属性我们是要去创建映射，所以这个 mapping 这个 mapping 要去写好，注意在这个地方 map in 后面是有一个 S 的，不要漏掉，随后写一下。那么在这里面就是包含相应的一些属性，也就是我们的 practice 写好。那么对于我们的这个 JSON 来讲的话，这个就是它的一个固定的结构，固定的一个写法，在这个里面你去为它去增加一些相应的属性。其实这个就相当于是一种表设计，随后我们来可以添加一些相应的字段的名称对吧，其实也就是我们的属性，比方说我们来一个叫做 real name 那么对于我们的这个 real name 来讲的话，我们可以为他去设计一个类型，给他来一个 type 这个 type 比方说它是一个字符串。那么我们就可以在这里写上一个 text 随后在它的一个下方我们去写上一个 index 这个 index 我们给它附上叫做 true 那么这个是什么意思啊？ index 其实就是代表我们当前要设置的这个 real name 它是否要被索引？如果说会被索引的话，那么我们就写上处，如果说你不需要让它索引的话，在这里改成 force 就可以了。那么默认的话它是一个 true 如果说你在这里设置完是一个 force 的话，那么它是，不会被索引的话，那么它也是不会被查询到的。


OK ，这点是需要去注意的。所以说你要是设置一些相应的字段的话，你要让用户去搜索到，你就设置为处，或者说你直接不填用它默认的就可以了。那么当然如果说假设你在这里面，你去把用户的一些密码信息你都放进去的话，那么你的 index 一定要设置为 false OK 吧。那么一般来说我们是不会放密码这种比较隐秘的私密的一种信息的。好，这是我们的一个 running 随后我们再来设置一个把它拷贝一份。在这里我们取一个名字，比方说这个我们叫 user name keyword 然后这个 index 我们来设置一个 force 好我们可以这样子去做。那么这样子的话其实我们最简单的一个属性其实就是收拾好了。


好。然后我们在发送之前这边 UI 我们改一下，这边写错了应该是183，这边是对应到我们的地址，点击 send 随后来看一下，这个时候我们是创建成功，创建成功之后返回到我们的这个 header 插件，刷新一下，新增加了一个 index mapping 点击查看它的索引信息，在这边我们就能够看到它帮我们创建了一个 mapping S 这个是 Doc 就是代表我们的一个文档。然后属性一个是 real name 另外一个是 user name 那么来看一下，如果说我们的 index 设置为是 true 的话，那么它其实默认会不去显示的，因为它本身默认就是true 。那么如果说你设置为 false 的话，它就会有对应的一些内容，text和 cue 的这两个都会帮我们去设置这种方式的话，我们就是通过一种自定义的方式去定义我们的一个 mapping 结构的。


另外在这边我们提一下，其实对于我们的一些属性来讲的话，它是可以去设置一些相应的数据类型的。在这边我们设置成了一个 text 以及是一个 keywordtext 和 keyword 是什么意思呢？其实它们本质上都是 string 类型，在曾经老版本里面其实是有 string 这样的类型的。但是在我们新的版本里面其实没有 string 了，它已经是被划分为 text 以及是 Q wordtext 是什么意思？就是说我们的一些文字类型的内容，如果说你是想要被分词被倒排索引的话，那么相关的这些内容涉及到比方说像产品的一些名称、产品的一些详情、描述介绍这种等等比较大的长的文本的话，是完全可以使用 text 去做一个对应的设置。


对于 keyword 来讲的话，那么它其实是不会被分析，也是不会被倒排索引的，它其实就是一种精确的匹配的一个搜索。比方说像一些订单状态，用户的 QQ 号、微信号、手机号，像这种一些数据类型，你想要去做到一个精确的匹配的话，那么你设置为 Q 的这种类型，那么就可以了，使用它的话是精确的，无需分词。那么当然有些同学可能会说像手机号、微信号、 QQ 的话，如果说支持一些模糊区域搜索，那么你把它设置为 text 其实也可以。在这里我们只是举了一个例子，就说 Q 的它是一个精确的，它是不会被分死的。这两个的话其实都是属于 string 类型的。
