那么这个是什么意思啊？假设我们现在在使用这个 ID 现在我们可能有 1000 条数据，那么 low stash 在识别之后会做一个一次性的同步，这个同步的话会把我们 1000 条数据全部都同步到咱们的一个 ES 中，这个 ID 的话我们必须要使用这个自增。


接下来的话，假设我们会有一些新的数据插入到数据库里面，这个 ID 会一直累加。它会从 1000 往后加，比方说一直加到一千零五十两千对吧，它会一直在累加。对于劳克赛事来讲的话，那么它其实会有一个定时任务。那么当它的一个定时任务到期以后，它会去做一个检查。检查以后会发现在我们的数据库里面，我们的一些新的 ID 一个 ID 值会超过我们原来的一个1000。当超过 1000 以后，那么它就会把我们这些新的数据同步到咱们的一个 DS 中。这个就是使用自增 ID 来做这种方式的话，它会有一个弊端，就是说它只能够做一个新增。如果说数据修改的话，它是识别不了的。所以往往我们会采用 update time 使用这个更新时间来作为我们一个数据同步的一个边界。那么这个时间的话，那么一开始其实我们也是在做新增的时候，那么它的一个也是同理的。那么一开始比方说我们也会有一个 1000 条的数据，1000条数据我们会一次性的同步到咱们的 ES 中，那么它会有一个时间，这个时间就是作为我们的同步的边界。当我们初次同步完毕以后，这个更新的时间其实是会被 logstash 记录的，它是会记录下来，下次等到定时任务到期以后，那么它会做一次检查。
