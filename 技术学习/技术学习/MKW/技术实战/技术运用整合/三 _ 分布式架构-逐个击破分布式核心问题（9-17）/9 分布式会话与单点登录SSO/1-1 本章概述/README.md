那么其次就是我们的一个拦截器，那么目前其实我们用户在访问相关的一些接口 API 的时候，我们并没有去做拦截和判断，那么其实我们是应该要去加一下的，这个的话其实就会涉及到一个权限的拦截。那么在咱们整合完分布式绘画了以后，其实我们就可以去实现这个绘画的拦截，去判断这个用户啊有没有登录过了，那么我们就放行，如果说没有登录过，那么我们就直接可以做拦截就可以了，那么这个其实也是和 Redis 整合，基于分布式绘画的。OK。那么最后一个是单点登录，那么这个其实也是一个绘画的共享，单点登录的一个场景就是一个公司，它可能会有很多的产品，有一个产品线，产品系，很多时候对于用户来讲的话，那么我只需要去登录一次了以后，那么其他的一些产品也其实也就是网站。当你再次去进行访问的时候，那么你就不需要再进行二次登陆了。


这个其实就是一个 session 共享 session 跨域的一个问题，其实也就是称之为是单点登录，也是可以通过 Redis 来实现这一个功能的。OK，那么以上这 3 个分布式会话拦截器单点登录就是在咱们本阶段本章所要去讲解以及是实现的相关的内容。
