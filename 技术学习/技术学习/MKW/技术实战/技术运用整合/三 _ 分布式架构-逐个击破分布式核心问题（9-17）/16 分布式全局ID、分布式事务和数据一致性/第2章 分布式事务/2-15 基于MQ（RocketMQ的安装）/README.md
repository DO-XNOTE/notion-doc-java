咱们看一下，里边就是 RocketMQ 的目录，比较熟悉的 bin 康府 lib 是吧，lib就是它的一些依赖的假包了，然后康复这里边是它的配置文件对吧，咱们几乎是不用动的进这个目录下就是它的一些启动的命令，咱们再回到官网，回到快速开始的那一章节。
快速开始这一章节，第一步要启动它的名称服务，启动它的内幕 server 咱们直接去执行 MQ 内部 server 就可以了。咱们在这个它的闭目下打开这个 power shell 然后去执行 MQ name server.cmd 运行一下。咱们看一下，他说你并没有指定 RocketMQ home 这个变量是吧，咱们给它设置一下，复制一下这个变量。然后回到桌面，点击我的电脑高级系统设置环境变量这块，咱们新建一个环境变量 RocketMQ home 然后把这个目录再给它复制一下，指到这个 rocky 的 MQ 的安装目录。就是这个是吧，咱们给它复制一下，然后给它粘贴过来，点击确定再确定。
咱们再回到这个它的官方文档，它要求是 gdk 1.8 以上是吧，咱们看一下当前的系统它的 gek 的版本 window 加 R 然后插入 cmd 再敲一下 Java 杠 jorcian 咱们当前的版本是 1.8 没有问题是吧，咱们再回到这个 power 上，再次运行这个 MQ 类目 server 这个设置完了，这个 power shell 要重启一下。如果你不重启的话，就像刚才咱们那样，它是不识别的，还是在这个闭目录下打开 power 扇再执行一下 MQ 内部 server cmd 是吧？ cmd 是在 Windows 这个系统下去运行的，所以咱们要选择这个 cmd 文件，咱们看一下。
内幕 server 已经启动成功了是吧，序列化的类型是 Jason 说明咱们的内部 server 安装成功，然后咱们还要再启动一个队列是吧，咱们看看。第一步，启动内幕 server 第二步，启动 broker 是吧。启动队列。启动队列的时候，咱们要用杠 N 去指令它的内幕色味是吧，这个就是它的内幕色味，内幕色味默认的端口是9876，所以一会咱们也要指定一下。那么还是回到这个闭目录，再启动一个 power shell 这块咱们要运行的是 MQ broker 是吧，点 cmd 然后杠 N 指定内幕 server 的地址， local host9876 回车。好，咱们可以看到这块 boot success 别启动成功了是吧。序列化的类型 Jason and name server 它的名称服务器是 logo house9876，说明这个队列也已经启动成功了。好，这一节就告一段落。下一节开始，咱们在程序里边去连接这个 Loki 的 MQ 完成咱们的分布式事务。谢谢大家。
