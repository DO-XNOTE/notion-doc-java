大家好，接下来咱们就看一下如何使用 UUID 作为全局的主键。首先咱们看一下 UUID 的概念。 UUID 它的意思是通用的唯一识别码对吧，咱们看一下后边的英文单词。 universally unique identifier university 是什么意思是全球的意思是吧，UUID它是全球唯一的 ID 对吧，那么它肯定是不重复的。如果咱们使用 UUID 去作为这个表的主键，那肯定是保证每条记录它都是不重复的。就是我们的实际应用当中是很少使用 UUID 作为这个表的组件的。


为什么呢？这个就是因为 UUID 它是有缺点的，它的缺点它只是一个单纯的 ID 并没有实际的意义是吧，而且长度也非常的长。 32 位在咱们的实际的业务系统当中使用了一个 32 位长的又没有任何实际意义的这么一个值，作为你的表的 ID 是吧，这个显然是不太合适的。但是它确确实实地解决了分布式系统当中 ID 重复的这么一个问题。
