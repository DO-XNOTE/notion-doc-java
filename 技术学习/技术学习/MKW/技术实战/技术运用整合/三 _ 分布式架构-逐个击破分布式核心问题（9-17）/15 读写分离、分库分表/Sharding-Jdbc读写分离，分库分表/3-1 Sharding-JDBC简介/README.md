今天的课程当中，咱们给大家详细的介绍了 my cat 分库分表，咱们可以使用 my cat 做分库分表，还可以做读写分离。那么 my cat 它可以算作是一个服务端的中间件代理，咱们需要启动 my cat 的服务，然后我应用端的程序直接连那个服务就可以了。对于我应用端的开发人员来说，这些后续的工作基本上都是透明的，完全不需要关心后面发生的什么事情。
从这一节课开始，咱们要给大家介绍另外一个读写分离分库分表的一个中间件，它就是沙丁 j d v c，它和买 cat 有什么区别呢？其实最大的区别就是沙金GDBC，它是一个客户端的代理，那么我们就不用单启动一个中间件的服务了，是吧？咱们直接把这个沙丁 GPT 的假包引入到我们的项目当中，就可以完成这个分布分表和读写分离的操作。那么咱们现在看一下沙金的 GDBC 它的一些简介。
首先沙丁这 DBC 它是一个开源的分布式的关系型数据库的中间件，对吧？它也是针对关系型数据库的。现在这个杀精 GBC 已经进入到了阿帕奇的孵化器，并且它在阿帕奇也已经有了一个官网，咱们一会可以看一下这个网站。它是客户端的代理模式，咱们不需要单搭建服务的，只要你后端的数据库配置好就可以了。
后端的数据库我有几台，我怎么分片的？我怎么规划的？你确定好，然后直接在我的应用程序那一端进行沙丁 GEV seed 配置，就可以完成度线分离分布分表了。它定位于轻量级的 Java 框架，以价包提供服务。那么这个山精练习 BC 只能应用于 Java 的项目，它是用 Java 编写的，对吧？你用其他的，比如说你用到 PHP 的语言，或者是 Python 的语言，它都是不支持的，怎么去理解它呢？可以理解为一个增强版的 GDBC 驱动，咱们想想咱们项目当中配置连接数据库的时候，你是不是要配置一个数据源呀？配置完数据源以后，然后把这个数据源加入到你的事务当中，然后再配置一些其他的，像 ORM 框架等等。
那么这个山精这 EBC 你就可以理解为它是一个数据源，咱们把所有的配置配置完以后都会暴露出一个数据源，然后咱们拿这个数据源再去进行你后面的这个配置，比如说像OIM、框架、事物这些都可以用这个杀精GDB、 seed 这个数据源去做配置就可以了。
然后它后边也是兼容各种主流的这种 ORM 框架。比如说Havernet，比如说 my Badcase，一会儿咱们在项目当中给大家使用 my badcase 做示例，然后这个就是这个沙丁 GDBC 的一个整体的架构图，这个咱们看一下下边是数据库层，是吧？它这个示例当中有三个数据库，然后上边是我们的 Java 应用，就是这个虚线框起来的这么一个，它是 Java 应用这块有两个绿色的部分是咱们的业务代码，然后蓝色的部分是沙丁、g、d、b、c，咱们的业务代码都是通过沙丁g、d、b、 c 去连接数据库的。所以咱们在项目当中通过配置文件，配置好杀进类etc，它里边的路由规则、分片、字段这些其实是和 my cat 里边的那些基本概念都是贯通的，咱们理解了买case，在讲解沙丁这 EBC 的时候，这些基本概念大家都是非常快速的就可以掌握了。
然后这个沙丁这 etc 的配置，咱们也可以统一的去使用一个注册中心，如果大家不使用注册中心的话，就直接在项目当中写在配置文件里就可以了。这个注册中心咱们可以使用 to keeper，这样可以保证所有的项目之间这个配置文件是统一的，这个就是它的一个整体的架构。然后山菱这 DBC 它提供了四种配置方式，其中最原始的就是这种 API 的方式，还提供了 e mail 这种方式。
最后是 spring boot 和 spring 命名空间，也就是 XML 配置文件的那种方式，在咱们的课程当中会着重给大家讲解 spring boot 和 spring 命名空间。这两种方式在项目当中也是使用最多的，因为在咱们的项目当中离不开spring，所以咱们把这两种方式给大家讲解一下。
最后它与买 cat 有什么区别？为什么这里边要给大家讲解这两种这种分库分表毒械分离的这个框架，它到底有什么区别？其中最主要的一个是服务端的代理，一个是客户端的代理，对吧？这个都是显而易见的，这个咱们怎么去选择呢？作为一个架构人员，你选择这种框架的时候，你要考虑到你公司的情况，比如说你这个公司 DBA 或者运维人员比较强，我的这个人员水平都比较高的话，那么我建议采用 my cat 这种方式做这个分布分表，因为买 case 它是偏向于DBA，偏向于运维的，那么如果你公司这种 DBA 或者运维人员相对比较少，能力也不是太强。
那么呢，我会推荐杀丁类DBC，它应该是作为客户端的代理，直接写到你的应用程序当中，通过你开发人员进行配置。如果是这种情况，那么我还是推荐你使用沙鲸这etc，这个就是不同的选择，要根据自己的情况，到底谁好谁坏也没有一个明确的定论，是吧？它就是两个不同的产品，就像大家早上吃早点一样，有人喜欢吃包子，有人喜欢吃油条，你就说到底说谁好谁不好，这个没有定论，就是要根据自己的公司的情况选择不同的分库分表的产品，然后还有一个不同点，这个可以说是他们两个在功能上面最大的一个差别了，就是买 cat 它不支持在一个库内的水平切分，什么叫在一个库内的水平切分呀？就是比如说我有一张表，一张逻辑表的概念，大家再快速的回想一下，比如我有一张逻辑表，是order，是吧？订单表我要水平切分，我如果使用买 cat 的话，我是不是水平切分？以后两张实体表要坐落在不同的数据库当中啊。
你在配置一张表的时候，你要配置这张表的 data NODE，是吧？它的数据节点是什么呀？配置了一个 data host，是吧？然后后边跟了一个schema，就是数据库的库名，咱们通过这条配置就可以看出你的一个逻辑表是不能在同一个库内切分的，你要坐落在不同的数据库当中。比如说我一个 order 表，我要切分成两张表，分别为 order 一和 order 2。那么 order 一和 order 2 是不能在同一个数据库当中的，但是我如果使用沙丁这etc，我是可以支持的。我就在一个库里边，我切成两张表，一个叫 order 0，一个叫 order 一，然后作为一个逻辑表对外展现，它的表名就是orders，这个可以说是杀精 GEBC 和 my cat 这个在功能场面最大的一个区别了。好，这个沙丁，这 DBC 呢？咱们就先给大家先介绍到这告一段落，然后从下一节课开始，咱们就要进入到代码层，看看怎么去使用山景这 EBC 做这个分布分表。
