前面几节课我们是通过了 fast d f s 结合到咱们的项目里面去实现了一个文件的上传，这种方式我们是依托的 fast d f s。那么它本身是一个分布式的文件系统，可以去存储文件，它可以去提供扩容的一种方案，它肯定是要比我们以前所使用的一种传统的文件上传的方式来的更好。它最主要的一个优势，优点就是水平扩容，我们是可以去无限的扩进行扩容的，这个是在我们平时使用传统的方式去做上传的时候实现不了的，但是 fast d f s 它也有一些相应的小缺点。

就比方说我们再去进行运维的时候，相应的它的一个复杂度会提高，运维成本也会提高，因为一旦我们的一个数据会成倍的增加，我们的文档会越来越多，越来越多，也就意味着我们肯定是要去做一个无限的扩容。对于运维来讲，那么它的一个运维的成本、复杂度等等各方面其实都会提高。除此以外，它的一个开发也是有一定的复杂度。在这里我们所谓的开发并不是指仅仅的一个上传，我们是指比方说我们现在上传了一张图片，要针对图片再要去做一些相应的开发。比方说我要去增加一个水印，我要去追加一些相应的图像的形成的效果，或者说是人脸识别 fast d f s 它是实现不了的，那么这个就需要由自己的一个技术团队去进行相应的开发，它的开发的复杂度也会提高。


此外我们不仅是可以上传图片、视频文件，我们也可以上传，现在其实很多的一些网站它都会有视频上传后的一个处理，比方说很多人会钟爱于拍摄vlog，对吧？那么你的视频上传以后可能会去调整相应的一些颜色、基调、曝光等等，这些的话就是对于视频的处理， fast DFS 它也不能够去实现，你就必须也要由自己自身的技术团队去进行针对于这些视频文件的一个后期处理，这个将大大增加开发的一个成本，所以这个就是使用 fast d f s 带来的一些小问题。


如果说像一些比较中小型的互联网公司在使用 fast d f s 的话，是完全没有任何问题的。但是一旦你的一个图片或者说你的视频等等的文档数据成倍的增加以后，其实我们就应该会考虑到使用第三方所提供的一个文件云存储。

这个的话在我们课程里面我们会来讲解云存储的阿里OSS，OK，我们先来简单的看一下 OSS 的话，它是由阿里所提供的，它会有一个SDK。这个 SDK 的话我们是可以去集成到咱们自身的项目里面去，它是可以提供不同的语音的版本，对于我们来讲，我们会使用 Java 去进行一个对接，使用起来会相当的简单。


此外它还能够提供强大的文件处理功能，就像我们之前所说的，它能够去针对于我们的图片增加水印、人脸识别，还有视频的一些处理，其实它都能够支持。这个就是它针对于文件的一种处理，针对于文件的一种计算，其实云存储都能够提供这样的一种服务。此外它不仅能够提供一些文件的处理功能，它还能够提供文件的一些权限，那么这些的话都是 fast DFS 所不能够提供的。


OK，在使用 OSS 的话，其实我们可以说几乎是零运维成本，我们只需要去购买它的一个服务，再进行一个对接就行了。那么像它能够支持一些扩容相应的方案，其实对于我们的一个海量存储的话也是没有任何问题的。另外 OSS 它也提供图形化的管理界面，它有一个控制台，我们也可以去通过控制台去看一下我们创建的存储空间，然后上传的一些图片也可以手动的去针对于图片去做一些相应的处理、删除等等，也都可以去实现。那么这个也都是由云存储所提供的一个管理平台，那么在我们使用 fast d f s 去做上传的话，一旦我们的数据量非常高的话，那么我们想要去查看这些文件，查看这些图片的话，可能我们会自己要去开发一个平台去进行相应的管理。这个的话其实使用 OSS 完全就可以避免了这样的一种成本的支出。


此外使用第三方云存储的话，它还有一个 CDN 加速的服务，如果说你开启的话，对于用户来讲，他们的一个访问的体验是相当的高的，如果说你使用的是自己搭建的 fast d f s，对于我们来讲，我们对外其实也是在云服务器里面去搭建的，对于我们的一个带宽来讲的话，也是在我们初期购买云服务器的时候锁定的，在我们使用了第三方云存储的时候，如果说你开启了一个 CDN 加速的话，用户在访问你的一个图片视频资源的时候。可以说是非常非常快。


OK，所以这些其实都是在使用像这种第三方云存储的一些优势。OK，当然它的一个唯一的小缺点就是收费。其实对于企业来讲，虽然这个使用第三方元存储要进行收费，其实你综合来讲，如果说你综合之前，如果说使用这种自建的 fast d f s 来讲的话，你的一个运维成本以及是你的开发成本是可以大大的降低，降低的同时你的风险其实也就降低了，因为风险管理其实也是企业管理中所需要去考虑的一个因素，所以说一般来说，一旦我们整体网站的一个业务量上升以后，我们是可以逐渐的从咱们的一个 fast DFS，我们可以去向第三方云存储去进行一个迁移的。像阿里的 OSS 它也是提供了这种迁移工具，也是可以去进行一个相应的使用的。OK，这些的话其实都是我们在后续会使用到的一个阿里的OSS，OK，那么当然其实第三方的云存储，你不仅有阿里的OSS，像腾讯和 t 牛云的话，其实也都提供类似的文件对象的一个存储，OK。
