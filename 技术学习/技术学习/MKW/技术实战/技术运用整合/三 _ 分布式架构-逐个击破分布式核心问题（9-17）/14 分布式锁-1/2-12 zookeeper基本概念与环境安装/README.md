在上一节的课程当中，咱们学习了基于 Redis 的分布式锁，大家一定要掌握原理。咱们在上一节的课程当中，在写这个 Redis 的分布式锁的时候，也是先要把原理弄清楚，弄清楚以后你在编写这个分布式锁的时候就非常的顺利了。


这一节咱们要看看另外一种分布式锁的实现，这种分布式锁是基于组 keeper 的。那好这个组 keeper 咱们要先聊一聊了，大家在项目当中没用过组 keeper 你用组 keeper 编写过哪些程序？我相信大部分人无非是两个，一个是大数据方面，hadoop组 keeper 作为一个组件还有另外一个途径就是使用这个分布式服务，就是阿里 double 这个框架，它是基于主 keeper 了。那我如果要使用这个 double 框架，我要去先安装主 keeper 那么好现在咱们就要基于主 keeper 去编程，用主 keeper 去实现分布式锁主 keeper 咱们先要给大家做一个基本的介绍。主 keeper 它也是存储一些数据，但是它里边的存储结构不像数据库对吧，我数据库里边是一些表去存储数据，这个 Redis 是 key value 减值队去存储数据。

咱们看一看它其实是一棵树有根节点，然后有自己的子节点，其实也比较像咱们的这个目录结构一层一层的，我这是一个文件夹，这是一个文件夹，这是一个文件夹对吧？这个结构也是这样。一个节点，但是它的这个节点里边是可以存储数据的。有的同学可能会问了，你这块红色和黄色是什么意思呢？红色节点它代表的是持久节点。持久节点什么意思？就是说你的和组 Q 绘画结束或者是组 keeper 重启，这些节点都是不会消失的，它一直存在，除非你手动的去删除这些节点。那么这个黄色它是瞬时节点。咱们先给大家解释一下这个瞬时节点是什么意思。瞬时节点它在你组 keeper 重启，或者说你连接阻 keeper 的这个会话如果断了，那么这个瞬时节点就会自动的去消失瞬时节点，它不可以再有子节点。如果你这个节点是瞬时节点，子节点是不可能存在的。为什么如果你自动消失了，你这个准时节点的子节点是不是也会跟着消失啊？就是作为这个顾虑组， keeper 规定它的瞬时节点是不可以有子节点的瞬时节点，它还有一个可以是有序节点。什么意思？就是说你在创建这个瞬时节点的时候，它的这个瞬时节点的名称是按照序号排序的，这一点也是非常的重要的。
