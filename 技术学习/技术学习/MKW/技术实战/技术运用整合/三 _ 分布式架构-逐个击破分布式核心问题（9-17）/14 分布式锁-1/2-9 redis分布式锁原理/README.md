但是 spring task 有一个致命的缺点就是什么呢？它不能够集群部署。如果你集群部署的话，那你定时任务就等于是重复了。比如我发送短信，我部署一台服务的时候，我定时发送一条短信是吧。如果我部署了两台服务，那怎么办呢？到了这个任务执行的时候，我两个服务都去发送短信，那我就发送了两条。那么为了避免这个任务重复执行，咱们就利用分布式锁解决这个问题。任务执行开始之前，咱们要先获取锁，看看哪一个服务获取到的锁，哪一个服务获取到的锁他去执行后边的任务，咱们另一台服务就不去执行了，就是利用这个原理解决分布式锁的问题。好，下面咱们先抛开这个定时任务部署，咱们先在代码当中去实现基于 Redis 的分布式锁。
