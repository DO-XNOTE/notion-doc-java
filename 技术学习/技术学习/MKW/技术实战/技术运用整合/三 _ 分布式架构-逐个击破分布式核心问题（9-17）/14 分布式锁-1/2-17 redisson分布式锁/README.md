会咱们会给大家进入到这个 ready 神的官网当中，看看它都有哪些强大的功能。在前面图文的课程当中，已经给大家做了一个粗略的介绍。它除了实现了 Redis 这种最基本的功能以外，他还重新实现了 Java 并发包里边的这个内容，就是这个 Java 肯克润的这个包。这个包下都是提供了一些处理并发的这些类，比如说包括锁，包括一些原子性的操作。但是这些包的内容都是单体应用都是基于一个 JVM 的。


那么如果我们多个 JVM 怎么办？我们就可以引入这个 ready 神这个价包。他把肯克瑞的这些包里边的这些处理并发相关的这些类又重新的实现了一遍，让这些功能可以跨 jam 去使用。它的功能可以说非常的强大。除了这个并发包，它还实现了像张小忠的一些集合类，像这个 list 像哈希 map 使得这些类也可以去跨 gvm 去使用，可以说是非常的强大。这个 regis 它里边也实现了这个分布式锁。所以其实在咱们的项目当中，只要引入了 ready 神，然后咱们就可以直接使用这个分布式锁了。它的这个功能和咱们前面介绍的这个 Q 瑞特这个夹包差不多。


Q 瑞特它是组 keeper 这个客户端的一个升级的版本，咱们在引入 QA 特以后也是可以直接的就去使用这个分布式锁的方法了，你是不用自己去实现这个分布式锁的。这个 Redis 其实一样，它也可以说是这个 Redis 客户端的一个升级的版本，它的内容也是比较多的，也提供了分布式锁，咱们也是只需要调用就可以了。


那么好咱们看看本节的内容，咱们会在项目当中引入 Redis 这个价包，然后进行这个 Redis 的配置。这个 Redis 它是 Redis 的一个客户端，其实它的这些功能都是借助于 Redis 的。所以咱们要进行这个 Redis 的配置，无非也就是说把这个 Redis 的这个IP 、端口给它配置到这个 Redis 当中。然后咱们就直接的去使用分布式锁了。
