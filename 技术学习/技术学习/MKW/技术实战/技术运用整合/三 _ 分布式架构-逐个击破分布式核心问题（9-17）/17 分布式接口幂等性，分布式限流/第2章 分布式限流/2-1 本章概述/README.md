OK 那咱回到课程里来，本套课程的微服务章节由我来跟大家担任船工长老。所以我迫不及待的先提前一个小节跟大家打一个照面，先用这个小章节跟大家做一个热身。在接下来的两个月时间中，我们将共同经历从分布式的架构转变到微服务架构的过程。那大家相信会跟我一起度过欢乐的两个月的学习时光，我们会幸福的言归正传。


本节课程我们讲什么内容呢？当当。这一节我们作为整个分布式章节的完结篇，我们来谈一谈啊如何做分布式的限流。这里主要是跟大家介绍几种常用的限流方案，它包括客户端的限流，也包括分布式的中心节点限流。我们接下来看一下本章的课程目录都有哪些。在本章的开始部分，我们先对分布式限流做一个简单的介绍。然后在后面的小节里以图文的形式跟大家从常见的分布式限流方案以及它的技术选型两个方面做一个介绍。说到分布式限流，不管你用哪个方案，所谓万变不离其宗，它的算法思想大体上是非常接近。那接下来我就带大家去理解常用的分布式限流的几种算法。再往后一个章节，我们就进入了动手环节。这里咱从由浅入深来安排进度先跟大家介绍一个基于客户端的简易限流方案。那它底层是利用了谷歌的工具包其中的一个 rate limit 类做的客户端限流，这其实是一个本地的限流方案，它并不能算是一个分布式的中心节点限流，不过它也可以作为一个轻量级的限流方案应用到自己的项目当中。我们搭建完这个 demo 以后，接着就趁热打铁去看一下刮瓦的预热模型，它在算法层面是如何来实现的。接下来我们将进入正儿八经的分布式限流方案了。


第一个方案我先跟大家介绍如何基于 ngx 做分布式限流。在前面的小节里，相信大家都已经深入的学习过 ngx 的用法了，这里我会跟大家介绍几个常用的 ngx 限流的方式，这其中有两个重点的方案要跟大家介绍。第一个是基于 IP 地址的限流方案，紧接着是基于最大连接数的限流方案。 ngx 作为一个网关层的中心化的限流方案，应用场景是非常广泛的。
