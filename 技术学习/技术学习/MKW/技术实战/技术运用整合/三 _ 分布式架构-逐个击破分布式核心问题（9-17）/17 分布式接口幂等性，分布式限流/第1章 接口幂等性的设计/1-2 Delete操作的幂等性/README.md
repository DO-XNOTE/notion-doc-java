hello 大家好，这一节咱们跟大家聊一聊迪丽特操作的幂等性。首先迪丽特操作的幂等性咱们分为两种情况，一种情况一种情况是你的这个 delete 操作它根据唯一的业务号去删除。另外一种情况就是你的这个底粒子操作没有这个唯一的业务号。其实不光是底粒子操作，咱们所有的接口都分为这两种情况去考虑，一个是有唯一的业务号，另外一种情况是没有这个唯一的业务号。


现在咱们先看一下 delete 操作的有唯一业务号的这种情况，有唯一业务号，咱们就根据这个唯一的业务号去删除数据。那么第一次删除的时候就将这条数据已经给它删除掉了对吧？当我们的接口出现问题的时候肯定要重试是吧，或者前端这个表单重复提交，这个时候就会进行第二次删除对吧。那么在执行第二次删除的时候，由于你第一次已经把这个数据删除掉了，我第二次再删除的时候找不到相应的记录了对不对？所以这个时候返回的结果就是 0 是吧。


delete 这条 SQL 执行的结果就是0，对业务数据其实是没有任何影响的。可以说 delete 操作也是一个幂等性的操作。如果想让咱们的这个 delete 操作的接口更完美一些，咱们可以在删除之前先根据这个唯一的业务单号去查询一下这条记录对吧。如果查询不到，咱们就给前端返回一个结果，说这条数据查询不到对吧，如果查询到了，咱们再进行数据的删除，这样是不是就可以更完美的实现这个滴滴的操作的接口了对吧？接下来咱们再看第二种情况。第二种情况就是我没有唯一的业务号对吧，然后我要执行比例的操作。那么这种情况就要看具体的业务需求了对吧，业务需求要不要求你的这个底下的操作具有幂等性。
