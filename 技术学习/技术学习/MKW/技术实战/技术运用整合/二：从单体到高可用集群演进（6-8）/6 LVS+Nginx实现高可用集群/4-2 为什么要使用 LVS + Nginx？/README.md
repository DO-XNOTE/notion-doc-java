那么通过上一节的一个拓扑图，大家肯定会发现 LVS 和 nginx 很像，但是我们为什么还要使用 nginx 呢？我们直接使用 LVS 和 Tomcat 的结合不就完事了吗？那么其实也是有原因的，如果我们单独只提到 lvs 的话，那么 LVS 的话，它是工作在 4 层的，那么 4 层的话它的性能会更高。它接受到的一个请求的话是可以直接去转发的。然后对于 nginx 来讲的话，他接收到请求以后，那么他还要对请求去进行处理，相应的那么 Nginx 的话，它就会有一定的系列的损耗。

如果说我们单个Nginx，它假设并发支持量是在1万左右的话，那么 LVS 的话，它可以支持的负载是 Nginx 几十倍，它的负载要远远的比 Nginx 更高，那么单个 Nginx 的话，它要承受全部的并发压力的话，那么肯定是不现实的，而且也是有瓶颈的。所以我们就会构建 Nginx 的集群，是需要使用多个 Nginx 共同的为用户去服务，去处理用户的请求。


那么当我们使用到 Nginx 的集群了以后，我们可以试着想一下，那么对于 Nginx 的一个集群来讲，它的前置的调度者充当 Nginx 的负载均衡，我们肯定不可能再会使用 Nginx 去充当前置的调度了，因为这样的话它还是会承受所有的并发压力的，所以我们就会使到 LVS 的话，它是不会处理请求的报文的，所以它的一个负载能力会更高，是相当的高的。对于 nginx 来讲的话，那么其实它会接受请求，也会去处理请求，所以它是一来一回的。 LVS 的话，它是可以只接受请求，但是它可以不去响应，它可以不用去处理，所以它的整个负载的话，肯定会比 Nginx 要来得更高。那么我们可以通过拓扑图来观察一下它们两者的一个区别。
