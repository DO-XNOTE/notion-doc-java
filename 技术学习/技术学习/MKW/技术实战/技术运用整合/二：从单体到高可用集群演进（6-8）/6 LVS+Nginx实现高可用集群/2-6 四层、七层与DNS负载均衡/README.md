上一节课我们是聊了一下 7 层网络模型，这节我们就一起来讲一下 7 层和 4 层负载均衡的一个概念和相应的区别，那么这也为我们后续的课程可以铺一下路。当我们使用集群的时候，肯定会涉及到负载均衡，负载均衡又分为 4 层和 7 层这样的一个区分，咱们这一节的话就来聊一下。前面我们已经是涉及到了负载均衡， load balance 可以简称为 LB Nginx，就能够作为一个负载均衡器来使用。负载均衡的话是建立在现有的企业网络环境里面，能够为企业网络设备或者服务器来带来更好的一些服务，那可以提高吞吐量，提升并发性能，增强服务器的处理性能，并且它也能够提高服务器计算能力，使得我们网络设备更加的灵活。

负载均衡的话，那么就是当有大量的并发请求来到服务器的时候，我们能够把这些请求分配到多台计算机节点上，让更多的节点来处理请求和响应，如此我们就可以大大缩减用户的一个等待请求的时间。那么 4 层负载均衡的话是基于 IP 加端口的一个负载均衡，原理的话是通过转发请求到后台的服务器，它是只负责转发，并且会记录当前连接是由哪个服务器处理的，后续这个连接的请求就会由同一台服务器去处理，其实也就相当于是长链接，这个连接一旦打开就会一直处于连接的状态，它的性能就非常的高了。

最后一个是 lvs，它是四层负载均衡，那么它是 Linux 内核的一个四层负载，和协议是没有什么关系，它主要是可以用于去转发一些请求的，它是基于我们的一个 cs 端的。后一个是 Haproxy，它也是一个四层负载均衡，它是支持转发功能，那么它灵活性非常的高，除了 4 层以外，它也能够在 7 层做 7 层的负载均衡。那么随后还有一个是 nginx， nginx 的话，它在新版本里面它也能够实现 4 层负载均衡。但是一般来说我们还是使用 nginx 做 7 层，它是以 7 层为主的。 Nginx 它是可以基于 HTTP 的去做相应的负载均衡。那么老版本的话是不支持是在 1. 9 之前，在 1. 9 版本以后的话，那么它其实是新增加了一个 stream 这样的模块。 stream 模块我们在之前的课程里面，在模块化里面其实有看到过，有提到过它就可以用于去实现 4 层协议的一个转发代理，或者说是负载均衡等等。

最后一个是 Haproxy，这个我们在上一个 ppt 也提到过了，它是支持 4 层和 7 层的负载转发功能，灵活性它是非常高的。下一个是 apache ，apache 的话，其实它的性能是远不如 nginx 高。其实我们在课程一开始我们也介绍过 apache的，它支持的并发不是很高，当并发达到百万级别以后，那么它的性能会越来越差。其实一般来说，我们在使用四层做负载的时候，我们一般会使用 lvs 的话，主要用于去做七层 Haproxy，它灵活性是非常高， 7 层和 4 层都能做。 7 层负载均衡的话，它基本上都是用于去处理 http 协议的，那适用于 web 服务器，比如说像 tomcat，阿帕奇，Nginx，只要是 web 服务器都可以。 4 层负载均衡的话，主要是用去处理基于 TCP UDP 协议这种负载均衡。


四层的话，它是用于转发请求，而不是去处理的，它可以把用户的请求直接转发给其他的应用，去处理器层的话，它是会处理请求的。像安吉克斯的话，就能够对响应中的一些，像比方说可以去在响应里面，因为它有过滤模块，它可以去处理 JS 、 CSS 等等文本的内容，它是可以去压缩，也可以去缓存。这就相当于是针对我们的请求响应做的一个处理。在这里我们可以去举一个例子，比方说我要去看演唱会，你得买票吧。
