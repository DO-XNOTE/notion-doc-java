先来看一下什么是 lvs，那么 lvs 的话它是一个高伸缩性，它也是一个高可用的服务。这个 lvs 的话它是建立在哪里？它有一个 build on，它是一个集群，这个集群的话是由 real service 所组建成的。其实这个 real server 在这里的话，我们画一下重点， real server 其实就是真实的服务器，那么真实的服务器也就是我们真实真正的要为用户去提供服务的，那么这些 real service 其实就是阿帕奇，或者说是恩迪斯，又或者说是 Tom kids，那么这个就是一个集群。然后我们的 l s 是带有 load balance，也就是说它是一个负载均衡器，它是跑在 Linux 操作系统上的，因为它叫做 Linux feature server，它不是一个 windows 飞书server，所以它只能够在 Linux 操作系统上去运行，在其他的操作手上是没有的。


那么它的整体的架构的话，那么其实是一个集群，那么它也是一个对于用户来讲的话，对于用户来讲，它这整个集群是透明的，然后用户访问的时候，对于用户来讲的话，它是一个单个的高性能的一个整体。OK，那么其实它的一个道理的话，和 Nginx 其实也是类似的， Nginx 的话它其实也是可以把所有的一些真实的服务器组建成一个集群，然后再去为用户去提供服务。对于用户来讲的话，我们的整体的架构其实全部都是透明的。

**那么首先用户是通过浏览器发起请求的话，到达服务器的一个LVS，那么就是在到达 LVS 的之前，其实会有一个DNS，这个 DNS 的话在云服务器上是会有的，在这里我们就直接忽略了。那么 LVS 的话，它是充当负载均衡器，也是一个调度器。那么 LVS 会根据自己的一些算法，比如说最常见的最常用的就是轮询，根据一些算法可以把请求发送到我们后端的某个真实的 web 服务器，在我们后面的服务器可能会有多个，那么 real server，那么在我们整个拓扑图里面，我们是可以简称为 rs 的。**


**用户访问 lvs 的时候，虽然会被转发到我们后端的一个真实的服务器，但是每台服务器它所提供的服务其实它都是一模一样的。所以用户不管访问到哪台真实的服务器，得到的服务的内容和响应，都是一模一样的。整个集群对于用户而言，那就是完全透明的，通过使用这种负载均衡的集群的话，是可以扩展我们服务器的一个负载的性能带宽，提高吞吐量，提高网络数据的一个处理能力，增强服务的灵活性以及是可用性。通过这样的一个拓扑图，其实我们也可以看得出来，他和 Nginx 其实是非常的类似的。如果说不使用LVS，使用 nginx ，那么它也是一个可用的网络拓扑图。**
