目的我们主要是为了更加精准的去响应用户的请求，以及是处理用户的请求。先来看一下 arp-ignore，这个是 arp 的响应级别，如果说很难理解的话，你就可以把这个当做是处理请求，也就是 request 接收。 request 你是需要去做一个处理和接收的。如果说在我们当前的某一台计算机节点上，我们配置了有多个 ip ，如果说你在一个 arp-ignore，它的一个级别你设置为 0 的话，那么在我们当前计算机节点上，所有的 IP 都能够去处理这样的请求，这个很明显在我们这里是不适用的，所以它还会有第二种级别。那么第二个是1，就是说当我们请求的目标地址到达对应的网络接口，才能够去处理响应咱们的请求。也就是说我们请求的是打个比方是幺九二点幺六八点幺五零，那么只有和这个 150 这个 IP 一定要一致，那么它才能够去处理用户的请求，这个才是我们所需要去使用的。


**如果说我们在当前的计算机节点上有很多的IP，但是你的 arp-ignore 设置了为 0，那么所有的 ip 都能够去处理用户的请求，这一点是需要去注意的**，所以这个我们应该要在我们当前的真实服务器里面去做到的一个配置，也就是如何去处理用户的request，它的方式我们要把它配置为一，其实也就是网卡的行为我们要去做设置的。

第二个是 arp 的通告行为，也就是 arp-announce，它会有三个级别，那么第一个级别是我们在当前的计算机节点上，如果有多个网络接口，都可以向外去进行通告，所有的网卡都可以相互的去接收到通告。


这个的话其实就相当于是在我们当前的计算机节点上，如果说我配置了多个网络接口，配置了多个网卡的话，那么所有的一些响应 response 就是都会进行返回的，它都会返回出去，那么这样子显然也不是很好。这个其实和我们上一个 arp-ignore，它的 0 可以说是相对应的。随后它还会有一个 1 和 2。在这里的话我们会采用二二的话术，指它只能够在自己网卡内进行通告，这是它的一个设置。另外它还会有一个级别为一级别。唯一的话是尽量的去避免本网卡和不匹配的目标进行通告，也就是说它会尽可能去避免。但是对于我们写代码的人来说，我们是没有尽可能尽量的这种说法，要么是 1，要么就是21，就是12，就是2，没有这种尽量尽可能这种说法。所以在这里我们只会使用 2 只在本网卡去通告。也就是说我们的响应只在我们的本网卡去进行。 0 的话，那么所有的网卡它都能够接收用户的响应， response 都能够接受盗版，所以 0 的话肯定不适用，所以在这里我们会使用 2 这个东西。


我们如何去进行配置？如何去设置网卡的行为？这里的话那么其实它也会有一段相应的配置，这个配置的话在这里我就直接拷贝了，

```java
# configration for lvs
net.ipv4.conf.all.arp_ignore = 1
net.ipv4.conf.default.arp_ignore = 1
net.ipv4.conf.lo.arp_ignore = 1
net.ipv4.conf.all.arp_announce = 2
net.ipv4.conf.default.arp_announce = 2
net.ipv4.conf.lo.arp_announce = 2
```
