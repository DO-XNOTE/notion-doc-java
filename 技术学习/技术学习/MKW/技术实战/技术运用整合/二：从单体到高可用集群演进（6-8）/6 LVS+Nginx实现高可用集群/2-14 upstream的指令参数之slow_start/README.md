那么上一节课呢，我们是讲了 Max_conns，接下来这节我们来讲一下 slow start，从它的字面意思上我们就能够看得出来，它其实是缓慢的开始，也就是缓慢的启动，那么使用了这样的一个参数了以后，那么呢我们就可以使得我们的服务器慢慢的加入到集群，而不是一下子的进去的，而不会一下子就启动起来，让用户的所有的流量都能够访问到。


因为我们有些服务器的话，它可能会在启动之后，那么有一些额外的配置，我不需要有很多的用户有很大的一个流量进来，所以往往我们会配置一个 slow start，而且使用了它以后，那么也便于我们的运维人员去启动一些监控的软件，可以去观察它流量从少到多的一个过程，从而可以更好的达到一个监控的效果。随后我们就一起来看一下。

那么它可以说当我们设置了一个时间了以后，那么当我们的这个 server 服务器使用了以后，它的一个默认的一个权重，它其实会覆盖掉的。假设它的权重我们为它设置的事实，当我们配置了这个 slow start 以后，它会把它的权重从 0 逐个一直升级到10，它是慢慢去升级的，它会在这一段时间里面去做一个升级，其实也就是会把我们一些不健康的服务器变成健康的服务器，也就是这样的一个启动的过程，那么它是在一段时间以后去进行的一个启动。


OK，那么默认的值的话它是0，那么 slow start 是 disable 的，默认是关闭的。这个参数的话是不能够使用到哈希，那么哈希的话就是 IP hash 以及是 URL hash，这是另外的一个负载均衡，这个我们还没有说，那么这个是不是用的另外随机的 load balance，它其实也是随机的，这种负载均衡它也是不支持的。它是适用于这种 wait 带上权重的，有了权重以后那么它才能够去支持。


OK，另外这个 slow start 它是不适用于单个server，它必须要使用在我们的集群里面，我们在一组 up 群里面，我们要必须要配置至少两个或者两个以上的服务器，那么才能够使用 slow start，包括 Max feels 以及是 fail time out 这两个参数我们会在后面再去说的。这三个参数都是不适用于单个服务器的。那么接下来我们打开命令行，我们可以来测试一下，看一下这个 slow start 这个参数能不能够使用。


那么我们修改一下 imooc.conf，在这个里面我们在它的一个文档里面其实也看到了，它是需要配置权重的，所以我们在这里面可以去配置一下权重。我们第一台配置权重为2，我们来配置高一些，配置为6。然后它的一个 slow start，这个我们直接拷贝一下，直接拷贝贴过来，直接把它可以设置为比方说设置为 60 秒，然后下方那么下方的话这两个这个 Max_conns 我们就去掉，我们都把他们的一个 wait 权重设置为2，然后前面这个注释我们开放一下，那么这样子我们就由三台 tomcat 组成的一个集群，那么其中这一台这台服务它如果说是从一个关闭的状态，那么启动了它加入到这个集群来的时候。那么其实它是一个 slow start，在 60 秒后慢慢的它的一个权重会从 0 慢慢的开始增长。


OK，随后我们来保存一下以后我们来看一下能不能够成功。当我们使用杠 t 命令，然后它会提示说这个参数它是一个不能够被识别的参数。 OK 为什么？那么主要是因为这个参数它其实是在**商业版本的 Nginx plus 里面才能够使用的，在我们使用开源版本的时候，其实这个参数是没有的，那么像有一些公司它会使用到商业版的，那么商业版是需要去付费，那么在商业版本里面就有可能会使用到这样的一个参数，所以大家的话可以去了解一下。那么如果说你手头会有一些正好有这种商业版的一个安吉克斯的话，那么你也可以去尝试着去使用，去测试一下。** OK 在这里的话，那么我们就只是演示一下它的配置是如何去做的。
