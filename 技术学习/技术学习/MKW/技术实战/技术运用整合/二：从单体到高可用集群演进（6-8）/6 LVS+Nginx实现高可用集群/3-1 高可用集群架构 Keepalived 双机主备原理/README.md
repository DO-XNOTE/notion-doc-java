那么前面几节我们是讲了 Nginx 把它部署到了云服务器，并且实现了一个反向代理以及是动静分离。接下来我们来看一下 Nginx 高可用那那么目前我们用户在访问网站的时候，其实是直接访问到 nginx，然后呢，  nginx 下是挂了一个 Tomcat 集群，那么这个是没有任何问题的。


那么 Tomcat 其实我们在集群环境下，那么他们其实是彼此协调。如果说有一台 Tomcat 的宕机了，那么其他的 Tomcat 还是能够继续为用户去服务的，那么这个是没有任何问题。但是如果说这个时候我们的  nginx 这一台它挂掉了怎么办？那么它其实是独立的去部署的，它是一个单节点。那么既然它是一个计算机，它是一台电脑的话，那么它也会存在宕机、死机、停电，电脑硬件损坏，导致我们当前 nginx 这个节点会坏掉，那么坏了以后的话怎么办？那么我们能不能去使用一个备用机去替换它？那么很明显是可以的。

让我们的备用机为用户去提供服务。那么当然用户在访问到备用机了以后，这一台 Nginx 备用机它也是起到了和主节点的 Nginx 一模一样的作用，它也是充当了一个反向代理，负载均衡网关，起到了这些等等的作用。那么用户请求到备用机了以后，那么所有的请求它还是能够继续去访问我们的 Tomcat 集群，那么这个其实就是  nginx 的一个高可用。


在我们部署的时候，我们其实是有两台 nginx，那么两台形成了一个主备关系之后，那么当有一台挂掉了以后，我们另外一台备用机就能够起到一个相应的作用，那么这个其实就是 ha ，也就是 high availability，这就是一个高可用的意思。

他们不能够同时的为用户去提供服务，因为这两台安静克斯的他们的彼此关系是一个主备关系，而不是集群。要区别于   Tomcat 的集群，那么   Tomcat 的集群的话，他们是同时的可以为用户去进行服务。那么我们的  nginx 主备节点的话，那么在同一个时刻只有一台，只有一台节点为用户去提供服务，因为他们是一个主和被这样的一个关系。

那么挂了以后，那么他这个时候他就会接管主节点的服务，它就会运作的，那么也就是说它会和我们的虚拟  ip 绑定到一起，这个时候它会为用户继续去提供服务。那么当我们的主节点修复好了以后，会重新接管以前的任务，以前的职责，备用机就会重新的恢复成为小弟了。那么这个其实就是  keepalive 的双击储备高可用的一个原理。
