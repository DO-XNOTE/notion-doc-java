那么它的话其实是一个非常核心的一部分的内容，这一部分的内容它主要是实现了它底层的一部分的一些通讯协议，它也为其它的一些模块和 Nginx 的进程等等的内容提供了一个运行时的环境。你可以把它理解成是一个 jvm 也可以，那么它是为其它的一些模块其中的一个环境，同时它也会协调其它的各个模块，随后那么它又分离了两个主要的模块，其中一个就是 http ，那么它是一个核心中的一部分。另外一部分就是mail，那么 mail 的话就是和我们邮件相关，但是这个和我们主体的内容没有多大的一个关联。


随后来看一下它各个组成的一个模块，它总共有以下这几部分。首先第一个是一个 events model，那么它是一个世界模块的话，那么我们在之前讲了，其实就是有一个依破，那么这个就是在 Linux 里面所使用的一个默认的事件机制，那么它其实是操作系统层面的一个事件处理机制。最后下一个是一个 phase handler，他的话其实是用于去处理客户端的一些请求处理以后，那么他还要负责去进行一些相应内容的响应，那么内容响应了以后，它其实会经过一个 output filter，它其实是一个过滤器，那么它会过滤掉一部分的内容，把内容再返回到我们的一个浏览器再去做一个响应。


其实我们在之前就已经是接触到了，那么在之前我们是使用了 gzip 去压缩了一些文本内容，那么压缩的过程其实就是一种过滤的过程，它可以把一些大的文本过滤成一些小的文本，再把一些小的文本再显示渲染到咱们的浏览器，那么它就是一个过滤。OK，那么下一个是一个 Upstream 的话，它是一个反向代理模块，它会把真正的请求转发到真实的一个服务器地址，去响应真实的内容。那么和 Upstream 相关的就是一个 


load balance，那么它是一个负载均衡器，它是可以去实现集群，然后去实现负载均衡的一些配置，其中那么它就包含了一些相应的负载均衡的算法。


最后一个就是一个 extend model，它是一个继承模块，其实就相当于是一个第三方的，那么如果说你要去实现一些第三方的一个模块的话，那么就会使用到 extend ，那么这一整块其实就是  Nginx 的模块化的一个体系。
