那么在我们浏览器去输入 IP 的时候，其实它是一个 HTTP 协议，所以它默认的，其实这个是会加上的，那么随后它还会有一个 80 端口，由于 80 端口本身就是默认的，所以在我们的浏览器里面你是可以不用去加的。那么在它的后方这个其实还会有一个斜杠，那么这个斜杠其实就是代表根的意思，就是 root 是这个意思，那么就相当于我们在编写一个 Ctrl 的时候，在 Ctrl 里面就是默认的，其实也可以去加上一个斜杠，那么道理其实都是一样的，它是代表一个根的路由。


那么当我们发起一个请求的时候，其实是请求到我们的一个 Nginx 请求来到服务器，那么在服务器里面的话，那么首先那么它会监听我们的一个 80 端口，它是会进行监听的，那么如果说接听到的话，那么它会把你相应的请求放到或者说找到某一台特定的一个server。找到某个server，那么其实这个 server 就是专门用于去监听 80 这个端口的。那么如果说我们是有多个不同的一个端口，那么在我们的后端，那么也同时会有多个不同的 server 去监听不同的端口的话，那么对应的它是都可以去找到的。


那么找到了这一台 server 以后，那么随后他还会去映们的一个路由，我们现在默认的路由其实是一个斜杠对吧，那么映射到了以后，他就会找到我们的一个 HTML 页面了，那么这个页面其实就是我们的一个首页。OK，那么在这里我们回到咱们的命令行，我们可以去看一下，

好，那么在这里面可以看到这一整个大括号，其实就是代表了一台server，在我们当前配置文件里面就构建了一个服务器，那么这个服务器是什么呢？它是用于去监听我们的 80 端口的。


OK，那么监听 80 端口的同时，它还会有一个 seven name，这个 seven name 其实就是你请求的时候的一个IP，那么我们的 IP 请求的是一个内网IP，那么这个内网 IP 其实你如果说是在 Linux 系统里面的话，你是可以通过 local host 去访问的，因为这个它是做映射去做接听的。它就是一个域名。如果说你现在已经是有一个域名备案好了，并且解释到这一台服务器了，那么同样的在这个地方你可以去写上你的一个域名，比方说你直接可以去写一个 3W 点 m . com，通过这个域名我们就可以找到这一台 server 了，那么随后在这里还会有一个 GNT 的端口，是一个80。那么当我们这两个条件全部符合了以后，那么在下方它会有一个location，那么这个老 k 系的话，其实就是一个映射。


可以看到在这里会有一个斜杠，那么这个斜杠的话它会找到我们的root，就是咱们的根，然后在旁边它会配置了一个 h 切妙，那么这就是一个文件夹，并且它是一个相对文件夹的一个壁纸，它在这个前方是没有斜杠的，所以它是一个相对目录。那么这个相对目录和我们的 config 其实是在同一级目录，我们可以出去看一下，也就是这个 HTML 和我们的 config 都是在同一级目录之下的。好，然后继续。
