在这里我们目前是有两个 work 的进程，可以看到这两个 work 进程其实它默认在这里是一个nobody，都是由 nobody 去进行执行的。咱们的一个主进程其实是由 root 用户去进行一个运行，主要是因为我们在之前讲进程模型的时候，我们的操作者其实也就是操作人。当我们去启动 Nginx 的时候，其实我们是会有一个启动的信号，这个启动的信号其实是发送给了master，那么 master 再把这样的一个信号传递给两个 worker 进程，让 worker 去执行。所以我们 work 进程执行的时候，它其实是一个 nobody 的用户去进行执行的。
