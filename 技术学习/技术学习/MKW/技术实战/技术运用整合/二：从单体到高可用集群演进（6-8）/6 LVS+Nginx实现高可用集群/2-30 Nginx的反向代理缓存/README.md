保存完以后我们再一次再进来，进来以后我们就可以去做一些配置了，那么这个配置的话我们可以配置在这个部位，首先它会有一个命令，那么这个命令它叫做 proxy cache path，是指我们的一个缓存主要去保持的一个位置。我们先写一下 proxy cache，再来一个path，那么它是指我们设置缓存的一个路径的地址。我们在这里加上一个注释，设置缓存保存的目录。好，那么在这里我们就可以去定义了，如何去定义？在这边写一下，我们就保存到我们的 nginx 里面，指定为user，然后 local 再到nginx。随后我们再可以去指定一个新创建的目录，这个目录没有创建，它会帮我们自动去创建的。我们取个名字叫做 up stream 下划线cash，随后我们再来加一个叫做 kiss 下划线15。那么这个是指我们的一个缓存，它可以去使用的一个共享空间。那么既然是一个共享空间的话，其实我们也可以为它去做一个设置，那么在后方加上一个等引号，那么我们要去为这个共享的内存空间去设置一个名字，比方说我在这里就叫做 mycache，然后再来一个冒号，写一个5M。

```shell
# proxy_cache_path设置缓存保存的目录 
#       keys_zone 设置共享内存以及占用的空间大小
#       max_size 设置缓存的大小
#       inactive 设置缓存的超时时间，超过此时间，缓存自动清理
#       use_temp_path 关闭临时目录
proxy_cache_path /usr/local/nginx/upstream_cache/ keys_zone=mycache:5m max_size=1g inactive=30s use_temp_path=off;
```

那么这是什么意思？这个是指我们共享内存这个空间，我们开辟出来的时候，我们使用 5 兆是这意思，然后它的名字是 my cash，然后这个 my cash 在这个下方我们在做配置的时候，我们会用得到的。那么随后再来一个，这边我们加一个注释，把注释给加上，那么这是设置共享内存以及占用的空间大小。好，随后下一个，那么下一个的话它会有一个叫做 Max size 这样的一个命令。


Max size 那么这是什么意思？那么这个是指我们的一个缓存整体的一个大小，我们是可以为它去设置的，比方说我们所有的缓存在这个目录里面，我们叠加起来以后，那么如果说你的一个云服务器的一个硬盘空间非常非常的大，你可以给它设置为一个t，两个 t 都可以，那么在这里我是在虚拟机里面，所以我就为它设置了一个g，这样子就可以了。


那么随后它还会有一个命令，那么我们编写一个命令，编加一个注释，这个是 Max size，在后方补上，这个是设置缓存大小。好，随后下一个，那么下一个的话它有个命令叫做inactive，那么这个是什么意思？其实从它的字面意思上就能够看得出来，它是一个未激活的一个意思。
