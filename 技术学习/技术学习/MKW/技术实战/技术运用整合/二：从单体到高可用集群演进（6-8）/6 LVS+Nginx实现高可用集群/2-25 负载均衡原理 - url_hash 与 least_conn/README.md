上两节我们是讲了负载均衡中的一个 ip hash 以及一致性哈希算法。这一节我们来看一下负载均衡中的 url_hash。从它的字面意思上我们就能够看得出来，它其实也是基于 URL 的一种哈希算法，只不过我们之前是使用的 ip 对 ip 进行了，是在这里我们会使用 URL 对 URL 进行一个，还是随后根据他获得的一个哈希值，我们再分配到不同的 Tomcat 的节点上去。

在我们的后方，我们的上游里面会有 Tomcat 123 有三台根据不同的 URL。首先第一个假设我们有一个请求，就叫做 course/list，去请求课程的企业列表，那么随后还有是一个 user info 去查看用户的信息，另外还有是一个 conn 账户信息。


那么对于这三个请求来讲的话，其实它们都是三个不同的 url，那么它会有一个规则，这个规则的话就是把这个 url 放到这个哈希的方法里面去得到的一个值了以后，再去和我们的节点数去求模求得的一个index，我们就可以指定到不同的一个路由去请求到我们特定的一台 Tomcat 。这个算法其实就是url_hash。使用 url_hash 的话，它会有一种情况，就是比方说假设我们的一个 account 在请求这个路由，在请求这个路径的时候，它可能会有一个大量的请求过来的时候，这台 Tomcat 的三会导致过热。这个时候的话你也可以把这一台 Tomcat 3 替换成一台nginx。这个时候我们的上游服务器里面就是有两台 Tomcat 以及是一台nginx。这一台 nginx 的话，在它的下方你再可以去挂多个 Tomcat ，组成一个基于 URL 的集群也是没有问题的。
