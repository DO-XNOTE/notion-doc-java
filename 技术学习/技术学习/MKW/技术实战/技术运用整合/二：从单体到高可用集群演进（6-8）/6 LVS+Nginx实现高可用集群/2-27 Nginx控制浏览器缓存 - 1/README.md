首先我们的用户会打开浏览器来访问到我们的一些网约内容，我们的网约内容其实是请求到了 Nginx 这一端。网页内容的话，它肯定有一部分的内容是静态资源，静态资源的话是这一部分内容的话是可以缓存到浏览器的。然后用户的请求到达 Nginx 了以后，不仅仅在 Nginx 这一端，它里面会有一些静态的资源文件，那么其实在它的一些上游服务器里面，它也会有可能包含一些静态文件的。这些静态文件是可以被恩迪克斯这一端去缓存的。所以在我们整个过程中，它只是包含了两个部分，第一个部分就是在我们这台在反向代理器的里面，它的一些静态资源文件是可以缓存到用户的。浏览器的另外一块内容就是我们的上游服务器，上游服务器里面如果说它包含了一些静态稳健的话，它也可以被缓存，只不过它是会被缓存到 Nginx 这一端。


对于这两者区别的话，对于浏览器来讲的话，其实浏览器里面用户去进行访问的时候，它其实是会加速单个用户的一个访问，单个用户其实就是当前浏览器的一个访问者，他去访问的时候可以加速他的一个访问。另外对于我们恩尼斯这一端的话，其实是可以优化我们在内网的一个传输，在我们的一个内网环境里面的话，像tomcat、阿帕奇， Nginx 都可以作为 Nginx 的一个上游服务器，它里面会包含一些静态的自由文件。由于我们在内网里面其实我们是分为多个服务器节点去进行部署的，所以在这个时候我们的安迪斯在拿到它里边的一些资源的时候，也会去做一个请求，它会有一个内网带宽的损耗，所以它是需要有一个请求和响应的时间。


如果说我们这一端的一些内容，一些静态文件会被缓存到 Nginx 这一端的话，那么其实就大大的加速了我们的一个访问，我们的内网的一个损耗就不会有那么多了，相应的也就会大大的减少我们用户整个请求的一个响应时间。这个其实就是我们在用户请求的时候，它分为了两个部分的缓存。OK，对于缓存的话，那么我们可以先来看一下。
