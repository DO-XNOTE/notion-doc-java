其实对于我们自己来讲的话，我们是 7* 24 小时可以提供不间断的服务的。但是使用这样的一个双击主备的高可用架构的时候，其实会存在一个问题，也就是说其实我们现在的话会有两个 Nginx  节点，那么它们其实是分别两个云服务器，我们是需要去采购的，单个云服务器的话要好一点的话可能要在十几万二十几万，那么我们要采购两台一模一样的配置的话，它们的价格肯定会很贵。


如果说我们的主和被这两台节点永远都不会发生故障的话，在使用双击主备的这种情况之下，所有的流量，所有的用户请求都会到达我们的 master 节点。我们的备用机备用节点的话，它是永远都不会发生一个作用的，也就是说我们会存在一个资源浪费的一个情况。资源浪费就是我们的一个备用机，它永远都不会处理用户的请求，但是我们每年都要去花费很多的成本，要去续租它的一个服务器的费用。假设每年是十几万，那么你十年下来就可能达到 100 多万，对于成本来讲的话，那么其实公司肯定是需要去考虑的，发生这种情况，那么老板肯定是不愿意去看到的。


那么如果说你自己是公司老板的话，你自己在搭建这样的一个架构的话，那么很明显你也不希望出现这样的一个情况，所以这个资源浪费我们完全是可以去避免的。可以举一个例子，在一个企业里会有老总和副总的话，那么其实他们都会有一些相应的业务，相应的事情要去做。老总挂了以后，那么副总就会暂替老总的位置，会去做一些事情，把相应的业务给拿过来做一个对接，那么其实老总和副总他们都应该要去做事情的。如果说我们按照现有双击储备的一个情况来讲的话，在这个企业里老总会做很多的事情，这个副总他相当于是一个空有职位，空战的职位，不做事情，师卫素餐，那么只有在老总挂了以后，他才会去做事情。老总一直不挂，那么这个副总就一直每天都没有事情干，就空拿着工资，这样子肯定是不好的对吧，但是你又不能够把它给开除，所以这就是双击主备所存在的一个问题，能不能去解决这样的问题，我们能不能够让这两个节点都能够为用户去提供服务，让老总和副总都可以去做事情。

我们来看一下，那么现有会有两个节点，一组一倍。这两个节点都是在 171 和 171 上，分别是两个独立的服系虚拟IP。那么我们肯定也会有虚拟 IP 会和我们的 master 捆绑到一起，它的虚拟 IP 我们定义为161，这个是我们目前所使用到的，然后这个 161 它会使得我们的备用机，把备用机也会放到我们的虚拟 IP 上。只有在我们的 master 挂了以后，这个虚拟 IP 161 会和我们的 172 捆绑到一起。


绑定到一起这是目前的一个双击主备，由于现在要采用一个双主热备，让这两台节点都能够发挥作用的话，那么我们可以这样子，我们可以再建立一个虚拟IP，这个虚拟 IP 的话，我们是一开始和 nginx 的一个备用机和 backup 放到一起，它的一个 IP 我们可以取名为162。也就是我们在一开始初期的时候，我们的 171 和 161 是放在一起去运作的， 172 和 162 又是放到一起去工作，这其实是两组。如果说我们的 nginx backup 这一个 172 节点，那么它发生了故障，这个时候的虚拟 IP 162 所有的请求，那么它其实就会和我们的 171 进行了一个绑定。就是从我们的这张图里面能够看得出来，这两台机子的节点其实是共同的，为用户去服务的，它们之间其实就没有组合被这样的一个关系的。那么它们是一个怎样的关系呢？它们其实就是一个互为主备的关系，它们是相对而言的。我们如果说以这一台 171 为主的话， 171 上的话，它自己这一台它是一个主，相对的 172 就是 171 的一个backup。如果说对于我们的 172 来讲的话，在 172 上 172 他自己就是他自己的一个master， 171 就是 172 的一个backup。这样子的话，它们的关系其实就是一个互为主备的关系。只要其中的一个节点发生了故障，虚拟 IP 就会把相应的流量导向它相对的另外的一个节点上去，OK，那么它是一个这样的关系。


对于我们的虚拟 IP 来讲的话，在最一开始，每一个 IP 它都会对应一个虚拟IP，如果说其中的一台发生了故障，那么它相对的另外一台在它的节点上会同时存在两个虚拟IP，有两个虚拟 IP 的话，那么对于我前端用户来讲的话，那么其实它前端用户请求其实一开始它会经过 DNS，也就是通过 DNS 轮询，它会把请求分发到不同的IP。这个不同的 IP 其实就是不同的虚拟IP。其实我们后面的一个运作的话，对于用户来讲他是透明的，他不用管我们后面有没有什么主备关系，他是不用去理会的，他只要关系到我的请求，只要能够发送到 161 或者 162 上就可以了。我们后面不管是其中哪一台发生了故障，它的虚拟 IP 都会自动的切换到它的备用机上的，这样子其实我们就形成了一个双主热备了。


OK，这个其实就是双组热备的一个原理，我们是通过两个虚拟 IP 分别对应了两组，它其实会有两组路由，这样子的话我们就可以更加高效的利用了我们的 nginx 节点聪熬，就不会造成我们的一个资源的浪费了，这一点是一定要去注意的。在这里也是需要去注意，进来的时候我们的虚拟 IP 其实是通过了一个DNS，那么这个  DNS 的话是在云服务器里面的。OK。
