 Memcache ，那么 Memcache 它和 Redis 都是一样，它们都是 NOSQL，它们都是分布式的缓存中间件，而且他们也都是以 key-value 的形式去存储的，但是 Memcache 的话，它的一个 key-value 的形式相对来说更加的简单一些，它只是支持字符串，但是简单也是它的一个优点。那么使用 Memcache 的话，它的内存使用率也是比较高的。另外它是一个多核处理多线程，这是它的一个优点。那么像 Redis 的话，它是一个单线程的，而且你要去操作使用的话，它其实都是利用的一个单核，那么 Memcash 它是多核，如果说你的服务器比较强劲的话，多核多线程利用起来的话，那么也是它的性能会更加的好。


当然如果说我们使用到 Memcache 的话，它其实也会有一些相应的缺点，比方说它是无法做到一个容灾的，这是什么意思？就是说它的一个数据是不能够去做到一个持久化的，像Redis，它的数据一旦可以去进行一个存储了以后，那么它就可以去配置持久化，它可以去保存起来的。


如果说我们的计算机节点发生了一些问题，要重启了，它宕机了，那么我们的数据在重启以后还是可以去恢复的，所以这一点的话是能开始的一个缺点。所以就是因为这样的一个问题的话，往往我们使用 Redis 使用的一个场景会比 Memcache 更好，也是更高的。

比方说第一个是单线程，那么它是单核区运行的，在存储大数据量的时候，那么它的一个性能的话是会有所降低的，但是小数据量的话，它会比 Memcache 更加的好。那么 Redis 的话它是一个单线程的话，也就是单台服务器，它无法能够充分利用多核服务器的一种，这个 CPU 利用不了全部。这个其实也是作者他是设置为了单线程，那么作者认为的话，使用单线程会更加的方便，更加的简单，多线程的话更加容易出错。这就是这个作者他所考虑的一个地方。而且在使用到 Redis 的时候，他也是建议我们事实的一种多实例的方式去使用的。


Redis 总的来说的话就是在大型的系统在存在有缓存共享，分布式部署，缓存内容庞大的时候，我们会使用到Redis，那么 Redis 它不仅仅是可以做缓存，还能够做存储，因为它是可以去持久化的。那么 Memcache 的话，它们只能够去用来做缓存。


Redis 的数据结构要比 Memcache 更加的丰富。在有时候去面试的时候，可能会有面试官会问你， Redis 和 Memcache 这两者的一个区别主要是什么？这个时候你就可以去回答一个持久化，因为持久化的话 Redis 支持， Memcache 不支持。还有一个就是数据结构，数据结构 Redis 更加丰富， Memcache 就比较简单，它只能够去存储简单的单一的 String 类型。OK？
