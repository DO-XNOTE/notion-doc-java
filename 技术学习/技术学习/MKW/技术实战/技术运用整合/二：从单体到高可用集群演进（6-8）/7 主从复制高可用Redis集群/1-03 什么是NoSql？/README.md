那么这一节课我们一起来看一下什么是 NOSQL。因为 Redis 的话，其实它本身是归类为 NOSQL 的，所以我们先得了解一下什么是 NOSQL。其实在 NOSQL 之前，其实我们使用的都是一些关系型数据库，像 MySQL , Oracle , SQLserver 这些其实都是的。那么 NOSQL 它的全称叫做 not only SQL，也就是说它不仅仅是SQL，它除了做一些数据方面的存储以外，它还会有其他的一些功能。

那么对于一些传统项目来讲的话，传统项目那么它使用的都是一些普通的关系型数据库就够用了。那么像 jsp 发送请求到 Controller 了，再到service，再到 DAO 层，那么这样子的话就可以和数据库去进行一个交互，这个时候它的访问量不会很大，也没有什么并发。所以传统项目往往会使用一个纯数据库就可以了，他不会去使用什么 NOSQL 的。


那么随着移动互联网还有是大数据和云计算的一个发展，普通的一个关系型数据库就不再适合把超大量的数据提供出去，做一个并发型的查询。那么这些数据的话，比方说有用户信息，聊天信息，地理位置信息，购物信息等等。因为数据库它单表的性能是有限的，如果说你做的好的话，你的一个优化以后能够达到 600 到 700 万的样子，那么普通一点的话可能就只能到达 4 到 500 的样子。但是对于一般来说， DBA 如果说在单表达到 300 万左右的时候，你就应该要考虑到后续的一个海量数据，那么你就要去做到一定的优化的准备了。那么一旦数据量庞大了以后，那么普通的数据库就不再适合于去做一些深挖，比方说像数据挖掘，大数据分析等等。所以这个时候 NOSQL 就横空出世了，因为它不仅仅是普通的SQL。


NOSQL 的一个横向扩展是非常的方便的，那么也可以称之为是水平扩展，因为在构建集群的时候，我们可以加节点，加机子就可以了，其实也可以理解为是  tomcat 集群，那样加一个节点就能够扩容，比普通的数据库要方便得很多。那么普通的数据库的话，像平时我们要创建表，创建字段，而 NOSQL 的话，它是一个舰只对的话，其实就类似于哈士map，那么这个就是 Redis 的这种形式，那么它的存储的形式数据结构是非常的简单的，所以 NOSQL 的一个可扩展能力也是非常的高的。此外，由于是键值对的一种形式，那么增加和删除缓存的话，其实能够做到非常的高效。相反数据库中如果说数据量达到上百万的话，那么增加和删除会显得它的一个性能会比较的低效。而且如果说我们是针对字段的增加或者说是删除的话，那么也是非常低效的，甚至你也可以说它是一个灾难性的，而且对于整体的数据会有很大的影响。
