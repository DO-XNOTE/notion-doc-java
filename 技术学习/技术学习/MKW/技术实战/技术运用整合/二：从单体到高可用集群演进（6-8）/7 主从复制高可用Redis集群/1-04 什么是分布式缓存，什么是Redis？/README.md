在高并发的场景下，通过分布式缓存我们是可以去提升数据的读取速度的，那么通过提升读取的速度，其实相应的我们整个整体的系统的性能也是随着提升了，那么在这里我们是需要去注意的。我们在这里所说的是一个读取，它是一个读操作，那么读操作我们是可以从缓存里面去读的。那么写操作的话，往往我们是指的是入库，是需要把数据写入到数据库的，这个也是遵循的一个二八原则。之前我们也是说了，因为在我们的互联网的环境下， 70%- 80% 的情况都是一个独操作，那么当我们的一个系统垂直的去进行拆分了以后，其实我们就需要使用缓存来提升整个整体的系统的一个毒性能，那么这个是可以归类为分布式计算领域的，所以这也就是称之为是一个分布式的缓存。


既然是读操作的一个技能提升了，那么相应的我们就能够为数据库的查询压力降低不少，节约数据库的开支，因为数据库是整个项目的一个瓶颈的所在。

所以一旦我们的数据被缓存了，那么在整个分布式系统里面，任意的服务器节点都能够拿到这个缓存中的数据，快速的可以去响应用户的一个需求。那么缓存的话它是基于内存的，可以是存到内存里面去的，那么内存的一个读取的效率以及是并发性的话，肯定是远远的要大于硬盘存储的数据库的，所以对于高并发的设计来讲的话，缓存是不可或缺的。那么它是可以让大多数的请求去命中缓存，而少量的请求在有需求的时候才会去访问到数据库，从而就可以增加提高我们整个系统的一个并发量，这样子也能够减少数据库的一个压力，为它进行一个降压。所以说我们这一个分布式缓存，它在分布式零计算领域的话，都是得到了广泛的使用，发光发热。
