那么首先我们会有一个客户端，这个客户端在我们之前其实使用的是一个 Redis client 这样的一个工具去连接的 Redis server。如果说我们后续再整合到 Java 里面去的话，在 Java 里面其实也会提供相应的客户端的。随后我们会有一个 Redis server，这个其实就是我们的一个Redis，它整个服务在启动以后，它是会有一个进程的。


在我们的 Redis 里面，在内部它其实会有两个东西，首先一个它会有一个多路复用器，这个我们上节课已经是介绍过了，它是非阻塞的一个模型，那么随后它其实还会有一个文件事件分配器，它专门是用于去分配一些事件的。在这个下面那么它会分为有三个不同的处理器，分别来看一下。


首先有一个连接应答处理器，随后是一个命令请求处理器，还有是一个命令回复处理器。如何去理解？首先我们先来看一个连接应答处理器。连接应答处理器的话，它的一个主要作用是要和我们的客户端去保持一个链接，像我们的一个 ready server，一旦启动了以后，那么其实我们就会有 read 这样的一个事件，和我们的连接应答器会捆绑到一起。它的全称的话其实叫做aedable，你就可以把它理解为是一种标志，它会有这样的一个事件，那么这个事件是会和我们的连接应答处理器捆绑到一起的。OK。


最后当我们的一个客户端和咱们的 server 要去建立连接的时候，这个其实也就是我们在一开始在命令行工具里面敲下了一个 Redis client，那么一开始的话肯定是需要和我们的 server 去建立连接，建立连接的时候，那么它其实就会发送一个如意的标志，其实就是一个 readable 事件。


那么这个时候的话，在我们的 Redis server 里面，它其实会有一个叫做 server socket，那么 server socket 和我们的一个客户端 socket 其实是对应的，它们是属于网络编程里面的一块内容，它们之间是一个 socket 的通信，在我们接触到 read 这样的一个事件了以后，随后我们就会交由咱们的多路复用序去进行处理。交给他去处理以后的话，其实它是一个非阻塞的，拿到了以后一旦接收它，就会把它放到我们这样的一个箭头里面去。


这个箭头的话，在这边其实我们可以称之为它是一个管道pipeline，或者我们也可以把它称之为是一个队列，它会往这里面丢进去以后，这个事件其实就会到达我们的文件事件分配器。这个分配器当它识别到它是一个 read 这样的事件以后，它就会和我们的这个连接应答处理器去做到一个匹配，也就是交由它去进行一个处理，它是一个read，相互进行一个匹配。这个时候的话其实就可以表明咱们的 client 和 server 这两端就是建立了一个连接。建立好连接了以后，这个 read 标识的话，这个事件它其实就会交由给我们的命令请求处理器。这个命令请求处理器的话，你就可以认为它是专门去处理请求的，也就是一个request，然后命令回复处理器，你可以把它理解为是一个有boss，也就是一个响应。


OK，那么随后我们在客户端可能要去，比方说我们要去 set 一个值对吧？ set 一个值的话，比方说 set name imock 这样子的话，其实它是一个命令，这个命令的话它的一个事件类型其实也是一个read，随后经过 server socket 再丢给多路复用器，拿到以后放到咱们的队列里面去，在交由文件事件分配系。

那么这个文件事件分配器拿到以后，它会进行一个判断，它会判断匹配是如意的事件，它这个时候是一个如意的事件的时候，就会让我们的命令请求处理器去处理咱们的命令，那么他就会去识别了，他会去识别当前就是一个 set name imok，所以他就要去做一个处理。他要把我们用户所设置的一个内容，把这个键值做一个存储，存储到咱们的内存里面去，那么这个其实就是一个命令请求的处理，就是一个request。


OK，当它处理完毕以后，随后的话它会分配一个 ，也就是写的一个标识，那么这个写的标识的话，在这边的话，其实我们就可以把它作为响应。因为我们的一个请求其实在处理完毕之后，那么在我们输入完毕一个命令以后，你可能会看到一个 OK 对吧。这个 OK 的话，其实就相当于是我们的一个命令回复处理器回写给我们的一个内容，所以它会用到一个  写的一个标记OK。那么这样子我们的一个写的标记其实是会和我们命令回复处理器是捆在一起的。 white 的话，那么其实它的全称是叫做 AE_Readable 这样的一个事件类型的。


OK。好。随后那么在我们的客户端这个地方，其实我们就需要去做一个回写，也就是OK，或者说我们在查询list，我们要展示 list 里面所有的内容的时候，那么它其实是回写的一个情况，我们要把内容显示在控制台的下方，那么它是一个 white 这样的一个事件类型。随后交由我们的多路复用器再丢给咱们的一个队列，让这个队列分配给我们的一个文件事件分配器，那么这个时候会匹配咱们的 white 事件，那么 white 事件是匹配到了。


随后的话，我们的命令回复处理器就会做一个回写，它会把我们的 OK 或者说是我们的一个获得的一个 list 数量， list 里面的内容等等，只要是一些需要展示的内容，它都会是作为一个response，就是把这个响应的内容回写给我们的一个客户端，在客户端上进行一个展示。
