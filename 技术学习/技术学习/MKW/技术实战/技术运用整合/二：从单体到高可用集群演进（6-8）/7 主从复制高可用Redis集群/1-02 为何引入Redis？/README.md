通过浏览器把请求发到咱们的后端，是通过 LVS 去接收的。接收掉请求了以后，它会分发给  Nginx , LVS 在这里它是一个 4 层的负载， Nginx 它是 7 层的负载。当我们的 LVS 如果说发生问题宕机了以后，那么其实我们也是搭建了一个备用机，它们是一个主备的关系，可以使用 keepalived 的去搭建一个双机主备，或者说是双主热备都是没有问题的。随后在我们的 engines 下方，它每一个 engines 其实都是挂了一个 Tom kit 集群，这样子的话整个集群就是 4 层和 7 层的一个融合，又称之为是一个多级负载。


这是我们用户的请求，所有的请求进来之后，不管是分发到哪一个节点，最终他们都会流向到咱们的数据库。数据库的话是用于存储数据，用于提供给用户去读取的。所以这个时候其实数据库是我们整个系统的最后一道门，那么是不能够被攻破的，是需要有一个屏障来保护它。


在这里是可以举一个例子，如果说现在我们在使用微博，那么微博上假设有一个流量明星，那么他表达说他结婚了，那么往往这个时候微博就可能会很拥堵，甚至是瘫痪。那么这是为什么呢？主要是因为上亿的流量涌入到了这个系统去查询，这个数据库肯定会崩掉，所以要引入缓存来提高数据库的一个性能。因为明星发布状态以后，后续的大多都是用户的读操作，写操作的话不会很多，那么这是一个二八原则， 80% 都是读请求。对于淘宝购物来讲的话，大多数情况下，那么你去查看的都是一些商品的详情，购买的动作其实是会相当少的，所以它是 20% 和 80% 的一个比例。


那么在咱们的这个图里面，其实我们可以在这个数据库的前方加一个屏障，这样子的话我们可以把一部分的数据前置，那么这样子用户的请求就不会访问到数据库了，他们就会访问这个屏障。那么这个屏障的话，在这里其实我们就会使用到分布式缓存Redis，这个 Redis 的话是可以存储一部分的数据，这样子的话当用户再一次的发送请求进来了以后，所有的流量就不会直接接触到我们的数据库，它会访问到Redis，因为 Redis 它是一个屏障，它相当于是一个防火墙，它可以保护咱们的数据库。那么这样子的话其实就提高了咱们数据库的一个读取的性能。


当然它的一个数据库性能提高了以后，相对来说咱们整个系统的性能也就提高了，而且它的一个吞吐量并发都会跟着上来，那么这个其实就是分布式缓存的一个作用，那么在我们这里的话，我们就会使用到Redis，

我们使用动物园里面的动物和管理员来做一个举例，那么首先有一个动物园的管理员，那么此外他管理着一个动物园。


动物园里面有大象，狮子，老虎，猴子，长颈鹿，吃牛，熊猫，还有是孔雀。在这个动物园里面的话，每一个动物它肯定会有不同的类型，或者说是不同的一个比例，它有多少只对吧。那么每一只动物它的一个现在的一个状态，它是幼年还是成年，他们的一个身高比例是多少，其实都是一些数据。那么假设现在有一个游客进来，这个游客会问我们的管理员说，现在我们的园区里面有多少个大象，那么这个时候这个动物园的管理员可能就会到这个动物园里面去溜达一圈，去数一下咱们的大象有多少台，数完了以后他再会告知到我们的一个客户，跟他说我们这个大象可能有个 10 只。


那么随后又有一个新的客户又过来了，他会问，那么现在我们动物园里面有多少只长颈鹿，长颈鹿的身高有多少？这个时候这个动物园的管理员，他又一次的会进来吧，这个时候他进来的时候还是会再去溜达一圈，再去数一下。可以试着去想一下，如果说我们的客流量很多，很多的客流量，很多的客户都会来询问咱们的管理员，这个时候我们的管理员其实就会很累，它能够支持的一个客流量不会很多，所以相应的我们整个系统它的一个并发就不会很大，因为他会累垮。那么这个时候我们能不能提供一个类似于账本的东西，举个例子，在这里我们可以提供一个类似于一个本子，或者说是账本的东西，这个管理员当他去查询完毕，溜达完毕咱们的动物园以后，把相关动物园找到的数据预先的可以记录在我们这个本子里面。那么这样子我们可以通过这个本子再一次的把数据返回，提供给咱们的一个客户。


那么这样子，如果说我们后续以后再有相应的客户来询问咱们的管理员，说咱们的一些数据是怎样，我们的一个动物是什么样的情况，那么这个管理员他就不需要再去溜达咱们的动物园了，这个时候这边的请求和响应就没有了，这个管理员会直接拿出一个笔记，掏出账本，他会从我们的账本上直接去读取我们相应的动物的数据，把动物的数据再一次的反馈给咱们的客户。


这样子的话我们这个管理员他能够接待的客流量就增多了，其实相应的咱们整个系统的并发性能就提升了，那么这是一个道理。可能会有同学会问，那么如果说我们现在的一些动物，它肯定会有一些新生儿为或者我们的一些新生儿会成长，这些数据的话，如何去及时的更新到我们的这个账本里面去。其实在我们整个动物园里面肯定会有一些维护人员，如果说有维护人员去进行维护的发现，比方说我们的狮子新增加了 2 个幼师，然后我们的熊猫以前是幼年的，现在成长了，另外它也长高了，所以相应的数据不就变化了吗？这个时候维护人员会把这些更新的数据直接会替换掉我们这个账本里面原先的数据。替换掉了以后，那么我们在这个账本里面的数据就变成了一个新的数据。那么如此以来，我们客户再一次询问咱们的管理员去拿到相应的数据的时候，这个时候它的数据它的缓存就得到了更新。这样子的话，其实我们的这个整个从我们这个举例的图里面就能够看得出来。


当我们使用到了 Redis 以后，分布式缓存其实就可以为我们提供这样的一个数据库的屏障，从而可以提高提升咱们整个系统的性能，提升咱们的吞吐量以及是一个用户的并发。OK？
