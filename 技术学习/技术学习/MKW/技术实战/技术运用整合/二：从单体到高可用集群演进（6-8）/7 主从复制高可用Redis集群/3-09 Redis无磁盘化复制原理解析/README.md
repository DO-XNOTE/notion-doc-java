那么其实它还会有另外的一种方式，叫做无磁盘化复制，那么它是没有硬盘的复制过程的，也就是说它不再会去触碰硬盘，那么呢，它其实会通过它自己的一个内存，就是说它 master 里面的自己的一个内存，它会从内存写入到另外的一个内存，在这里是可以看到它其实是内存之间的传输，那么这种传输的话其实是属于一个 socket 的方式去进行的一个传输，那么它其实也就是基于网络的。


那么为什么会有这样的一种方式，那么其实主要是因为它的一个服务器的一个磁盘的类型，那么其实也我们也是知道，就说云服务器的话，它是由硬件组成的，其中就包含了一个磁盘，也就是你的硬盘。那么硬盘的话它分为很多种类型，那么最常见的话就是 SSD 一台硬盘，另外就是一些普通的机械硬盘，那么如果说你是使用的是最普通的机械硬盘的话，那么它的磁盘读写是相当的低下的，所以它就会出现一个无磁盘化的复制，那么这样子就避免去进行磁盘的读写，从而提高它的一个效率。那么也就是说你的一个磁盘，你的一个硬盘类型是普通的机械硬盘的话，那么它其实它的一个 IO 就比较低。另外如果说你的一个内网的带宽，它的一个网速非常高的话，那么你就可以采用无磁盘化复制的这种方式来替换掉我们之前的一种方式，那么也是可以的。这种方式的话它默认是关闭的，你可以去在它的配置文件里面去做一个开启，开启了以后那么就可以去使它的无磁盘化复制的。
