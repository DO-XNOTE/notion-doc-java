那么前面我们是一起学习了主从复制以及是哨兵，那么通过实现哨兵或者组成的话，其实我们是可以去提高毒的并发，但是单个 master 它的容量是有限的，那么如果说它的数据达到一定程度的话，那么肯定是有一定的瓶颈，所以在这个时候那么我们就可以通过水平扩展的形式来把它扩展成多个 master slave，成为了一个多主多从。这样子其实它们就是一个集群了。

那么在这一局的话，我们一开始就来一起学习一下什么是比亚迪斯集群，也就是 Redis cluster，那么它是可以支撑多个 master slave 的，那么它也能够去支撑海量数据的一个存储，实现高并发以及是高可用哨兵模式的话，那么它其实本身也是一种集群，它是能够提高读请求的并发。


但是容错方面可能会是有一些小问题，比方说 master 在进行同步数据给 slave 的时候，这个时候它其实是异步复制的，假设我们的在复制的过程中，这个 master 它挂掉了，那么 slave 上的数据肯定就没有 master 上的新。当我们的一个数据同步的时候，数据同步肯定是有一定的时间的延迟的，可能会有个几百毫秒，或者说是 1- 2 秒，那么在这一段时间内的数据就可能会被丢失。那么当我们老的 master 重新的恢复成一个 slave 以后，那么新的数据就会丢失。所以这个就其实就是在哨兵或者说是储存模式中所存在的一个小问题，
