展开来看到目前其实我们在刚刚的首页里面，其实可以看得出来，它分为了 3 个类别。第一个是一级大分类，它就相当于是一个root，是一个根分类，随后是一个二级分类，再到一个最小的三级分类。


当然，如果一个网站做得非常非常复杂，再往下分，它可能会有四级、五级分类，会有很多。但是在我们这里，我们从一级、二级到三级，有三级分类就已经是绰绰有余了。好，随后继续下就是一个负ID。负 ID 是什么意思？负 ID 其实就是指我们的每一个类别，它会有一个上级分类。比方我现在是一个大分类，大分类它就是一个root，它是一个根分类，它是没有负 ID 的。对于我们 root 类别的下一个子类别，它是会有对应的负 ID 的，它其实本质上查询每一个类别的关联都是根据负 ID 去进行关联的。


OK，我们会在后面看一下数据库的里面的真实的数据。以真实的数据为例，我们就能够理解负 2D 的一个作用了。随后下方会有一个图标，这个图标是什么？来看一下。在这里图标是指的是一个logo，我们在网页里面其实是可以看得出来的。在这个部位可以看到每一个分类的左侧都会有很小的一个图片，这个小的图片就可以让用户更加形象的看到当前的一个分类是一个什么内容，这样子其实用户的体验会比较高。好。最后下一个是一个口号slogan，这个口号 slogan 又是一个什么东西？这个我们就要看到咱们页面的下方了。页面往下面滑，可以看到每一个分类的旁边会有一句话，这个其实就是它的slogan。再往下我们可以缩小一些c、s、 s 样式，放大之后会有一点小变化。可以看到下一个分类是饼干，它会有这样的一句话OK，我们可以再往下面看，是和肉相关的。这些文字的简短的描述就是一个 slogan 口号。


再往下看是一个分类图和背景颜色。分类图和背景颜色其实也就是和我们下方这些每次向下滑相关的。它的图片下方背景色就是对应在我们数据库设置里面的背景颜色了。OK，这一块内容就是我们的商品分类，商品分类我们要去实现，我们该怎么去做？我们是不是应该在这个地方首页一次性把所有的类别全部都查询出来去显示？很显然这种做法不是很好，因为其实可以看得出来，我们在一开始页面加载的时候，我们没有必要去加载那么多的数据。在前期一开始加载，我们只需要把所有的大分类全部都展示就可以了。如果用户后续再把鼠标移过来，我们可以进行一个懒加载。


根据当前鼠标所选择的某一个 root 分类，就是最大的分类。根据他的 ID 再去查询这个分类下所有的子类别，再把子类别查询出来，随后在中间进行一个渲染和展示。以这种方式做，用户的体验是非常好的，而且在首页加载的时候也是非常快的，这一点我们可以参考一下。


淘宝，我们把淘宝打开一下刷新。在淘宝的其实和我们也是类似的，都是一些分类，而且都是一些大分类，是 root 分类。把鼠标移过来之后，在中间会有一些子类别进行相应的展示。OK，其实和我们也是差不多的，我们可以打开F12，把鼠标移过来，移到network，这个是网络好，我们清空一下。随后来一个刷新，现在我们放大一下，好点击清空。在这个下方其实是网络的一些连接，有请求在这边都会有一个展示。我把鼠标移过来，你会发现我鼠标第一次移过来的时候，在下方加载了很多的一些内容，这个也是在用户把鼠标移过来之后，它才会进行的一个加载。我鼠标移开，我鼠标再移过来，你会发现它没有再加载了。也就是它只会加载一次，第二次再去加载是没有任何效果的，因为他已经把相应的内容在中间的部位进行了一个展示了。


OK，我可以把清空鼠标再移过来，你会发现没有任何的加载了。当然，如果我把鼠标再移到上方家电，它又会有数据进行加载了。所以它淘宝的分类其实是一种懒加载。OK，好。对于我们来讲，其实我们也是可以这样子做的，并且在我们的首页其实也是这样子去实现的。我们可以按 F12 打开到network，我们刷新一下，放大，把这一块清空，刷新一下，鼠标移过来，你会发现进行了一次加载。鼠标再移过来，没有任何的加载。鼠标往下移，移到顶杆，你会发现它会进行了一次加载。OK，懒加载的这种方式，我们会使用懒加载的方式去实现我们的一个分类的显示。


OK，好。关于这一块内容，我们可以先到咱们的 index Ctrl 里面，我们先把它的一个基本需求可以先去写一下。可以理一下。我们的一个基本的思路就是首页分类展示需求。第一个是查询大分类，应该是第一次刷新主页查询大分类，渲染展示到首页。第二步，如果鼠标上移到大分类，这大分类其实就是一级分类，或者是根分类都可以，则加载其子分类的内容，如果已经存在子分类，则不需要加载括号，这个就是懒加载模式。OK，好，这个就是我们的一个基本需求，我们下一节就会根据需求来完成我们的功能。OK。
