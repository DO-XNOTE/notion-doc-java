虽然如此，在我们的订单状态里面，其实还会有很多的一零状态，代表订单一直处于是等待支付。相应的，如果用户一直不去进行支付， 10 的状态会越来越多。但是 10 又不是非中泰的订单状态，所以我们要把 10 改为终态的状态，也就是订单关闭。OK，订单关闭如何去实现？我们就可以去使用定时任务实现它的一个订单超期的自动关闭。对于一个定时任务来讲，其实 springboot 就为我们提供了一个 task 或者是一个 job 的概念，我们可以去通过配置就可以实现它的一个定时任务了。所以这一节我们就一起来看一下如何在 spring mode 里面去构建一个定时任务。打开我们开发工具，在在 configure 里面，我们去创建一个类，这个类的名称就叫做 order task，或者你可以把它取名为 order job。其实都可以好，因为它是一个job，是一个任务，所以它需要被 spring 容器给扫拿到。我们要加上一个注解，叫做component，这是一个组件的意思。最后我们在这里面去定义一个方法 public boys or two close 自动关闭订单。


好，写了这个方法以后，既然是要去执行定时任务，所以相应的我们应该要为他添加任务的一个注解。也有一个 schedule 的，我们可以进去源码，在源码里面可以往下面翻。看到有一个 c r o n，这个就是一个表达式，它默认是为空的。在表达式里面该如何去写？其实可以看得出来它有一个example，这个就是它的一个表达式，它里面其实涉及到的是一些时分，秒，年月日，还有星期。相应的其实你都可以去设置的。如何去设置？在这里加上一个括号c，把 c r o n 写上冒号。里面可以写上相应的表达式。
