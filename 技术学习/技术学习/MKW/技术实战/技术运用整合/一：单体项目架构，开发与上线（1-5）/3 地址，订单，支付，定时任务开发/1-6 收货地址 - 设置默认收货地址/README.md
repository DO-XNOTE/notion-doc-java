关于收货地址的几个功能增删、改查。我们在前面几节其实都已经是讲了，接下来这一节我们要讲一下。最后一个就是设为默认。设为默认其实就是把之前的一个默认的地址改为非默认的，选中的地址要改为一个默认地址。在这里我们可以先看一下前端。在前端其实有一个方法叫做 set default address，传入的一个 address ID，随后会发起一个 post 请求，这是它的路由地址传入的参数和删除是一样的，一个是用户的ID，另外一个就是对应的 address ID。


好，现在我们就可以来编写一下我们的后端了。还是一样，我们先要写一下service，拷贝一下，改一下。把，改成 update user address，处 b 储备default，OK，注释改掉。这是修改默认地址传入的参数，这是不变的。好，我们来实现一下。事物不要忘记要加上使用required。在这里面其实会分为两个步骤。第一个步骤首先是要查找默认地址，设置为不默认，这是第一步。第二步是根据地址 ID 修改为默认的地址。这两个操作我们先来做第一步。第一步是要根据当前的一个用户 ID 以及是否为 default 这两个属性去进行查找。我们还是一样使用 pojo  来进行搜索。


user address query address 定一下，用一下传入两个参数。第一个参数是 set 一个 user ID，第二个是 is default。在这里我们直接设定为 1 就可以了。来一个 yes or no，这是我们定义的枚举，应该是 yes 点 type 好，这两个参数是我们的条件好。


随后我们就可以去发起一个搜索的请求，通过 map 点SELECT，这里面其实可以 select one，也可以使用select，这个也可以。一个是返回的是list，另外一个是返回的一个单独的一个对象。其实这两个都可以。我们使用 select 好了，把对象给放进去，他查询出来的就是一个 list User address，它是一个list。好在这里我们就可以通过一个 for 循环，或者可以通过一个 list 点 get 0 也可以了。但是我们在这里我们直接使用一个 for 循环好了。这么做的目的主要是为了以防万一。可能我们数据库里面会出现一些数据的紊乱，可能会一个用户的默认的地址可能会有多个，以防万一。出现这种情况，我们可以通过一个 for 循环，就可以把其他的一些默认的设置为非默认的地址就可以了。
