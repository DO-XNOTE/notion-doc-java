前面两节课我们是讲了订单的流程，订单的状态以及订单的相关表设计。这节课我们来讲一下支付中心。


**支付中聘其实就是老师我所做的一个封装的接口，把支付宝和微信支付的接口封装到了一个项目里面去提供给大家去使用的。**


我们先来看一下用户这个时候要去做支付，要做支付，他会直接去调用第三方的支付平台。支付其实就是我们自己开发者的一个服务器，我们自己的服务端，它是和第三方的支付平台去进行一个交互的，它有请求也会有响应。但是支付如果你要去进行对接，它其实必须要有一个前提，你要有一个企业的资质，你没有企业资质，你是不可以去进行调用的。所以我才会在这里做了一个聚合的一支付中心，也就是在支付和微信支付宝中间的支付中心。它是一个独立的项目，也是一个独立的工程，是可以去进行部署的。请求是可以发送给支付中心，再发送到第三方支付。它包含了微信和支付宝里面的一些调用的接口。

OK，在这里是有请求，有请求也会有响应。微信和支付宝收到我们支付中心过来的一些请求之后，他是会把结果响应给我们的。支付中心的响应给他了以后，再会响应到我们自己的开发者服务端，也就是最终他再会以一个页面的形式，把成功的一个结果展示给我们的用户去看。


如果你没有支付中心，按照正常流程，你是可以在自己的项目里面去集成支付宝的 SDK 或者微信的SDK，在自己的项目里面可以直接去接收到第三方支付的一个响应，中间是没有支付中心的。OK，其实在我们的一个课程里面，本身会涉及到这种支付，支付我们都是真实的交易，是会真正的去扣款的，而不是直接在数据库里面把订单他的状态直接改成已付款。要对接微信和支付宝支付是必须要使用企业资质，我们一定要去强调一下，个人是肯定不能够去对接的，哪怕你去使用一些类似于沙盒的这种应用，其实也是不能够做到一个在真正的生产环境上去使用的。所以老师也是为了保证大家学习的一个流畅度以及支付的真实度，我们在这里就提供了。


一个包装好的支付中心，而且企业资质的，大家没有企业资质其实都是没有关系的，可以使用老师提供的接口，这些接口都是被封装在支付中心里的，这样大家就可以无阻碍的去学习的。不过毕竟是企业的资源，它是。
