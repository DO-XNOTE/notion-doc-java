我们可以从这图里面先看到。首先我们有用户会访问咱们的浏览器，随后请求才会到达我们的后端。在我们的后端里面，这是一个单体项目，并且它是一个外包。在包里面它包含了一些相应的内容，比方像 server IT，也就是 MBC 的这一层架构的后端代码。当然也包含一部分前端代码，比方像 js, css 以及是 html，但是 html 其实是 jsp，它是由 jsp 渲染而来的，它会渲染成 html。但是它的渲染的过程其实是在我们的后端去做的。也就是只要有用户来访问我们的服务器，所有用户请求后所得到的页面，它全部都是在服务器去进行一个渲染的。无疑这对于我们的扶持来讲，其实是有一定的压力的。


如果我们在前期一开始用户量比较少的情况下，其实基本上是没有什么问题。一旦我们的用户越来越多，比方到达了上万百万的时候，我们所有的页面都在服务器去渲染，这对于我们的服饰会造成非常非常大的性能影响，是需要去注意。在我们传统做 Java web 开发的时候，我们浏览器里面请求到所有的页面都是通过 URL 去进行跳转的。我们再来看一下前后端单页面的一种交互，这个其实也就是 MVVM 的开发模式，是在前端的。首先用户会访问我们的浏览器，当然浏览器会请求到我们的后端。在这里面我们后端所有的一些内容其实都是以一个接口形式所存在的。但是浏览器所访问到的内容都是一些静态页面，这些静态页面其实都是一些 H5 的页面，这些内容这些静态资源全部都在我们的一个静态资源服务器，也就是Nginx。在早期可能会用到阿帕奇服饰，现如今我们都会使用非常主流的反向代理服饰，也就是Nginx。


我们的用户访问到我们的页面以后，这些页面会发起一些请求，这些请求就是用户请求，它是会请求我们的后端。这个时候其实可以看到我们的 server 的这一端，也就是我们所说的 MVC 模式架构。后端代码其实全部都是在另外一台服务器，这台服务器其实就是一个 Tomcat 静态资源，和我们的后台的一些动态代码，其实就做了一层分割分开来。这个其实就是一个前后端分离的一种开发模式。


在我们的前端和后端进行交互的时候，它们之间其实是通过 restful 这样的一种请求，其实就是 restful web service。通过这种方式去请求到之后，会获得一些相应的数据，它们之间的一个数据交互形式，全部都是以 JSON 的形式去进行交互的。


OK，这个是用户通过浏览器去进行访问。如果在陷入厅，其实我们很多用户在使用手机的时候，手机端也是非常主流的一个客户端，我们会有一些页面，如果我们的页面兼容H5，其实也能够在手机浏览器上去进行打开的。所以我们手机也是可以通过 H5 的形式去访问到我们 Nginx 里面所有的一些静态页面的。


随后还有一种方式是手机APP，不管你是小程序、 iOS 还是安卓，只要在我们手机端上有相应的客户端软件，我们也可以发起请求来访问到我们的服务器的。来看一下iOS、安卓和小程序都有。其实手机端和我们服务器之间的通信，它也是通过 restful 去进行通信，交互也是通过JSON，都是这样子，一模一样的数据类型这样子。其实不管我们的客户端的样式有多种，它是有多样化的。其实对于我们后端来讲，后端只是有一套代码去维护就可以了，不需要复制多份去进行开发，我们只需要去针对不同的客户端去做一些接口上的微调就可以了。好。其实我们使用前后端分离其实有很多的好处的。这样子做我们可以使得我们的分工明确，前端和后端开发人员，他们的效率可以提升一个档次，使得各自领域的开发人员他们术业有专攻，并且其实我们的团队也是解耦了，前端归前端，后端归后端。不同团队所关注的内容都是不一样的，后端代码只需要一套就可以了。团队是可以进行并行开发的。如果我们换成之前的传统开发模式，所有人都是耦合在一起，并且我们是相互阻塞的，我们必须完成上一个接口，才能够对接下一个页面。


在前后端分离的这种开发模式里面，其实也有一些小问题，比方沟通，因为我们的团队多了，人也多了，我们一定会少不了沟通。另外，我们在发布的时候，其实我们部署其实也是各自发布的，稍微会有一些复杂，需要做到版本的一些统一。相应。其实既然我们的前后端也分离了，如果我们会产出一些BUG，我们也可以精准定位，理清我们的边界，有些 bug 是前端的，就是前端也不会踢给后端，就不会造成这种踢皮球的现象。另外就是需要注意，我们所有的一些静态资源文件，全部都是部署在 Nginx 里面的。


这种方式其实就是一个前后端分离模式。我们再可以强调一下，前后端分离不仅它是一种开发模式，也是一种架构模式，可以称之为是前后端分离架构。使用前后端分离的时候，我们是需要去注意，我们的前端和后端项目是要区分开来的，前端和后端的项目是两个不同的项目，放在两个不同的服务器，并且各自也需要去独立的去部署，并且都有两套不同的不一样的代码库，而且会有两个不同的开发团队共同的去进行开发和维护。从我们的图里面其实也是可以看出来这是两个服务器，有两套代码。这个就是我们所需要去理解的前后端分离开发模式。
