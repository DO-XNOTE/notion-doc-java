上一节课给大家介绍了数据库设计工具 PDMan，我相信大家也去下载了，进行了一个使用。这节课我们接着来看我们在这里我们可以看到它会有一个模型的版本，点一下了以后其实在里面定位，其实就是你每次做的一个修改，它会有一个记录，只要你同步以后你每次的版本都会在有一个相应的展示，其实它就是类似于一个版本控制的性质，和你在使用s，p、 n 或者是吉特也是类似的。


在这里我们可以先来去修改一下咱们的这张表，我们双击一下。比方我们在这里面再来新增一个，新增一个用户名 username，对于旁边来讲，它的一个类型默认的是一个字符串。后面的一些等等的设置我们就不去管了，直接 control s 保存一下。随后我们再来到模型版本，点击以后，在这边这个时候会看到，在这里原先这一个点是黄色的，现在出现了一个红色，他会说当前的内容和上一个版本上一个版本它的内容是有一个变化的，但是这个版本我们还没有保存。随后我们来看一下，在这里有一个保存新版本，我们在这里可以去进行一个点击保存一下，这个时候它的版本号就累加 1 了。


零点二这边的版本描述是来一个可以随便写，其实新增用户名字段，点击确定，这个时候保存了。保存完毕之后再来看一下，它会提示这个时候的版本目前它并没有同步，所以我们是可以去进行一个同步的。如何去同步？点击一下以后可以看到这边会有一个同步，我们只需要去点击。以后，它就会把我们的一个可执行的 SQL 脚本直接放到我们的数据库里面去进行执行了。


在这里面其实也可以看出来它是修改我们的用户表，只不过我们在之前创建用户表的时候，我们的表名是中文的，我们并没有把表名使用它的一个英文。其实表明无所谓。在这里我们仅仅只是用于去做一个演示。可以看到我们是修改这张表，添加了一个column，叫做 username。随后用户名的类型是 varchar 32。它有一个comment，也就是为它增加了一个备注，叫做用户名。OK，它的位置是 after reversing，在它的一个锁的后面第二个字段。好，我们在这里点击同步点一下，点击了以后会提示我们的数据库同步成功。随后我们可以回到咱们的数据库，在这里面我们可以来刷新一下。好，这个时候我们来放大看一下。我们可以看到这个时候在这个下方新增加了一个 username 这样的字段了。


OK，这个其实我们可以通过咱们的 PDMan 所做的一个同步的设置，在这里它还会有一个任意版本的比较，这个其实就是不同版本之间为我们留下的一个记录。每次你做完修改以后，我们都是可以去查看的。点击一下会有一个初始版本和增量版本，这两个可以各自去选择，当然我们现在版本比较少，如果版本多是可以任意切换去查看的。选择了两个不同的版本了，以后我们可以来点击比较，点击一下，在这里他就会把这两次他们的差异在这里就会进行一个展示。这个就是我们刚刚所看到的一个 SQL 的一个脚本。OK，这个就是一个比较好。我们再来看一下。目前其实可以看得出来我们脚本的一个迭代，或者我们每次数据库的一个迭代，其实它都属于是一种增量的设计，它仅仅只是针对于我们数据库某一张表里面的一些字段做了相应的一些修改，其实这是一种增量的迭代。


它还有一种方式来看一下，我们在这边可以点击同步配置，其实我们默认就是一个字段的增量，字段的增量其实也是为了我们上了生产以后，如果我们的项目发布了，相应的在生产环境里面也会有数据库的。数据库它很庞大，我们是不可能人工的，手动的通过数据库工具去修改相应的字段的。我们全部都是通过这种增量方式，也就是通过每次的一个可执行的脚本，直接放到扶持里面去运行，去修改我们的数据库表达。这种方式就是数据库字段增量的一种模式。

这个时候我们来看一下，点击当前这个版本，你会可以看到在我们的脚本里面，由于我们现在使用的一种方式是全量的，其实我们选择的一种模式，他刚刚其实我们看到了，我们先关掉。它是称之为重建数据表，是数据库，其实对于我们的一般的数据来讲，属于全量增量，这个是全量，注意一下，点击看一下。


在这里面可以看得出来，首先它是会创建这样的一个表，对把这张表进行了一个删除，这个其实就是全量的一种方式。另外它还会有一个音色感，它其实相当于是一个临时表，他会先创建一个临时表，把我们当前所有的数据放到临时表里面去，放进去了以后，我们再重新去创建一个新的表，在新的表里面又去把我们相应的一些字段属性给放进去。这就是一个全量的设置。


其实会有一个弊端，假设我们当前这一张数据库表，它有上百万条的数据，我们在执行操作的时候是非常慢的。因为你先要去把所有的数据放到一张临时表，你再查询出来，放到一张新的表里面去，它的耗时是十分久的，所以这一点一定要注意。我们在这里讲一下，对于我们来讲，我们所有的一种同步方式，全部都是以字段增量的方式去做的。OK，当我们点击的切换以后，现在是由全量变为了增量。刚刚其实可以看得出来，在这里面我们是修改这张表， job 把 update time 把更新的时间字段给删除了。OK，这个我们这节课所需要去注意的。一个增量和全量的设计在我们企业里面开发也是需要去注意的，尤其是我。当我们的数据库发生了变化，我们要去使用到我们的一个生产环境的数据库的时候，一定要注意，必须要使用增量的脚本去执行的。
