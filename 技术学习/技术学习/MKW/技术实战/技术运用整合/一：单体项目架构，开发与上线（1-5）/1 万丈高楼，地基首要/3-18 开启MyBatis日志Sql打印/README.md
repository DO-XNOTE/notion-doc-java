我们是不需要去写 SQL 语句的。我们前面是通过日志来监控了 service 执行的时间，在这里我们能不能让日志来打印一下。我们每一次调用数据库的一个 SQL 语句，其实也是提供了相应的功能的，我们是可以开启这样的功能，在每一次我们进行调试的时候，**把相应的 SQL 语句是可以打印出来的，但是这一点是需要去注意，仅仅只适合我们在开发环境的时候把相应的 SQL 语句打印出来。**


***如果我们在生产环境上，你要把所有的 SQL 语句去打印出来，日志会非常的多，会非常的冗余，也是没有必要的，因为用户量一旦大，请求会增多。很多的重复性的 SQL 语句都会打印在空气台。所以我们仅仅只会在测试或者开发环境去进行相应的一个 SQL 日志的输出***。所以我们来看一下如何去进行输出。
