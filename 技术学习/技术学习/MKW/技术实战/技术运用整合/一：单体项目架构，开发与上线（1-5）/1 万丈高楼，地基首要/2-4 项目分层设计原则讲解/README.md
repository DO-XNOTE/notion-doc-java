假设现在我们有一个工厂，这个工厂是专门去生产汽车的，但是生产汽车的时候，在汽车在生产出来之前，它其实会有很多的组装的构成。我们来看一下。假设我们现在有很多的一些部件，我们有轮子，我们也有一些底盘，还有是我们的安全座椅，另外方向盘，车内音箱，蓄电池及是发动机。当然还有很多的一些组件，这些组件对于一个汽车厂商来讲，它不太可能会在自己的工厂里面去生产这么多的一些组件，它其实是会把每一个部件外包给其他的公司去做的。在我自己公司里面，我可能只会生产 1 到两样，甚至是只有几个东西，随后在我这里进行一个整合，也就是进行一个组装。也就是在我们生产之前，我们会先拆分，把我们所有的组件一个包出去，让其他的一些外包商去进行开发，开发完毕之后再到我自己，最终我再进行一个聚合，一个整合。


整合成一台汽车以后，它就是一个非常不错的产品，我们可以把它流入市场去进行一些相应的销售。这个其实就是在我们一开始前期先进行拆分，最后再进行一个聚合的目的，它的一个好处其实就是把我们每个组件进行了一个公用化。比方现在我们要生产三种不同的汽车，比方有一个吉普大型诺基，要是吉普自由光以及是自由侠。在这个时候假设我们的轮子其实都是一样的，这个时候我们只需要向同一个厂商去订购所有的轮子就可以了，我就不需要额外的重复的再去造轮子去生产了，拿过来直接去组装就 OK 了。


其实我们在开发项目的时候，和我们在生产汽车其实是差不多道理。在一开始我们要先把我们的一些业务模块等等间距进行一个拆分，去进行一个分层，最终在我们去运行的时候去做一个聚合，道理都是一样的，这样子就可以降低我们的耦合度，也能够实现我们各个组件的共用化。

通过 Maven 去进行一个项目的聚合，其实就是把我们项目去进行拆分了。首先一个我们最终要去运行的一个包，其实它是一个外包或者是一个架包，它其实是通过很多的一些零碎的组件去进行聚合的。比方在我们的一个项目里面，我们会有一个 common common，它里面都是一些比较通用的方法，工具类、枚举等等都会在我们的 common 里面，它是单独的一个包，我们可以称之为它是一个子模块或者是一个子工程。外面的这一整个框其实就是我们最终所要去聚合的一个场所，它其实是一个Pom， Pom 是我们的一个聚合工程，所有子模块都是在我们的聚合工程里面的。这是第一个，是一个common。随后我们会有一些portal，像一些和我们实体类相关的，比方有entity，bo，vo 和数据层交互。我们通过数据库表逆向生成的一些内容。一些实体类，我们统称把它当做是一个Pojo，它其实也是一个包。它们两个，一个是 Pojo，一个是 common，都是不同意义上的一种组件。


随后再来看我们的map。 map 其实就是数据层，或者它也是一个 dao 层。因为我们会使用Mybetis，所以我们在这里定义的时候，我们就定义为 Mapper 了。如果你使用的是像 jpa 或者是 Havena 等其他的一些 19 层框架，在这里可以把它定义为是一个 DAO 道层，也可以。


随后再来看我们的业务层，也就是 service service。它当然也可以独立成是一个价包，是一个子工程，一个子组件。当然还会有 control 的，这个是接受我们的请求去处理我们请求。我们还有更多的一些共用的组件，都可以让我们的 Maven 去进行拆分和聚合都是没有问题的。在我们的整个项目开发过程中，其实在我们前期我们也会通过每本去进行拆分和聚合的，通过每稳的分层，其实可以使得我们项目层次会更加的清晰。当然其实在我们后端编程的时候，也会导致我们的一个数据层、业务层以及是控制层，它的一个工作会极度细化。这样子可能会有 3 个不同的后端开发人员去进行开发和维护，也就是俗称的一个拧螺丝。当然在一些中小型初创公司，肯定这三层是由同一个工程师去进行开发和维护的。


那么多个不同的项目，其实是可以通过每本的依赖去实现代码的公用化，这样子就可以不必去重复的去造轮子了。就像打个比方，现在可能我们有一个 ERP 项目，还有一个 CMS 项目，这两个项目的时候，在开发的时候，他们可能会有一些共用的加包。他们只需要去引入这样的一个 common 的依赖，就可以使得我们没有必要再重复的去编写额外的代码了。这个就是一个组件，它的规模化。如果你正在一个传统企业里面进行一些相应的项目开发的时候，可能会出现一种情况，同一份代码可能会出现在多个不同的项目里面。当这样的一份代码要去进行修改，你可能要去修改多个项目，没有去很好的使用。利用每晚去进行项目的拆分和聚合。如果一旦你使用这种分层的思想了，以后，相应的共用化的组件就可以相互的去进行依赖了。你只需要去修改一个地方，就可以同时被多个不同的项目去进行调用。这个就是分层和聚合的一个目的。


简单的小结一下，每晚是可以通过拆分项目为多个不同的子模块，可以按需让其他的项目去进行依赖。不同的子模块聚合在一起，最终就是一个可以去运行的项目。并且它们可以使用不同的组件相互不同的去进行依赖的。
