这节我们就带着大家一起使用 idea 开发工具来构建一个 Maven 的聚合工程。首先点击这里 create new project，创建一个新的工程。打开以后在这里面我们的一个左侧选项，我们是需要把眉本去选定的，因为我们是通过眉本去构建咱们的项目的。随后在这里是有一个JDK，当你的开发工具安装完毕，配置好以后，相应的 SDK 其实都会有，在这里面可以使用 1. 8 就可以了。下方是一些相应的美文的模板，我们不用去管，在这里是不用去打勾的。随后在下方会有一个next，直接点击就可以了。随后我们可以到这里来看一下 group ID，直接填上 com 点 imooc 可就可以了。


最后下方一个其实就是我们的一个项目名，我们的项目是和吃有关的，所以这样子福迪杠 d v 名称根据自己的情况直接去命名就行。版本号这个 version 自定义，使用它默认的 1. 0 也就行了。下一步继续，我们在这里点击 next 可以看一下会有一个项目名，在直接把横杠再贴过来。下方的一个保存地址，自己可以去选择一下。当我们选择完以后，我们把窗口可以缩小一点，在下方这里的 next 就是我们点击 finish 点击一下。这个时候可以来看一下我们的项目其实已经是生成了。这个就是我们目前的一个项目。这个是它的一个 Pom 文件，在 Pom 文件里面其实我们是需要去注意的，因为我们在上节课，我们已经是说了，我们要去构建一个聚合工程，这个聚合工程在外层，在它的最外部其实是一个顶级工程。这个顶级工程其实它有一个项目的类型，来看一下。


其实在美文里面会有一个 packaging 这样的一个标签，这个是指我们的一个打包方式，由于它是一个聚合的项目，在最外层的，所以我们要给它来一个命名，叫做Pom。其实 packaging 它其实有三种方式，一个是Pom，还有 Jar ，其次还有 War ，总共是三种。在这里我们使用 Pom 就可以了。这样子我们来看一下，现在我们目前最外层这样的一个结构其实就已经是有了。


这个其实就是我们的一个在最外层的聚合的工程，但是在我们聚合工程里面，它其实内部还并没有其他的子工程，也就是它的一些组件。其实现在还并没有按照我们在上一节课的 PPT 里面的一个项的内容，比方像common，Pojo，还有是数据层 map control 的 service 等等，我们会按照一定的顺序，按照他们的依赖一步一步去进行构建。所以我们第一个我们应该要构建一个common。


如何去构建，我们是需要去选中我们项目的，在这里鼠标选中我们的项目，右键来你有一个什么。在这里可以看到它会有一个module，这个其实就是 Maven 里面模块的一个意思，我们通俗点讲，它是属于我们当前项目的一个子工程，按照每本的说法，它就是一个 module 模块，点击一下以后你会发现它的弹出来的窗口其实和我们之前在构建，在创建项目的时候其实是一样的。在这里的选择以及的 JDK 选择都是一样。直接点击 next 到下一步。在这个地方你会看到在这里它会有一个parent，这是多的一个，就是指它有一个父工程。我们的 foodie-dev 其实就是我们的一个父工程了。


当前我们所需要去固定的属于它的一个子工程，它们的关系其实是一种继承的关系。在这里我们是需要为我们当前的一个子工程或者是子模块去定义它的一个名称，取个名字叫做 foodie 杠DV。再来一个杠common，大家在自己构建的时候， DV 可以去掉的，在这边我是为了进行一个区分，所以我是加上了一个DV。再点击next，下一步在这里它的项目名或者是模块名，它会自动把横杠给过滤掉，重新加进去就可以了。再点击 finish，来看一下，随后在我们的项目里面，其实它就会自动的把这样的一个新增加的模块放到我们的聚合工程里面。我们一起来看一下它的一个结构，我们可以在这里点击之后按一个星号，它会自动的展开，可以看到在展开了以后，在这里其实会多一个福迪。


-dev 杠 common 这个就是我们新创建的一个子工程，这个子工程里面的结构其实可以看得出来它里面的内容和我们在外部的副工程，其实它们的结构都是类似的，只不过它的 Pom 文件。我们来看一下。它的 Pom 文件里面它的默认的一个 packaging，当它没有去写，它默认的打包方式，其实就是一个架把。我们可以在这边写一下，它的一个默认打包方式，如果你不写也其实也是可以的。默认打包方式为价的形式。在这里我们可以把直接注释掉。我们是需要去注意所有的依赖，所有的组件，只要是一个子模块，它们都是互相依赖的，它们都是作为一个架包存在于我们的顶级工程里面的，这一点是需要去注意的。



好。随后我们再回到我们副工程里面，也就是我们聚合项目的 POM 文件，我们来打开看一下，在这个里面大家可以发现原先是没有一个module，现在它其实多了一个module。这个指是指所有的子模块，子工程或者是一些组件，只要你是依赖了我们的一个父工程，相应的所有的模块其实全部都会在我们顶级工程的 POM 里面出现。比方 common 就是我们刚刚所构建的，一旦你构建了它就会在这里出现一个module，也就是所有的 module 其实全部都会出现在这里，后面它会一个一个的往下面去进行排列的，这一点是需要去注意的。


OK 现在其实对于我们来讲，我们最初的一个聚合工程，以及是它所包含的一个子工程，其实我们就已经是构建好了的目录结构，其实也都有。随后我们在我们加上一些相应的注释，我们来简单的来说一下。首先一个在聚合工程里可以分为顶级项目。顶级项目其实我们也是称之为是一个顶级工程，或者是父工程与子工程，这两者的关系其实就是父子继承这样的关系。子工程在每本里称之为模块，也就是模9。模块之间是平级的，是可以相互依赖的。好，这是第一点。


第二点，子模块是可以直接去使用复工程里面的一些资源的，可以使用顶级工程里所有的资源。资源其实就是一些相应的依赖，其实依赖是可以去使用的。子模块之间如果要使用资源，必须构建依赖，其实就是把坐标放到你所想要去引用的模块里面去就可以了。其实就是一个构建关系好。

第三点，一个顶级工程是可以包含很多不同的一些子工程，是可以由多个不同的子工程共同组合而成。这样子它最终形成了一个可以运行的项目。好，现在我们第一部分，我们的一个顶级工程，以及是我们的第一个子工程，现在就已经是构建好了。
