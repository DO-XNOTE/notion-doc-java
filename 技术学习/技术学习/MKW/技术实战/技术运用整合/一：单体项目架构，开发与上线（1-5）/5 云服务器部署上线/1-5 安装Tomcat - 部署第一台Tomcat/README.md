另外一个就是咱们的数据库，我们会部署 MariaDB，那么这两个 Tomcat 以及是一个数据库，我们其实都会去做一个相应的部署。那么对于数据的话，那么用户他首先会先访问到我们第一台 tomcat，随后在这一台 Tomcat 的里面，由于我们的前端会发起一些请求到后端，所以数据会请求到我们第二台 tomcat，那么随后我们要去响应一些相应的数据，所以请求又会落到 MariaDB。那么拿到一些数据了以后，相应的我们会把这些数据再反馈到前端，由前端的页面去做一个相应的确认。那么从这张图里面就可以看得出来，其实我们这三端两个 tomcat 以及是我们的数据库，它们之间其实都可以做到一个相互通信的一个效果。那么对于我们部署来讲的话，其实我们就应该要把这三端部署在不同的服务器节点上。那么举一个例子，第一台 tomcat 我们会部署在 192.168.1.81 这一台服务器节点，那么第二台我们会部署在 82 上，那么数据库我们会部署在 91 上。那么这三个 IP 都是不同的IP，都是分别对应着不同的三台服务器节点。


那么要达到一个数据互通的一个效果的话，我们前面也说了，我们必须要保证我们的内网是要互通的，也就是我们要相互 ping 通，只有相互拼通了，那么我们才能够让每一关都可以达到一个数据流转，数据互通的效果，那么这一点是一定要去注意的。
