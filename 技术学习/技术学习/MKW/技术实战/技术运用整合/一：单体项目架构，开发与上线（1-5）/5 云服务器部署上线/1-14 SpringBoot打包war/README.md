哦，万事俱备，只欠打包，这一节的话，我们就一起来进行一个打包。关于 spring boot，它其实它默认的打包方式是一个价包的形式，因为在我们的  pom  文件里面，我们打开看一下，在 pom 文件这个部分，它其实是会有一个package，它默认其实是一个价包，那么你可以不用去写，你不用去写的话它默认的话它就是一个价包。那么在我们打包的时候，我们在这里我们会使用哇包的形式去进行一个打包，那么在这边我们可以去写一下，打包方式有两种，第一种是一个价包，第二种是一个外包。


那么我们在这里为什么要使用外包呢？我们先来看价包，价包的话，那么它其实是一种服务化的概念，那么在后续课程里面，后续的话大家会接触到微服务 spring cloud，后续接触 spring cloud，等到大家接触到 spring cloud 以后，那么大家我在进行部署的时候，我们每一个包都会打成架包。所有的服务打包都是以价的形式存在的。


OK，war包的话，那么它其实是一个应用程序的概念，那么既然是应用程序的话，它其实里面是可以包含页面的，但是在我们的项目里面，我们并没有去包含页面。所以外包的话它也可以作为一个接口，也可以作为一个服务对外去进行提供，也可以向外提供服务和接口。OK，那么所以我们在这里我们会使用哇包的形式去进行一个打包，因为毕竟在我们一开始初创型的小公司，我们都会以一个单体的形式去做，单体的话一个外包就够了，等到做到微服务化了以后，进行我们的业务拆分，那么再进行打包部署了。所有的一些服务都会打成一个价包。


那么要打成外包的话，我们需要有一些相应的步骤去做的，因为我们是不可以直接我们直接在这个里面去改，是改不了的，我们都可以这样子，我们先把这个拷贝一份，原来的先注释掉，把这个改成一个War。你把这个改成War的话，它是不可以直接去进行打包成袜的，你打包了以后，那么也是不生效的，所以我们要去做一些额外的配置。我们先来写一下，加一个注释，这是第一步打包。
