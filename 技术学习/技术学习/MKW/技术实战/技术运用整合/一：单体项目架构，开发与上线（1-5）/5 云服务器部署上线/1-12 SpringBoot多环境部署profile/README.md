那么第一个就是我们技术团队在进行编码的时候的一个环境，我们会称之为是一个开发环境，那么缩写就叫做 d b，其实就是一个development。那么第二个是一个测试环境，那么这个环境往往是我们在发布之前，我们会提交一个版本，让测试人员去进行一些代码的测试，那么这个环境我们会称之为是一个test，是一个测试环境。


那么随后还会有一个生产环境，这个生产环境的话就是把我们的一些所有的一些代码就去以后打包发布到我们的生产环境，提供给我们的用户去使用了，那么这个环境我们会称之为是一个生产环境，那么它其实是一个prod 就是一个product，是一个生产环境，是真正的用于去生产的。那么在有一些细分的公司的话，可能会在测试环境和生产环境之间还会有一个预发布环境，那么这个我们在这边就直接省略了，我们呢就讲这三个就可以了。


那么我们在做一个多环境部署的时候，其实我们往往会涉及到一个 URL 的更改，比方说像我们本地使用的数据库和测试环境，以及是生产环境使用的数据库的地址都是不一样的，那么所以我们如果说要去做一个环境发布部署的时候，往往我们要手动的去更改我们的 URL 地址，这样子其实是会非常的麻烦。如果我们有 100 个URL，那么你是需要去修改 100 次，很显然这样子去做的话是不太好，所以 Spring 它会为我们提供一个 profile 多环境的支持。那么使用了多环境支持以后，那么我们很简单的就可以通过激活不同的环境来使用不同环境里面的一些配置就可以了。
