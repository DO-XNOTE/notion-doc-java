# 5．Redis的持久化操作有哪些？

Redis提供了两种持久化机制 - RDB(Redis Database) 和 AOF(Append Only File)。

RDB 是Redis的内存快照,会周期性的根据配置保存一次内存数据集到磁盘。它的优点是备份速度快,且体积小。缺点是数据可能会丢失最近一次快照后的更新。

AOF 是把所有对数据库执行的写操作命令记录下来,在Redis重启的时候就可以重放这些命令来恢复原始的数据。AOF 的优点是可以最大程度保证数据不丢失,缺点是文件会变得很大。

在实际使用中,通常两者结合使用:

- 使用RDB进行定期备份,保证可以进行快速恢复。
- 使用AOF记录每次写操作,以便在崩溃后快速恢复最近的更新。
- 通过配置混合持久化的策略,让AOF在进行重写时生成快照,替代RDB。
此外,还可以Enabled AOF 持久化但不激活文件重写,这样就只会追加操作而不会重写,减少文件占用空间而又保证数据安全。
