# 3．什么是CAS？CAS的本质是什么？什么是原子性？Compare and S wap两步，为什么能保证原子性（涉及到CPU指令）？

CAS(Compare And Swap) 是一种保证原子性的算法技术。

CAS 的本质是利用 CPU 的比较并交换(Compare And Swap) 指令实现的。这个指令可以让 CPU 原子地执行读-改-写一个内存位置的操作。

CAS 算法包含三个操作数:

1. 需要读写的内存位置 V
1. 进行比较的预期旧值 A
1. 拟写入的新值 B
当且仅当 V 的值等于 A 时,CAS 通过原子操作将 V 的值改为 B,否则不会执行任何操作。

CAS 的两步操作:

1. 比较(Compare):将内存位置 V 的值与预期旧值 A 进行比较。
1. 交换(Swap):如果 V 的值与 A 相等,则将 V 的值更新为新值 B。
这两个操作由 CPU 的比较并交换指令原子地执行。因此,CAS 可以保证操作的原子性。

当多个线程同时使用 CAS 修改同一个值时,只有其中一个会成功,其它都会失败,从而避免了冲突。这使得 CAS 可以在不使用锁的情况下实现原子操作。

所以 CAS 的关键在于利用 CPU 的原子指令来保证其操作的原子性。


请问CAS在并发控制中的应用场景有哪些？

CAS在并发控制中的主要应用场景有:

1. 实现原子计数器。利用CAS可原子地增加或减少计数值,解决多线程环境下计数值累加的问题。
1. 实现并发队列。比如构建基于链表的阻塞队列,利用CAS原子性地修改队头和队尾指针。
1. 实现线程安全的collection。比如ConcurrentHashMap通过CAS实现修改元素的原子性,保证并发读写 safety。
1. 实现无锁数据结构。比如引用计数法中原子地增加和减少引用数,解除GC难题。
1. 实现乐观锁。比如版本号方式的乐观锁,CAS保证原子性更新版本号和数据。
1. 实现并发算法组件。比如背后的并发工具类AtomicInteger、AtomicReference等广泛用CAS实现。
1. 实现非阻塞同步组件。比如非阻塞队列、事件广播器等通过CAS实现引用转移,避免锁竞争。
1. 实现并发编程模式。比如线程安全的单例模式、工厂模式等,利用CAS避免同步开销。
所以总体来说,CAS通过原子指令提供了一种无锁编程范式,在并发控制中有很广泛的应用前景。

你能给我解释一下CAS是如何实现原子操作的吗？

CAS可以实现原子操作,主要原理是:

1. CAS操作涉及到三个参数:内存地址、旧值和新值。它先读取该内存地址的值,将其与预期的旧值做比较。
1. 如果两者相同,CPU会替换新值,否则不做任何操作。这个过程是原子的,因为它是一个CPU指令。
1. 在执行CAS指令的过程中,CPU会锁定总线,阻止其他任何CPU对这个内存地址进行修改,从而保证了这个“比较-替换”操作的原子性。
1. 其他CPU请求访问这个内存地址时,会被CAS操作占用的CPU锁定,只能在CAS完成后才能访问,这就防止了多个CPU在CAS指令执行期间并发地修改同一个内存地址。
1. 如果比较失败,CAS返回失败表示期待值与实际值不匹配;如果成功,返回成功表示已成功替换为新值。
1. 所以通过一个CPU原子指令实现了“读取-比较-修改内存”这一整体原子操作,在操作过程中防止了其他线程的干扰。
总之,CAS之所以能够实现原子操作,关键在于它是一个硬件层面的原子指令,在执行这一系列操作的时候,能够防止其他操作打断或插入。这就给并发编程提供了一个很好的同步机制。

除了CAS，还有其他方法可以实现原子操作吗？

是的,除了CAS外,还有一些其他方法可以实现原子操作:

- 互斥锁:利用互斥锁保证只有一个线程同时进入临界区,可以实现对共享数据的原子操作。但是充分利用cpu需要避免锁定开销。
- 可重入锁:特殊的互斥锁,允许同一线程多次获取锁,用于保护嵌套代码的原子性。
- 信号量:通过定义多线程间的同步点,控制同时访问资源的线程数量,实现对资源的并发访问限制。
- 读写锁:区分读锁和写锁,允许多个读锁持有但只允许一个写锁,更高效但较复杂。
- 原子引用类型:如AtomicInteger等java内置的原子类,通过CAS实现对基本类型的原子操作。
- 无锁算法:如参考计数法实现对象的原子引用计数等。
- 线程同步操作:如java中的synchronized方法和代码块。
- 悲观锁:加大粒度的独占锁,但代价高。
- 非阻塞算法:如乐观锁、无锁集合等利用CAS等无锁操作实现同步。
所以总体上,除CAS外,互斥锁、读写锁、信号量、原子类都可以实现不同程度和粒度的原子操作,但性能和复杂度各不相同。

cas 的代码举例

这里有一个使用CAS实现线程安全计数器的代码示例:

```java
import java.util.concurrent.atomic.AtomicInteger;

public class CasCounter {

    private AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        int current;
        int next;
        do {
            current = count.get();
            next = current + 1;
        } while(!count.compareAndSet(current, next));
    }

    public int getCount() {
        return count.get();
    }
}


```

主要步骤:

1. 使用AtomicInteger声明并初始化一个可原子更新的计数器count
1. increment方法使用CAS操作实现计数累加:
1. getCount直接返回原子计数器的当前值
1. 在多线程环境下,每个线程对increment的调用都能原子完成+1操作,避免计数错乱
这个示例体现了CAS如何用循环CAS操作实现复杂的原子修改,这里是加1但也可以扩展到其他修改场景。只要保证循环CAS的预期值和预期更新值一致性即可。

所以CAS提供了一种无锁编程思想,基于硬件原子指令实现高效的并发控制。
