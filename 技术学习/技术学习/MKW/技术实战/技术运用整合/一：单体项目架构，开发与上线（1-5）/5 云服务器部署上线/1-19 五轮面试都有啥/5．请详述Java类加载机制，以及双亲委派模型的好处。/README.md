# 5．请详述Java类加载机制，以及双亲委派模型的好处。

Java类加载分为三个阶段:加载、链接和初始化。

加载阶段会通过类的全限定名找到class文件,加载到内存生成一个Class对象。

链接阶段会进行验证、准备和解析三个步骤:

验证:确保加载的Class符合JVM规范,没有安全方面的问题。

准备:为static变量分配内存并设置默认初始值。

解析:将符号引用转换为直接引用。

初始化阶段会初始化类变量和静态代码块。


Java采用的是双亲委派模型来进行类加载。其好处主要有:

1. 避免类的重复加载
双亲委派模型确保了任何一个给定的类都只会被加载一次,存在多个类加载器时不会重复加载。

1. 保证Java核心API的稳定性
Bootstrap ClassLoader作为双亲委派模型的最顶层类加载器,用于加载Java核心库,保证了Java核心API的稳定性。

1. 不同类加载器可以实现相同类的不同命名空间隔离
类加载是由类加载器和类的全限定名共同决定的,所以不同的类加载器可以加载相同名称的类,实现隔离。

1. 保证Java安全稳定运行
非法的类无法被加载,保证了java的安全稳定运行。



双亲委派模型(Parent Delegation Model)是一种在面向对象编程中用于实现继承的模型。

其主要思想是:每个对象都可以有一个父对象(Parent),对象可以委派父对象来响应访问请求。当访问对象的属性或方法时,如果对象自身不存在该属性或方法,它就可以将请求委派给父对象处理。父对象再递归地将请求委派自己的父对象,直到请求被处理或抛出错误。

双亲委派模型的主要特点包括:

- 子对象可以通过委派扩展父对象的功能,实现继承关系。
- 相对于类继承,更加灵活动态。子对象可以在运行时改变父对象。
- 查找属性/方法时会遍历父对象链,形成就近 resolution 机制。
- 父对象可以有多个子对象,实现共享继承。
双亲委派模型常应用于基于原型的语言如 JavaScript、Self、Io 等,是实现继承的一种常见方式。相对于类继承,它更加灵活,子对象可以动态改变继承链。
