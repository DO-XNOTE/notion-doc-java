大家好，我是大木。前面我们一直在探讨如何利用索引提升查询效率，但是同学们务必知道，索引也不是万能药，它也是有开销的。开销主要是增删改的时候索引的维护开销。索引越多，维护开销也越大。所以在条件允许的情况下，应该尽量少创建一些索引。


这节课我们来探讨针对重复索引、冗余索引以及未使用的索引该怎么样处理。这其中存在一些坑，如果处理不当的话，甚至可能会导致灾难。所以这节课的内容还是非常重要的。

order by 字句无法使用索引，对吧？这是为什么？这是因为对于 index(from_data)，它是一个非主键索引，而非主键索引，它会在 b 加去的叶子节点存储主键值。也就是前面说的非主键索引会先去查询主键是什么。于是 index(from_data)这样的一个索引，某种意义上来说，某种意义上来说，就相当于是， index(from_data ,emp_no)。因此这种情况下，我们的 order by 子句也可以使用索引。


而为什么 index(from_data ,to_date) 反而使用不了索引呢？这是因为， index from date to date 某种意义上来说，就相当于是， index from date to date emp_no。而现在我们的 where 条件里面并没有 to date，也就是把 to_date 给跳过了，只使用了 

from_date 以及 emp_no，不符合最左前缀原则。所以 order by 子句没有办法使用索引。
