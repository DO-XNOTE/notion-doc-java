比方现在我们创建了一个组合索引，作用在name、age、 sex 这三个字段上面。会有哪些限制？

首先，当查询条件不包括最左列的时候，是没有办法使用索引的。在这个例子里面，最左列就是 name 字段，如果你的查询条件不包括 name 字段，就没有办法使用索引。

比方说这里 where age 等于5， and sex 等于1，没有办法使用索引。


第二，查询条件不能跳过索引中的列。比方这里我们使用 name 等于大木 and sex 等于32，跳过了 a 级字段。这个时候我们只能使用到索引里面的 name 字段。而 sex 字段因为跳过了 a 级字段，是没有办法使用到索引的。

第三，如果查询里面有某个类的范围查询，或者是模糊查询，列右边的所有列都没有办法使用索引。比方 name 等于大木 and age 大于32， a 级字段用到了范围查询，所以 a 级字段右边的所有列都没有办法使用索引，也就是 sex 相关的条件都没有办法使用索引。


**okay 这三个限制其实就是著名的最左前缀原则，指的是索引按照最左优先的方式匹配索引**。


不满足这三个条件的时候，都没有办法完全的使用索引。因此，当使用 b 去索引的时候，索引列的顺序是非常重要的。这里面的几个限制其实都和索引的顺序有关。在做性能调优的时候，我们常常也需要根据查询条件去调整索引需要的字段，甚至需要去额外创建一些使用的列相同列，顺序不同的索引。


再来看一下哈希索引，它是索引的优势在于一般来说它的性能比 b-tree 索引的性能要好一些，只要哈希不冲突，时间复杂度就是 o1。但是它的限制也比较明显。
