大家想我们现在的数据库引擎是Innodb，而对Innodb，一个非主键索引，它的叶子节点，存储的是索引加组件，而组件索引的叶子节点，存储的是组件加表数据。而MySQL，它是以页为单位的。同学们还记得一个页的默认大小是 1024 K，于是在一个页里面，非主键索引，可以存储更多的条目。于是对于一张表，比方有 100 万条数据，使用非主键索引，比方扫描的配计数，可能只有 100 个，而使用主键索引，可能需要 500 个。于是使用非主键索引的性能反而比使用主键索引的性能更好。同理， MySQL 之所以在存在多个非主键索引的时候，选择最小的非主键索引去使用，也是为了在一个配置里面存储更多的数据，从而减少扫描的次数。
