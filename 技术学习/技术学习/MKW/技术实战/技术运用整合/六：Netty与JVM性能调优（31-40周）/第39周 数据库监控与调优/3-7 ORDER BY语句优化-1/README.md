大家好，我是大木。这节课来探讨 order by 语句的调优。在项目中，我们经常会用到 order by，该怎么样优化？要想优化 order by 语句，最好的做法就是利用索引去避免排序，这是 order by 调优的首要原则。不过这句话有点难以理解，同学们可以先记住它，一会儿就知道是什么意思了。来看一下这节课的安排。


大概是这样子的。首先我们会先做几个实验，从中去总结哪些情况下 order by 子句能够利用索引，哪些情况下 order by 子句用不了索引。接着来探讨 MySQL 排序的原理。

还记得前面说过， b+tree 里面的关键字，也就是你的索引值都是按照顺序排列的。如果索引的顺序正好和 order by 子句所需要的顺序一致，这个时候 MySQL 就不需要针对 order by 子句专门去做排序，只需要直接按照索引的顺序取数据返回就 OK 了。这就是为什么前面说最好的做法是利用索引避免排序。本质上就是利用索引本身的有序性，让 MySQL 跳过排序的过程。理解这一点之后，下面来做实验。

又或者你可以画出来一颗 b+tree，然后去分析能不能使用索引。如果 MySQL 能够利用索引避免的排序，当然是皆大欢喜的， MySQL 不需要针对 order by 去做专门的处理。但问题是，如果我们编写的 SQL 像后面几条 SQL 一样，根本没有办法利用索引避免排序。这个时候， MySQL 就得针对 order by 子句去实现排序功能，从而去满足我们的需求了，对吧？这种情况下，我们已经知道会在 explain 结果里面 extra 字段展示 using filesort，但是 useing filesort 的原理是怎样的？我们来探讨一下。
