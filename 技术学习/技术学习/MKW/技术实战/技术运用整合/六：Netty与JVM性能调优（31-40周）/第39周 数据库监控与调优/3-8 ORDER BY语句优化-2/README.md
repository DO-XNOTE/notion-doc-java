全字段排序是针对 ROWID 排序的一个优化，它和 ROWID 排序的步骤大致是一样子的。区别在于全字段排序，它是直接取出 SQL 中需要的所有字段放到 sort buffer 的，而不是像 ROWID 模式那样只取出来ID、 sort column 放到 sort buffer。由于 sort buffer 已经包含了查询需要的所有字段，所以在 sort_buffer 排序完成之后就可以直接返回了。简单对比一下全字段排序和 ROWID 排序。



全字段排序带来的好处是性能的提升，一上去就把需要的所有字段扔到 sort buffer 了，不需要两次 IO 才能返回想要的数据。但是全自带排序的缺点也是比较明显的，就是一行数据，这样的空间一般会比 ROWID 排序多一些。如果 sort buffer 比较小，就比较容易导致写临时文件。

第三，应该设置合理的 Max_length_for_sort_data 的值。这个参数如果设置的太大，各种排序 SQL 都会利用全字段排序，可能会导致大量的内存占用。而如果要写临时文件，又会占用大量的硬盘。而如果设置的太小，又会导致各种 SQL 都去使用 ROWID 排序，从而产生两次IO，性能上可能会差一些。大木个人并不建议随便修改这个值，除非你能够 hold 住它。


第四，可以考虑调小 max_sort_length 的作用是指定在排序的时候最多取多少个字节。比方有一个超长的 text 字段或者是挖下字段，它只会取前这么多个字节去排序，而忽略后面的部分。比方你设置 Max salt lens 是 10，它只会举前面 10 个字节去进行排序，后面的部分全部忽略掉。调小 Max sort length 有助于减少 Filesort 的占用。
