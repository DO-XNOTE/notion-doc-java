这样就可以提升性能。


假设现在 b 加剧索引是一个主键索引，比方小二查询 ID 等于 5 的数据，现在索引里面搜索到 ID 等于5，再到表述这里面获取数据。而如果你的主键就能够覆盖所有的查询条件，就不需要到表数据里面获取数据了，直接返回索引就 OK 了。


而假设我们的索引是一个非主键索引，可以通过索引拿到数据的主键，之后再通过主键到表数据里面获取数据。如果非主键索引能够覆盖你的所有字段，就可以省略两步。首先是通过非主键去获取主键，之后再通过主键获取数据，对吧？这是覆盖索引的好处。下面我们来做一个对比实验。


现在我们的 salery 表里面创建了一个组合索引，作用在 from data 以及 to date 上面。而我们现在的 SQL 语句是 selection from salaries， where from date to date。我们的组合索引是没有办法覆盖我们的查询条件的，可以 explain 一下。
