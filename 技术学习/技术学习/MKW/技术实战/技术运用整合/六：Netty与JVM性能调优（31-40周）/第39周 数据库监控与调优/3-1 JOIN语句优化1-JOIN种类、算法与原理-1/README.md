这种算法大量是这样玩的。比方现在有三张表T1， T2 以及T3。如果使用 n r 键算法，执行过程大概是这样子的先查询出 T1 里面符合条件的数据，弄一个 for 循环，遍历这些数据，在循环里面查询出 T2 表符合条件的数据，并使用 reference key 去匹配，也就是你 join 时候的 own 字段。接着待查询出第三张表里面的数据去做匹配。由于第三张表是个全表扫描，所以这里直接弄了一个 for 循环 T3 表的所有数据去做判断，如果满足条件就返回给客户端。不难发现， nesty 的 loop join 算法是比较粗暴的，外层的循环结果就越多，内存循环扫描的次数也越多。
