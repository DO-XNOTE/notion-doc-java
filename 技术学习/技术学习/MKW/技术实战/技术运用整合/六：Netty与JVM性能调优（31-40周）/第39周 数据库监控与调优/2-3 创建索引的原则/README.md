OK，回来。对于 update 以及 delete 语句，为了调整使用的字段，一般也需要创建索引。比方这条词口语句就应该为 emp number 创建一个索引。当然了， e m p number 是主键，不需要额外创建索引。而对于这条 delete 语句，就应该为 first name 创建索引。这是因为 update 的语句以及 delete 的语句。它需要先根据 where 条件查询出对应的数据，再对这些数据进行更新或者是删除。创建索引可以提升查询的效率。

第一种不建议创建索引的场景 where 子句里面用不到的字段，一般不需要创建索引。因为索引的作用是快速的定位到想要的数据，如果你的查询条件里面根本都用不到，就没有必要为这个字段创建索引了。


第二，如果表里面的数据非常少，也没有必要创建索引。比方有一张表，里面只有 100 条数据，根本没有必要创建索引。因为这种情况下，是否创建索引对查询效率的影响并不大。

第四，对于频繁更新的字段，要考虑创建索引的开销。因为维护索引是有开销的，在修改或者删除数据的时候也需要更新索引。如果某一个字段修改非常的频繁而查询很少，不建议为字段创建索引。
