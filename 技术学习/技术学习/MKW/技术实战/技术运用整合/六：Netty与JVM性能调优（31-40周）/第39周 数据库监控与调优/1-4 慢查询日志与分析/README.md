long query time。又来指定 SQL 的执行时间超过多长才记录到慢查询日志，默认是 10 秒。生产环境里面，我们一般都会修改配置的值，比方把它设置成 500 毫秒，只要思考超过 500 毫秒，就会记录到慢查询日志了。


第3 low queries not using indexes 表示是否要把没有使用索引的 circle 记录到慢查询日志里面去？默认情况下是关闭的，一旦打开，只要 SQL 没有使用索引，就会记录到漫查性日志里面去。它会忽略 long query time 的配置。生产环境里面建议关闭，因为对于生产环境，我们更加关注的是某条 SQL 执行的是不是够快，而不是它有没有使用索引。而对于开发环境，弹幕建议打开这个配置，这样可以方便我们在开发过程中去定位到没有使用索引的SQL，进而急躁的去介入去调优。
