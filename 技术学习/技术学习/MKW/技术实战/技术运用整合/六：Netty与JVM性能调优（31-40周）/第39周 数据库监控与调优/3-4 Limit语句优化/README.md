大家好，我是大木。这节课来探讨分页查询的优化。到分页查询相信大家都不陌生，在 MySQL 里面想做分页查询，使用 limit 语句就 OK 了。里面的语句可以接受两个参数。第一个是offset，它用来指定返回结果的第一行的偏移量，表示想要跳过多少行。


而 set 用来指定返回多少条数据。比方有这样的一条 SQL select the thing from employees。我们现在想要对这条效果做分页查询，每页展示 10 条。如果想要查第一页，就可以写成 limit 010 执行。点击output，看一下需要花费多久。 92 毫秒，记录一下。查询第一页的时候花费 92 毫秒。当 offset 等于 0 的时候，可以省略，直接写成 limit 10，效果是一样的。


如果想要查询第二页，我们就可以把 offset 写成10，表示跳过 10 行取 10 行。想要查询第三页，就写成 limit 20  10，跳过 20 行取 10 行。以此类推。可以发现利米的语法非常的简单，相信同学们也早就掌握了。


但是 limit 有一个问题，当 offset 非常大的时候，查询就可能会比较慢。比方我们想要查询第 30001 页的数据， offset 是 30 万，这个时候来执行一下 sql，看需要花费多久。

下面来探讨第一种优化方式。

这两种方式是目前业界有关秘密的优化最常用的两种方式。而如果你能够想办法获得查询的起始主键值，甚至是起始主键值以及解除主键值，那么就可以使用方案 4 或者是方案5。
