在咱们的工具篇里面已经探讨了五六款监控工具，照顾到了 springboot 玩家以及传统部署模式的玩家。这些工具有的是为 spring boot 量身打造的，比如 spring boot actuator，有的只能适用于传统布式模式下的项目，比如 PSI probe。但不管是 spring boot 项目还是传统项目，都需要良好的监控，在定位问题的时候才能有的放矢。同学们可以根据自己项目的情况，合理选择这些工具，好对照表格简单回顾一下。
课上我们先探讨了 Skyworking 这款工具。课上花费了很大的篇幅去探讨，着重探讨了加 y 镜头的方方面面。加白邮件特既能够支持 spring boot 项目，也能够支持传统部署的项目。还记得对于 spring boot 项目，只要 Java 杠加 y 件，侧跟上加 y 件的价包的路径就 OK 了。而对于传统项目，可以修改一下他们 get 的 shell 脚本。


当然了，传统项目课上没有去演示，因为太简单了。之后课上探讨了 springboot actuator，这是一款 spring boot 专用的监控工具，使用比较的简单，只要添加 spring boot starter actuator 的依赖就可以用上了。常用的几个配置课上也做了比较详细的演示，同学们务必记得。


讲完 spring boot actuator 之后，课上用 spring boot admin 实现了 actuator 的监控数据可视化，从而帮助我们更加高效方便的使用actuator。除了 spring boot actuator 以外，课上还以手机的方式提供了数据可视化的另一种选择。用 promise 加上 Grafana 去实现监控数据可视化，对吧。


之后课上探讨了 Java melody 这款工具也很有意思，界面比较的丑，功能却非常的强大，加把 melt 的使用比较的简单，配的也很少。课上做了详细的注释，使用起来也是非常省心。遗憾的是他和哈克瑞数据源有一点兼容性的问题。再之后课上探讨了他们开的manager。和前修改工具不同，他们开的 manager 不仅能够做监控，还能够管理他们cat。同学们要记得这玩意儿的几个角色以及 context 的配置。此外还需要知道他们看的 manager 几个实用的功能，比如怎么样分析内存泄露，怎么样通过API。
