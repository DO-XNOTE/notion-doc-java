这几个来探讨 skyworking 的动态配置。关于动态配置，我又专门写了一篇手记来看一下。动态配置是 sky booking 六点五点零才艺如的能力实现的效果。当配置修改之后，不需要重启， sky walking 就可以生效了。目前 sky walking 并不是所有配置都支持动态刷新的，只有这个表格里面列出的配置才能动态刷新。要想实现动态配置，需要借助一个额外的配置。服务器引入之后，架构图就变成这样了。简单来说就是 Skywalking 定时从配置服务器里面读取配置，弄了一个定时任务，定时的修改配置的值。


sky walking 支持好几款配置服务器，比如 dynamic configuration service、阿波罗、Kacos、zookeeper、e、d、c、 d 以及console。虽然支持很多款配置服务器，不过实现的目标是一样的，用法其实也都差不多，只是配置不同而已。下面我们就来用 Nuckles 作为我们的配置服务器，来为 sky walking 实现告警规则的动态刷新。要想使用 Nuckles 管理 sky walking 的配置，首先需要搭建一个Nuckles，到这个地址就可以下载 Nucks server 了。如果网络不好，也可以用我提供的百度盘地址。下载完成之后就可以搭建了。我已经下载好了。根据你的操作系统执行相应的命令。如果你是 Linux 或者 Unix 或者 Mac 就执行这一条。如果你是windows，就执行 CMD 这一条。我们来安装一下，切换了 Knockers 的 b 目录，执行点杠 start up s h 杠 m stand alone。启动好了，访问 local host 8 八 4 八杠knockus，输入账号纳克斯，密码也是纳克斯，点击登陆就可以看到纳克斯的首页了。
