这节课来对前面讲解的内容做一个实战练习。我们的任务有三点需求。首先为项目设置采样，每 3 秒采样一条。这这种诉求在实际项目中很常见。默认情况下， sky walking 是不采样的，所有的监控数据都会上报给 sky walking。这意味着在高并发的系统中，所有的数据都上报，很可能会影响到微服的性能。因为每次上报微服都要请求Skyworking。所以在实际项目中，我们往往会做一个抽样，限制上报数据到 skyworking 的速率。


当然了，我这里要求的 3 秒才上报一条显然太少了，不适用于生产。实际项目中可以根据项目的需求设置合适的值。如果你的项目不设置采样，发现性能并没有太大影响，不设置也是一个不错的选择。关于采样，还记得 Zippking 也有一个采样率的概念，对吧，默认是10%，同学们要记得前面的知识。


第二个诉求，我们需要打印出 circle 的详情，包括 circle 的参数。还记得在前面的演示中，我们没有经过任何的处理， Skyworking 也是会收集到应用执行的 circle 的，只是没有收集到 circle 的参数。而在实际项目中，收起 SQL 的参数在定位一些 Mysql 调用时非常的常见，因为很多情况下 Mysql 是参数相关的，可能某一些参数做客执行的很快，但是某一些参数执行的就非常慢。
