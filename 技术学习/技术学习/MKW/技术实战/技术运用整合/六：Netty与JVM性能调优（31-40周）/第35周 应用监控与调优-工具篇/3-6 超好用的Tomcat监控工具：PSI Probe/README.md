上一节我们探讨了他们 cat manager，这款工具不够强大，但实际项目中，特别是传统项目，对于管理他们 cat 以及监控他们 cat 是有很大需求的。这一节来探讨 Psi probe，利用这款工具可以满足我们监控和管理项目的需求。官方定义 Psi probe 是一个先进的汤姆盖的管理与监控工具。略认为这是一个增强版的汤姆盖的manager。他的 get up 地址在这里， wiki 在这里，官方文档在这里，感兴趣的同学可以看一看。好话不多说，来玩一玩 Psi probe 吧。关于 Psi 的搭建和使用，我专门写了一篇手记，一起来看一下。

在使用 Psi probe 之前，务必检查一下 Psi probe 是否支持你所使用的 Tom cat 版本。支持的 Tom 凯的版本可以在这里查看。课上所使用的 Tom 凯的九点零点三零在这里可以看到，也是支持的。**可以发现大木的课还是比较强调官方文档的，每一个组件都会贴一下官方文档，从我的第一套视频课程就是这种风格了。我贴官方文档的目的主要有两个，第一是未来这些软件都会发布新的版本，经过课程的学习，同学们应该有能力跟随官方文档去研究新特性，并去使用更新的版本。第二，官方文档是第一手的文档，里面有第一手的学习素材，是最好的文档。**


**大木也真心希望在未来某一天，大家能够逐渐成为自己的老师，跟着官方文档去自学，就像大幕一样，成为自己的老师对吧？会不会有点狂啊？总而言之，真正的高手绝对不是说看两个视频，读几个源码分析就能练成的，必然是经过 1 万个小时的磨练。而从官方文档开始磨练，就是一个很不错的起点，希望大家能够逐步重视起官方文档。**


好，回到正题。确认过版本之后，就可以正式搭建 PSI probe 了。首先到地址下载，如果网络不好，可以使用百度盘地址下载，我已经下载好了。下载完成之后需要配置账号和角色。可以发现这一步和上一节讲的他们看的 manager 非常类似。并且 PSI probe 也内置了四种角色，分别是probe、 user power、 user power、 user plus 以及manager。贵这个词念贵，不要读成 GUI 好吧。不同角色所具备的权限可以到地址查看。可以看到 manager 具备了所有的权限，其他的角色具备的权限相对少一些。你可以根据自己的需要为你的用户分配不同的角色。


好，我们来配置一下。找到他们 cat CONF 目录的他们开的 users 插票。用 ID 打开可以看到上一节我们的配置已经可以让 admin 用户使用 PSI probe 的所有功能了。我们就保持这样吧。接着看第三步，把下载的外包扔到他们 cat 的外部 app s 目录中，最后启动。他们 cat p s i probe 就搭建好了。把 PSI probe 的外包扔到他们 cat 的外包 APP 目录里面，最后启动 Tom cat，切换到 b 目录执行，点 gun start of s h 启动好了。


访问 localhost 8080 p s i probe web 三点四点二，输入账号和密码，账号 admin 密码123456。点击登录就可以看到 PSI probe 的首页了，目前是英文的。如果想要使用中文，可以点击下面的中国国旗，这样就变成中文展示了。应用程序这一栏可以看到他们 cat 所部署的应用，并且有一些统计信息，点击可以查看详情。比如我们点一下萨摩瓦，可以看到相应的图表，展示了请求的情况，但是目前没有请求。还可以停止应用，重新载入应用，或者是卸载应用。和 Tom cat manager 类似，返回这里的请求会话属性以及属性数，也可以点击详情。在右侧可以点击查看更多的详情。比如有哪些 server let，有哪些过滤器，我们的应用 context 有哪些参数等等。数据源这一栏可以查看当前他们 cat 里面的数据源。就目前来说， PSI probe 支持这么多的数据源，基本上业界主流的数据源都是支持的。不过我们现在是一个空的 Tom cat，所以这里展示没有任何数据源。发布这一栏可以发布应用，和 Tom cat manager 的功能基本上是一样的。日志一栏可以查看日志详情。 Catalina out 是他们 get 的日志文件，点击可以查看详情。


Pro blog 是 PSI probe 的日志文件，也可以点击这个按钮下载日志，或者点击右边的按钮下载压缩后的日志。线程一栏可以查看当前他们 cat 里面线程的运行情况。这里美中不足的是，它不能区分出哪个线程是属于哪个应用的，而是罗列到了一起。


集群一栏可以监控集群模式下的 Tom cat。系统信息，可以查看当前服务器的信息，包括系统的概况，内存的使用情况。系统的信息主要是一些运行中的环境变量。操作系统的信息。硬件的信息。大人硬件的信息会比较的慢。包装器控制展示的是 Java service wrapper 的信息，如果你不了解 Java service wrapper，可以查看这篇文章，它可以用来把任何的 Java 程序设置成服务，最后还有可信库，大家可能不知道是什么，我们可以点击一下英文就知道是什么意思了。


叫 trust store，这是 SSL 里面会用到的。如果你的 Tom cat 配置了证书，或者是开启了HTTPS，这个时候就可以配置一个 trust store，好回到中文状态。连接器可以查看 Tom cat 连接器的运行情况。证书一栏和他们开的 manager 一样，也是他们开的。启用 T2S 使用 HTTPS 的时候，证书的管理。最后快速检查，可以检查出当前数据源内存储以及应用是否有问题，如果有问题也会提出来。


总的来说， PSI probe 使用起来还是非常简单的，只要搭建一下就可以玩起来了，同学们可以搭建一个试一下。最后回答手机 p s i probe 不仅能够监控 Tom cat，还能监控其他的 web 容器或者是服务器。在这个地址里面有列出来，监控他们看的 5- 9 借 boss 3- 7 以及 spring 的 TC server， where may well 的微 fabric TC server。如果你的项目使用了这个列表里面其中一款 web 容器或者是服务器，并且正在面临着项目发布难、管理难等等问题，不妨尝试引入 PSI probe。这节课就到这里，谢谢大家。
