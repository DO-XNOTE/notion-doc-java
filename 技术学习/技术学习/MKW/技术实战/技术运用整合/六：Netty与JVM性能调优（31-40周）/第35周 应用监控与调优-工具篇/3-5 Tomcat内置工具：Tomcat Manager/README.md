这几个来探讨用汤姆 cat manager 监控汤姆cat。 manager 是汤姆 cat 内置的一款工具，从名称就可以看出来了，是用来管理汤姆 cat 的。从功能上来讲，他们开的 manager 并不是很强大，但是作为一款内置的工具，掌握一下还是很有必要的。我们到 Tom cat 官网下载一个 Tom cat，这里我们下载一个 time 开的9，点击 ZIP 就可以下载了。当然了，同学们也可以根据自己的操作系统下载对应的文件。我已经下载好了，解压。他们 cat 将目录切换了 Tom cat 的 b 目录，执行 start of SH 启动。他们 cat 报了一个没有权限，我们赋予一下权限。


c h m d 杠 2755 新再次执行 start up s h 启动完成之后，访问 local host 8080，点击 manager app 就可以进入他们看的 manager 了。这个时候可以看到一个登录的弹窗，目前我们还没有配置用户名和密码，所以我们可以先点击取消。点击完取消之后会弹出来一个错误页，目前是不允许访问他们看的 manager 的。

所以在生产环境里面要慎用。我们点击一下，他说现在没有内存泄露的问题。好，我们现在把萨摩袜给停掉，然后再次点击发现泄露，可以看到这个时候它就会有提示了，说 sum 袜这个应用虽然已经停掉了，但是内还在内存里面，是有内存泄漏问题的。如果某一个时刻它 m k 的运行不太正常，我们是不是就可以点击发现泄露按钮，看到这样的结果之后就可以重启，他们看到了对吧？其他的几项都是对 T2S 的一些诊断，如果没有使用HTTPS，一般是用不到的。最后，服务器信息这一栏可以展示当前服务器的一些信息。好，回到他们开的manager。最上面有一个伏击状态按钮，点击可以展示服气的一些状态，比如服气自身的信息，当前内存的占用情况。这里还有一个简单的监控，我们用 get 方法访问了这样的路径，是地址访问的，花费了 10 毫秒。在这里还有他们 get 性能相关的一些监控信息，比如最大线程数，当前使用了多少线程数，繁忙的线程数有多少等等。


再次回到上面，点击完整的服务器状态，可以查看更多的信息，非常的详细。他们看到 manager 除使用界面的形式访问以外，还可以使用 API 的形式去访问。点击管理者帮助，可以看到所有 a p i 的文档。 a p i 的形式是这样子的 manager text 跟上命令的名称，然后问号，再跟上参数。 deploy 命令可以部署应用参数可以指定路径。 service info 命令可以查看服务器的信息。 thread dump 命令可以 dump 线程。考虑到英文不好的同学，我特地找了一个中文文档，地址在这里，同学们也可以做一个参考。


总的来说，他们开的 manager 的 API 还是比较实用的，能够返回他们 get 的执行细节。好，我们现在是使用 logo host 的 8080 去访问他们 get manager 的。如果是远程访问，比方如果我想使用我本季的 IP 冒号 8080 去访问他不看的manager，该怎么办？来试一下幺九二点幺六八点一点幺零幺冒号8080，点击 manager app 可以看到。现在又出现了一个错误页403。在出现错误的时候不要慌，其实这一行已经告诉我们答案了。他说默认情况下，他们开的 manager 只允许本机访问。如果你想修改这种限制，你需要修改他们开的 manager 的 context XML。我们来改一下。


找到 web BBS 目录 manager web info connect XML，用 ID 打开这里的allow，表示允许哪些 IP 对应的机器访问 top care 的manager，我们这里就改成任何机器都能访问。改下正则。这里需要提醒一下，实际项目中尽量不要这样配置，因为任何 IP 都能访问，是会存在安全隐患的对吧？好，重启他们cat，叫了刷新输入账号 admin 密码123456，可以看到依然能够访问。好简单。


总结一下。这节课详细探讨了他们开的manager。他们开的 manager 是一款简单应用了他们开的管理工具，虽然它功能并不强，界面做的也比较的丑，但是还是能够在一定程度上满足我们监控应用的需求。除此以外，他们家的 manager 还有一点限制，那就是只支持传统的部署模式，对于 spring boot 价包的部署模式是没有办法使用 top cat manager 的。这也是一个小小的遗憾。这节课希望同学们能够掌握两点，首先是他们开的 manager 如何使用？第二，这节课也体现了一个排查问题的技巧，但当报错的时候，可以从错误提示中分析线索。一般来说，成熟的开源软件都会有良好的错误提示，如果你能够很好的分析错误提示，问题往往解决了一半。
