好，话不多说，下面我们就来参照文档的说明，用 sky walking 去监控与分析应用的性能瓶颈。简单起见，我就用咱们课上第一阶段单体架构的代码进行演示了。如果想要间隔，后面微服务拆分后的系统也是一样的，没有任何区别。要想使用价位agent，首先需要到 sky working 包里面的 agent 目录，把这个目录拷贝到任意的位置。找到 agent 目录，拷贝到任意位置。比如桌面粘贴，需要修改 config 目录下面的 agent config 文件，把 agent 点 service name 修改成我们的微服务名称，打开找到 config 目录 agent 点 config 修改 agent 点 service name 改成比如 foodie 杠 DV 保存。如果你的 Skyworking 和微服务部署在不同的服务器，还需要修改里面 collector back and service 的值。当然我们不需要修改，对吧？我们接着看。


我们在启动项目的时候，需要添加一个 j v m 的参数，叫做加 y agent 冒号跟上 skyworking agent 点价的全路径，也就是价包的全路径。把它拖到终端里面去获得全路径copy。打开idea，点击 edit configuration，输入 gun Java agents 冒号跟上全路径。 apply okay 启动应用，可以发现项目在启动的时候，他会打印他找到了 skyworking agent 点价了，并且读取了目录下面的 agent 点 config 文件。


启动完成之后，我们来访问一下这个项目的 rest API，做一些测试。这里我使用了 idea 的一个插件，叫做 rest for toolkit。安装之后，它就会自动的识别出当前项目 spring MVC 的所有 rest API，并且你随便选择两个，点击send，它就会自动的发送请求，非常的方便。我们随便访问几个。


把握好了，这个时候再去访问 Skyworking 的首页，选择最近 15 分钟，会发现 Skyworking 上展示了当前系统的总体状态。它有两种视图，一是服务视图，展示了当前项目的CPM、c、b、m，可以在一定程度上体现出当前应用的吞吐量或者并发数。还可以展示最慢的几个端点，以及应用上面 API 响应时间的统计值。


第二是数据库视图，它可以展示出数据库的一些信息，比如数据库的平均响应时间、数据库的吞吐量、数据库的服务质量以及数据库的响应时间。一般来讲，我们会比较关注数据库的响应时间，数据库的吞吐量以及响应时间的统计值。还有最关注的项目里面最慢的几个。Rest。 a p i 点击拓扑图可以看到请求的去向。他说用户向负 d d e v 微服务发送了请求，负 d d v 又请求了数据库。这个图还是很有意义的，首先可以帮助开发人员更好的理解所维护的微服务在整个系统中的位置。当然，这个图上只有一个微服务，对吧？对于架构师，也可以很好地帮助他去检测当前项目的实现和他的规划是否一致。
点击微服务，还可以展示出详情，也是一些应用的性能指标，比如响应时间、c、t， m 等等。追踪是整个 sky working 里面最重要的一个导航。在页面里面会详细的展示指定应用。有哪些 API 被访问了，总共花费了多久？每一个 API 在每个阶段花费了多久？比方 notify merchant order paid API 总共花费了 1161 毫秒，其中有 1135 毫秒花费在了 spring MVC 上。而这个API，它又操作了数据库，点击可以查看详情。


说执行的数据库是什么。 SELECT 1，这是 Mysql 的一个存慧性校验的SQL，这是真正去执行业务的SQL。以及提交每一个阶段花费了多久都一目了然。这样一旦发现我们的应用性能达不到要求，就可以通过左边去分析哪些 API 耗时过长，通过右边则可以分析是 API 的哪个环节耗时过长，是 API 执行的慢还是数据库访问慢，都可以迅速分析出来。这样我们定位性能瓶颈的时候就可以有的放矢了。


除此以外， Skywalking 还有告警的能力，后面我们会详细讲。还有指标对比的能力，任选两个指标进行对比，不难发现 Skyworking 的功能真的是非常强大。好，回到文档前面，我们是在 idea 里面去启动的，如果我们的应用是用架袜钢架的方式启动，该怎么办？来演示一下，把项目停掉，执行 Maven clean install 杠 d skip tests，忽略掉单人测试构建项目。构建好了切换到 API 项目的 target 目录。可执行的价包。在这里 Java 杠 Java agent，跟上， Skywalker agent 点价的全路径，再杠价，最后跟上可执行的价保名称。这里需要强调一下，杠 Java agent 这个参数一定要在杠架参数之前，否则将无法使用 Java agent 实现对应用的监控。可以看到应用在启动的时候依然打印了 sky working 相关的日志。
