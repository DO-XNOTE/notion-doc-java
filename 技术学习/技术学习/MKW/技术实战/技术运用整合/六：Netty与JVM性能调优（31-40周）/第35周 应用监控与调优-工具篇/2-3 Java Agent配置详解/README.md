这节课来详细探讨加微 agent 的配置。前面我们修改了 agent config 文件里面的配置属性，但事实上， a 推荐的 config 文件里面有很多的配置属性，这些配置属性可以帮助我们去指定如何收集微服务的监控数据，并且加 y 减还支持了好几种方式去指定这些配置属性的值。所以我专门总结了一篇手记，一起来看看。


从这个表格可以看到，加 y agent 支持了这么多的配置属性，我把每一个配置属性的作用都做了整理，同学们可以根据项目的需要，参照描述一栏的说明，合理去修改配置属性的值。至于配置的使用方式，咱们前面是直接修改 agent config 文件的。除直接修改配置文件以外， sky working 还提供了多种方式去指定配置属性的值。第一种是系统属性杠 d 启动的时候加上杠大 d skywalking 点儿，上面表格里面的配置属性的名称就可以了。演示一下 edit configurations，跟上杠大地skywalking。注意 skyworking 的前缀千万不能少 agent 点 service name 等于比如x。 Apply OK。重启应用，随便访问几个API，刷新 sky working 的首页，选择最近的 15 分钟，这个时候就可以看到这里已经有一个服务叫做 x x x 了。


第二种方式，代理选项演示一下 edit the configurations，把这里的杠大 d skywalking 点儿给删掉，改成等于号就 OK 了。代理选项是不需要加上 sky walking 点儿这种前缀的。我们把 service 的名称设成 YYY apply okay。重启应用，随便访问几个单点，刷新 sky working。选择时间最近的 15 分钟。可以看到现在已经有一个服务叫 YYY 了，说明我们的配置是生效的。


第三种方式系统环境变量。细线的同学会发现 agent config 文件里面有很多的大写字母，每一个大写字母都表示 skyworking 会读取名为大写字母的环境变量。如果读不到，会使用冒号后面的值作为默认值。现在我们来定义一个环节，变量到了大括号叫 s w a p name 冒号。如果读不到环境变量的值，就把它设为 n n n。保存 edit configurations 把前面设置的代理选项给删掉，怎么设置环境变量？在这里有一个 environment variables，点击加号 s w a p name，把它的值设成 MMM okay apply okay 重启项目。


随便访问几个单点，刷新 sky working 选择最近的 15 分钟，可以看到这边已经能够显示 m m m 服务了。如果我们把环境变量给删掉，重启，这里就会多出来一个的服务。回到文档。当然了，我们刚刚演示的时候都是用指定符的名称去做测试的，对于其他的配置属性也是一样的玩法。同学们要能够触类旁通。 Java agency 的配置姿势这么多，如果组合使用，它们的优先级关系是怎样的？我在这里也做了总结，代理选项的优先级是最高的，第二是杠 d 的方式，第三是环境变量，最后是配置文件。


实际项目中，我们应该选择使用哪种方式？个人建议将不会经常变化的配置直接写到 a 键的 config 配置文件里面去，而将容易变化的配置用文档里面。这 3 种方式的其中一种，在启动应用的时候指定不建议混合使用。因为一旦混合使用，优先级关系就会变得更加复杂了，不利于大家管理自己的项目。好，这节课就到这里，谢谢大家。
