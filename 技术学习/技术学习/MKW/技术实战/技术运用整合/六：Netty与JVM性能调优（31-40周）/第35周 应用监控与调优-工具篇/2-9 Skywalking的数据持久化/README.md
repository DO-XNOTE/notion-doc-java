目前来说， Skyworking 和 elastic search 配合使用的案例要相对多很多，所以下面我们就来使用 elastic search 存储 sky working 的数据。至于其他的存储方式，套路也是大同小异的，无非配置不同而已，希望同学们能够触类旁通。好话不多说，来演示一下。


要想使用 elastic search 存储数据，需要先搭建一个 elastic search。我这边已经准备好了，将目录切换到 elastic search 的b。目录。切换好了，执行点杠 elastic search 就可以启动了。如果想要后台启动，可以杠d。有关 Elsa 设置相关的知识，在课程的第三阶段有非常详细的讲解，如果同学们感到生疏，也可以复习一下。


启动完成之后，访问 logo host 9200，可以得到正常的结果，说明 elastic search 已经启动成功了。然后找到 Skyworking 安装包的 config 目录，在这个里面有一个 application wear mail，我们用 idea 打开。首先需要注释掉这一段，表示让 sky walking 用 HR 存储数据。然后我们需要解开 elastic search 相关的内容。如果你用的是 elastic search 6 或者更低的版本，解开这一段就可以了。如果你用的是 elastic search 7，需要解开这一段。对，我们只需要解开这一行就可以了。配置 elastic search 的地址9200。如果你的 elastic search 有多个node，那么用逗号分隔就可以了。修改好配置之后，启动 sky walking，切换到 sky walking 的 b 目录，执行 start up s h。


在启动了访问 LOGO host 8080，可以正常返回 skyworking 的界面，访问 logo host 9200 cat indices 可以看到现在我们的 elastic search 里面多了很多的索引，这些索引都是由 Skyworking 自动给我们创建的，但是目前索引里面还没有任何的数据。


下面我们启动项目，随便访问几个API，打开 skyworking 首页，刷新选择时间范围最近的 15 分钟可以看到能够正常监控，拓扑图追踪等等都是正常的。刷新 indexes 可以看到现在索引里面已经有数据了，说明已经实现用 elastic search 存储数据。好，这节课就到这里，谢谢大家。
