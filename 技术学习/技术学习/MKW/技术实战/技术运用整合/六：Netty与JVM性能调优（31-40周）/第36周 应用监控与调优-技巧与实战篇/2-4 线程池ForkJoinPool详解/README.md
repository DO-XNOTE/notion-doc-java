我们把一个大任务拆分成了若干多个互不依赖的子任务。图上画了 8 个我们的 fog join pool，里面弄了两个线程，这些子任务被放到了两个队列里面。每个队列都会有一个线程来消费执行。线程和队列是一一对应的，并且在正常情况下，县城会议 r i f o 也就是后进先出的方式，从自己的队列里面获取任务去执行。
如果发现自己的队列已经空了，任务全部消费完成了吗？检查其他的县城里面有没有尚未执行的任务。比方这里的 focus showing worker thread 1 把自己队列里面的任务执行完了，就会查看 thread 2 里面有没有尚未执行的任务。


如果有，就使用 f i f o，也就是先进先出的方式，从队列里面窃取任务去执行。也就是线程在执行自己任务的时候，是后进先出的，从顶端消费。而执行别人的任务是先进先出的，从底端消费。这样可以减少争抢。最后当所有任务都执行完成之后，返回结果。经过分析可以发现，使用 focus 可以充分利用线程实现并行计算，同时还可以利用工作窃取提升性能。
