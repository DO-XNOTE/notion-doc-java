大家好，我是大木。这节来对所相关的知识做一个总结。课上主要介绍了两大类锁，第一类是 signals 的，第二类是加白油条 concurrent 包里面的锁。课上介绍了完全的lock，元全的 read write lock 以及 stand lock。这三个锁性能是依次递增的，但是使用难度也是依次递增的。实际项目中，同学们应该按照需求去选择，不要过分的追求性能上去就用 stand lock。我个人的建议是触机尽量使用简单的实现，比方如果用 think nice 的能够实现，就不要去用后面的锁。

如果用人群的 lock 能实现，就不要去用人群的 read write lock。当项目出现明显问题之后，再去考虑怎么样优化，因为代码越简单，维护起来也越容易。此外，课上探讨了所调优的五板斧。这章的内容细节比较多，对工作经验 3 年以内的同学，只要了解个大概就可以了。但是如果你已经工作 3 年以上，建议掌握所相关的所有知识点。毕竟要想成长为高阶程序员，所相关的知识是绕不开的。好，这节课就到这里。
