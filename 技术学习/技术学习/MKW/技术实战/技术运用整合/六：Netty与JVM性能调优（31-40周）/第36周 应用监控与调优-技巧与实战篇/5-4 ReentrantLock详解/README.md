我们就可以打开源码， command F12 搜一下，

在调用 condition 方法的时候，必须要在 lock 的保护之下，也就是操作 condition 的代码必须在 lock 点 lock 和 lock on lock 之间。来看一下，肯定写怎么使用。


在这段代码里面，我们先创建了一个lock，用这一个 lock 创建了两个condition。下面的方法分别针对 condition 1 以及 condition 2 进行操作。 await condition 1 以及 await condition 2 用来阻塞线程。下面还有两个 signal 方法用来唤醒线程。


这里再次强调一下，操作 condition 的代码一定要在 lock 和 unlock 之间。运气下看看，来分析一下。首先在这里我们弄了一个线程，调 await condition 1，所以它会打印这段日志，打到这里，执行到这里阻塞了。因为这里我们又创建了一个线程，于是它又去调await，肯定是2，执行到这里也阻塞了。再之后， main 方法睡了 1 秒，又去掉 signal condition 1，尝试唤醒。注册了 condition 1 的线程，也就是 thread 0，于是就打印了这一段。在之后又睡了一秒去唤醒 thread 1，于是打印了这么一单。这里的 994 毫秒和 2002 毫秒是有误差的对吧？从这段代码不难发现， signal 的使用还是比较清晰的。而且如果和 object 的 wait 方法， notify 方法来对比，你会发现 condition 的功能要强大很多。


传统的 wait 方法， notify 方法只不过相当于一个 condition 对吧。而这里我们可以使用多个condition，并且可以让多个肯定性协作起来。所以实际项目中，如果想要实现线程之间的协作，个人更加推崇使用 region 的 lock 的condition。这节课就到这里，谢谢大家。
