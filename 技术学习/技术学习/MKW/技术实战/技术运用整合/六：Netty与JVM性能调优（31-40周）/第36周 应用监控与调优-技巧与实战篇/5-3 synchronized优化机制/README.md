这段代码里面， think nice 的作用于一个局部变量。前面讲过，出现线程安全问题的因素有两个，第一是要存在共享数据。第二，多个线程要同时操作共享数据。而这段代码里面， object 是一个局部变量，不存在共享数据。因此即使多个线程执行 summer method 方法，也不会有线程安全问题。 JDK 可以通过逃逸分析检测到这种不可能存在共享数据，也不可能存在竞争的代码。自动消除掉这个锁。
