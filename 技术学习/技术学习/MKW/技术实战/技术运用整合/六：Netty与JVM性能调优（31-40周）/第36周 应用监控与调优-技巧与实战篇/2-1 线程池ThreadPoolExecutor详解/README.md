线程池一共有这么多的状态，这张图描述了状态的迁移。 running 指的是当前线程池可以接受提交的新任务，也可以处理阻塞队列中的任务。 running 状态的线程池调用 shut down 方法，就会把线程池的状态转变成 shut down， shut down 表示不能提交新的任务，但是可以处理阻塞队列里面的任务。 running 状态调用 shut down now 会把线程池的状态转变成stop。 stop 表示不能提交新的任务，也不会处理队列中的任务，它会直接中断处理中的任务。


tiding 指的是如果所有的任务都已经终止了，就会进入到这个状态了。如果线程池是在下的 down 状态下，那么阻塞队列为空，并且线程池里面执行的任务也空的话，才会变成tiding。如果线程池是 stop 状态的话，那么线程池里面执行的任务是空的话，就会进到 tiding 了。 tiding 状态调用 terminated 的方法，会进入到 terminated 的状态，表示线程池彻底终止这个状态还是比较清晰的。好，这节课就到这里，谢谢大家。
