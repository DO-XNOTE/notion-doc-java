大家好，我是大木。这一章来探讨异步化，异步化也是实际项目中一个重要的调优措施。说到异步，相信大家都不会感到陌生，和异步对应的是同步调用，指的是程序按照定义的顺序依次执行，每一行程序都必须等待上一行执行完成之后才能执行。而异步调用是指程序在执行的时候，不需要等待异步调用的语句返回，就可以执行后面的操作了。

主要是当你的某个业务请求非常的耗时，体验很差的时候，可以将其中一些不影响主流程的业务异步执行，这样就可以减少请求响应的时间，从而提升用户体验了。举个例子，我这里有一段示例代码，这段代码模拟一个非常耗时的业务，方法叫Biz。在 Biz 方法里面做了三件事。首先调用 sub beads 1，子业务，需要花费 1 秒。紧接着调用 save pelog，保存了一条操作日志，又花费了 1 秒。最后执行 sub bizarre，又花费了 1 秒，运行一下看看。整个业务需要花费 3 秒才能返回，耗时非常的长，用户体验也会比较的差。


如果这里的 sub beads 1、 several p log、 sub beads 2 这三步，如果其中的某一步或者多步，它的执行结果并不会影响主流程，就可以把它改造成异步执行了。比方这里的保存日志是否成功，不会影响主流程，就可以把它改造成异步的。
