大家好，我是大木。这节来总结对象池。关于对象池，希望大家至少能够掌握以下几点。首先，要能够了解对象池的适用场景，知道什么时候该用，什么时候不该用。第二，关于 comments pull two。要能够熟练操作 factory 的核心API。毕竟使用 commerce pull two 的时候，基本上都是要实现 factory 接口的，因此操作 factory 的核心 API 是一项必备技能。


第三，希望同学们能够熟练掌握 object pool 的核心API。尽管一般来说不需要自己去实现 object pool 接口，可以直接使用 generic object pool 或者其他 comments pool to 内置的对象池。但是掌握 object pool 的核心 API 对分析源码是有帮助的，所以建议同学们也要掌握，至少应该留个印象。


第四，要能够知道对象池的配置和注意点。位置在咱们课上并没有展开，但是也是有一些门道的。比方对象池不宜过大，否则回收的时候压力会比较大。一般来说，对象总数在 200 左右的对象池已经比较大了。总数上限的对象池在我的职业生涯中还没有见到过，不过这里的数字只是经验之谈，同学们还是要根据项目的需要去设置大小。又比如如果设置了 Invict 的周期不要太小，否则也会影响对象池的性能。频繁执行定时任务对性能是有影响的。


另外，建议同学们还是要花一点时间试玩一下这些配置，至少要留个印象，用到的时候知道到哪儿查，毕竟我可是花了大半天的时间，通过翻源码才总结出来那些配置是干嘛的，默认值是什么。这里也要吐槽一下啊，不知道为什么官方文档竟然没有任何配置相关的说明，市面上的文档总结的也是非常不完善，所以只能靠自己了。anyway，希望大家能够玩一玩这些配置。好这节课就到这里，谢谢大家。
