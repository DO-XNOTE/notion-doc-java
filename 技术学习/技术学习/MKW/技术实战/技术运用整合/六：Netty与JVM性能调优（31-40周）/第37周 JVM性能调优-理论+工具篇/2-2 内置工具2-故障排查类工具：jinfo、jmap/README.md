大家好，我是大木。讲完两款内置的监控工具之后，来看一下内置的故障排查工具。故障排查工具稍微多一些，有 6 款，我们按照从简单到能的顺序进行讲解。第一款jinfo。


j info 依然是一个实验性的工具，虽然现在依然非常流行，但是从 jdk 9 开始，接音符的部分功能已经可以使用。一会我们要讲的 j h、s、 d b 或者是 jcmd 去代替了。另外需要提醒一点， j if 在一些版本的 j t k 里面，特别是比较老的 j t k 版本，比如 j d k 6，对 windows 的支持不是很完善，支持的参数比较的少。我这篇文章是基于类 Unix 系统去编写的，同学们如果使用的是 windows 操作系统，如果发现课上讲解的某些参数在你的操作系统里面不支持，不用诧异，一切以你操作系统下面执行的接 info 杠 h 的结果为准。OK。接 info 工具的作用主要有两个查看 JVM 参数以及调整 JVM 参数。说到查看 JVM 参数，同学们还记得使用前面讲解的 JPS 杠v，是可以查看虚拟机在启动的时候所指定的参数的对吧？但是如果想要动态了解那些并没有显示指定的参数的默认值，这个时候接 info 就可以发挥作用了。


好，来看一下接音否是怎么玩的。它的命令格式是这样子的，有不少的参数，同学们可以看一看。这里我准备了 6 个势力，我们来对照势力玩一玩。首先用 GPS 看一下当前运行的进程，这里我们测试的进程使用26089，然后不带参数执行 j info， j info 直接跟进程号。这样展示的结果有两部分，第一部分是系统属性，其实就是打印了 system 点 get practice 的结果。第二部分，是 j v m 参数，从这些参数里面可以获得很多的信息。比方 26089 进程对内存最大，新时代最大，使用的是 G1 这款垃圾收集器等等。应该使用 Cease Props 织打印系统属性，杠 sees props，这样就不打赢 VM 参数了。你也可以使用， come flags，只打印 g m m 参数不打印，系统属性好。


接着看。杠 flag 的玩法有好多种。第一种方式，你可以用杠 flag 后面跟上一个名称去打印指定 g v m 参数的值。比方现在我们想要查看 use g e g c 参数的值是什么，就可以这么玩。杠flag。又是 g e g c，可以看到这个参数是启用的杠flag。这个参数还有一个使用场景，比如我们现在想要查看的某个参数，在这个结构里面并没有展示。比方我们现在想要查看一下线程栈的大小，里面并没有这样的一个参数。我们也是可以玩的。 thread sack size，可以看到线程站的大小是 1 兆。你想查看的参数可以到地址里面去查，同学们还记得前面的福利okay。

好。经过演示，我们发现建音符的能力还是比较强大的，为我们赋予了查看参数的能力，特别是查看 j v m 参数的能力。这样在定位 j v m 参数相关问题的时候，就能够有据可依了。另外，建英否还提供了动态修改 GM 参数的能力，这样在做一些调优的时候，不需要重启应用就可以生效了，为调优 GM 提供了便利。


好，下面来探讨第二款故障排查工具JMAP。 j map 和 j info 类似，也是一个实验性的工具。同时部分功能也可以使用 j h、s、d、 b 或者是 j c、m、 d 去代替。然后在一些版本的j、d、 k 里面对 windows 操作系统的支持也不是很好，需要以 j map 杠 h 的结果为准。


j map 这款工具的作用是展示对象的内存映射，或者是对内存的详细信息。建 map 的命令格式是这样子的建 map 跟上参数，然后进程。我们来玩一下 JMAP 杠 class States，26089，可以展示 26089 进程类加载的统计信息。你也可以使用 find lesser info 去查看指定进程里面等待回收的对象的信息。我们来玩一下。Finalizer。info，可以发现当前没有对象正在等待被回收。继续看杠histo。这个参数也非常的常用，它可以展示指定进程里面对应的直方图，也就是竞争里面指定的对象有多少个。 histol 还有一个子命令life，这样查看的是指定竞争里面存活对象的直方图。


最后最关键的一个命令dump，它可以实现转储加巴堆。所谓转储加巴堆，指的是把加巴堆里面的对象做一个快照，存储到一个文件里面去。这个文件也叫堆dump，或者是堆转储文件。 dump 命令又有 3 个子命令，我们来玩一下 j map 杠 dump 冒号 life 表示只转储存活的对象， format 对 b 表示以h、p、r、o、 f 这种格式去转储加把堆指定 file 等于一个文件，比如 my dump。点h、P、R、O、f，最后跟上进程26089。执行完之后，就可以在当前目录下面找到一个叫 my dump 的文件，在这里这个文件里面存储的是 26089 进程当前堆内存的快照。


至于怎么样分析文件，后面我们再来探讨。好。不难发现，建 map 这个工具可以让我们了解 GM 的内存分配信息，同时还可以生成对内存的快照，这也是我们实际项目中排查内存分配相关问题的一款利器。最后也为大家提供一个小小的拓展知识点。要想去 dump 堆，除使用 JMAP 以外，还有其他 4 种方法，同学们可以对照这个说明玩一玩。
