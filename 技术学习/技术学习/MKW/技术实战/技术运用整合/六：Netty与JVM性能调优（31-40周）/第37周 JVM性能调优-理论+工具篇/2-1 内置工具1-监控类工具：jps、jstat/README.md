这套手机主要整理了两大类的工具，

一是 jdk 内置的一系列工具，

第二类是一系列的第三方工具。

先来探讨 JDK 内置的相关工具，


这些工具从功能的角度来可以分成三类，

第一是监控类的工具，第二是故障排查工具，第三是可视化工具。而从可用性以及授权的不同，又可以分成三类。


一是正式支持的工具。正式支持表示这些工具会有长期的技术支持，不同的平台、不同的 JDK 版本之间，这些工具可能会有一定的差异，但是总体来说还是兼容的。


第二类工具是实验性的工具。这一类工具会被声明是实验性质，并且不会有技术支持，某一些工具甚至会在某一个新的 JDK 版本里面突然就没了。但是事实上这些工具也都非常的稳定，而且功能是比较强大的，完全可以用在生产。所以千万不要因为一看这个工具是实验性的就不学了，OK？第三是商业授权工具。


这一类工具主要指的是 g m、 c 以及 g f r 商业授权工具。用在商业环境里面是要付费的，但一般来说在个人开发环境里面是可以免费使用的。在咱们这套课程里面会探讨到这么多款的工具，下面我们就来一个系统的学习这些工具。先来看一下 JDK 内置工具里面的监控类工具。


第一个工具叫 jps，这是一个用来查看 j v m 进程状态的工具，它虽然是一个实验性的工具，但实际项目中用的非常的多。来玩一下GPS，可以看到这台机器里面所有的加法进程。这个工具还有很多的参数，比方杠 q 让输出的结果只展示进程号， j b s 杠 q 只有进程号，后面的这一堆就没了。又比如杠m，可以展示传递给妹方法的参数，你还可以多个参数配合使用。比方 j p s、杠 m l，这样既可以展示传递给妹方法的参数，又可以展示启动类的完整包名。这些参数里面用的比较多的是杠m、杠l、杠小v。另外还可以提一下，在类 Unix 机器上有一个和 GPS 类似的工具 PS 杠 EF graph。


Java，使用这个命令也可以查看 Java 进程。但是经过对比不难发现， j p s 使用起来比 p s、杠 e f 要方便一些，而且展示的结果可控性也要高一些，对吧？此外，你还可以使用 GPS 去监控远程服务器上的进程，指定一下 host ID，这么玩儿。但是要想查看远程服务器上的进程，需要做一系列的配置，这个配置还是有点小复杂的。所以有关如何连接远程服务器，后面我们专门划一节课来探讨。同学们目前知道有这么一回事儿就 OK 了。下面我们来看一下第二个工具 this date。这款工具用来监控 j v m 的各种运行状态。它也是一个实验系的工具，实际项目中也非常的常用。它的命令格式稍微有点复杂，这么长，我们不妨对照手机来玩一下。
