标记清除算法的优点是实现起来比较简单，缺点是存在内存碎片。另外，使用标记清楚算法的时候，分配内存的速度也会受到影响。这是为什么？大家想假设现在有一个比较大的对象，由于现在有很多碎片化的内存空间，想找到一块连续空间，就需要去遍历空闲链表，从而去查找哪一块内存可以存放这样的对象。在极端场景下，需要把整个链表全部遍历完，才能知道这个对象该分配到哪里去，又或者根本没有办法分配便利。空闲链表是需要时间的，所以分配内存的速度会受到影响。


标记整理的优点是没有碎片，它的缺点是整理存在开销，因为标记整理需要把对象集中起来，放到内存的一端，过程需要计算以及时间。并且对象越多，占用的内存越大，整理的开销也会越大。复制的优点是性能好，没有碎片。一般来讲，复制算法比标记清除以及标记整理性能要好一些，因为它不像标记清除或者标记整理那样，需要标记哪些对象是存活的，哪些对象可以回收。而只需要找到存活的对象，移动这个对象就 OK 了，所以性能上会有一些优势。复制算法的缺点是内存使用率低。从这张图就可以看出来了，我们一次只能使用整个内存的一半，内存使用率最多只会达到50%。
