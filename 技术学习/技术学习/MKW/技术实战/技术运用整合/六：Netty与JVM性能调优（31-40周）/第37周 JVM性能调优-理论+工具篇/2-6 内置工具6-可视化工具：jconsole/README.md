大家好，我是大木。这节课来探讨第二款可视化工具 j 空手。空手是一款基于 JMX 的可视化监控以及管理工具。这款工具非常的简单，也非常的省心，来玩一下。要讲启动借空手，只要执行借空的命令就可以了。 JCONSOLE，你可以选择本地的进程，也可以连接一个远程的进程，这里我们选择一个本地进程。关于如何连接远程进程，我们放到后面专门探讨。


这里我们选择 39548 进程，点击连接，这样就可以监控以及管理进程了。可以看到这 console 里面有 6 个子页面，其中概览展示了虚拟机运行数据的概要信息，包括堆内存使用量、线程个数、类、加载个数以及 CPU 占用率的曲线图。内存这一栏监控虚拟机内存的变化趋势，可以简单认为这是一个可视化的 jessiator 命令。这里我们可以看到堆内存的使用量逐渐变大。你还可以看其他的选项，比方查看 G1 垃圾收集器的异电源空间的变化。你还可以点击执行 gc 去通知 jvm 执行垃圾回收。


线程这一栏用来监控应用线程的个数、波动以及状态。当遇到线程停顿的时候，可以考虑使用这个页面的功能进行分析，相当于可视化的 jstack 命令。在这里还可以看到每个线程的详情。

**点击检查死锁，可以帮助你自动分析有没有死锁**。建工应用当前已经加载类的个数的变化趋势。可以看到我们的应用总共加载过 4746 个类，但是有 164 个类已经被卸载了，目前还加载着 4582 个雷。


讲到这里，同学们应该可以发现，概览这一栏里面的曲线图和内存线程类的那几个页面的信息是一致的。事实上，概览其实是这几个页面的信息聚合。好。接着看 VM 概要，展示了当前应用的一些概要信息，比方虚拟机的版本、虚拟机的厂商、虚拟机的进程号、当前应用活动的线程、曾经最多的线程数等等各种信息。


最后， im a bean 用来展示当前应用被g、 mx 管理的being。我们可以点进去看一看，可以看到每个属性的值。你还可以点击刷新，强制刷新 bin 的监控结果。不难发现，这些控作使用起来非常简单，非常的傻瓜化，同时功能也非常的强大，可以是实际项目中监控应用以及排查问题的一款利器。但是它不是最强大的，在下一节我们将会探讨一款更加强大的工具。
