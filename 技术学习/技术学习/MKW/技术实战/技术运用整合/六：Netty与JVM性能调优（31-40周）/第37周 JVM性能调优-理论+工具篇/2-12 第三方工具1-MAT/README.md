大家好，我是大木。这节课要探讨的是一款第三方工具，叫做 memory analyzer tool，也叫 memory Analyzer，简称mat。它可以作为独立软件存在，也可以作为 eclipse 的插件去存在。从名称就可以看出来了，它是用来分析内存的，它能够帮助我们快速找到内存泄露的问题，同时也可以帮助我们做一些内存上的调优。


m a t 的主要功能，同学们可以看一下。好在正式上手 m a t 之前，需要普及两个概念，第一是浅堆和保留堆，以及相关的一系列术语。首先，什么是浅堆？所谓浅堆，指的就是对象自身所消耗的内存。第二，保留级。所谓的保留级就是当对象 x 被垃圾收集的时候，有垃圾收集器删除的一堆对象，我们知道对象之间是有关联的对吧？某一个对象，它被回收的时候，也会收集它所关联的那些对象，只要这些对象都不可达，就会被回收了。为什么叫做保留级？这是因为如果 x 不被回收集合，里面的对象也都会保留下来，所以被称作保留级。


第三，保留堆保留的指的是某个对象保留级里面所有对象浅层大小的总和，也要想让对象保持活动时候所需要的内存。如果的通俗一点，那就是保留的。指的是对象 x 所保留的内存大小，也就是由于 x 还活着，导致了多大的内存没被回收，这就是 x 的保留堆。第四，是前导对象级的保留级。它指的是当某一个前导对象不可达的时候，所被释放的那些对象。比方 x 对象已经不可达了，有一切对象又关联到了 x 对象，他们也都不可达，这些对象也会被回收，对吧？这就是前导对象的保留级。好，下面我们来看一下。


对象引用图。 a 引用c，b， a 引用c， c 引用e， c 引用d，以此类推等等等。我们来用这里的术语描述对象之间的关系。第一行，他说强大对象是 e 的保留级是 e 和g。这是什么意思？我们来分析一下。假设 e 还可达，不能被回收，它将会导致 g 也没有办法被回收。因为 e 引用了g，所以 e 的保留级就是 e 和g。第二行。假设 c 没有办法被回收，它将会导致d、e、f、g、 h 都没有办法被回收。所以 c 对象的保留级是c、d、e、f、g、h。同理，由于 a 和 b 不能被回收，那么将会导致a、b、c、d、e、f、v、j、 h 全部不能被回收。所以 a 和 b 的保留级是c、d、f、g、h。这是前堆和保留级。


第二个术语，叫做支配术。支配术可以帮助我们快速的了解对象之间的依赖关系。我们来分析一下这张对象引用图。 a 引用c， b 引用c， c 引用 e 等等等。这种引用关系里面从起始节点到某一个节点的每条路径都必须经过 x 对象， x 支配了对象y。从这个图里面就可以分析出来， e 对象支配了 g 对象，因为从起始节点到达 g 的路径全部都会经过 e 对象，而 g 对象就没有支配 h 对象。这是因为从起始节点到达 h 的路径有两条， f 和g。在支配的时候有直接支配和间接支配。对于某一个对象，路径最短的支配者就是对象的直接支配者，否则就是间接支配者，这就是支配数。


理解这两个概念之后，就可以安装 mat 玩起来了。点击这个地址，根据自己的操作系统去下载就可以安装了。如果你的网络不好，也可以使用这边提供的百度盘地址进行下载。当然了，你也可以直接在 Elix 的插件市场上安装。 Mat 也是一样的。我已经下载好了，我们来安装一下。解压。安装非常简单，双击就可以打开了。


Overview 是对这个软件的一些介绍， Totorize 是对这个软件的一些教程。 Workbench 是这个软件的工作台，也是我们平时操作的地方。点击 file a query keep dump，这样就可以连接上本地的进程。比方我们连接 12875 进程，点击 next 会让我们做一些配置，我们保持默认就可以了。 finish 它会连上进程，并去做 Java 堆dump。做完堆 dump 之后，会自动给我们分析堆内存，产生报表，非常的方便。但是在实际项目中，我们用的更多的是 open hip dump 菜单，让 mat 去帮助我们加载已经存在的对 dump 文件。我们点击选择一个对 dump 文件，比如前面的麦当佛h、p、r、o、f，这样也可以产生相应的报表。界面分成了好多部分，同时还有好多按钮。我们回到手机看一下。


左上角叫做inspector，翻译成中文叫透视图。它可以用来展示对象的详细信息，比如内存地址、加载器名称、包名、对象名称等等等等。在这一块会展示类的近代属性和对应的值，对象实力的属性和对应的值，以及对象所属的类的继承结构。左下角的 keep dump history，它会展示最近分析过的对大盘文件，在这边有一个功能选择栏，有很多的按钮，可以做很多的事情。比如概览展示堆的直方图，查看支配数，用 OKR 查询对象等等。这里面的饼图可以用来展示保留大小里面最大的那些对象。我们来看一下点击可以展示出来这一块区域占用了多大。左侧会展示出这么大的内存。里面对象的一系列详情好不到。


手机 actions 会展示常用的一些分析动作，比方查看对内存的直方图，查看支配数，查看开销最大的对象，以及查看被多个 class loader 所加载的类等等等。 reports 里面则是一系列和报表相关的工具。立刻 suspects 可以用来查找内存泄漏的问题，并且可以对系统做一个概览。 top components 可以列出那些占用堆超过 1% 的所有组件。最后还有一个 step by step 里面有一个 component report。这个功能其实是一组功能的集合，它可以用来分析某一个包或者是某一个 class loader 下面的对象所存在的问题。比方可以分析再把 u 条包里面的类的实例，对象的一些使用情况，例如包里面对象的重复字符串空集合的使用率等等各种问题。
现在我们已经对 Mat 的主要功能有了一个概要的认识，下面我们来挑几个有特色的功能演示一下。第一个是支配数，这样就可以查看支配数了。在对象上面右击 immediate dominators，这样就可以查看这一个对象的直接支配对象，点击在这里还有一个帮助。从这里的说明可以发现，它会展示是哪些对象的存在，而导致了这里的 default listable bin factory 不能被回收。


我们点击finish，这样就可以知道了原来是根对象的存在导致了这个对象不能被回收。好，关掉。我们再次右击。有一个修 return 的set，它可以查看这个对象所持有的保留级，点击finish，这样我们就可以知道对象的保留级有这么多。关掉 top consumers 也非常实用，点击它可以看出来当前各个包的占用情况。并且在这里还有个 biggest objects，可以分析出当前比较大的对象。如果你的低占用不太正常，可以到这里看一下。好回来。


还有一个特别实用的功能，叫做 leak suspects。点击它可以帮助我们分析可能存在的内置泄露问题。在这个下面，他甚至可以给我们展示出来他怀疑的那些对象。比方在这里，他说这个对象是由克拉斯 loader 加载的，占用了这么多的内存，可以发现居然占用了 10% 以上的内存，他就怀疑这个对象可能是有问题的。假设你的项目确实出了问题，你就可以点 details 去分析具体的原因，干掉。另外，这里的功能菜单也都非常的实用，比方在这里有一个 Java basics。


点 GC roots 可以查看所有的 GC root，点 system properties 可以查看系统属性。点 thread details 可以查看线程的运行情况，点 finalize overview，可以查看垃圾收集器的各种情况。总的来说， mat 的功能非常的丰富，可以在内存分析这个领域它做到了极致。课上我们也只是演示了 m a t 常用的一系列功能，其他的功能同学们可以多点在这里。弹幕也给同学们布置一个作业。请同学们按照 help 菜单 help contains 点击这个里面的说明，去通读一下 m a t 的帮助文档。至少要把 task 里面提到的菜单都去亲自点击一下，玩一玩，至少要留一个印象。这里面的内容也都非常的实用，后面的实战也会去用到。这节课就到这里，谢谢大家。
