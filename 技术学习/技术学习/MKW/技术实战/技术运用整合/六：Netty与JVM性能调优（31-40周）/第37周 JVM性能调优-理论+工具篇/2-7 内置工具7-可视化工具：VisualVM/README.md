大家好，我是大木。这节课来看到 visual VM，也叫接 visual VM，前面在讲解线程池调优实战的时候已经用到过工具了，对吧？这节课我们就来详细探讨一下。在正式讲解 visual VM 之前，必须提醒一下不同版本的 visual VM，它的界面布局、使用体验甚至是功能都是有差异的。这里大部分使用的是当前最新的版本。二点零点一建议同学们在学习的时候使用和弹幕相同的版本，避免踩到不必要的坑，等有能力之后再去选择自己喜欢的版本。


OK，回到正题。微信 v m 官方说自己是一个多合一的 Java 故障排查工具，它也是目前最强大的监控以及故障排查工具之一。一会儿同学们就会有直观的感受了。下面我们就来对照手机来体验一下 j d k 8 以及更低的版本。 visual v m 是内置在 j d k 里面的，要想使用只要执行 j v m 命令就可以了。还记得在线程值调优时代一节，我们就是这么玩的对吧？因为当时我们使用的是 j d k 8。但是从 j d k 9 开始， visual VM 的用法就会有所不同了。官方把 visual VM 给独立出来了，所以需要手动下载，点击这个地址就可以下载了。


目前官方提供了两个下载方式，一种是 ZIP 文件， ZIP 文件能够适用于各种操作系统，第二种是 DMG 文件，这种方式是 macos 专用的。这里但不建议大家下载 Zippo 包的方式。如果网络不好，也可以使用这里提供的百度盘地址加速下载。下载完成之后解压就可以玩起来了。我已经下载好了，解压切换到 b 目录。对于 Linux 或者是macos，执行 visual VM 文件就可以启动了。
对于windows，双击 visual VM Exe enter 启动。笑了。和接 console 类似， VISUV m 也会自动带出当前启动的本地进程。当然你也可以远程连接。这里我们依然选择一个本地进程去玩儿。至于如何远程连接，我们也放到后面专门去讲。这里我们就选择 3219 进程去玩儿。


双击可以看到有 5 个子页面 overview 展示了应用的概要信息，相当于可视化的j、p、 s 以及 j info、 monitor 监控。这里的图表展示了 CPU 内存类线程的曲线图。 perform g c 按钮可以通知 jvm 执行垃圾回收，我们可以点击一下，可以看到内存占用下降了。
