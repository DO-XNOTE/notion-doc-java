因为反编译的是 class 文件，这样就可以看到 class 文件里面的内容。我们可以把它输出到一个 TXT 文件里面去。好，我们用 idea 打开一点TXT，来分析一下反编译之后的结果。可以看到一个 class 文件包含了几部分。第一部分是类的一些描述信息，我们记一下，它记录了 class 文件存储的位置，什么时候修改过 class 文件的 MT5 值，以及从哪个价位类里面编译出来的。第二部分还是一些描述信息，主要描述了这个类是用什么样版本的 JDK 编译的， 52 的表示是JDK8。第三部分是产量池。第四部分，是字段的信息。最后第五部分是方法的信息，可以看到方法的描述符，方法的访问权限。

在这个阶段里面， jvm 会先执行 class init 方法。 class init 方法是由编译器自动收集类里面的所有静态变量，它的复制动作以及静态语句块去合并而成的，也叫类构造器方法。 class need 方法里面的代码执行顺序和源文件里面的顺序是一致的。我们来运行一个例子看看。这里我们运行一下，可以看到等于1。我们使用了一个静态变量，在这里又有一个静态代码块，先把它赋成0，又把它赋成1。 class 妮的方法会把这两段合到一起，变成一个卡西尼的方法。所以 i 等于1。我们再把顺序调一下，再次运行，可以发现变成 0 了。


第二个特性子类的 class init 方法被调用之前，会先调用父类的 class init 方法。第3， jvm 会保证 class init 方法的线程安全性。此外，在j、 v m 初始化代码执行的时候，如果实例化了一个新对象，会调用 INIT 方法对实例变量进行初始化，并去执行相应构造方法里面的代码。我们再来运行一个例子，看看，在这个类里面有静态代码块，有构造方法，还有魅方法，运行的顺序是怎样的。应急下看看。可以看到，先执行了静态代码块，执行了 main 方法，再之后执行了构造块，最后是构造方法。我们再来看一下这个例子，在 j m test 5 里面的面，方法实例化了一个 SUB 对象，而 sub 类又继承自 super 类。 JVM test 5， super 以及 SUB 都有静态代码块。


构造方法以及构造块来看一下是怎么样打印的。可以看到首先打印了 GFM test 5 静态代码块，也就是这一段并没有去执行构造块以及构造方法的代码。很好理解，因为我们没有去 new GFM test 5，当代码执行到这一行的时候，他去 new sub 对象 new 之前，他去执行这里的 classy 尼的方法对不对，于是他就去执行这里。但是这里又说了子类的 class 尼的方法。调用之前，会先调用父类的克拉斯尼的方法，于是会先去调用 super 类的 study 方法，再去调用 sub 类的 study 方法。调用完成了之后，又去调用构造块。因为在 test 4 里面我们可以看到构造块执行的顺序在构造方法之前。于是先执行 super 类的构造块，再去执行 super 类的构造方法，最后再执行 sub 类的构造块以及 sub 类的构造方法。
