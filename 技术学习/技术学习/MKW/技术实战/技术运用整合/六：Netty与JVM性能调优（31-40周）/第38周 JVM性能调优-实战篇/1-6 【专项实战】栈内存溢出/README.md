怎么样让他能够创建更多的线程？比方我想让他创建1万个线程。要想实现这个目标，我们可以分析一下。


首先我们应该能够想到去减少 xss 配置，这样每个线程这样的内存就会少一些。我们知道栈是线程独享的吗？这样对于相同的内存总量就能够创建更多的线程了，对吧？第二，机械的总内存减去操作系统所占用的内存，减去堆内存减去。方法去内存减去程序计数器占用的内存，再减掉直接内存。剩下来的内存就由虚拟机站以及本地方法站去分配了。

因此我们应该想办法尽量的减少这些内存的占用，这样占能够分配的内存就会越大，能够创建的线程就会越多了。


第三，应该尽量杀死其他的程序，为应用腾出内存。第四，操作系统对创建线程的数量也是有限制的。比方 Linux 操作系统，它的限制有这么 4 个，我在这里总结出来了，同学们可以看看。

你需要把限制设置的大一点，不然比方操作系统最大只允许你创建 2500 个，你根本别想创建1万个线程。好结果分析不难发现，要想让应用运行更多的线程，既然是把堆内存、栈内存以及元数据空间大小都设的更小一些，对吧？这还是挺有意思的。正常我们在做调优的时候，都是把某个值设大，而这里居然是把值设小。


最后这里也说明一下，由于这个类在运行的时候会造成机器假死，并且对于 Mac 操作系统，它限制最多创建的线程数是 2048 个，所以在我这台机器上运行是得不到咱们想要的结果的。而前面给大家看到的 Linux 服务器配置又特别的低，
