所以配置这些参数选项也是学习 JVM 的难点之一。但事实上，这些参数还是有一定规律可循的。这一节就来探讨一下 g m m 参数选项的分类，以及各类参数的用法。针对这个话题，大漠专门编写了一篇手记，一起来看一下。


这篇手记也是基于 jdk 11 去编写的，当然了，对 jdk8 也可以兼容。总的来说， j v m 把它的参数选项分成了四大类标准选项、附加选项、高级选项以及不该去使用的选项。先来看一下标准选项。标准选项是用来执行常见操作的。这些选项各种厂牌的虚拟机都会去支持，不管是我们熟悉的 hotspot 虚拟机，还是其他的虚拟机，比如 openJ9 虚拟机、猪肉虚拟机等等，都会去支持。要想查看标准选项，只要用 Java 杠 help 就可以看到了。 copy 粘贴下有这么多的标准选项。标准选项的格式是不统一的，有的以杠开头，有的以杠 key 等于 value 开头，有的以杠杠开头。一切以加为 help 的结果为准。在结果里面，它会展示配置的格式。我在手机里面也总结了常用的标准选项，有这么多，同学们可以看一看。


再来看一下附加选项。附加选项在以前叫做非标准选项，结果到了 jdk11 里面改了个名字，叫做附加选项了。这些选项是 hot spot 虚拟机里面的通用选项，但是其他厂牌的j、v、 m 不一定会支持这些选项，并且这些选项在未来也可能会发生变化。
附加选项都是以杠 x 开头的，用 Java 杠大 x 就可以查看所有的附加选项了。 


Java 杠大 x 有这么多的附加选项，附加选项的使用格式还是比较统一的，大部分附加选项的格式都是以杠 x 跟上属性名就 OK 了，比如杠 x carp、杠 x Int 等等。但是也有例外，比方设置最大定内存就得用这种格式。在 Java 杠 x 的输出里面，也会展示出每一个选项的配置格式。


常用的附加选项有这么多，这里面有很多选项，前面的课程已经用过了。比如杠 x perp 以编译模式运行，杠 xInt 以解释模式运行，杠 x mix 的以混合模式运行以及杠 xshare Mod。在演示 visual Vim profile 的时候，我们把 x share 设成了 off 关闭掉的类数据共享，把应用搞崩溃掉对吧？附加选项里面其实用的最多的是几个设置内存的参数。比方 x m n 设置新生代的初始值以及最大值， x m s 设置对内存的初始大小， xmix 设置对内存的最大小， xss 设置线程帧的大小。
好，接着看。


第三就是高级选项了。高级选项是给开发人员去使用的，它可以让我们控制 JVM 的各种行为。所有的高级选项都是以杠叉叉开头的，你可以使用这个命令去查看高级选项。 copy 粘贴有这么多好几百个。也可以使用 jhsdb 去查看高级选项。演示一下 copy 输入 jps 查看当前运行的 Java 进程。粘贴输入精灵号 enter 报错了， control c 退出。使用 Java 杠 version 查看一下当前的 jdk 版本，十一点零点七。还记得前面讲过接 hsdb 没有办法在 jdk 十一点零点七下面正常工作。所以我们可以用 Echo dollar Java home 展示一下当前的 Java home，然后用 export Java home 等于，十一点零点四修改环境变量。


最后再用 Java 杠 version 查看 Java 版本号，已经变成十一点零点四了。做完这些之后，还需要确定我们的 Java 应用也是使用 j d k 十一点零点四运行的，所以我们 file project structure 把接力给版本也改成十一点零。点四。 apply OK。重启。


之后再用j、t、 s 拿到进程号，使用接h、s、d、 b 连接上进程，进去之后输入flags，这样也可以拿到高级选项。好回到手机。高级选项的使用格式非常的统一，如果选项的类型是不论用加号表示打开这个选项，用减号表示关闭这个选项。比方如果想要配置这里的 zero t lab，就可以用杠叉叉加号或者减号， zero t lab 就可以了。而如果选项的类型不是不能一律用杠叉叉选项，等于值的格式好。


由于高级选项非常的多，所以j、d、 k 又对高级选项做了一个细分，主要分成四大类高级运行之选项、高级 g t 编译器选项、高级可服性选项以及高级垃圾收集选项。在这篇手机里面，我也整理了每一个细分下面的常用选项。比方常用的高级运行时选项有这些，这里面有很多，相信同学们也已经见过了，比方 max g c pause minutes。
