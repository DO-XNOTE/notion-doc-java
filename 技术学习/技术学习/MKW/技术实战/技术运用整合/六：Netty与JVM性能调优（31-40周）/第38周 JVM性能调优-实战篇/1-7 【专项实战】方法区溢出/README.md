第二种可能是夹杂着类的种类太多而导致溢出。对于这种情况，主要可以分为以下几种。第一种可能是使用 CG library 或者其他动态代理的操作库，生成了大量的动态类。假设你使用过c，x、f， x 等等框架，就比较容易遇到这种问题。因为c、x、f， x 都大量的使用到了动态代理，甚至包括spring， hibernate 等等框架底层也都用到了 CG library，增强的类越多，就需要越大的空间去存储类的定义。


第二种可能。如果你的项目使用了 jsp，并且 jsp 特别的多，也可能会出现这种问题。因为 jsp 是在第一次被访问的时候才会被编译成 Java 类。在极端场景下，可能应用在启动的时候是没有问题的，但是后来访问了一些页面，对应的 jsp 被编译成 Java 类了，结果竟然把方法局给打满了，报内存移出了。


第三种可能。脚本语言动态类加载也可能会导致这种问题。像大木目前经手的一个项目，就是一个混合编程的项目，里面用到了 Grovi 脚本，就遇到了由于动态类加载而导致原空间溢出的问题，感兴趣的同学可以看一下这篇文章，和弹幕在项目里面遇到的问题基本上是一样的。当然了，如果你不用 groovy 脚本是不会遇到这种问题的。
