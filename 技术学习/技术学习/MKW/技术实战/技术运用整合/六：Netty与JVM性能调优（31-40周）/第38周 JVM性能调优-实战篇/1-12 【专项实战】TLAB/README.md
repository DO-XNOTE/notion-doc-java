JVM 会为每一个线程分配一块 TLAB 区域。 TLAB 其实占用的是 Eden 电源的空间。


Jvm 为什么要设计TLAB这样的一块区域，主要还是为了加速对象的分配。我们知道对象一般都是在对内存分配的，而对内存是线程共享的，因此可能会有多个线程同时在堆上分配空间。于是每一次的对象分配都需要去做同步处理。 JVM 底层采用了 case 以及失败重试的方式，从而保证分配不出问题。但是如果竞争非常的激烈，在堆内层分配对象，效率势必会下降。而实际项目中，对象分配可以是 Java 里面最常用的操作了，对吧？我们 new 对象的时候都要去做对象的分配，因此 jVm 设计了 TLAB。由于 TLAB 是线程专用的内存分配区域，而且会为每个线程分配 TLAB ，所以可以避免对象分配时候的线程冲突，并提升对象分配的效率。
