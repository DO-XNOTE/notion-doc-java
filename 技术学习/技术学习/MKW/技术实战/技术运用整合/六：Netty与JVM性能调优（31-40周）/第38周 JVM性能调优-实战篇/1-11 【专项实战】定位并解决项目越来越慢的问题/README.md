我们把项目先停掉，然后来分析一下。导致项目变慢的。

第一个可能是垃圾收集的 stop the world，时间过长很好理解，比方垃圾收集非常的频繁， stop the word 的时间越来越长，这样也会导致你的项目执行越来越慢。


第二种可能是项目依赖的资源导致项目变慢。比方你的数据库数据量增加的特别快，于是查询变慢了，这个时候你可以去做一些 MySQL 优化，或者是分库分表等等。又比如网络出了问题，比方网络抖动，或者是网络被攻击了，也可能会导致项目变慢。


第三是科特开始慢了，导致即时编译器停止工作，用解释模式运行，这样项目也会变慢。


第四，线程争抢过于激烈，导致你的代码老是抢不到 CPU 时间面去执行。


第五，服务器出问题了。比方操作系统出问题了，这个时候重启一下一般就 OK 了。重启大法针对操作系统的问题特别有效。又或者服务器上其他的进程把服务器的资源都抢走了，于是导致你的项目执行很慢。比方你的服务器中毒了，被人挂了木马。
