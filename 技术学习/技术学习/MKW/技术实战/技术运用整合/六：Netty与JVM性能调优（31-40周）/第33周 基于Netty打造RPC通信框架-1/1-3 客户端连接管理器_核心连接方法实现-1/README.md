好了，我们环境说完了，紧接着我们这节课也是实际编码的第一节课。我们还记得上节课老师已经把整个的整体的我们要做 RPC 的模块跟大家梳理清楚了。我们现在想做一个RPC，我们应该从哪方面开始入手？其实你就可以把 RPC 这个东西当成一个 NET 的通讯模型一样，客户端首先去建立连接，去请求 server 端发送一条请求， server 端解析完了之后给我回送一个响应。就是这么一个非常非常简单的过程。
我们可以先从客户端开始，我们现在使用RPC， RPC 有一点就是我们的连接，我们需要怎么去管理起来，比如我们写一个 net 的 client 端，我们每次去创建一个 client 端的时候，我们是不是要对连接做一个管理。所以我们从我们整个的 RPC 自研的实现的过程，我们就从 0 开始，从我们的叫做 connector manager 就是我们的连接管理器开始。我们要从连接管理器入手。连接管理器的目的无非就是我们建立的所有的连接都是要通过它去帮我们去管理的。首先我们粉红色的表示什么？这里边粉红色的就表示我们实际的实体类，绿色就表示我们可能需要的方法。后面的黄色的就是方法。具体的实现的步骤和细节，老师在这里已经给小伙伴们都列出来了，包括后面有好多好多。我们整个 RPC 框架，根据老师所画的图一步一步的去进行编码，这样你脑海里有一个全局的一个框架。
