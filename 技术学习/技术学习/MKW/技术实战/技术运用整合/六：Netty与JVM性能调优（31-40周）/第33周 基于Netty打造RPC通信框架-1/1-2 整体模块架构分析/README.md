如果你想做一个最传统的 RPC 通用框架，是我们的 remote call，远程的程序调用 process call 其实就是我作为一端，你作为另外一端，我调用你，你执行给我返还结果。说白了，我调用方，我是 client 端，或者我是 consumer 端，你是服务的提供方，你是一个 server 端，你是一个 provider 端，你提供服务给我去远程调用。这两个过程其实是非常单纯的，我们只需要实现一个 server 就好了。至于后面，我们肯定会有像 double 这种，它会有注册中心registry，我们把提供服务的那一方把服务去 export 去暴露出去，暴露给zookeeper。然后我们 client 端，也就是 consumer 端，主动的去做一个import，做一个拉取服务列表的工作，去针对于它有的服务缓存到本地，生成一个 proxy 代理去进行相互的这种 RPC 直连的调用，这是double。所以在实现框架。
