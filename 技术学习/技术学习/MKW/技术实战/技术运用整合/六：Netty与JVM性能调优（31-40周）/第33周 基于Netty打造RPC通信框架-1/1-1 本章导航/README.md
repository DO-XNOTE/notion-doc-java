我们接下来看一看我们后半部分的内容有哪些。其实我们就要做 3 件事情，只要做好了这 3 件事情，你就能够跟着老师从头到尾的去实现一个 RPC 控制框架。让我们看一看到底是哪三件事情？首先第一件事情，我们既然要基于 Nike 去写一个 RPC 通用框架。

首先第一点，你自己是不是在脑海中应该构建一个最简单的 RPC 通信的程序。起码你要知道肯定会有可岸段，肯定会有 server 段，怎么去通信。如果不基于native，我们只能去使用这种 IO socket 的方式，我们可能会用到 socket 里边的 output swim 解析出来，相互的去传输数据。
除了这个以外，可能还会用到一些动态代理对不对？像我们的 client 端，肯定得需要类似于这种 proxy 的方式。如果熟悉动态代理的小伙伴们肯定会知道，要实现一个 Java 里边反射的一个接口，就是我们的 invocation handler。好了，还需要哪些技术？比如编解码，比如反射，都需要用得到。
我们最开始的第一块内容，老师是以图文的方式给大家展现的，因为其实比较简单，我没有必要花很大一块时间去一行一行的领大家去敲。我希望把课上的时间放到最有意义的事情去做。源码也会给大家小伙伴们花最快的时间在脑海里建立一个 RPC 通讯模型。接下来第二件事情就很简单了，我们先对整个的RPC，我们要基于 Nike 去做这件事情。我们来看一看我们要实现哪些模块。我们到时候老师会给小伙伴画一个模块的架构图，做一个设计和分析。第三件事情，老师会给大家一个非常非常完整的，从 0 开始基于 Nike 去完成一个 RPC 通用框架。我期望你跟着老师走。完了这一个过程，你真的就对 Netty 非常熟悉了，起码可能你会对设计模式非常熟悉，以后你再去面试或者在工作中都会有一定的自信。 OK 这就是我们整个后半部分的课程介绍。其实就这 3 个点，希望小伙伴们一起跟我去坚持走下去。小伙伴们加油。好了，这节课我们就讲到这。
