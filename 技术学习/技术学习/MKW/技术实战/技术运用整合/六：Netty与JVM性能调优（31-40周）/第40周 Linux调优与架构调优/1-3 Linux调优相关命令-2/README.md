大家好，我是大木，下面来探讨 Linux 调优常用的一些命令，通过这些命令可以帮助我们了解 Linux 的运行情况，进而在操作系统层面定位性能瓶颈，做到 Linux 系统。相信大家首先想到的是它有一堆命令，对吧？咱们这套课程也不能免俗，也是基于命令去玩的。考虑到命令非常的多，而且市面上的文章非常的散，质量也是参差不齐的。因此大漠花了整整一周的时间，对照官方文档整理了一套手机来看一下。课上将会探讨这几类命令。首先是一些入门级的命令，接着是系统级的命令，用户管理相关的命令，cpu 调优相关的命令，进程相关的命令，内存调优相关的命令，磁盘调优相关的命令以及网络调优相关的命令。

第一是杠杠help。通过杠杠 help 参数，可以看到命令的简易帮助文档。比方 shutdown 这个命令的帮助文档可以使用 shut down。刚刚help，他会告诉我们下车档的命令格式是这样子的。有这么些参数，但是刚刚 help 它有两个问题。一是展示的结果太简单了。因此对于复杂的命令，杠杆 help 的参考价值是比较有限的。


第二，有一些命令根本没有杠杆 help 参数。比方 top 这个命令就没有杠杆 help 参数。他说你传入了一个不合适的参数。
