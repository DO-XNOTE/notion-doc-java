Prometheus 刚启动的时候，状态是 inactive 的，因为这个时候还没有加载这个规则。当 Prometheus 针对告警规则进行计算，发现满足条件的时候，就会变成 pending 当 for 的条件。也就是这里规则已经达到了持续的时间，这个时候就会变成Firing。
监控实验我们发现，当监控指标达到告警规则的条件之后，就会触发告警。但目前的告警能力就像玩具一样，对于生产环境显然是不够的。实际项目中，我们一般都希望告警以短信、邮件或者是其他的方式通知到相关的人员。就像 Zabbix 的高级。如果想要实现这种目标，就要用到 alert manager 回答PPT。
