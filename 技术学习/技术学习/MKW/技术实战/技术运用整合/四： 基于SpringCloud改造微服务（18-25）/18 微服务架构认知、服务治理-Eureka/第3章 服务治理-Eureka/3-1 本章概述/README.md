我们则是通过视频的方式进行讲解。同学们如果更关注实战部分，其实倒也是可以跳过图文来直接看弹幕视频的。但是我更建议大家在看完视频以后，一定要再回过头来学习图文课程中的理论知识，夯实知识的根基。因为我们如果出门，面试官他其实是不会问说这个功能是怎么用的，还不太关注这些应用层面的事情。他更希望了解的是这个功能底层是如何实现的这些偏底层的细节。所以从这个角度来说，理论知识同样也是非常非常重要的。


在全链路中，我们将要全面深入尤瑞卡的七个核心流程，它们分别是注册、服务发现心跳续约、服务下线以及剔除和自保这七个流程，它构成了尤瑞卡中服务生命周期的所有状态的流转。接下来一个环节是源码品读，我们学习不仅要知其然，也要知其所以然。所以我特别安排了源码阅读的环节，带大家深入源码学习，有瑞卡里面的核心流程服务注册心跳检测和服务续约。它在底层是如何实现的？一方面，我们可以学习到开源项目中的一些小技巧，打个比方竞争状态下的同步控制啦，或者是如何应用设计模式对我们的代码层次结构上进行解耦。另一方面，我们可以迅速积累阅读开源项目源码的经验，这个对我们以后的工作学习都是大有益处的。


接下来我们要从架构层面去探讨如何对注册中心进行高可用的改造，我们会动手搭建一个双倍份的注册中心。接下来就是万众瞩目的电商项目改造环节。同学们可以把这个环节理解成老城区改造计划，我将要大刀阔斧地对现有的电商项目进行一番彻头彻尾的改造。同学们知道它最早是基于 sprint boot 搭建的一套单体应用。对不对？在后面一个章节，我们利用了很多的中间件技术以及分布式的技术来丰富这个应用。那在微服务的章节当中，我并不会对他的业务流程做过多的改造，而是从这个架构层面将整个项目基于 spring cloud 提供的组件改造成微服务的架构。最后一个环节课程小结与作业。正所谓学而实习之不易跃乎。那大家学完了之后要进行一定的复习来巩固知识。我将要提供一个思维导图形式的小结，让大家强化每个环节中学到的知识，并且布置一些轻松愉快的小作业，让大家课后来完成。好的。以上便是尤瑞卡环节所有的课程目录，接下来我们就要进入激动人心的学习。
