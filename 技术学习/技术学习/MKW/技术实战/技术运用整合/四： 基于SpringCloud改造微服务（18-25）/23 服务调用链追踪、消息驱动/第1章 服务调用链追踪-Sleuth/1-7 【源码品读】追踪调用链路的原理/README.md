为什么在 sloth 的文件夹里面可以看到很多其他组件呢？比如说 web 比如说 spring cloud 的一代目网关组件 zoom 还有 job scheduling rx Java reactor high streaks 甚至连 GR PC 都有。


为什么？这就要说到 sloth 的特殊性了。作为链路追踪的关键一环，咱 sloth 的核心指标是什么？是要在上下游调用链路上打标。那么就是说不管你上游是 GR PC 还是 rx Java 也不管你下游是 zoo 还是一个 web 调用它要做到左右风园，老少通吃。也就是说它必须针对每种不同的调用方式开发一套自己的适配器。因为它既然要在不同的组件、不同的调用方式之间传递这个链路追踪的信息，那必然要对不同的协议以及不同的底层通信通道有所适配。比如说我们通过 web 调用的形式，那它底层是用的 HTTP 协议，那它自然和其它的组件有那么些不同。


打个比方，如果它的下游是一个 messaging 组件， messaging 是谁呢？ messaging 是指消息组件，也就是 Kafka 和 rabbit MQ 那你通过 web 形式的调用和通过 Kafka 和 rabbit MQ 形式的调用，它在底层的通信机制上一定是不同的。
