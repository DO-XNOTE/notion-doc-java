网的各位同学们，大家好，我们终于来到了整个微服务体系阶段课程中的最后一节。在这大结局里，老师要跟大家讲一点不一样的。这一节当中，老师不再是老师，我想以一个朋友的身份和大家分享一下我过去这些年工作经验中的感悟。所以我把这一节课程的名字叫做放飞自我。这里有我刚刚开始职业生涯时候跌过的跟头，也有后来在工作中的思考和顿悟，还有我过去这十年间的思想变化、工作方式的变化，希望能对大家以后的工作有那么一点的启发。那我们现在就开始放飞自我的过程。

第一句我想说的话就是走到聚光灯下，我想大部分技术人员还是比较害羞的，不太善于表达自己，而且我们都喜欢确定的事情，不喜欢在什么事都没有确定的时候承担责任和风险。所以面对需求变更或者新的机会，可能我们内心会有那么一点点的退缩。我没有做过这件事儿，需求还没有搞明白。不好意思，我没有这方面的经验，可是我们技术人员又是非常勇敢的。尽管我们心里这样说，甚至嘴上这样说，可当你真正的埋头在这件事情上工作的时候，我们可以把结果做得非常漂亮。可是做出好的结果我们又开始谦虚了起来。我们心里总想着自己只是一个写代码的，好做好本职工作也就可以了，这些功劳。还是你们来讲，不要叫上我。其实我们都渴望自己的工作成果，自己的能力可以被大家认可，只是由于自己心里的小害羞，只能默默的在幕后贡献战斗力。对于技术人员来说，从幕后走到聚光灯下，告诉大家自己做出的成绩似乎是一件非常难的事。


还记得我刚到阿里的第一个主管跟我说过这样一句话，在幕后摸摸的码代码其实就是技术人员的舒适圈，我们必须要跳出这个舒适圈，才能获得更多维度的成长。就像健身人士锻炼肌肉一样，我们要从不同维度锻炼自己的个人能力。所以同学们，我们可以尝试着做这样一个改变，从幕后走到聚光灯下，大家主动的勇敢的去承担一个项目，我们为结果来负责，勇敢的表达自己的想法，让别人知道自己的工作成果，扩大自己的影响力。


为什么鼓励大家主动走到聚光灯下，因为摆在我们眼前的一个残酷的现实就是会教的孩子才有奶吃。我们技术人员都是勤勤恳恳的，老黄牛的类型，非常的低调，写起代码来可谓是人狠话不多，但是一声不吭，老老实实的干活，真的在职业发展上对我们有利吗？ please dont be too nice 不要让别人成为我们的代言人，别人说什么就是什么。我们交付的项目，我们做出的东西，自己的劳动成果是一定要让别人知道的。表达能力、发言能力可能是我们技术人员的短板，但这并不是不能锻炼的。如果我们做了一个 10 分的项目可以打满分。但是由于我们的低调不善言辞，从我们嘴里讲出去，可能只有五毛钱的特效了。


就像阿里的晋升答辩一样，你会发现两个人在同一个组里工作，所做的项目也相同，他的工作内容基本上又是非常相似的，但是答辩出来的晋升结果却是截然不同的。像前面那个例子，我项目做得非常漂亮，做了 10 分，但是我讲出去只有 5 分，这是非常可惜的。我们可以换一个思路来讲这个问题，如果现在有一个项目，它并没有完成，我们大概做了 5 分。当我们向别人做 presentation 的时候，但是我们可以照八分的效果来讲，这三分并不是吹嘘，而是你对未来业务的思考。所以我们要让别人真实的知道你的付出和思考，不要因为自己的害羞或者低调，让你的工作成果打了一个折扣，这一点我们真的要学习印度工程师。


大家知道，整个美国的互联网圣地硅谷已经被印度工程师占领掉了，从现在硅谷华人和印度人可以平分秋色的场景已经再也见不到了，现在全部是印度工程师的天下。如果我们说讲究低调谦逊的中国人是这种做十分讲出去只有五分的话，那么印度工程师恰恰就是做五分可以讲到八分。所以同学们如果想要在职场上获得好的发展，真的不能过分低调，要大胆秀出来。这大胆秀出来的部分，可不是叫大家去吹嘘，5分做到 8 分。这其中的三分可不是吹出来的，要靠什么？要靠多思考和多想。


大家可以试着问自己这样几个问题，你目前工作在的这个项目，它有什么现存的问题？这个项目或者说业务，它未来还有哪些改进的点？从业务角度，你有没有思考过他还有哪些没有被挖掘的需求？如果这些问题在你心中都没有答案，同学们可能要花点时间反思一下。没有问题并不是一件好事，它可能代表着我们平时在工作中思考的太少。有的时候在工作中我们也要适当的跳出程序员的视角来看待问题，去思考作为一个技术人员以外的职责，关注和思考技术背后的那些真实的业务。有了自己的思考，有了想法，我们就要把这些知识和智慧总结沉淀下来。请总结可以帮助我们梳理工作中的思路，总结出一套适合自己适合业务的打法。而且这个总结他的听众不只是我们自己，他不光是要做给自己看，有时也要适时的把总结分享出去，让别人知道你做了哪些思考，这也是扩大自己影响力的一个机会。


比如说阿里里面非常流行什么叫周报，每周你要把自己当周的工作总结，所思所想，写成一份周报发给整个部门。其实很多同学就是借助这个机会，对部门老板那里留下了非常好的印象。多思考、多交流，多让别人了解自己，分享自己的想法，有更多思维上的碰撞。似乎这些东西看起来并不像是一个正儿八经的技术人员。我们眼中的技术人员是什么呢？那发明 Linux 和 get 的大牛 Linux 曾经说过这样一句毒鸡汤，相信大家都听过叫 talk is cheap show me the coat 人家大牛都发话了，多说无意，写 code 才是真本领。所以这句话被祖祖辈辈的程序员奉为圣经一样。


不过我们仔细转念一想，这为什么叫毒鸡汤？大家看看这句话是谁说的，Linux说的，我把它类比一下。就像马爸爸说，我这个人生做过最后悔的一件事就是创办阿里巴巴。再比如马爸爸说我觉得这个人每个月赚 2 万块钱是最幸福的，这口鸡汤喝起来是不是有那么点咸。同学们。所以每句话，我们要看说这句话的人是谁，就像 Linux 这种业界丰碑一样的存在，他说的任何一句话都能变成圣经。但是我们要好好思考一下。真的并不是所有人都是 Linux 的，她的成功也绝对不仅仅是努力而已。作为普通的技术人员，我们真的要认清现实，有的时候闷头干活真的不是一件对职业发展有利的事情。
所以我的观点是 talk is not cheap 什么样的 talk 才 not cheap 那就是融入了自己的思考、自己的想法，还有总结的 talk 所以满天流星夸夸其谈的 talk 是一文不值的。但是融入了个人思考和想法的 talk 是值得与大家分享交流的。


说到分享和交流，我这里还想提一个叫说服力的关键词，因为老师在外企工作的时间也非常长，所以有大半时间都是在跟各国不同文化的人进行交流。其实老美相对于中国人来说，在沟通交流当中非常的坦诚，他们的沟通方式是 I see what I mean 我所说的就是表达我自己的个人意愿。但是这种方式如果在咱中华文化下，你这样做可能会被人认为是楞头青。


对不对？其实技术人员大多是这种性格，说什么就是什么，我想说的话，我的喜怒哀乐甚至都表现在脸上。为了一个技术问题，为了自己在 it 业界的信仰，比如说 PHP 究竟是不是最好的语言，为了这些问题都可以争得面红耳赤。其实这么直来直往可能并不太容易让别人接受自己的意见和想法。那我们沟通和说服力最终要达到的目的是什么？就是合作，让别人接受自己的想法。


老师有一套非常简单行之有效的方式，不能说她对谁都灵验，但是一定可以提高你的沟通成功率。那就是在表达一个不同观点的时候，我们可以先给予肯定。比如说我觉得你这个思路非常有道理，那我们顺着这个思路往下思考。咦这里别人就不会抱有一个非常敌对的态度跟你沟通交流了，因为你先给予了对方肯定。


在接下来就是表达自己意见的时候了，我们可以找出一个很小的突破点。比如这样说，我觉得如果在这个功能点上可以采用怎么样的技术方案，你觉得会不会更合理呢？你有什么更好的意见吗？这样说是不是你就把对方引导到了，一起去寻找，一起去思考更好的解决方案这条道路上来的。


从心理上来讲，我们都不喜欢自己的想法被别人否定，这样也会激起我们的反抗意识。不妨我们可以在沟通交流中试一试这种方式，先肯定他人，再把这个话题引向一同去寻找更合理的解决方案这个方向上来。那最后我要说出这两个词，正直和勇气。有人的地方就江湖，在职场中，我们难免会碰到那些没有真本事却靠巴结别人上位的人。不过要我说，跪舔不可取，他害人中害己。在一个真正做事情的公司里，这种人真的存活不下去的。什么是我们技术人员安身立命的方法？那就是靠实力说话。这里有我们过硬的技术实力，还有我们的人际沟通、leadership总结归纳、敢于担当等等技术以外的软实力，有过硬的实力，我们才有足够的勇气走到聚光灯下去，要资源拿结果，不要停止奋斗，也不要停止不断的挑战自己的极限。


其实技术人员的职业生涯远比我们想象的要长得多。有真本事，有真技术。即便到了咱技术人员口中的那道坎，35岁，你的路子也一样的广阔。就像老师一样，同学们听我的声音可能感觉到我年纪并不是很大。其实我真的已经接近刚才说的程序员口中的那道坎了，所以我是正儿八经的中年大叔。不过我拒绝这个称呼，我在自己的工作岗位上一直是疯狂输出的 DPS 角色。为什么？因为我心中住着一个执剑少年。同学们，和我一起再做一回执剑少年吧。
