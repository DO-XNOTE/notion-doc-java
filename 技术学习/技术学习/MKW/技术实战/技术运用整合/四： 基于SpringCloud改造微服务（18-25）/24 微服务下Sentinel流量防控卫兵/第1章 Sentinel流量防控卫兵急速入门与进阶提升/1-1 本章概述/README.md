小伙伴们大家好，这节课开始，我们一起来学习下一个非常大的服务模块微服务架构设计与落地。在这里我们跟小伙伴们一起学习哨兵三层的技术。那接下来我们来看一看对于这个哨兵我们都需要学习哪些内容。其实老师在这里主要分为两大块。第一大块主要是哨兵的一个基础，就包括我们官方的一些文档，还有一些对应的 demo 示例，到时候跟大家一起来进行学习分享。然后其实第二大块就是对于哨兵的一个持久化的规则。那这一块我们后面再去说。首先来看一看这个第一块哨兵基础我们课程的这个大纲目录是什么样子的。最开始我们要对于哨兵整个一个全景做一个全面的分析，来看一看哨兵具体有哪些功能，有哪些实际的这个作用让小伙伴们知道。然后接下来对于哨兵的这个快速的急速入门，我们用一个非常简单的一个流控的 demo 跟大家把哨兵说清楚。那接下来就是对于哨兵的这个核心的 API 了。那在这一块对于官网，它里边有非常全面完善的文档，五种非常主流的这种流控降级，包括我们对应的热点参数、自适应以及黑白名单这五种流通的策略，文档上写得非常的详细。当然老师在这里也会给小伙伴们提供一些文图的资料，帮助小伙伴们快速的去学习。


接下来我们来看一看对于哨兵控制台的一个使用。其实我们所有哨兵流控的一些规则配置都是要通过控制台结合着去应用的，包括在实际的这个生产环境中。只不过我们的哨兵的 dashboard 控制台，它可能是基于内存去存储实际的这个规则的。当我们的这个服务挂掉了或者是控制台挂掉了之后，那可能这些规则都没有了。那怎么办呢？后边我们可能会把这些规则持久化到一个 data source 中。那这个 data source 可以是做 keeper 也可以是我们的携程的阿波罗或者是阿里的 nuckles 都可以。当然在这里我们会选择这个携程的阿波罗跟大家一起来配合着去做持久化规则的讲解。


那 sentence 与我们 spring 框架怎么去整合呢？在这里它是使用了这个 LP 的方式跟它完美去整。然后接下来就是阿里巴巴的这个 double 框架跟我们的这个三次进行一个整合。那这个就是基础部分的一个课程目录。 OK 那我们这节课先讲到这儿，接下来我们就开始快速的去进入哨兵的学习了，感谢小伙伴们收看。
