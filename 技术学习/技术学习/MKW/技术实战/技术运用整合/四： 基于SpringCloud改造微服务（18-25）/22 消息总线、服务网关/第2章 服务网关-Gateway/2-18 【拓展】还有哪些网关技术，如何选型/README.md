hello 木客网的各位同学们，大家好，在揭晓前，我们来讨论一下那些年咱曾经追过的网关技术，并且在这些技术中如何做出一个适当的选型。朋友们有请第一位选手，他它是一个组合，nginx 和 lua 。咱前面应该学过 nginx 和 lua 在限流中的应用，对不对？同学们如果提到这两个知识点还是一脸懵逼泰勒懵逼的状态。这是什么？


那可能就要去学。而实习之翻到分布式章节，把它先复习一番了。 OK 咱在分布式章节里面学到了 Lua 可以跟 Redis 搭配使用对不对？但是卢尔这种非常短小精悍的脚本，它可以见缝插针，甚至可以跟 nginx 网关层搭配来一起使用做限流等等一些小的辅助工作。所以说在**业界里面我们的 NGINX 和 Lua 搭配的脚本可谓是黄金搭档**。除了是老搭档以外，他们还有一个共同的特征是**开源免费**。所以 nuginx 和 lua 的组合广泛应用于各种互联网产品当中，甚至有很多大厂他对 nginx 做了一些深度的定制，也应用到了自己的生产环境上。那如果大家去国内公司任职的话，会发现他在互联网公司的使用真的是相当广泛。但是有优点当然也有缺点了。

那咱作为 Java 方向的研发人员， nginx 和 Lua 对我们来说相当于一个新的编程语言。但是咱看 **spring cloud 的 gateway 它可以完全使用 Java 的方式来编写网关层的指令**。所以从这个入门的门槛来说， gateway 是远远低于 NGINX 和 Lua 的。毕竟咱说长江后浪推前浪，一代更比一代强了。所以 gateway 作为新时代下的网关组件，它自然有一些新框架的特性，那就是一定要易于上手和学习。nginx 和 lua 虽然都在业界久经考验，但是毕竟都是上了年纪的技术了，所以还是需要一些学习曲线来适应它。那它除了这些优点以外，还有一个**最重要点叫软负载性价比高**。


什么意思？软负载非常好理解，它其实是指 NGINX 和 Lua 都是应用在软件层面的网关层组件，它并不需要硬件系统的支持。同学们可能会纳闷了什么，网关组件竟然还需要硬件吗？没错，同学们你们猜对了，大千世界无奇不有。正有一款大名鼎鼎在国外广受欢迎的网关层组件，它确实需要硬件来加持。啊那究竟是谁呢？让我们热烈掌声隆重。有请 F4 的升级版 F5 括弧，它可不是一个组合，它是 F5 networks 公司出品的一款网关层的组件。老师在的这个项目正在使用 f5 组件做网关层。这个组件说起来可牛逼了，它什么功能都有，只有你想不到的，没有它不能提供的。但是有那么两个缺点，一个是贵，第二个是麻烦。那咱先来看一看它都有哪些功能。

这个 f5 同学它的特长实在是太多了，我就挑几个说的出口的来跟大家介绍一下。第一个，负载均衡。这不用说你会我会大家会。 f5 的负载均衡策略也是属于高度可定制的，因为它同样的也是一款可编程的网关层组件。 OK 除了负载均衡以外，它还有流量控制。那这个流量控制并不需要集成其他技术或者另一种脚本语，它本身就是 f5 的一个内置的功能。


除了这两个优点以外，有一个最受大家欢迎的点就是它的高可用，而它的可用性可以达到 5 个九百分之九十九点九。想知道很多互联网公司，尤其是外企，它对自己网站的可用性指标是非常非常重视的，所以拿 f5 组件贵，它自然有贵的道理。在可用性方面它确实是一枝独秀。那它还有什么优点呢？这里就要提到全链路监控，那这里并不需要集成其他的组件。全链路监控本身就是 f5 中一个内置的属性，它让你的所有网络请求无可遁形，各种指标、大盘报表想看就看，非常的方便。


除此以外，它还有一个压缩技术叫 big IP 压缩流量。我们平时发送网络请求的时候，一个包可能就是几 K 大一点的几十 K 如果大家不传文件的话，就是这种短小的包。但是当你的用户基数堆上来以后，这个流量依然可以达到一个比较大比较恐怖的数字。那所以在 f5 里面我们可以对用户流量做一个压缩，这样的话可以减少大量的带宽消耗。因为实际上大型互联网公司它的出网入网带宽都是相对昂贵的，甚至很多的都是专线。咱不说减少百分之五六十，你即使减少个百分之二三十也能省下来一大笔费用的。


OK 那除了这些以外，当然了还有编程路由和拓补路由，这两点其实都比较好理解，咱的 gateway 实际上也可以当做一个编程路由。咱们在 f5 中可以使用它的脚本语言对路由规则进行高度的定制。好除了这些以外，还有一个比较被大公司看中的特点就是攻击防护、网络安全。在这一点上面， f5 弥补了开源软件的一些空白。大家如果使用开源的网关层组件，面对网络攻击，比如 DOS 那可能一方面大家需要借助其他技术来做网络防护，或者是定制你的开源软件，添加一段自己的网络防护的程序，但是这也是非常大的工作量。


f5 不一样，它对攻击防护和网络安全都有内置的功能，比方说动态黑名单或者是识别到此攻击，它都有一套规则和玩法来识别你的流量，是处于真实用户还是恶意访问请求。 Ok. 除了这些以外，还有一个老板非常喜欢的功能，因为老板不喜欢细节，他只喜欢看一个什么报表和大盘，就是统计结果。所以 f5 真是抓住了老板的心，什么统计报表应有尽有。那咱说的这八个点，只是 f5 的一部分特长，它真正的功能还远远不止这些，这些功能其实还是基于技术角度出发做的。


总监。那如果你的老板他并不懂技术，但是他来问你小李小张，你们为什么选择 f5 作为网关层组件呢？刚才说的这些东西我觉得挺好，但是都听不懂，你能不能说人话？好吧。那既然老板诚心诚意的问了，咱就大发慈悲的告诉他们，那咱就用朴实无华的语言来总结一下 f5 的人性化标签，那你要给老板洗脑，为什么要买 f5 了？那第一点你要告诉他，咱是个软硬件二合一的网关层组件。就是说你可能需要招聘一些专业的网络运维工程师来不仅说维护你的软件编写路由规则等等，还要维护它的硬件。它的潜台词就是说成本可能会有所上升。那咱把钱花出去了，总归得有落地有声，对不对？要有点成效。所以 f5 它还具有超强性能。

当然了这个性能还是从单机角度来考察的，咱们说，花钱的总归要比免费的好那么一些。但是咱如果从集群的角度来说，达到同样一个 QPS 的承载能力，你 f5 的花费肯定会比  nginx 或者是 gateway 这种开源软件，它的花费是要高的。 OK 除了这些以外，你还要跟你的老板说 f5 的功能只有你想不到，没有 f5 做不到的，各种企业级的网关应用场景应有尽有，各种统计结果、报表也是非常丰富的。但是这些都有一个前提，就是说只要你肯掏钱，出门三不遍，有钱便是胆，只要你肯花钱， f5 绝对帮你摆平所有网关层的烦恼。


所以从以上几点，咱总结出来两条事实，第一点，在技术领域，便宜不一定无好货，咱有很多优秀的开源软件也是可以满足我们企业级开发的要求了。但是好货一定不便宜了。这货就是指的 f5 了。那第二个事实， f5 这家公司靠着卖他的网络服务，已经成了纳斯达克的上市公司了。这个市值也接近 100 亿了，括弧美元。所以谁会是 f5 的忠实粉丝呢？我们总结了这样三点，第一点，你这个公司肯定不能差钱，所以很多小公司只好望而却步了。
