hello 互康网的各位同学们大家好。上一章中，咱学习了如何利用 spring cloud 中的 config 组件从远程 github 拉取配置文件，并且可以在运行期将 github 上文件的属性变动推送给业务方。但是大家发现这里面有一个问题没有，**咱在刷新配置的时候，是不是只能一台服务器来刷新啊？那这个效率低的实在是看不下去，咱有什么办法来解决这个问题没有呢？那必须是有，咱这一节就来学习一款专门为康费格量身打造的兄弟组件。这兄弟是谁呢？我们这就来揭晓。**


下一页让我们隆重有请一位开公交车的老司机 bus 没错，那这个 bus 就是公交车的那个 bus 被称为叫消息总县。你看这个名字起的透出一股势不可挡的王霸之气，听起来好像还是蛮厉害的。总线有一种武林至尊，宝刀屠龙，号令天下莫感不从的气势。 OK 那咱这就去看一看本章的大纲，领教一下这个 bus 都有什么能耐。这一章的总体篇幅是比较短的。第一部分，咱先来理解一下消息总线四个字是什么含义，然后再简单介绍一下 bus 组件。接下来第二部分我将给大家介绍 bus 的体系结构以及它的接入方式。尽管 bus 是 spend cloud 的一个组件，但是它不像尤瑞卡这样独立，它需要借助一个消息组件，比如说卡夫卡，比如说 rabbit MQ 那咱在本章的课程中究竟选用哪一种接入方式呢？我们就选最常用的 demo 里采用 rabbit MQ 的介入方式来搭建总线式的消息驱动服务。
