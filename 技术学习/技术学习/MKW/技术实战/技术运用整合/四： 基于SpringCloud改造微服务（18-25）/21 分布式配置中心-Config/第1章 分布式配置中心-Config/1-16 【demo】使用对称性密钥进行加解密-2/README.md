hello 慕课网的各位同学们，大家好，在上半场中，咱们把准备工作都已经做足了。那下半场我们就来做这样几件事，第一件，加密一个属性，然后把它保存在 github 当中。
第二件事就是在我们的项目里，通过配置文件拉取到这个加密属性，然后进行解密。 OK 在这之前我想问大家一个问题，加密和解密应该在哪里进行呢？同学们可以好好想一想这个问题，咱的解密和加密过后的属性是用在 config client 这个项目当中，但是加密和解密也是它来进行吗？实际上不是的，我们只要记住密钥在哪边，这个加解密就应该是由谁来执行的。那对于这种对称性加密来说，我们只有一个密钥是存在 config server eureka 这里的对不对？那因此来说加密和解密一定都是由它来执行的。也就是说由配置中心先将这个属性进行加密，然后把加密好的数据藏在给上传到 github 上，等到我们的客户端去拉取这个属性的时候，实际上也是通过 config server 进行了一层解密，解密完了以后再进行使用的。
