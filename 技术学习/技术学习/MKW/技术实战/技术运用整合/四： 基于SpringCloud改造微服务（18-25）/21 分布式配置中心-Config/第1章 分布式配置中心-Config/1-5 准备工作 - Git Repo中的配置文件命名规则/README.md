那我们接下来看今天的内容主要有三点，第一点，我们到 github 上创建一个代码仓库，这一个步骤平淡无奇，我会使用自己的 github 账号创建一个公共的 Repo 来存放这些资源文件，我们接着看。第二个步骤这一步就有点撂了，给文件起名，就像给人起名是一个样子的。所以我们的配置文件要和你的项目生辰八字、风水都要贴合，可不能随便乱起，待会儿就去带大家看看这里面的门道。


最后一步，我们创建好了配置文件以后要做什么呢？当然是在配置文件中添加属性了我们在这一节中统一使用 YAML 的格式来配置文件的属性。好。那我们现在转战到浏览器上开始秀操作了。我这里已经登录了一个全新的 github 账号，我们点击 new repository 首先给我们的项目创建一个新的 config Repo 那它的名字就起一个最简单的叫 config ripple okay 类型要选择 public read me 文件，不需要直接点击 create 接着咱就去创建一个文件，点击 create a new file 创建文件的第一步要怎么？给这个文件起名字，前面说了咱能乱起吗？不能，咱要和项目的生辰八字贴合对不对？那怎么个贴合法？我这里插播一条广告，我们来看一看文件起名都有哪些默认的潜规则。


在 spring cloud config 当中文件的名称和后台的三个属性是相对应起来的。哪三个属性呢？我们先看头两个，它的名称叫 application 和 profile 按约定俗成的规则。这个 application 就是我们的应用名称，比如叫 eureka consumer 或者叫 high strikes fall back 也就是项目的名字。那么 profile 是谁呢？它是项目所处的环境。我们平时开发当中知道有一个开发环境通常叫 DEV 有的公司还有一个集成测试环境，叫 integration test 或者叫 staging 等。我们上了生产这个环境就叫 production 一般来说我们可以把这个 profile 字段用来区别不同的环境。比如说我这里跟大家举一个例子，我创建一个文件，它叫 application 杠 profile 那它是一个 YAML 的配置文件。比方说我们创建了一个项目，这个项目的名称牛气冲天叫什么叫 holy sheet 它所处的环境是 DEV 环境。那么这个文件名我们可以把它命名成 holy sheet 中横杠 DEV 电压，某。


Yaml. 咱还可以把这个文件命名成 props 的格式，这里具体采用 YAML 还是props ，全看各位客观的心情了。因为 spring cloud config 组件不管你用哪种格式来定义它，都会把它 mapping 成后台的一套标准的 KY 6 格式。对我们代码是无感知的。


咱说了这两个属性以外还有一个是谁叫 labellabel 我们通常把它指代为代码分支的名称，你可以在自己的配置文件当中指定你想要拉取的这个 git Repo 下的某一个分支，这样它就会到这个分支下拉取对应的配置文件了。当然有同学问，如果我的配置文件前面还有路径怎么办？也就是说在这个 Repo 下，它是经过了几层子文件夹之后才到了你的配置文件。没关系，这里还有一个 puff 也就是路径的参数可以在你的配置文件中进行配置的。这样的话我们的 config server 就知道该从哪一个路径下面去拉取文件了。


OK 我们看完了文件的命名规则，接下来就去创建项目中所需要的几个配置文件。那么这里遵循前面说到的命名前规则，给这个配置文件起一个名称，它的 application name 首先我们要给出来，虽然这个 application 还没有创建，咱玩没造出来，可以给它先起名字，那为父就给你赐名 config 杠 consumer 有朝一日，当你落地的时候，要记得呼喊正确的名字，拉去自己的配置文件。


接下来一个 profile 咱给它起名叫 DEV 那咱接下来要来命名属性了。第一个属性，我们给它命名叫 info 一个冒号回车 profile 这是什么意思呢？咱给大家起一个 DEV 也就是告诉大家我现在所处的 profile 是 DEV 实际上这个属性打酱油了。那接下来给它定义一些不打酱油的业务属性。 OK 那接下来咱来定义一些业务属性，我想怎么才能定义一个比较有意义的业务属性呢？就跟它定义一个属性叫name 。不过赋予它一些崇高的含义，大家有没有了解过圣经里面有一个非常著名的史图叫做保罗。不过保罗是他在归信基督以后在生产环境中起的名字。那么他还没有归信基督以前，也就是在开发版本他叫什么名字，那它的名字叫扫罗。所以咱在 DEV 环境里就把这个名称定义为扫罗。那我们再定一个其他属性，比如叫 words words 就叫 god bless me。 Okay. 那咱在定义完成了这些内容以后，我们看看是不是还漏了点什么？这个文件名称是不是少了一个小尾巴，它的格式是 YAML 好，那我们配置完之后，把这个文件先提交上去，直接 commit new file 好完成。


那么接下来咱再创建一个不同的文件 create new file 这个文件的名称叫做 config consumer 杠 trod 证明它是一个 production 的文件。那这里面的内容我从前面创建好的文件里面把它 copy 一下，复制过来，把这个 info profile 改成 production 那么名字是不是就要变了，我们的使徒扫罗现在变成了使徒保罗，那他的口号我们把它改成 god bless you OK 滑到下面提交直接 commit 到 master branch 大家还记着，我前面提到了一个 label 对不对？这个 label 就是指的 branch 的 name 如果大家不指定一个 label 的话，它默认会去哪个 branch 来拉文件呢？那当然就是当前的 master branch 了。
