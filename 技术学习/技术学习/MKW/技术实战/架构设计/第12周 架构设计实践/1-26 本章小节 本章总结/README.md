# 1-26 本章小节 本章总结

1-26 本章小节 本章总结
相信课程这一周的内容在视频浏览、代码研读和贴纸实战中快速度过了。这个章节是对DDD领域驱 动设计的整体介绍和分享。 希望通过这个章节的交流，可以给小伙伴们提供一个上手DDD项目的信心，不再有那种不明觉厉的 感觉。大家可以自由发挥领域驱动设计中的那些理念(比如分而治之、读写分离、控制反转、隐形 的概念显性化)、方法论工具(比如嘴炮贴纸、小学生组句游戏)、设计模式(工厂、适配器、外 观、策略、组合)、风格(事件驱动、管道过滤器)来简化日常工作中的开发和设计需求。 但请谨记，架构设计的本质是解决问题，我们可以随意地剪裁方法论，用其中20%-30%的套路来简 化我们的沟通和设计成本即可。以无招胜有招。
