它一般都是用声明式代码来编程，其次它都构造的是异步处理管道，所以说你简单的可以认为说响应式编辑其实就是一种异步处理的范例，就它基本上所写的情绪都是在处理异步的，这种函数和消息这种东西基本都是e，不是编程的一个样板，一个这么理解。


其次它是可以针对任何内容创建数据流，这些内容可能是你的变量、用户的输入各种内容的属性，或者你的一些数据区域，比如缓存，或者从数据结构里面来的，对吧？监控这些流，观察这些流上的变化，然后根据流的信息和它的变化来采取不同的措施。这就是一个响应式编程的，是过程大致的要做的内容和工作就在这里。


最后响应式编程你看它前面说的是对流量进行处理，对不对？它其实我们也讲到它是一个管道，就是pipeline，对不对？你要建立这种pipeline，它其实是可以任意的对这些流进行组合、创建或者过滤。也就是说你对流上可以做的任何一个操作，你都可以把它施加在这些流上面。


流有很多操作步骤，很多这个操作方式，你可以任意的在这个构建的这个管道，就是这个 pipeline 上面，你可以对它进行操作，就得出你想要的一个结果。前面就是响应式编程的其他要素，大家以后在这些实践工作当中或者学习当中，可以把在真实的这个编程案例与我们今天在讲一中讲到的几个要点互相印证。你就会更加透彻地明白这几个关键点对你整个响应式编程的深远影响。
