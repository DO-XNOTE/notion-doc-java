# 1-9 响应式宣言：起源

不管是谈到这个响应式编程还是响应式架构，中间有一个很重要的东西，前面我们都没有聊，这是我们现在题目上面所写的响应式宣言，它是一个什么样的东西呢？稍后再讲，在讲政治的响应式宣言之前，简单聊一下为什么会有响式宣言这件事情发生啊。强势宣言它的难用序也比较简单，但是也比较有趣，也值得大家在探究更多的这个架构设计或者模式这种东西之前，好好的思考一下整个业界的架构模式的这种演变历程，也就是整个的一个 it 发展史，其实最初的所有的这种软件都是靠硬件支撑的。就是眼镜跟硬件之间这二者结合得非常非常紧密，也就是说更多的时候甚至可以说是硬件的发展促进了软件的更新。


举个简单例子，在最初的这些程序，不管是 Java 还是别的语言，都是靠非常强劲的这种内存或者说 CPU 来支撑各种高强度的运算，那个时代就是典型的摩尔定律的黄金时期。摩尔定律讲的就是说这个晶体管人越多，它这个运转能力就越强，对吧？而且会发展得很快。因此如果你把你的程序设计建立在一种对单体的机器要求非常强劲的这种需求之上，那就是 OK 的，没有任何问题的。


所以说以前大家不知道有没有听说过大机这种东西，或者小型机、中型机、微型机这种区分，那就是根据你的计算的能力，单体计算机的能力来划分的，但是那种更多的时代是在it，相对于现在的讲道是比较早一点的时代，对吧？那个时候就纯粹靠 CPU 的能力说话，但是随着这个互联网的兴起，这也一切就会逐渐的发生了改变。


首先计算机 CPU 这一块，特别是也发展到多核时代，这个多核时代就是要求你的程序能够善用这个多核，否则的话你这些都是浪费的，你虽然有多核，但是你这个默认定律也要突破，以后没有什么太大的意义了，对吧？默认定律可能有时候不太生效了，你利用不了这么多核的东西的话，那其实就等于浪费。


那怎么来解决这个问题呢？对，传统的这个程序设计，从架构到软件的撰写，它都提出了一个新的命题和挑战。那怎么来解决问题呢？首先是想说是我们要更多地利用多核设计来改进我们的程序。基于这种思想才有了这个响应式宣言，它倡导的是我们一种新的思维模式、新的设计方法来应对这种硬件上的改变，以此促进了我们的软件的改变。


就是这样子一个道理，详式宣言，它主要就是定义了四条黄金指导原则，来指导我们完成这种价格风格的转变以及软件编写上，它也是有非常重要的实践意义。通过这种新的指导原则设计出了新架构或者新系统，那么它应该更好地利用了新的这种硬件特点。比如说像多核这种表示在软件编写的时候，你要更多的想的是如何利用好多核这种能力，而不是这样传统的这种依赖于强劲的 CPU 做计算这种思维方式，从而这个转变非常大。小程序它定义的这四条仿金法则就是。
