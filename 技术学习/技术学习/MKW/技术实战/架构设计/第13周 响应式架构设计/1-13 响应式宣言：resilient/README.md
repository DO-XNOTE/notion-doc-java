# 1-13 响应式宣言：resilient

现在我们来看一下第二个原则， resident recently，在中文里面我们把叫回弹性，这个回弹性简单的理解一下，就是你拿出一个球，比如说不是特别硬的这种球，然后气在消的时候，你捏它一下，它很快就会复原，这就是回弹性表示什么？表示它很快可以复原，就是它这个系统的复原的能力怎么样？你简单理解，就是这样子的一个意思。

当然这个说的是比较通俗的，我们来看一下，具体我们来怎么来看这个回弹性。简单来讲就是系统即使出现了问题，就是有失败的时候，我们依然可以保证这个响应。就是即使系统内部可能有一些小小的问题，但是我们也要想办法能够返回用户一些信息，每个组件的恢复都被委托给了另外一个组件，因此在必要时可以通过复制来保证高可用。


这个就是说你系统里面有一个组件出了问题，那正常情况下要让整个系统保持运转，那怎么完成呢？那必须得有一个对它能够重启并且恢复的这么一个组件，那就是说它把自身的这种恢复能力依赖于另外一个组件。那这个主题就抓不住这种事情的，所以很多分布式系统设计的原理就跟这个是一模一样的，我们通过这种复制的方法把它放在另外一份，通过一些人工的手段设计好之后，把这一份资料，把这一种服务的能力复制几份，当自身出了问题之后重启一次，然后从那个地方把状态接管回来，这样子我又可以对外进行服务了。


这就是不是像一个球一样，你把它按下去，它要恢复原样？我们系统当中有一部分组件它不工作了，但是我通过另外一种手段可以让它继续工作，这是我自己的复原能力，就是这么简单的一件。这种就是前面讲过的，即使当系统上出现一些问题，但是它的客户端不大会受到影响。这其实跟我们前面有在别的讲座当中讲过的，咱们的这个分布式系统的理念是一模一样的，所以说你不要简单看说咱们讲这个响应式编程，但其实它很多理念包括分布式都是融合在一起的。


大家不要单纯的把一件事情就是简单来看它，其实它是跟很多设计方法和理念都是相辅相成的，确保系统在出现故障时保持响应能力，这个也是有看咱们讲这个问题了，对吧？如果你作为一个客户，那你永远希望店铺不打烊，对不对？我随时要用的是系统，它不能死机。当你系统不管有错误还是错，对秀用户来讲那都没有意义的，用户需要的是你随时能够给他想要的东西。所以说才会出现我们这种响应式编程的响应式架构和分布式，这种系统能够保存它的高可用性和它的可恢复性，这就是回弹性对它的要求。当然这种做法就是前面有讲过的，通过这种复制隔离或者委派的这种方式来实现的。复制这个担心我觉得相对来说这是比较容易理解的。


前面讲过的比如一份资料存储多份当 a 不可用的，我用 b 转移一下，对不对？隔离的意思就是说我尽量不要让 a 的失败去影响到b，这是我们的一种隔离手段。前面在我们的防水层设计里面也有讲到过这个道理，就是一样的委判，就是当自己出问题的时候，你可以把这个错误的功能转移出去，就是原来本来在一个系统里面是由 a 来服务的，现在 a 出了问题，那我把这个系统的功能可以转移到b，提供相同的功能，那就是这样子一种方式，对吧？所以说系统出问题的时候保持及时响应性，这就是我们对回弹性的一个基础理解，它不仅适用一种高可用的任务关键性系统，任何不具备回弹连带系统都会在发生失误以后或者错误就丢失掉。


及时响应性，一般来讲，咱们回弹性都是通过这样复制这种遏制隔离及委托的方法来实现的。前面已经讲过这个点了，我们再重复强调一下这件事情，让失败的这种扩散尽量被限制在一个很小的范围内，这就是我们前面的隔离原则，对不对？其他组件互相隔离，保证了这种失败不会危机的整个系统同时把它的恢复的这种能力也不是委托给自己，而是一大堆其他的部件。所谓通过以上种种手段，我们总结一下，对吧？来保证系统的回弹性，同时也就是响应性的一个基石。
