# 1-8 响应式编程：声明式

响应式变成了最后一个特征，就是声明式，这个声明式其实坦率来说还是挺难理解的，所以我们这里重点来讲一下第一点就是我们所谓的声明式，大家要搞清楚这个声明式它是相对于说另外一种比较常见或者主流的投影机装置的这种所谓的命令式来做比较的，得出自己叫声明式，那声明式的特征定义就是它的目标不太一样，声明式编程的目标它是描述希望得到的结果，但是并不指定如何去获取这种结果。

这句话听上去比较难以理解，那简单来说这个话其实也很好理解，大部分人所受的这个教育，其实是或者说我们的编程习惯都是性相应图灵机这种模式，就是你同时指定这个编程的过程，就是过程先把这个东西一样的堆栈，然后对它做一个加法，就有两个数，得到一个结论，相加，对不对啊？这就是一种典型的这个命令式的这样一个，但是声明式并不是这样子，它更多的时候是一个希望得到一个什么样的目标，就是得到一个什么样的结论。这种来引导了我们的这个编程，所以这思考方式就是完全不一样。


第二它是一种数据结构的形式来表达程序的执行逻辑，这点如果你是长期使用这种命令式编程的方式来理解程序，或者理解这个程序逻辑与运作方式的，其实挺难理解这件事情。就说白了就是你写了一个数据结构，这个程序都已经跑出结果来了，这点其实对于很多经典的这个命令式的这个编程导师也罢，还是普通的无名之辈也罢，都觉得挺奇怪的，为什么会是这样子一个结？相当于我只设计了数据结构集，然后结出来结果，但其实因为传统的这种命令式的想法都是你不但有个数据结构，你还得是有一个执行的顺序，或者只是他们这些数据结构或者数据对象之间怎么交互，对吧？你想得到一个结论，但是声明是编程不是这样子的。


最后整个声明式编解的它是不只听步骤，这个前面已经知道，就是我们说过它已经不是命令式的这种，所以它是没有步骤的，因为它只有数据结构，所以它没有步骤。而它怎么执行的那生命式编程，它是依赖于它这个语言，就你要使用这种生命式编程，肯定也有英文语言，对吧？用这个语言它的运行环境来完成了这个所谓的就是得出一个计算结果，但是这个中间它不指定每一个句子的执行步骤，就是把这些东西我只要一个结果，但是怎么得到这个结果，我都是交给这个运行环境来完成，就是由运行环境决定了很多东西。


那其实大家需要充分理解的一件事情是什么呢？这里我们所说的是声明式编程，它更多的是针对我们所谓的命令式的一个不说叫对立版，就是概念是比较截然相反的一种定义的方法或者理解的思维方式。也罢，因为传统的投影机都是基于命令式的，所以说如果一个跟它不一样的东西，那就是什么？那就是我们遵守的声明史。

所以说在声明时的实际的这个操作或者实践当中大部分的方式，这种所谓的声明式编程的，其实说白了就是函数式编程。因为它不使用任何的可变的状态，也不需要指定执行的顺序。逻辑式编程就是我们的命令式这种，因为需要具体的执行。除了函数式编程，另外一种就是比较常见的和大家熟悉语法，就是这个规则引擎应用的东西，规则引擎里面就是怎么样所谓的逻辑编程，它是生命式的，因为它只需要通过这种各种的所谓的factories，就是你对的 true or false，还执行一些的入，就是要执行什么样的，你把这种 action 响应的这样一个内容来描述我们要解决的问题就可以了。


如果说大家对这个规则引擎很熟悉的，就很容易去理解这件事情。规则引擎里面你可以看一下，大部分都不是以这种命令时候的过程式为主导的，对不对？所以说这一点就是所谓的声明式编程的一个特点，大家可以简单的把它理解成为我们大部分所说的生命日，可能在函数识别简单中体现得比较明显，因为就是大家可能不是特别熟悉，但当可能个别同学熟悉的就是这种基于规则引擎的这种语言，和自制性环境也是一样。他们最共同的特点就是不太在意这个执行过程，因为他把这个执行的步转化过程都委托给其所在的这个环境群，就是这样子。


所以说你的思维方式需要发生一个很大的转变，才能够理解这种函数式编程或者逻辑式编程的这样一个差异，就是跟传统图例集中命令式的方式的差异在哪里？一定要转变这种思维方式，才能为你后面真正进行这个响应式编程打下一个非常好的良好的思想转变，这就是我在开头讲的说，你要获得一些新的技能或者新的编程技巧，那么你必须要调整你的思维方式，这一点其实就是一个很大的一个变化，这一点不要看只有一页 PPT 来描述的，但是其实它对你整个去理解这个响应式编程，或者说声明式编程也要有花更多的时间精力去理解它，在这个地方你可以答应我在抛砖引玉课后呢。大家最好是自己再去找更多的这个例子或者书来理解一下什么是生命史。或者你简单一点去理解什么是函数式编程。
