# 1-23 响应式架构优点：弹性

这个特点其实在之前讲我们的响应式架构的这个特点的时候就已经讲过，但这个总我们可以再从另外一个角度来看一下弹性，它给系统带来优点好处是在哪里？第一个故障的应对就是我们的整个程序因为有这种弹性，导致他对失败的容忍度非常高。而且在运行时的自动修复，我们再重温一下前面所讲的内容，就是当这种高弹性的强势架构系统里面，如果有一个组件出了问题，理论上它是不会导致整体系统失败的，因为它有自我故障上报的能力，系统也会提供对应的修复能力，而且是自动的。所以说这种情况下设计师的系统相对来说对错误会系统异常，它的容忍度相对就较高，这给系统带来的稳定性和高可用性是非常大的用处的。从语言的设计到整体执行的环境都是非常有意义的。

当你在运行期要作为价格，我是考虑这种高可用特征指标的时候，就特别希望有一个能自动应对故障的机制，隔离失败也是前面分享过的，将故障限制在一个相对可控的区域，表示即使这个区域出了问题，或者这个公点出做故障不会影响整体的系统运行，不会引起激烈故障，激励过程就是一个错误，导致一层一层的小错误变成放大的大错，不断的被放大，不断的影响其他的系统，最终让整个系统变得不可用。


故障隔离多大意义就在于此，把错误限定在一个相对较可控、较小的范围之内，让更大的系统还是可以得到保障。所以说故障的隔离本身也是非常重要的，像以前我们讲的这种架构设计模式，也都提到过故障隔离，这个故障隔离在这种弹性系统里面非常有意义。消息驱动的松耦合这个其实已经是反复强调的东西了，但时候不妨再强调一下，因为使用这种消息驱动的架构，那么发送方和结束方是可以独立变化，这个我想不言而喻，因为它们之间并不存在上内存生态这种强耦合或者其他的一些关系，就是一个发生宝，一个接触法，所以说它可以读裂变，彼此的影响都降到最低，意思就是说消息的发送方做出一个变化，接受方要不要变其实完全无所谓，接受方要不要完全不受，发生了控制。他们平时可以独立升级，不管是升版本还是升系统，还是增加机器，减少机器，彼此都不会影响对方，这也是一种弹性。


所以特别是我们作为一个架构师，在设计系统的时候，考虑这个系统设计的弹性指标是非常重要的，因为系统一旦陷入一个错误死循环当中，你就无法拯救。但是如果相反，你的错误一旦出现，可以很快地被消灭掉，那就证明这个系统本身设计弹性很好，而响应式架构就是专门有很多功能是支持这种新老弹性的。所以说使用响应式架构在你的系统当中自然天然地获得弹性。
