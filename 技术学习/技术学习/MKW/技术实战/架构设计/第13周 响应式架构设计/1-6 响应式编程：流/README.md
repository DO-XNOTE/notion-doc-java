就是它是在时间分布和数量上无限的一系列动态数据集合体，或者一个按照时间排序的事件序列。这个就是前面其实我已经大概说过这个意思，你不要把它单纯的认为说它必须得是按照什么你找到一个数据，因为大家平时写程序或者说惯性的思维方式的这个做设计的时候，想到就一想到一个流，里面就是一堆数据给你，这个本身是没有什么错的。但如果在响应式变成这个场景下，你可能稍微就有一点小小的偏差。因为它其实是有两个含义在里面，一是有时间这个概念，如果说你是普通的，比如像 Java 程序的，其实你从一个所谓的流里面去读东西的时候，你根本不在意这个时间这件事情的，这就是一个最大的区别。


但是在这个响应式编程中，所谓的这个世界它是一个序列的概念，就是它是一系列的这个东西组成在一起，但这些序列它怎么完成，是通过时间来控制的，说出这个时间这个维度的东西，在这个流里面你要理解它是非常重要的，在响应式变形当中，对吧？然后你如果特别是这个 Java 这个里面这个集合类对大家的很多误导，但不能说是故意刻意引织你来做这个误解，在这个地方它不要狭隘地被理解成是一个集合体，它也并不是用来存储元素的，对吧？这个它的流，它只是让你能够去操作这些特定的元素，提供一种规范，就是你通过这个流，你本身它并不是去读这个里面的数据，不是一，它就跟 Java 里面这种特定的这个集合体它是不太一样的，你通过 Java 这个集合你是可以读到一些数据的，对吧？这是最大的一个作用。
但是在详细编程中这个内容的概念它只是让你能够去操作这个中间的元素，这个流里面至于是什么元素，其实理论上并不太关心这个事情。它只是提供了一系列的操作规范，让你能够去操作这个流的这个元素。所以说这个事情是你要分开去理解的一个就是说你要操作的这个流里面的这个元素，这个其实是从流里面读出来的，你以为它是一个数据的集合，对吧？但它并不是这样子的，它本身是不存储这个元素的，你读的这个元素只是通过这个流的哪一些操作规范，你把它进行了一个操作而已，是这样子的。


第二是一个非可变形，这个非可变形其实是什么意思？首先理解非可以变这个本身的含义，非可变的意思就是它但出现以后就是不可改变的，你如果这对它做了任何修改都是不可能的。同时在加法当中有一个非可变类，对吧？就这跟这个含义是一模一样的，在响应式编程中所谓的非可什么意思？就是数据流它作为一个源，它是不可更改的，就你读到什么东西它就是什么，你只能带它进行操作，你不能改变其本身就是这个含义。


最后一个也就是老生常谈的响应式编程中的数据都是异步的，对吧？因为我们要读的这个数据，其实你看上去它是一个数据流，但它并不是说这个数据里面这个元素前面我也讲过，对吧？它并不存储这个元素本身，而是你读到的东西，可能你要读什么方式不是同步操作的，就说不是说哪一个东西你马上得到一个操作，这都有及时的响应，而是他来了一个，不管是你把叫消息还是叫事件或者叫数据读到这之后触发一下一个操作，但这个操作它并不会阻塞你后续的一些动作，而是以一个异步的形式进行了一个响应。


这就是响应式编程中对流的几个重要的点的一个顶端概念，特别是这个地方要注意一下你对这个流的理解上不能出平台，最重要的点就是流本身它不是存储元素的，而是提供了这个操作规范，这点特别重要，很多人第一次去理解这个流的时候就容易出现偏差，就在于它把它跟那种集合类搞混了。


没错，你可以把集合内变成一个流，但都不能说这个流就是一个集合内，对吧？这我指的是加入规范里面，对不对？首先集合类里面存的是元素，但是流本身并不是流，只是让你去操作这些集合内形成的一个数据流的一个规范，所以说大家一定要注意这二者的区分，特别是响应式编程当中，这个流里面出来之后，你可操作的元素上，还有元素的本身这个时间概念也是很重要的，请大家注意这点。
