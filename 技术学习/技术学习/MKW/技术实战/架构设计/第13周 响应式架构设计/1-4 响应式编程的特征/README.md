就说因为我们整个小游戏编程更多的说的是一种声明式，对吧？然后它取得这些值根据的都是我们外面传递的消息或者流里面的事件产生的一些动态行为，就这意思，同时我们也不再关注系统的状态变化，而使用时间变化的功能。


这个话怎么理解？就是说因为我们整个响应式编程它都是使用流来改变的，流里面这个东西是什么东西呢？就是一系列随着时间而推进的消息或者事件，所以说整个响应式编辑就是针对这种事件或消息形成了理由之后，对它做到一个功能性的反应，对不对？所以说整个系统就是一个随着时间变化的功能组合在一起的这么一个系统。然后在这个流当中我们监听的是流上事件或者一些消息，对不对？那数据流它流出来的这个事件是按照时间顺序来进行发生的。同时因为我们是一个流，前面讲过的我们是要形成一个pipeline，在整个编程过程当中，那所以说在这个 pipeline 当中，你不断地对这个流动的信息或者事件或者消息再进行各种相应的修改或者计算，然后传播这种变化，因为你整个系统可能是由多个节点构成的这么一个pipeline。


那在每一个 bug banner 上要执行自己相应的任务，所以说它会自动去传播这种变化，那最后一点也是一个显而易见的这样一个特点，就是我们一切都是使用的异步数据源来进行编程。这里两个点，一是异步，这个异步这件事情前面一定反复起来，我在这种响应式编程当中，因为我们一切都是异步操作，这本质就是一个异步系统，所以说这一点是非常突出的一个特点。二，就是前面也有讲过我们是对流进行一个编程，这段所谓的数据流其实是一个很广泛的含义，你并不要把它单纯的认为就是一个数据在里面这么一个流，而是说一个广义的含义。什么意思？就是说所谓的数据就只要是含有任何数据的流，它都是一个数据流，这里面可能它是一个事件，可能是一个消息，也可能是一些在变化的东西，或者真的就是一个数据源，都有可能是一个数据源。简单来说在强调变详细变成就是对这种异步数据源进行编程。我们把这个异步数据进行各种加工作，代机形成一个pipeline，一个管道，在这个管道上经过各种变化得出一个我们想要的结论，这就是所谓的响应式编程。
