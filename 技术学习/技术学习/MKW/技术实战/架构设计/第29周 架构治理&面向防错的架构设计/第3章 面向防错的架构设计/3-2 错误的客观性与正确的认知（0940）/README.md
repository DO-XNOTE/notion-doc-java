犯错误都是会有的事情，但是我们需要的是什么呢？需要的是我们要能够认识到这个错误。如果犯了错误还死不悔改，死皮赖脸地觉得我没错，那这以后也不是具备任何可以改进的空间。认识到自己的错误，然后就可以去修正这个错误。犯错误不要紧，改正了就好了。


修正错误之后，我们就要从中吸取相关的降序，如果不吸取教训，也意味着我们这个错误就白犯了。俗话说花钱买教训，那教训都买不回来，那这钱就白花了，那这错误就是我们要花的这个钱。所以吸取教训之后，我们要从这个中自我成长，要让自己成长为一个我，下次可以不复再犯这个同样的错误了，我们不在一个跟一个坑里摔第二跤，我们不摔第二个同样的跟头，所以这就是我们需要的自我成长。这样子的话，在后续犯错误再重新走这个闭环的时候，我们就能够更好地推动自我成长。但是一定要经历这些吗？一定要不停地犯错误才能成长吗？我们有没有办法成长了之后，我们就少犯错误，或者是说我们将这个犯错误这条路就给它堵死，然后直接从自我成长指向这个认识错误。


有没有这样的一个办法？这样子的话我们是否就可以不犯错误？我们可以认识到一些错误，然后修正这些错误，吸取教训，自我成长，那怎么样子才能够在不犯错误的情况下认识错误呢？就是看别人犯的错误，犯在别人身上我们当然就不疼了，但是我们可以从他们犯的错误中吸收相关的经验，然然后自我成长，这个就是可以从犯错误这个点上获得到的一个捷径。

它传递存在着各种各样的不确定性，他要我们要明确这个差异性。这里其实这两个点我觉得大家可能都会听得很多了。比如说像阿里中最有名的一句口号叫做拥抱变化，每一个人都要去拥抱变化，但拥抱变化有几个阶段，第一个阶段是被迫的拥抱变化，第二个阶段是主动的去拥抱变化。第三个阶段是什么呢？是去创造变化，用变化改变一切。既然都要变化，为什么你一定要做在最后成为了那个被牵着走的人？为什么不能磋商之前，你去成为去创造变化的人，你，你去把一切做了改变，这时候别人才就跟着你去走。所以在防错设计中这一点也是非常关键的，我们为什么一定要等错误产生了之后，我们才去拥抱他们？我们是不是可以在错误发生之前，我们就去创造某些变化，从而让错误不再发生？这就是我们在防错设计这个点上最最关键的一点。当然信息它的不确定性它存在两点，一是差异性，二是误导性。我现在说的这些知识可以，其实可能我脑子里想的已经存在差异了，然后你听到的我的这些信息可能到你脑子里吸收的又是存在差异了，所以我想的和你想的就已经存在很大差异了，甚至我想的到你想的可能只有百分之二、三十的东西是一致的，剩下的东西都是不一样的。


甚至说我们的信息还有那些误导性，我想的和我说的会存在偏差，你听到的和你脑子里吸收的又存在偏差，那所以这里有些我可能我认为是 a 的，到了你的可能变成b，但就存在着相关的误导，那这些信息的不对症器都有可能带来这个世界上的一些变化，也可能带来一些我们未知的一些情况，这些情况就可能会引起相关的一些问题。


那更具体的来说，就比如说就是经验，经验是你从其他人那里，或者你从你自己这边学到的相关的知识，但这些知识也可能会存在的一些差异性，大家同时也可能会存在这些误导性，你曾经的经验误导了你。现在。就比如说如果我们一直使用的是Windows，那我们的所有的关闭按钮都在右上角，但是这一天你换了Mac，你就会发现你之前的经验不好使了，因为所有的关闭都在左上角，这时候就会导致你曾经的经验反而是你的绊脚石，你需要去改变，需要去适应。


其次是行动，我们要从经验中提炼出相关的指引，然后围绕着这个指引去做好相关的行动。经验是行动的一个最好的一个蓝本，我们曾经这么做过，那我们下一次行动就会参照之前的经验去进行相关的一个行为。当然行动多了，我们就对这个事情有了一个更立体的一个体感，那它就形成我们对应的认知。


所以从经验到认知，这是一个最常规的一种演变方向，但是我们也可以反向而来，我们其实并没有相关经验，但是我们可以通过学习、通用思考先形成一些相关的认知，然后用这个认知去指导我们的行为，从而在行为中获得相关的经验，并通过这个经验反向的去硬成为我们的认知。通过这样子的一个闭环来使我们更好地来完成中间这个行动，通过经验和认知双向的一个保障，让我们的行动可以做得更好。


那在防错这个领域也是一样，我们用一个防错的一个思维，防错的认知去提高我们的一认知水平，然后去做执行，然后获取在防错架构设计中的经验，再通过这个经验反向地推动我们的认知进一步的成长，最终让我们能够更好地做好架构设计。好了，到这里我们这一小节就结束了，这一小节的主要内容就是让大家认识到这个世界的复杂性、动弹性，认识到每一个人都可能犯错误的客观性。那在下一小节里，我们将去聊一聊防错设计它是怎么来的？它的起源是什么？它的历史是什么样子的。好，各位架构师们，我们下一节课不见不散。
