所以这也就引出了我们可能会犯错误的一些点，比如说潜意识，我们的习惯、认知和文化两个两拨人，他们的习惯不一样，那可能在面对同一个产品的时候，他们的觉得体验就是不一样。


比如说 Windows 的关闭按钮在右上角， Macro 的观标者在左上角，那这二者就意味着不同人在这里就会有不同的习惯，那使惯了 Windows 的人使用 Mac 就会觉得很不适应，使惯了 Mac 的人去使用Windows，它也会很不适应。


而文化这块儿，你中国文化跟欧美文化本质上也会有较大的差别，那所以你完全进入到一个欧美设计出来的这样的一个产品，比如说像亚马逊，你用起来就会觉得哇它很不好用，但是你用淘宝，用京东你会觉得很顺。但是反过来让他们去用中国设计的一些产品，他们又会觉得你设计的好复杂，所以这就是文化上的差异带来的一些对于产品的一些认知上的变化，那这些变化也可能会导致说在使用与习惯、认知、文化相悖的这样一些产品，就可能容易犯一些错误。第二类就是粗心，马大哈，粗心大意，这是个人之常情，很常规的错误，我们从小都是犯着这种错误过来的烦躁，尤其像机械性的重复性劳动极易的引起烦躁，但是烦躁就会引起你的注意力下降，注意力下降就会导致你容易犯错误而经验缺失。这也是一类非常常见的一些错误的原因，因为经验决定了我对于重复行为的约束。做一件事情上我有经验，那就意味着我在做下一次的时候，我更容易去成功，这也是连续创业者他更容易成功的一个原因。当你缺失了这个经验的时候，你就会要去探索，但是探索同时也是伴随着风险的，风险越大，那你犯错误的可能就会越大。


当然还有一些，比如说像干扰导致干扰，它有很多种，包括产品的设计，包括个人经历，包括其他人因素的干扰。那当产品设计示意不合理的时候，比如说他明明应该在 OK 那个按钮上加重颜色，结果他明明在取消那个按钮上加重颜色，你一不小心就点了取消，那这种其实都是一种干扰。


还有就是像气息缺失，上下游的信息缺失，这是其他错误的引发源头，因为我们没有了一些上下文信息的时候，我们只能去靠猜猜呢，就太容易犯错误了。当然还有像试探新行为、破坏型行为和黑天鹅这种非预期的意外性事件，这几类都是说我们在一些行为上的判定。试探新行为方向往是一个善意的，它是个经验缺失或者一个新事物前的一个常规行为。但是破坏型行为，就比如说刚才写的善酷跑路，它就是一种很恶意的行为，它也是我们所需要关注的一个点。


当然还有一些像黑天鹅。什么是黑天鹅？说在之前欧洲认为黑天鹅全都是白的，只要有一天他们发现了澳洲的黑天鹅，它就指在预期以外的小概率事件，但这个事件却有非常强的颠覆性。你想，因为大家原来都觉得天鹅都是白的，所以天鹅的名词就是洁白，但是到了后面发现了黑天鹅。


在原来的这些所有的认设定，所有的认知都被颠覆掉了，所以从这些时代我们就能够推演出我们犯错误的十宗罪，它也是我们在平时最容易犯错误的那些点。

比如说你刚才提到的粗心、懒惰、烦躁、试探和潜意识，那这些都是我们在人性，在这种我们的习惯性上，在这些上面，在于我们的人的特点上，比如说粗心、懒惰，那可能就是人的一些个性。这样烦躁和试探，这也是出于人的一个本性，当你去做一些很复杂很繁琐的事情的时候，很容易的就陷入到烦躁的一个状态。


那除了这五宗之外，我们还有五宗罪，其实这五宗罪相当于比较客观，比如说经验缺失，当你没有这相关的经验的时候，你本身就是个小白，那小白就会容易犯一些错误。其次这信息不对称这块儿也是一个很客观的一个事实，我现在说的跟你现在收集的本身来说，这个信息即使咱们面对面的沟通，也容易产生不对称。第三就是外部的干扰，外部的干扰也会导致我们犯很大的错误，尤其比如说当你专心致志地做一个事的时候，当外面特别吵的时候，其实你也容易犯错误。其次就是恶意行为，擅苦跑路是吧？恶意的去攻击、漏洞利用，这些都属于恶意新行为。当然还有最后的黑天鹅，那这十类就从主观和客观两个层面上界定出了犯错误的适中可能，也就是我们认为的十宗罪，那围绕着这十宗罪，我们是否就也有一些业界已经是形成形的一些方法可以借鉴？当然是有，所以业界就会有防错的十大原则，

虽然这十大原则跟十宗罪不是一一对应的，但是十大原则都能够面向于诗中罪的，这诗中可能会带来错误的这种原因，它都做了一些相关的保护。


比如说我们刚才之前一直讲的断根，它就是从根本上去切断犯错误的可能，让不再有机会进入到犯错误的阶段，或者进入犯错误的功能。而保险则就意味着我们通过双重保险来确保一件事情，它出现错误的概率降到最低。比如说我两个功能都有 50% 的错误，那我中国双保险，那整个它的犯错误的概率就会降到25%。当然我们平时肯定没有这么多的犯错误的概率，如果你比如说一个东西，一件事物你犯错误的概率是10%，两件事都是10%，那你犯错误的概率直接就下降到了1%，而现实中往往可能比这个凯奥低。


再有就是自动，通过自动的执行某一些功能，某一些触发，它能够帮助我们来完成相关的事情，而避免因为人的介入而产生相关的错误。而相符，它就意与我们最早看工业的那个案例是一样的，它必须形状一样才能插进去，形状不一样它就插不进去，通过形状相符来确保整个的一个使用过程中不会出现错误。


还有顺序，就比如说我们的新手引导我们的工单体系，我们的上一步下一步，像我们的 PPT 也是一样，我们上一步，下一步我们可以通过不停地按右键就能切到下一个元素，我们按左键就回到上个元素，它通过这样的方式就可以有一个明确的一个顺序，这个顺序就能够帮助我们保证我们的行为不容易犯一些错误。


当然还有像隔离账户体系的隔离，像我们现在的新冠病毒，那也是说我们大家把大家都隔离开，那这个病毒就不会再传染了。其实功能也是一样，当两个功能完全隔离的时候，它是就不会出现从一个功能意外的闯入另一个功能这样的错误，而复制就是这是我觉得我们的每个作为程序员最常见出得了怎么样子才不会超错呢？ control 4 control v 贴过来它就不会错误，反而你照着抄就有可能打错字。而像标识和警告，它俩是两种比较类似的这样的一个原则，但标识更多的还是起于提示的作用，而警告则是告诉别人你做了这个事情可能会出现问题。标识不一定代表它背后一定是问题的，警号及背后一定是问题。


最后就是缓和，缓和包括两类，一类是对于结构的缓和，一类是对过程的缓和。比如说像高速公路，它就针对一个过程的缓和。但是比如说我们平时的这种降级服务，那它就是对于结果的一个缓和。当事情发生了之后，我通过一些层面上去兜这个底，防止它出现问题。而防错的十大原则其实是之间不是割裂的，不是孤立的，它们之间可以两种组合，形成更符合实际使用场景的这一种设计。
