# 3-5 软件领域的放错探索，你以为的正确（1018）

hello，各位慕课网的未来架构师们，欢迎回来，在上一小节里，我们为大家介绍了在工业界中的那些防错的一些案例。工业界中的防错真的是五花八门，从小型的电器到大型的这种高速公路的巨型工程，其实它都有各种各样的一些防错的设计思想在里面，目的也是为了尽可能的让大家可以不犯错误，让大家能够更安全地使用这些工业性的产品。那么在这一小节里，我们就将带为大家讲解一些软件领域的防错探索，我们看一看在软件领域我们又做了哪些探索，可以让我们在使用软件的时候尽可能的不犯错误。比如说我们还是先来聊一聊就是这个防错在软件上面怎么定义的，就是我们在产品设计上避免用户进入会发生错误的阶段和功能，就是这个功能设计起来的时候，就用户就没有进入到那种犯错误的这种阶段的这种效果，

那这个就是所谓的断根设计。那举一个例子，比如说我想在淘宝上买一款鞋子，然后我选了这个 36 码，其实我想选红色，但是我发现红色已经没有了，那这时候它在软件设计上就将红色灰掉了，这意味着我就不可能说我下了一个单，它实际上库存已经没有货了，它发不出货，然后它还要给我打电话说我退单退款。这样的话就会出现一个沟通性的成本，如果我不愿意退，那它就很麻烦。


那所以它在这样的一个防错设计下，就可以避免用户进入到一些该进入的这样的一个阶段，就是比如说无货还进入到了一个要发货的一个阶段，那其实对于双方来说都不是一个好的一个结果。其次就比如说我想在这个某外卖平台上去买这样的一个外卖，然后它会提示我你要添加餐具中的商品才可以下单，它的这样的提示就可以什么呢？就可以避免我在没有选餐具的时候我就下单成功了，然后商家也根据我们的没有选择就没有给我餐具，结果到了之后我就没有办法吃了。所以通过这样的一个防措，就可以让用户能够避免因为忘记选餐具而导致买到了餐之后无法可持这样的一个状态。

比如说像微信框，我们要去微信去删除，它会有个二次提示，防止我们第一次去划过去就删掉了，这样子的话就会有大量的误操作。但是有了二次确认我们就会明确出，诶，我们是不是有个误操作，如果是误操作我们就会取消掉。


还有像手机关机的二次确认，就是关机的键是很容易被误碰到的，那这种情况下就会导致我们明明还在使用的过程中，结果就出现了手机被莫名的关机，这样的肯定不是我们希望用到的。那在自动这个原则下，我们最常见的就是以前我们的播放或者是我们的下载软件，当它网络切换从 Wifi 切换到 4G 的时候，如果不能自动停止下载，那可能流量就会被浪费掉。现在来说这个功能其实就没有那么常见了，但是在以前的时候，这是个非常常见的功能，因为那时候流量还很贵，现在的话大部分流量是用不完的，所以网络切换的时候，它只会有变成从原来的自动停止变成了一个提示。


大家有没有发现这个从软件设计上的这样的一个演变，它是很有意思的，以前我们希望是用一个自动这样的一个原则去让它停止下载，然后让人去再去切换，这样的话可以避免流量被消耗。而现在我们不再用这种原则了，我们变成了一个提示，而这个提示其实是很弱的，这样的话我们第一保证的不是流量了，我们第一保证是用户体验，因为他认为对于绝大多数用户来说流量都是充足。


还有就实际上秒杀倒计时，它会自动跳转到下载下单页面，这样的话就不需用，当秒杀倒计时完成之后，我再点击还要花一个时间，这样的话等到我下单的时候可能就已经被抢光了，所以通过这个方式来提高用户的体验，让用户更有抢秒杀的这种参与感。

我们做好相关的差异化的一个隔离，这样的话用户就不会用到不该他们使用到的一个范围，比如说像全新体系，就是一个非常经典的隔离原则，可以避免说我用到了你账户下的一些资源，然后你用到了我账户下的一些资源，这样子的话就会产生出大量的问题。那通过这样的一个隔离，就能够差异化的处理不同的级别，不同的账户之间的用户的体验。


还有比如说像满减区域和折扣区域的分离，那有的人就可能喜欢满减，而有的人就喜欢折扣，他觉得满减给我特别好的一种感觉，是吧？我满 100 减50，有的人就觉得你满减还不如折扣来的爽利，我就 5 折，比你更关更干脆。但每个人可能会有每个人的倾向，那我们通过当他俩分离，就可以更好的便于用户进行相关选择，还有像顺序这种顺序，它其实就有点像是我们的引导下一步这样的一个方式，可以避免用户因为不知道要怎么用而产生了混乱。比如说向新用户引导，我们就会告诉用户你先点a，再点b，再点c，这样子的一个方式就能够帮助他更好地熟悉我们的系统，熟悉我们的游戏等等。


还有像比如说我们的工单体系，它可以是引导着用户，诶，我这一步我填完了之后，我下一步要谁审批那个人，审批完之后我再下一步要找另一个人去处理，它就会通过这样的方式将整个的一个事件处理变成一步一步的执行。这样子的话拿到这个工单的人就不会因为他不知道下一步要找谁而犯错误，它就可以很清晰很明确地将这个事情一步一步地做到位。


所以我们轻易发现所有的这种原则就是它都是在尽可能地让用户去更好地使用这个产品，它去选择用户最有可能犯错误的那个点，去做好对应的设计，然后让用户在这个点上尽可能的不要去犯错误。但是其实很多时候我们自以为的是这样的一个引选择，很多时候可能也会带来一些适当其反的效果，这里则就要注意向习惯性和交互性的一个平衡。

原来它这个右下角的这个话筒，它是在输入框这个位置的，那就会有大量的用户在打字的时候不小心摁到这里，然后它就会弹出让它输入语音，然后值得取消。所以在后续他们把这个按钮从输入框这里换到了右下角，但是换到右下角这似乎是一个很好的一个选择，是吧？但是右下角结果会导致一个什么问题？会导致一个我打字是没有问题了，但是我在发送的时候，我总是会摁到这个小话筒，因为在原来发送这个按钮就是在最右下角的，而现在最右下角这个位置被这个话筒所占据了，那这时候就会出现我打字没问题，然后发送它弹出了个语音，我可以把语音取消掉，然后我再重新去点这个发送。


而这里的误触性其实还是蛮高的，所以这里体现的是什么呢？用户原来的习惯是发生在最右下角，结果现在右下角被另一个交互性的按钮所替代了，而这时候就会出现用户开始频繁的不习惯，这种状态下就会导致这一次的防错，其实它是把原有的错误防掉了，但是它产生了一种新的错误，所以这也意味着我们要在产品设计过程中去思考这里平衡。
