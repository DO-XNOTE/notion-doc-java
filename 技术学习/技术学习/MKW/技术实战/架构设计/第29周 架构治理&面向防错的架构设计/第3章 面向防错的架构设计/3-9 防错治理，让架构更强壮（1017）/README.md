有没有在第一时间做拒绝 fell fast 在最快速的就失败，告诉它你这东西就不好用，就不行，你就不能过。这个点是要考虑到的，但是在这个点上其实没有，是吧？因为它格式没不通过，它要保存成功了。


第二就是不是可以尽可能避用使用，避免使用 JSON 这种复杂的格式？因为本身来说 JSON 你一旦比较大的时候，它根本就看不出来你这里面一些细节小词错误在哪里，你只能贴出来到一些 JSON 工具里去格式化一下，去看看是不是有问题。但是有时候在线上你去维护这个系统的时候，它本身那个工具你平台不具备这格式化的能力，那这时候就一旦 json 出了问题，或者你觉得已经在别的地方改完了，你再过来追微调的时候，其都有可能调出问题，那是不是有更好的方式？是不是比如说用配置代码化这种方式去解决这个问题，亦或是说我们将 JSON 转换成一线下mail，或者用 JSON 转换成一些文本，用简单文本是不是能更好一些？其实这些都是我们可以去思考的一点。


第三就是说是不是我们能够在一屏展示，因为你的 JSON 太大的时候，当你一屏展示不下的，你就要上翻下，翻来翻去，你就特别容易有漏的地方，特别容易看花了眼，没有办法一目了然嘛。那最后一个还有什么呢？就是我们是否有这个复制粘贴这个这样一个功能有两种思路，老师这里写的是关闭 paste 功能，这样的话它就没有办法贴进来，它只能是说通过一些方式录进来，为什么呢？因为你在贴的时候，你往往从别的平台贴过来，它可能有一些特殊符号，那这些特殊符号是看不见的，但是它会生效，它在进行角色转换的时候可能会失败，那这样的话就可能会导致你录的时候是正常的，但使用的时候是不正常的。所以要么在复制的时候把这些特殊字符给过滤掉，要么就是关闭掉，这个尽量让它去手录。但是手录也有另一个问题，手录就是你容易犯错误，所以这里最好还是针对性的，针对场景去探讨。


如果它是一些小型的功能，建议就是关闭掉，这样的话就完全杜绝掉。尤其是在一个文本框里面，你前面后面有一个特殊字符，完全看不出来，关闭掉之后它本身又很短，你手打一下其实是很容易就能确定下来，但是如果比较大面积的，那就要考虑是否是它本身就可以帮这些特殊字符给过滤掉而避免这些问题。所以这些最终映射到我们的防错原则。


第一就是断根。如果我们第一时间做了格式校验，格式不通过就拒绝，那它就不会有机会写入一个存在格式问题的JSON，这是一个第一个种断根。第二个，如果我们尽可能地避免使用 JSON 复杂的格式，那它就不会因为 JSON 中缺少引号而导致了故障。


最后一个关闭了 paste 的功能，那它也就彻底断掉了由于负的粘贴而引入了特殊字符带来的这种故障。当然它还有其他的一些原则，比如说相符，我进行格式校验，格式通过才能过，其实这也是一种相符，我尽可能在一屏展示，尽量不要跨屏，一目了然，简实相符相当于我的文本与整池的屏幕屏显是相符的。还有就是隔离，那我们尽可能的让线上和线下做好相关的隔离，比如说用配置代码化，那是不是就是线下和线上之间它只有一个发布流程，去保证可以做好双端的隔离，而避免因为我们直接在线上修改配置而导致的故障。当然最后还有警告，就是无论哪里出了问题，我们都要给用户以提示，告诉他你这样不行，那样也不行，从而避免他进入到犯错误的空间，约束他的预期行为。那这些就是我们从这个故障中复盘出来的它以及相关联的一些防错的原则，那我们接下来就是肯定要利用这些原则将这个故障对应的功能进行落地，比如说添加格式校验，如果真的必须用JSON，那我们还继续保持字，但 JSON 呢？尽量一屏显示，然后把特殊字符给干掉，通过这样子就可以尽可能地避免掉 JSON 配置 JSON 出现错误而导致线上故障。

那是不是说我可以在不同的页面里去操作，这样子的我回滚就没有继续的按钮，发布其实也没有回滚的按钮，我在选择发布继续的时候，它不可能回滚，我在选择回滚的时候它不可能继续发布，这样子是不是就可以将我们的发布回滚页面点回滚，结果却触发了继续发布这样的场景彻底的就断更了。


第二个就是继续增加确认弹窗，做二次提示，这是一个很低成本的一个修改，但是它可以起到一个警示加保险的一个作用，是吧？它可以 20 次的确认，当你第一次可能是误操作，但第二次确认你就会发现我是个误操作，我就可以取消了，并且这个弹窗可以阻塞他当前的一个行为，他会向他的注意力圈吸引在弹窗上，从而达成一个信心就注意力的一集中。这样子的话，即使他在回滚之中很焦虑，很着急，他压力很大的状态下，仍然能够将他的注意力集中在当前的页面上。


第三就是我们刚才遇到这个问题，发布中是不是可以中断？我们是不是可以及时的止损，不让它继续发布了？这样子的话我们就能够快速地停止我们的损失，像刚才那样，就是他没有办法及时停止，只能继续发布，蒙受着双重折磨，一方面线上还在拼命的出损失，另一方面我们只能眼恨睁睁地看着他先发布再回滚。


所以这些我们与我们的原则去匹配，第一个是断根，这块其实体现还是比较明显的。第二个就是重复，是不是说我们在弹窗的时候做二次提醒，差异本身又起到一个警示，也是保险这样的一个效果，那当然警示也是在提示在里面，包括发布中可以中断，其实这也是一种断根的一个效果。当然这里还可以继续挖掘出其他的可以运用出来的断更的一些原则，这些我就交给各种同学们在课后去完成。
