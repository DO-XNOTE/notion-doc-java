# 2-7 向死而生的质量中台（1026）

hello，各位未来架构师们，欢迎回来。在休息之前，宋大伟同学为我们分享了关于他们公司质量中台的建设，以及质量中台面临到的问题。其实他提到的这些僵局其实真的还蛮有体感的，因为中台本身作为一个独立的一个团队，然后抽离出来去支撑整个公司的这种可复用的能力，其实其本身的压力就是比较大的，因为它需要考虑整个公司所有业务对于中台的一个诉求，它去抽象出一整套更合理的架构，并通过这套架构来支撑全公司的业务。


那也意味着他需要去做很多很深层次的东西，而这些深层次的东西一方面来建设就是可复用的能力，另一方面也能够让大家更容易地去定制自身的一些个性化的逻辑，但是当这个复用和扩展能力没有达到预期的时候，那合作之间出现的僵局就出现了，这就是宋大美公司在之前遇到的问题。

这个架构师他来接手了这个原有的公司的这个质量。中台来了之后，其实我们都以为星光上人要烧三把火，他肯定要把我们烧一烧。其实来了其实他还真做了一些事儿，比如说他带着我们跟我们一起去聊，去聊这个整个的一个现状，包括我们之前的工具平台的一个现状，包括大家各个业务的一个发展的现状，还有之前的这个中台为什么呢？最后推不下去了，最后被大家各种吐槽，各种抱怨，他把这些情况都做了一下分析。
然后也来看了一下大家当前的架构设计和架构代码，去来评估了一下当前的债务，去明确了一下当前的代码质量是什么样。那我们到底有哪些重复的轮子？各个团队之间有多少这种重复的轮子，以及大家团每个团队都有哪些是定制的质量部分，这些质量定制它带来的效果是什么样？为什么会定制？在什么样的背景下去定制，以及分析了整体的组织架构，各团队当前的人手是什么样子的？每个团队都有多少人？然后多少人是在做什么呢？哪些人跟中台其实做对接的是接口人。


那如果在原来中台没有中台之前，那各个团队投入在这种工具平台投入，在这种轮子的建设上划了多少人？那当前如果我们在重启中台，我们可投入的资源有多少？每个团队能有多少人能够支撑过来？哪些人是可以实现过来的，有哪些人是可以虚线过来的？通过这些之后，基本上的情况他就知道了。然后他又重新地和各个团队的 TL 和架构师一起去开了一个会，去明确了下关于中台的一个定位，

它中台不再是以一个完全建设可重复项能力为主的这团队了，它常来就说我们中台是以一个以核心业务为内核，支持类核心业务的一个这样的一个中台，也就是我们现在先明确公司最核心的业务是什么，所以我们所有的中台的建设会聚焦在核心业务。


而对于其他非核心业务，大家还是可以以自己的思路，以自己的方法去建设自己的一些测试平台。如果觉得你觉得中台有能力，你可以复用，你可以拿去用，但是如果你觉得中台知识不好，你也可以自己去建设。所以这个就和之前的完全建设可复用的能力，大家都用了一个统一的中台有了天然的差异。


第二就是什么呢？最大程度上复用核心能力个性化的差异，以业务身份去做区分，所以这样就可以先把核心业务通过中台支撑起来，然后类核心业务都通过业务身份的介入，然后差异化的来实现部分他们个性的这种业务。这样子的话整个的一个定位就比较清晰了。核心业务为主导，因为毕竟它是公司最核心业务，最挣钱的业务，那我们再把很多围绕着核心业务延展出来的业务也支撑起来，中台能覆盖的范围已经很大了。第二个组织保证，那这里就需要老板来背书，他找了公司老板沟通了几次，然后明确出要重组新中台团队，其中需要一部分人实现过来，可能主要的还是老的那些中台团队的人实现过来了，然后每个团队会有一两个人去虚线的汇报给组织中台团队，他们的目的是能够将中台跟业务团队紧密地结合起来。通过虚线汇报可以第一方面能够达到人员控制的效果，第二方面又可以双方共享人才。


第三块儿执行路径仍然是以核心业务为出发点，深度的去建设这个平台，中台本身也是一个平台，然后其他业务则和核心业务进行差异的一个梳理，然后来完成中台的一个下沉。如果他们的业务跟核心业务差异非常的大，就比如说刚才说的这种传统的 ToC 电商交易和 ToB 的这种面向接口的这种下单，它肯定存在较大的差异，那这里就要明确出我这个差异化之后，我还能不能有没有办法进入中台？我有没有办法从中间半拉进入到中台？明确的梳理好这个差异点之后，就可以去分析是否有下沉的空间了，那在整个的一个过程里，我们中台就经历了一个长租的发展。

没有的核心业务其实它已经有一个比相对来说比较成型的这样的一个体系，中台就在这个体系之上进行了能力的个抽象通用，将这种通用能力抽象起来，然后在保持在原有的核心能力支撑不变的情况下，开发出了一套扩展能力，然后你的业务就可以利用这套扩展能力去开发它们的定制点，它们可以将它的定制点通过扩展能力接入到中台中来，并且做好了相关的一个隔离。


其他业务在使用这个中台的时候不会影响到核心业务，核心业务也不会因其流量突然变高或怎样影响到其他的业务。通过这个隔离来完成了双方的一个各自的保护，然后并且通过定制点开放出来，能允能够允许与核心业务差异化的这些业务来完成他们的一个前台业务定制，这是我们目前已经能够实现的这样的一个中台。
当然我们还有一些建设中的一些能力，比如说多租户的能力，我们是否未来也能够说在公司内部面向于各种差异化的业务，比如说像 ToB 业务，那这类就会以 SaaS 的方式开放给我们更多的外部商户，那，那其实就需要这种多租户的能力，那多租户能力在中台中建设也意味着我们这种 ToB 的这些能力也能够逐步地向中台去靠拢。
那既然我们要覆盖更多的业务，我们就要更多地加强这种业务身份的这样的一个能力，包括刚才我们提到的隔离保护能力，我要继续的去做好做透，让各个业务之间不要有互相的影响。当然在未来我们也希望能够通过流程编排将各种业务它能够更好地、更轻量化地进入中台，它甚至说我们可以通过流程编排低代码的实现差异化的前台业务。


第二就是一个中台其实没有办法解决所有的问题，所以通过中台演化出垂直中台，通过垂直中台来解决垂直领域，因为每一个垂直领域都可能有其特殊性，那通过垂直中台解决垂直领域问题，能够更好地来帮助我们适配公司更多的业务，通过种种的这些发展与建设，我们相信未来我们的中台一定会越来越好。
