第一阶段就由我们的何大亮同学为我们带来关于联调的那些事儿，那在联调中都会经历什么呢？我们又有哪些事儿需要联调？通常来说联调就是一个人与人之间坐在一起去调试的一个过程，那基本上来说大项目跑不掉了，对吧？一个大项目肯定是跨了多个人、多个团队，甚至还有跨公司，那也意味着我们大家都要坐下来好好的去调一调这个代码，调一调这个架构，免得自己开发完了。觉得我是这么理解的，那他应该也跟我一样，实际上可能会有差异。


其次就是跨团队的这些项目，跨团队项目既涉及到不同的人，每一个人都可能有他自己的理解，有他自己的使用方法，通过联调将所有的信息拉齐拉平，来确保其最终效果的质量。所以只要有合作的地方，就需要有联调。那么联调中又要关注哪些事儿呢？首先第一点就是进度，联调毕竟是在提测之前最后一个环节，那这个环节里面我们也是要保证我们的进度的。其次明确其中的风险，并且控制好相关的风险。联调中的风险最大的一个值就是进度的风险，以及联调中发现的这些质量的一些风险。
我们要提测肯定要达到质量的一个水准之后才能提测，不可能把所有的问题都扔给 QA 去兜底，所以在这个过程里，这个点也是我们需要去关注的，当然最终也是要有对应的结果，有对应的成果，我们不能联调了好几天，最后发现什么都没调出来，那这个不就开玩笑了吗？所以在联调中最重要的是我们要去关注好两个人合作之间的边界，在围绕着这个边界去打造更好的一个合作模式，除了大量同学为我们带来的这一摊子事儿，我们还有大洋同学，洋大洋这次为我们带来关于流程相关的一些分享，那在流程中最重要解决的是什么问题？

在这之前课程中我们提到跟流程相关的基本上都是为了约束我们的行为，将我们的行为约束在一个已经确认明确其不存在大规模风险的这样的一个过程里，并且通过步骤一步一步地限制住这个行为，避免行为产生一些预期之外的行为。同时做好相关的信息记录与信息触达，让所有在这个流程链路上的人都能够获取到其需要的信息，那么这就是我们这个流程它作为我们带来的一个价值，当然它也起到了相关的管控的作用，而管控中最关键的一点就是准入什么样的架构，什么样的代码，我们可以让它达到上线标准了，这都是我们需要在准入这个点上去做好卡点的。


但是卡点太多又会带来很多很纠结的问题，所以大洋同学为我们分享的就是围绕着流程与卡点的这些事儿，在整个流程过程中，它还起到了相关的审批的一个效果，通过审批效果收获审批的这样的一个工作流来起到审批信息的触达，也通过其他的这样的一个工单来达到触及流程相关人这样的一个效果，并且所有的审批触达都存在相关的记录，所有的行为都被记录到整个流程系统中去。


这样子在后续排查问题去分析我们的行为的时候，我们也有相关的数据。而针对准入来说几方面是它可以对我们代码做好相关的扫描，针对静态的一些问题，针对一些我们需要关注的复杂度的一些问题，通过扫描可以帮助我们非常好的，非常低成本的来发现这些问题。然后做好相关的卡点，在它可以确定在什么样的质量下我们也能上线，什么样的质量下我们不能上线，以及可以有一些实验性的流程验证的过程的节点，通过这些节点来构成我们的实验室，确定在整个发布流程中我们的实验室的一些执行与项目的一些验证，那这些都可能为我们的研发过程带来一些提效，带来一些风险的降低。当然卡点越多，其带来的阻碍力，其带来的沟通成本也都会越多。而大洋同学为我们分享的就是关于流程卡点，关于这些在他们团队中落地的经历的那些事儿。
