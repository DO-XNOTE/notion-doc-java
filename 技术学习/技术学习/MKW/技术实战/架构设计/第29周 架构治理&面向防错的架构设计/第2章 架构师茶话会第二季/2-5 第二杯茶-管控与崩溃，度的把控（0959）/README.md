成语大于成本吗？这个得到了大家一致认可，都认可卡点的在长期来看它的收益是大于成本的。那在短期来看，每个人就开始说了短期这个，我们现在这个进度压力这么大，那这卡点我怎么搞啊？后总结出来一句话，就是债务决定成本，收益摆在那里，但是成本是由债务来决定的，债务越高成本越高。


那我们目前经历的这些问题，就是说我们其实有大量的技术债务，因为创业公司它一直是在努力地向前跑，努力地向前冲，那在这个过程里面其实我们还是欠下来的技术债务，所以在短期来看我们成本还是比较高的。那更近一点去看长期它能有什么收益？质量提升这块儿认可吗？大家都认可，无论从架构上还是代码上，它的质量都是提升的，但是它有成本也有成本，周期变长，那也意味着我们在这个代码和架构的质量提升换过来的这个变长的周期是否值得？这个其实我们要去评估的它到底变了多长，然后还有竞争力这个坏味道和新债务其实是能够去做控制的，因为卡点它把增量的部分卡住了，大毛孔规范的扫描，很多东西都会控制得住了，所以从这点上看，它其实是有收益的。


包括线上的稳定性，然后包括我们在整体工作中的这种习惯和我们伪造的这些的积累，包括债务的清还等等，都其实是有一些积累的。当然总体来看人员成长也是有达到了通过卡点倒逼人员成长的这样的一个效果，所以总体来看收益还真是远大于成本的，成本更多的还是在于研发周期的控制，而针对短期来看，其实最大的就是直接影响到我们工作的，直接顶在脑门上的就是这个债务偿还压力，而这个偿还压力也意味着需要去加班，真的你为了不影响进度，然后又要花大量时间在这个债务偿还上，那只能去加班了。在这个上面还会有涉及到精力的分散和进度，压力的变大。然后也还有一些人因为这些事就觉得太难受了，我在这干不下去了，我想走。这个人员存在着流失的风险，一些情绪在弥漫，而精力也是在消耗，但是它也带来了另一方面的一个收益效果，就是它的线上问题真的是收敛了，通过这个线上问题比原来大概降低了30%，这个效果还是非常好的，并且债务的控制和代码质量这两方面也是有了相关的一个收益，所以短期来看它的收益还是有，但是它的成本其实远比长期来看成本要高得多。
