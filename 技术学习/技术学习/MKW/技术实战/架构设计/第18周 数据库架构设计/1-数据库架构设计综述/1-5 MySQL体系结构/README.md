这一块并不属于 MySQL 的体系结构分层。我们指的 MySQL 体系结构，一般指的是 MySQL server 单的一个组成部分。此外，还有一些数会把体积结构划分成两层，分别是 server 层以及存储引擎层。而 server 层会包括连接层以及 SQL 层，存储引擎层不变，而文件系统层则被排除在外。不去讨论这种划分方式，其实是一种简化的划分，我们课上还是把它划分成 4 层，下面我们来逐个探讨。

连接池的玩法大致上是预先在连接池里面放入一定数量的连接，当需要建立数据库连接的时候，只需要从连接池里面取出来直接使用就 OK 了。使用完毕之后再发回到池子里面去。但需要注意，这里我们指的连接池并不是应用端的连接池。比方你的数据库连接池，比方你在项目里面使用的数据库连接池，而是 MySQL 服务端的连接池，它是由 MySQL 去实现的。不过连接池和我们平时熟悉的数据库连接池原理其实是类似的，都是管理数据库连接的池子。

其中 free space 区域是空闲空间，它其实就是用来分配 Row Records 的一个区域。当 free space 这个区域的内存全部分配完成之后，就表明这个页的数据量已经最大了。我们需要创建一个新的页，也就是分页,此外，在 free space 里面还会维护一个链表，当  Row Records  里面的一个数据被删除的时候，会放到 free space 里面的列表里面去。


另外， file  Trailer 部分是和 file header 里面的 checksum 相对应的。当页在内存里面被修改，还没有同步到磁盘之前，我们需要把它的 checksum 给计算出来。而由于 file header 是在页的最前面，所以 checksum 会首先同步到磁盘。而当 page 从内存同步到磁盘同步完成之后， checksum 也会被写到页的尾部。如果同步成功，页的头部和尾部的 checksum 应该是一致的。而如果写了一半突然断电了，或者是因为其他的情况没有成功的同步到磁盘。


file header 里面的 check sum 记录的是修改过的配置的 check sum，而 file trailer 里面的 check sum 记录的是修改之前的 check sum。那两个不同就说明同步中间出了错。其他的几个组成比较简单，同学们可以看一看。OK，这是页。最后是行，
