大家好，我是大木。这节课来探讨 MySQL 事务的隔离级别。虽然多数场景下我们并不需要去修改事物的隔离级别，但它是面试的重灾区。而且这种问题回答好了，很能够体现一个人的数据库功底，所以是同学们必须要掌握的知识点。话不多说，进入正题。现在花 2 分钟的时间做一个简单的铺垫。


首先，为什么要使用事物？这个问题有点多余。因为没有事物，我们就没有办法保证数据操作的安全性了。比如你银行卡里的余额，如果在操作的时候没有办法保证数据的安全性，数据可能不准确，还得了什么样的操作才是安全的。这就引出事物的 ACID 特性了。
ACID 是原子性、一致性、格力性、持久性这四个特性的英文首字母的组合。
