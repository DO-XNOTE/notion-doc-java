其中层次模型和网状模型在上个世纪七八十年代比较流行，不过现在已经逐渐被关系模型给取代了，现在再去学习这两个玩意儿，意义不是很大，所以课上我们不去讨论。而面向对象数据模型目前还比较新，支持这种数据模型的数据库系统还比较少，所以我们也不去探讨。对面向对象数据模型，感兴趣的同学可以看一下。

比方规范化设计，消除冗余，从而让你的数据库符合数据库的范式设计。数据库的范式设计大家应该是比较熟悉的，什么？ 1 范式 2 范式 3 范式 4 范式完美范式对吧？在架构式一期里面也有比较详细的探讨。再次强调，数据库设计并不是越规范越好，合理的冗余很多时候能够大幅度的提升性能。再比如，还可以对关系模式进行必要的分解，从而提升性能。比方垂直分解，对于平时说的数据库垂直扩容，也就是把一个字段超多的表，按照字段拆分成两个或者更多的表，每一个表只包含其中的一部分字段。此外，还有水平分解，对于平时说的数据库水平扩容，也就是把一个数据量超大的表拆分成若干多个表，结构相同的表，每个表里面只包含原先大表里面的部分数据，从而减少每个小表的数据量，提升查询性能。当然了，这里我们只是简单地提下水平拆分以及垂直拆分，在技术选型这一阶段还会探讨。
