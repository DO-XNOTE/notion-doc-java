全同步复制是主节点写入 Binlog 之后，就等着，直到事务在所有从库都提交之后才返回客户端。这样的好处是能够确保数据实时同步到了所有的从库。但是这样玩数据库更新的效率会差很多，你的增删改操作可能会有比较大的延迟。而半同步复制则是异步复制和全同步复制之间的折中。它的玩法是这样的主库提交事务之后不立即返回，而是等待至少一个从库接收到 binlog 并写入到 relay log 之后才返回，从而保证主库出问题的时候，至少有一个从库的数据是完整的。
