在这章我们探讨了分布式ID、分库、分表、中间件、分布式事务以及数据如何迁移。希望同学们经过讲解，至少要能够掌握。首先，生成分布式 ID 的几种思路，还记得课程上探讨了 5 种方案对吧？**分别是 uuid。基于 Redis 之类的工具，利用数据库本身的自增能力、号段模式以及类 snowflake 算法。**这些思路是必须要掌握的。另外，希望同学们对课上提到的几款分布式 ID 生成组件留一个印象，例如**滴滴的 tiny ID，美团的leaf，百度的 uid-generator 等等**。将来要是你的项目需要用到分母式ID，可以拿这些组件作为候选项对比一下，选择一个比较适合自己的用到项目里面去，或者也可以借鉴这些组件的设计去二次开发。



第二，要能够掌握 mycat 以及 ShardingSphre 这两款分布分秒中间件如何使用。这本应该是架构师一期课程的教学目标，这里只是再提一下。第三，要能够了解分布式事物的几种思想，也就是这篇文章务必通读，
