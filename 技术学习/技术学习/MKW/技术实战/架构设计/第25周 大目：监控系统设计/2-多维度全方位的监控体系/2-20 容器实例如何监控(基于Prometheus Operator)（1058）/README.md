首先，可以由 prometheus operator 生成 prometheus 的配置，之后， prometheus server 就可以使用这个配置去抓取各个 pod 的监控信息。如果配置了告警规则，并且在持续了 for 那么长时间之后就会触发告警，把告警推送给 AlertManager，再通过邮箱等渠道把告警推送给受件人。


最后， Grafana 负责可视化 prometheus server 里面的数据。总体来说， prometheus operator 的架构并不复杂，而且目前同学们只要知道 prometheus operator 能够生成 prometheus 的配置，并且能够管理 prometheus server 以及 AlertManager 就ok了。至于它具体怎么样实现的，我们稍后再说。
