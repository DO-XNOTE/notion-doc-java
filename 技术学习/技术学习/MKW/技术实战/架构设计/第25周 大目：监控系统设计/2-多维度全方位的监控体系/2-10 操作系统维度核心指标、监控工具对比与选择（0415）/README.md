一是 CPU 相关的指标，包括 CPU 使用率以及 CPU 队列长度。其中 CPU 队列长度指的是当 CPU 忙不过来的时候，就会产生任务队列，等待 CPU 资源去处理，队列越长说明 CPU 越忙。

第二类是内存相关的指标，包括内存总大小，使用了多少内存，剩余的可用内存、虚拟内存大小、腋交换频率等等。其中页交换频率是指内存和虚拟内存之间的数据交换的频率，越低越好。


三是磁盘相关的监控指标。包括磁盘总大小、使用的磁盘大小、剩余的磁盘大小、磁盘使用率、磁盘队列长度等等。其中磁盘使用率是指磁盘处于读、写等工作状态，占总时间的比例。磁斑使用率越大，表示磁斑越忙，而磁盘队列长度类似于 CPU 的队列长度，队列越长的表示磁斑越**忙。以弹幕个人的经验来说，在生产环境，磁盘队列的长度一般要保持在个位数，如果达到两位数，一般就要做调优了。**


第四是网络相关的监控指标。在微服务大显其道的今天**，微服务之间的通信都需要经过网络，相关的问题其实是线上我们经常要排查的问题类型之一。**有关网络，一般要监控网络的带宽消耗、丢包率、错误率等等。其中网络带宽消耗包括了网卡每秒接收到的数据量以及每秒传出去的数据量。如果接收到的数据量或者传输出去的数据量，这两个数据里面的其中一个或者两个和网络带宽相当，说明带宽已经达到了性能瓶颈。这个时候要么进行扩容，要么就得想办法把流量降下来。


总而言之，操作系统的核心指标其实就两大类，一是计算资源，指  cpu 、 2是 io 资源，包括内存、磁盘、网络，这是核心指标。下面来看看常用的操作系统监控的工具对比以及如何选择？
