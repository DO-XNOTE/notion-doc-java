上节课我们已基于 spring boot actuator 实现了 Prometheus 监控信息的暴露。但是在上节课，我们的 Prometheus.yaml 是这样写的。我们在 Prometheus.yaml 里面配置了微服务的具体地址。这种方式其实并不是很适合生产环境，因为在实际项目中，我们往往会有 n 多个微服务，每个微服务还会有 n 多个实例。如果采用这种方式，微服的地址发生变化，我们都要去修改。

Prometheus.yaml 有一天我们又开发了一个新的微服务，我们也得去修改 Prometheus.yaml，大家可想而知有多麻烦。

各个微服务把自身的IP、端口等等各种信息注册到服务发现组件里面去。这样服务发件组件里面会记录各个微服的地址。

Prometheus server 端会通过服务发现组件找到每个微服实例的 actuator Prometheus 端点，并自动抓取监控信息。


OK， Prometheus 它支持多种复发键组件，比方 Consul、eureka、 K8S 等等。

不管使用哪种复发性组件，这架构基本上都是一样的。我们课上使用的服务发件组件是 eureka。还记得前面我们已经把 Jenkins ci test project 的项目注册到了 eureka 上面去了。现在应用已经启动好了，我们访问 localhsot:8761，可以看到微服务已经成功注册。下面我们就来让 Prometheus server 通过 eureka 找到微服务，并去实现监控。
