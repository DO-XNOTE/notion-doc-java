随着互联网的飞速发展，项目是越来越复杂，需求变更越来越快，因为压力越来越大，故障的定位也是越来越复杂了，我们往往需要引入监控系统。引入监控系统可以帮助我们解决这么几个问题。首先是系统的可视化很好理解。利用监控系统，我们可以洞悉系统当前的运行情况。比方内存c、p、 u 磁盘的占用率，应用的QPS 接口调用的成功率，SRA等等，利用监控系统都可以清晰的展示出来。二是科学管理。应用监控系统有助于企业科学的进行管理。这个很好理解，不多说了。三是复杂问题的定位。在微服构建的系统中，往往会包含 n 多子系统之间的协作和调用。举个例子，某一次调用涉及到了三个微服务，请求调用 a 的某一个API， a 去调用b，又调用了 c 的某一个API。如果没有监控系统，我们得分别远程到 a、b、 c 这三个微服务所在的服务器或者是容器里面分析日志，才能知道到底哪里出了问题，是a、b、 c 层面的业务异常，还是a、 b 之间的网络有问题，或者b、 c 之间的网络有问题。而借助监控系统，比方利用日志监控工具，就可以在统一的地方去分析所有微服务的日志，从而快速的找到出问题的日志。再比如利用调用链监控工具，我们可以快速分析出是调用的哪个位置出了问题，测试站问题的排查有据可依。


OK。第四，引入监控系统，有助于提前预防问题。举个例子，监控系统，它展示了当前的 IO 硬盘CPU 的占用情况，如果监测到内存占用持续接近满负荷，我们就可以提前扩容，从而防止应用性能瓶颈的产生。第五，快速介入问题。举个例子，我们可以在监控系统中配置告警规则，比方配置 API 错误率达到 30%，就推送告警给相关的人员。这样一旦规则满足的话，监控系统就会自动推送告警，这样相关人员就可以快速介入并去修复故障了。第六，节省成本，引入监控系统也有助于企业去节省成本。举个例子，在没有监控系统的时代，往往需要值班轮班，总得有相关的人员去盯着系统有没有问题。而有了监控系统之后，可以轻松的实现 24 小时无人值守，出问题自动通知，这个时候人工再去介入就 OK 了。


还有第七，监控系统也有助于企业的形象提升，也就是提升科技感。有同学可能会笑了，这个好像不重要，但是以弹幕的经历来说，我们团队在 19 年上半年的 KPI 有一项就是科技感的提升。所以形象这个东西在大厂里面其实也是比较重要的。OK，这是监控系统的几点主要作用。简单来说，就一句话，利用监控系统有助于帮助我们了解系统的情况，从而让相关的操作有据可依。相关操作包括了提前预测问题，快速发现问题以及快速定位等等。OK，相信经过这节的讲解，大家已经知道为什么要引入监控系统了。好，这节课就到这里，谢谢大家。
