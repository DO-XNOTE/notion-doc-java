大家好，我是大木。目前市面上有非常多的监控系统，另外很多企业还喜欢自己造轮子，打造属于自己团队的监控系统，所以领域可以是百花齐放。但是各种监控系统之间还是有很多共性的，这节课我们就来探讨这些共性的部分，去探讨监控系统的典型流程，以及实现监控系统的常用方法，从而为大家在挑选监控系统以及设计监控系统的时候做一个参考。

一个典型的监控系统，它的流程大量是这样的。首先，监控系统会针对需要监控的指标进行数据收集，之后对这些数据进行分析，从而预测可能存在的故障风险。比方前面说的，分析内存的占用，从而预测可能存在的内存瓶颈，让项目提前扩容。而经过数据分析之后，发现系统的指标超出正常阈值，就推送故障告警。之后相关人员通过监控数据去定位故障并解决问题。


最后，当故障解决完毕之后，监控系统还可以发挥余热，用来做故障相关的数据分析以及问题复盘。比方可以为故障复盘提供数据支撑。例如在某某时间点发现流量不正常，平时是多少，突然变成了多少，什么时候解决问题了等等。这样我们的问题复盘就可以做得非常的清晰了。这是监控的点击流程。
