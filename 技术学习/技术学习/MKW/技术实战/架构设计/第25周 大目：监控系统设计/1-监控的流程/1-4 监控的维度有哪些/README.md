大家好，我是大木。这节课来探讨监控的维度，以及不同维度的监控应该承担的职责。目前市面上有非常多的监控系统，但总的来说可以分成 6 个维度。一是微服实例维度的监控。在维度重点监控微服实例的运行情况。例如微服实例所占用的 CPU、内存、磁盘，微服务的 QPS，微服务的 web 容器线程池、数据库连接池的运行情况，各个线程的状态，哪些线程是运行中的，哪些线程是 waiting 的，还有各个线程的 CPU 占用时间等的。

二是容器实例维度的监控。在维度重点监控容器占用的内存、CPU、磁盘、网络端口占用等等。

在这个维度，重点监控  JVM 的运行情况，例如 CPU 内存、GC 吞吐量堆限制栈的情况。四是操作系统维度的监控。在这个维度，重点监控操作系统的 CPU 占率、内存、网络、磁盘、工作负载、端口占用等等。

实际项目中，我们往往会用到各种各样的中间件，例如数据库中间件、消息中间件等等。针对中间件，我们也需要监控起来，不过不同中间件的监控指标不太一样，要看具体的中间件是什么。最后是系统整体的监控，也就是监控系统整体的运行情况。在这个维度，我们会从宏观上进行监控，例如监控整个应用所有微服务的日志，监控系统内部的调用链路等等，大量是 6 个维度，不同的维度我们会使用不同的监控工具，具体的工具我们会在后面详细探讨。总而言之，一句话，我们要让监控的力度丰富并且细致，我们要监控一切。这节课就到这里，谢谢大家。
