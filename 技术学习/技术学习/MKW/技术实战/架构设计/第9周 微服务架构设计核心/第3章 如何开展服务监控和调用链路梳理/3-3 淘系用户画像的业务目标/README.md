它就是从你个人的行为数据上面分析出你这个人的特点，从而给你去做一些精准的推送，它相当于是透过你的**业务买点所获取到的行为数据来刻画你这个人。**那我们大致从四个方向上跟同学们去聊一聊行为数据对业务的一些重要的助力。比方说第一个漏斗模型，那我们这里又谈到漏斗模型，那这一次漏斗模型并不是说用户的访问流量，那咱这里评估的是一个转化率，比如我们的下单这个动作其实可以从很多个入口里面来触发，比如说购物车，或者是你个人电商站点里面的收藏夹里面的收藏商品，以及各个不同页面，它有一些根据个性化推送的商品，那么这些都可以促成一个下单。


那我们收集到的行为数据就可以去获取到你每个不同的入口，它的转化率是多少？那从我们行为数据这里分析出来的转化率，那可以把它排排坐，从高到低搭成一个漏斗，那么漏斗最上部的说明你的转化率以及转化单量是最高的，那我们根据这从高到低的作词，可以梳理出一条什么呢？主链路。


那什么是主链路？咱前面提到过，就是完成你业务所需的最小系统可用度，那这是一个方面，另一个方面就是且在电商场景中钱和什么相关？转化率，钱和转化率是息息相关的，那运用这个漏斗模型，我可以去识别出转化率最高、转化总量最高的这样的关键链路，给它投入更多的资源，在这个链路上面发展更多的新式的玩法。


OK，那这里是一个应用方向，那还有接下来就到了真正的用户画像的部分，我可以通过你的行为数据做一个群体画像。那什么是群体画像？其实它就是透过你的行为数据去识别出有相同类型行为习惯的用户这一个群体它所存在的一些特征。比方说我跟同学们举这样几个例子，让大家理解一下什么叫做群体化。


讲那午夜后购物， 12 点之后购物的这些人，它有什么通用的消费习惯？那这其实就是一个时间维度上的一个群体画像。咱看到这个能想到的什么呀？睡得晚？ 00 后、 95 后对不对？那从这里我们可以大致刻画出一个相对年轻的一个用户群体，那我们再往下举例子，比方说客单价比较高的人群，他有什么群体特征？什么是客单价？就是说你每笔订单下单比较高的这个人群，那什么样的人群他具有这个高客单价的特征？很容易联想到妈妈团给小孩子买东西，中国的父母对小孩子是非常慷慨、非常大方的，那这是其中另一个维度。


我们再看，还可以通过群体画像去刻画地区性的这种群体，他们有什么消费习惯？那比如说广东可能比较热衷于购买厨房调料、煲汤之类的，那像四川可能更热衷买花椒、辣椒之类的这种地区性的消费习惯。拿到这些群体画像之后，我们其实要做一件什么事，去分析你的群体偏好，它有什么购物的倾向性等等这些信息，那我们既然要透过行为数据做一些精准的用户画像的推荐。那不光我们需要群体画像，还需要一个什么呀？个人的一个精准画像。比如说你这个人的消费数据当中，你所收藏的商品，以及触发复购的最多的商品类目是什么？比如说某个人特别喜欢吃花生，那它复购率最高的商品是花生，其实就是分析出这么一个简单的结果。再比如我可以分析一个人的购物习惯，它在哪些时间点会比较活跃的去浏览商品，哪些时间点是他购买欲最强的时间段？我可以做一些特殊的用户推荐，在某些特定的时间段给他推送过去。那我们把群体画像和精准画像合二为一，就可以得到一个非常精确的用户画像来做你的精准推荐，在什么样的时间点给他推荐什么样的类目，能最大程度地促进这个下单的这个动作的完成，那这里相对来说是处于一个比较微观的层面。我们再从宏观层面来说行为数据这里还有一个用途，就是让我们的业务人员可以去实施的掌控你的大盘数据，比如你的每天的 PV 以及日均的交易额、交易单量等等。那你当日的复购率是多少？你浏览特定商品的所有用户，他最终促成的订单也就是转化率是多少，这些数据是相对比较重要的运营指标从哪里来？就是从你用户的行为数据上面去把它挖掘出来。那讲了这么多的业务场景，那我们再来看需要挖掘什么样的行为数据才能达到这样的业务场景分析？我们就像小学的时候写作文、写文章，那老师都会告诉你们要有四个要素。什么要素？时间、地点、人物事件，对不对？

行为数据，这里它的要求和小学生作文题目是一模一样的，它也区分别需要这四个要素齐备。比如时间，那你当前的这条用户数据，这条行为数据，它发生的具体的时间点是什么？以及第二个要素，地点。那第一点是指什么呀？


有两个不同的维度，第一个维度是指你触发这个动作所在的地点。那咱前面说过，我们做用户群体分析的时候，可以去分析地区性的消费习惯。OK，那这个地点就隐含着你下单的地点，浏览商品的地点，那这是一个方向，还有一个方向是什么呀？你的页面元素，你的这个场景是在页面上具体的哪个元素上面触发的？这可以去帮助你分析一些，比如说复购率最高的业务场景，是购物车还是收藏夹，还是你的用户推荐等等。那我需要知道的就是你当前的业务动作，它在哪个具体的页面，哪个具体的空间上面发生。


那再往下看人物，这个人物有两层意思，第一个是匿名用户，也就是非登录用户，比如电商行业当中，你浏览商品详情去搜索商品，其实并不是需要大家登录操作的。那么这里我抓取的是一个匿名用户，再往下登录用户，这个才是比较有业务价值的数据，我要捕捉你当前有账号的这部分登录用户，他的用户习惯，那分析他的习惯为什么呀？干嘛要分析他？因为我们的这个用户后面咱在注册电商场景的时候，都会提供自己的个人信息，你的性别、你所在地以及你的年龄等等这些信息。


那它是这些用户的元数据，通过你的用户行为数据，再加上他背后的这些性别、年龄等等个人的数据，我们就可以把你的用户行为套用到你用户的一些个人信息特征上面，从而得到某些相似群体，那它都有什么共通的用户行为？OK，这就是咱前面说的群体画像，那时间、地点、人物都有了，接下来干嘛呀？就是指你接下来的这个事件是什么事件，在咱们的这里就其实就是业务数据，它做了什么样的操作，它操作的哪些业务数据？这是整个 log 当中的事件描述的部分。


那提到这 4 个点，同学们回想一下，我们平时在写后台业务代码的时候，会把这 4 个点全记录下来吗？恐怕不会啊，我们经常的会记一些关键的log，比如说你的商品被上架了，但是商品上架好像没有人物这个特点，对吧？是谁上架的？他 log 里面可能没记。再比如你一个用户做了一个添加购物车的操作，那我在后台业务代码当中记录这样一个log，但是好像并没有记录它是从哪个页面触发的。


那同学们再仔细琢磨一下我们后台的一些业务，埋点业务日志是不是很难找到这四要素全部齐备的？没错，那这里跟同学们讲这个用户行为分析的这样的一个场景，其实并不是依赖于你后台的业务日志，我们有一个更通用的一个叫法，叫无痕埋点。
