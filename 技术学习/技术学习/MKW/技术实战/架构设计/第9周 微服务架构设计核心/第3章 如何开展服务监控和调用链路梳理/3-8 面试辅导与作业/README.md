那么因此在我们谈到每一个买点方案的时候，这里要有一个各方功能的侧重，那它的优势是什么？适用于哪些埋点场景？那同时它有哪些劣势？那第二个面试问题，来考察同学们线上排查问题的方式、方法、手段，比如线上故障排查从哪些方面可以入手？OK，当你被问到这个问题的时候，其实最省事的一个方案，也是最直接的一个方案，就是使用链路追踪那一个线上故障，当一个 error 发出来的时候，最简单的我在 error 里面去找一个什么信息，一个 global trace ID，那它是你当前这个请求上下游链路串联起来的一个全局唯一的ID。通过它我能找到什么信息？能找到所有上下游调用链，在这一整条链路上面打上的所有信息，它都是带上了这个 global trace ID，因此线上故障排查有没有经验，就看你的方式、方法、手段。那如果你达到这个点上，那你肯定在线上排查问题方面肯定有自己的经验积累。
