# 2-3 本章小结

配置管理是微服务领域一个很重要的功能点，在多年前我们对配置管理的理解还停留在“剥离成配置 文件”，但这种方式还是将代码业务与配置项混合部署，即便将属性项通过java启动脚本传入，也必 须在配置文件中声明相关属性。
随着微服务理念的发展，现在“配置管理”已经不再只停留在“配置”这个层面，他也是作为一项基础服 务，就像订单、商品服务一样，通过注册中心将自己注册成为一个服务提供者，这种方式可以彻底 将配置资源从应用层面和环境层面隔离出来。
