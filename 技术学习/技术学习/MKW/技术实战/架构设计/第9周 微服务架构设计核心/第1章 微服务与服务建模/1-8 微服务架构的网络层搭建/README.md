那首先抵达的是 Nginx 吗？不是的诶，我们访问慕课网的首页，最先的请求抵达的是咱的DNS，那么 DNS 这里做第一层的负载均衡，它会返回给你多个地址当中的一个地址。


那么我这里假设咱的 DNS 上面配置了两个地址， 110.120.119. 10 还有点 11 这两个地址，那么这个请求拿到具体的这个 IP 地址，它会访问谁？那同学们看，这里其实不是Nginx，它会访问一个 l v s 所暴露出来的虚 i p，那并不是所有公司都使用 LVS 的，那有些公司，比如老师现在在公司，他们就使用的是F5，也是一种网关集群。


那 LVS 是咱的 Linux 里面最常用的一种虚 IP 的生成的方式，那咱这里 DNS 返回的 10 和 11 网关就对应着咱 LVS 里面的两个不同的集群。那我为什么这里画两个？每一个 IP 后面我画了两台服务器？因为这就是砸 LVS 的一种主从架构的模式，那当然在真实环境里往往是不止两台的。那么为什么我们这里要把 LVS 做成这种一主一从的？这种形式很简单，咱在做所有架构设计的时候，要考虑的一个问题就是要消灭单点。什么叫单点？你单个服务器，如果它挂掉了，那么你的服务就会变成不可用，所以每一个环节一定要去避免这种单点的存在。


那么我们看这个 LVS 虚 IP 背后，它真实的 IP 是什么呢？这里才到了我们的反向代理层 NGX 集群，那么我们的 Nginx 集群这里通过 LVS 对外虚化出了同一个 IP 地址，所以咱说这个网络部署结构没有想象的简单，对吧？咱在 Nginx 和我们的用户访问请求之间，还隔着一个 LVS 或者是 F5 以及最外层的 DNS 的复载均衡。


那到达了NGX，我们是由 Nginx 集群直接路由到我们的微服务网关吗？同学们想一想会直接路由到吗？一般来说在我们的大型应用或者是大规模的互联网公司这里，通常你的 Nginx 集群当中还会牵扯到多层的跳转，也就是说一个请求从最外层的 Nginx 集群到达微服务网关之前，通常还会经过几层的转发，并且在这个环节当中，一般我们还会做一个什么呢？一个 API 的鉴权服务，来确保你当前的访问请求是一个授信用户，或者是说授信站点发出来的。

好，那我们接着往下说。一个请求，抵达了微服务网关之后，接下来就交给咱的微服务网关，我们也叫做业务层网关，那由业务层网关经过一些路由以及谓词断言等等，把你的请求真正的路由到了后台的服务集群当中，那这里老师就要说到这中间的这条横线了。通常来讲，在大公司当中，我们会根据你的服务特点把它分为安全级别的服务，还有可以对外访问的服务。比如上面的这两个服务，我们是可以把它暴露给外部用户的。因此在网络部署结构上面，我们会划分一个单独的区域叫 unsecure zone。从名字当中我们就可以看出来，当前的这个网络分区不是一个完全安全的网络分区。什么意思？就是说它可以直接承接外部抵达的用户请求的流量，那从安全级别的角度来说，我们把这类服务打包在 unsecure zone 这套网络环境中去，那对于一些非常私密的服务，比如你的底层的交易链路，或者你底层的财务清算系统，那这部分数据第一个非常的敏感。


第二它不会暴露给任何的外部访问的用户。那从数据安全的角度，我们会给它单独成立一个分区，叫做 secure zone，也就是当前分区的所有服务。它不是一个对外服务，那由外部转接过来的网络请求无法抵达到这个分区。那通常来讲，如果你处于 unsecure zone 的服务，想要去访问 secure zone，那么我们会需要专门的团队，网络安全团队去审查你的访问需求，打开特定的防火墙的白名单。那对于 secure zone 这里的数据访问策略，通常来讲，我们是说只进不出，只允许数据进来，不允许数据出这个网络分区。okay，那这整幅图就是我们经过简化的微服务架构下的网络体系搭建。
