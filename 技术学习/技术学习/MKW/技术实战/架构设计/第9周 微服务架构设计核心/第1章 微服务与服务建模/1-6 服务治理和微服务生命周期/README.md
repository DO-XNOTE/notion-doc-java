不过另一方面，即使你是一个有过 RPC 这些框架使用经验的开发同学，那你也未必知道服务之力，因为服务治理是微服务框架后面的一个背后的一个流程，那我们平常在工作当中使用微服务框架调用微服务，可能并没有去花心思了解这微服务框架背后是怎么来运作的，那么这一节里我们就从服务治理来入手，看一下微服务的一个生命周期。


那这里我们先拿一个简单的服务调用跟同学们举例子，一个用户这里发起了一个服务调用请求，这个请求首先被路由到了我们的微服务的网关层，那这个网关并不是指它的反向代理层，那它是更往下层的一个业务层网关，我们可以把它理解成像 spring cloud 当中的 Gateway 或者是 Zoo 这类的微服务网关。

[那微服务网关这里把这个请求路由到了一个订单服务的服务集群当中，我在这里下了一个订单，那订单里面你买了什么商品？](/d7cbf4fb862c4c41ab73f7062bd87fc8)是不是要从订单微服务发起一个调用，从商品服务当中获取这个商品的信息，锁定库存等等。OK，那么我们的问题就来了，那同学们平常在写代码的时候，只是简单的发起一个远程调用，你并不知道它背后发生了什么。[那现在问题来了，我问大家这个订单服务是如何知道你商品服务都有哪些可用的服务器列表的？](/d7cbf4fb862c4c41ab73f7062bd87fc8)咱的这个前提是说，你的订单到商品之间的这个访问请求不会经过网关转发，它是由订单服务直接发起抵达商品服务的。那这个点对点之间，两个微父集群之间，他们怎么知道互相对方有哪些服务器？它的地址是什么？它是否是处于一个可用状态？这是第一个问题，我们接下来看第二个问题。


假设我们现在做双 11 的大促了，我的这个商品服务有点扛不住压力，[我想要对这个商品服务的集群做一个弹性的扩缩容，这里有可能往集群当中加入新的服务器，也有可能去下线掉一台服务器。那我在商品微服务集群当中，我的这个服务器增加和删除操作如何反映到订单微服务](/d7cbf4fb862c4c41ab73f7062bd87fc8)？我怎么知道哪些是你新加入的服务器，哪些服务器现在被下线掉了？如果这些服务器当前的可用状态没有同步到订单微服务这里，那我发起的远程调用就可能是失败的。OK，那这是两个问题了。接下来我们再来看一个问题，如果我的这个商品微服务出现了一些网络故障，导致服务不可用，或者说发生了一些 out of memory 这样的内存泄露操作，导致服务器宕机了，[那对于这种宕机和下线操作，我如何把这个信息同步给订单服务？那以上的这几个问题，同学们如果能把这些问题解决掉，那就算是玩转了。](/d7cbf4fb862c4c41ab73f7062bd87fc8)


服务治理就是这么简单。服务治理其实它的概念我用一句话跟大家去说一下，它就是指我如何去维护一个当前可用的服务器的列表，那维护在哪里？我不管，反正我要有这么一个地方来维护它。同时我这个列表要能反映你服务当前的状态，也就是说你的服务器下线，上线，增加服务器，删除服务器，我要体现在这个服务列表上面，这就是服务治理。


那么我们接下来，那我们再回到这个问题上，你的这个可用服务的列表它保存在哪里？从当前这幅图里面，同学们看到有一个地方可以保存这个列表吗？诶，好像没有啊。那我们的微服务治理框架是如何来处理这个问题的？来我们接下来看，

[怎么说？我的商品服务每隔一段时间，比如说 15 秒、 20 秒，每隔一段时间我会向咱的注册中心发送一个心跳包， heartbeat package，那这个心跳包实际上就是你的商品服务去上报自己的状态，我觉得自己状态还很好，我还能继续喝，那这样的一种状态，那万一你这边喝断片了怎么办呢？](/d7cbf4fb862c4c41ab73f7062bd87fc8) @哎，你当前的这个服务续约服务心跳没有在固定时间内发送到你的服务注册中心，那这种情况下你的服务注册中心这里就会去做一个动作，叫做服务剔除，它会每隔一段时间跑一个批处理，那么这个批处理就会去查看你过去一段特定的时间内，比如方说一分钟内我都没有收到你服务的心跳包，那么这种情况我就会将你的服务从自己的服务列表当中把它给删除掉，这就是辅剔除。


okay，那么[通常来讲，咱的各种注册中心都有一个开关，可以去手动的来控制服务剔除功能的开启和关闭。那么对于某些注册中心，它还有一些高级版的功能，比方说你在短时间内服务剔除所剔下线的服务达到了一个特定的比例，比如说 80% 的服务都在过去的一分钟之内被服务剔除掉了，那么通常这种情况是属于一个非常极端的情况，那这个架势一看不对，这会不会因为短暂的网络抖动导致服务暂时不可用？那我可不能把所有服务都踢下线？](/d7cbf4fb862c4c41ab73f7062bd87fc8)这种情况下，有些服务注册中心就会开启一个服务自保功能，一旦服务自保功能开启，那么它的服务剔除这个功能就会被相应的关闭，那么这里是一个被动剔除的这样的一个操作。
