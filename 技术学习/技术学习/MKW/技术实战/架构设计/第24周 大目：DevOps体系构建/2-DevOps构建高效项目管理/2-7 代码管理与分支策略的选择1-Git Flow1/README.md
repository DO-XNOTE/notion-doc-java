而三个临时分支分别是 feature 分支、 release 分支以及 hotfix 分支。 feature 分支是特性分支，用来开发一个新的功能，一旦开发完了之后就可以合并到 develop 分支了。


release 分支是发布实施分支，当我们要发布的时候，就会基于 develop 分支创建 release 分支。当 release 分支完成之后，又会合并到 master 分支以及 develop 分支。**最后 hotfix 分支是 bug 修复的分支，用来修复生产环境上非常紧急，需要立即修复的问题。完成之后也会合并到 master 以及 develop 分支**，可以发现临时分支可以同时存在多条比方，同时存在 2 个特性分支，存在 3 个 release 分支，都是 OK 的。
