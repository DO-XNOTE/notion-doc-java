大家好，我是大木。现在我们已经知道如何用 redmine 去管理项目了。在此基础之上，你还可以为 redmine 引入 scrum 以及看板，从而让项目管理更加的敏捷。要想实现这一点，需要为 redmine 安装scrum。插件。地址在这里插件的下载地址，

[https://redmine.ociotec.com/projects/redmine-plugin-scrum/files](https://redmine.ociotec.com/projects/redmine-plugin-scrum/files)

点击这里的文件，里面有每一个版本的 scrum 插件。我们课上使用目前最新的零点二零点零，点击就可以下载了。下载下来是一个压缩包，对于网络不好的同学可以到这里去下载。下载之后把它解压到 redmine 的 plugins 目录，并执行这条命令，

`RAILS_ENV=production bundle exec rake redmine:plugins:migrate`

[https://redmine.ociotec.com/projects/redmine-plugin-scrum/wiki](https://redmine.ociotec.com/projects/redmine-plugin-scrum/wiki)

就可以安装插件了。来操作一下。

首先解压，编码，解压后的插件放到 redmine 的 plugins 目录。还记得前面我们已经把 redmine 的 plugin 目录挂载到了这个位置对吧？最后用 Docker PS 查看当前运行的容器，再用 Docker Exec 杠 IT 并发小。进入到 redmine 容器 CD plugins，可以看到容器里面已经有 scrum 目录了。最后执行一下这条命令安装插件。

`RAILS_ENV=production bundle exec rake redmine:plugins:migrate`

安装完成之后，需要重启一下 redmine, exit 退出容器 docker ps ，查看运行的进程。 Docker restart 跟上容器ID，重启 redmine。容器重启完成，回到浏览器刷新，可以看到重启之后在 redmine 右侧多了很多的警告，这是因为 redmine 插件还没有初始化导致的。我们来对照警告初始化一下。


先来解决这个问题点击之所以有警告，是因为这里的 sp 自定义字段没有候选项而导致的。这里的 sp 是 sprint 冲刺的意思。我们来添加一下自定义字段，点击自定义属性，新建自定义属性，选中问题。下一页格式是字符串名称，填入 sp 就 OK 了，点击保存了，你也可以输入其他的名称。之后再次点击 sp 自定义字段，就可以看到下拉菜单了。选择 sp 并点击应用，可以看到刚刚的 sp 自定义字段插件未配置的警告已经消失了。
