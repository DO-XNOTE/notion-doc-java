大家好，我是大木，来探讨使用 JIRA 管理项目。 JIRA 是世界范围内非常流行的项目管理工具，它的使用体验、功能性、可维护性、插件支持等等各方面在同一领域都是数一数二的。要说它的缺点，可能就是商业授权需要掏钱。它有三种使用方式，一是 server 版本，这个版本可以下载下来，自己部署安装。下载地址在这里。

[https://www.atlassian.com/zh/software/jira/download/data-center](https://www.atlassian.com/zh/software/jira/download/data-center)

目前的长期支持版本是 8. 13. 5，另外短期支持版本是 8. 16. 0，建议线上使用。长期支持版本根据自己的操作系统下载安装就 k 了。不过在这里有一个说明，他说 server 版本已经停滞销售了，停止销售的时间是 2021 年2月 2 号，也就是刚刚停售没多久。而到 2024 年2月2日则会直接停止支持。它的替代品是 data center 版本。不过目前来说，国内 90% 以上采用 JIRA 的企业使用的都是 server 版本。这意味着这些企业在 2024 年2月 2 号之后都会面临着迁移的成本。这是 server 版本一个被官方已经抛弃的版本。

第二种方式， data center 版本也可以下载下来，自己部署安装。下载地址在这里。 data center 版本也包括了长期支持版本以及短期支持版本，也是根据自己的操作系统下载安装就可以了。说假设想要安装这个版本，就点击 get started，选择一下自己的操作系统，点击 submit 就可以下载了。


第三种版本， cloud 版本，地址在这里。其实就是直接使用它们的云服务，三种版本的功能基本上是一致的，但是 UI 以及使用体验是存在差异的。有关 JIRA 的版本政策，我找到了两篇解读文章，面试这里，以及这里。总的来说，和这两篇文章类似，我个人对 data center 版本持消极态度。首先，对老客户来说，目前国内更多的企业采用的是 server 版本的JIRA。刚刚说过了，可能 90% 以上的企业采用的都是 server 版本的JIRA。未来要么迁移到 data center 版本，要么迁移到 cloud 版本。而迁移老的数据，怎么样迁移，这是一个很大的挑战，因为要花费大量的人力以及时间，而不同的版本在使用体验上又会存在差异。既然这样，我做迁移也得从 0 开始，让大家去适应。我换一个软件，也是让大家从 0 开始适应，成本都很高。


总而言之，迁移是一个很大的挑战，你会遇到各种各样的阻力。比方大家适应了 server 版本，不想迁移，用习惯了。比如你迁移了之后，大家都要从 0 去学。一刀切的方案是非常激进的，尽管有三年的缓冲期，但是三年之后你就不得不面临迁移了。还有一点，对于 server 版本的用户，在购买了原生 server 版本的商业授权之后，还需要另外购买 get center 授权或者是 cloud 版本授权，这其实是一笔不小的费用。所以大家如果在项目里面选用 JIRA 管理项目，要慎重一下，**至少对于新项目不要使用 server 版本，而应该使用 data center 版本或者是 cloud 版本，可以避免掉牵引的开销。**


如果同学们想要领略 Jerry 的风采，对于 server 版本，可以参考一下系列博客，比方简介肌肉是什么，有哪些功能，怎么样用 JIRA 进行敏捷的开发管理，写的还是不错的。而对于 deck center 版本或者是 cloud 版本，可以看一下这篇文章，这是 JIRA 官方文档。由于 JIRA 不同的版本有不同的使用体验，所以课上我们就不去展开探讨如何使用 JIRA 了，否则三个版本演示一下，可能几个小时都过去了，意义不大。这节课就到这里，谢谢大家
