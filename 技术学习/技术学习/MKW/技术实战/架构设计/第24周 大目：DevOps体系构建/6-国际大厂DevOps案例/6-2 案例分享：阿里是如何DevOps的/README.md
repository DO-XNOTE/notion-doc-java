大家好，我是大木。这几个来分享阿里是如何 Devops 的。首先，在阿里内部普遍采用了一体化的应用交付平台，叫做 Aone。 Aone 包含了需求管理、迭代管理、任务管理、代码管理、流水线管理、缺陷管理、测试计划管理、版本管理、知识库等等功能。可以这样说，咱们课上前面讲的一系列工具的功能， Aone 都包含了，并且提供了相同的使用体验。在 2017 年的时候， Aone 正式都要开放了。

开放版本阿里云效。由于云校的使用方式和内部的 Aone 非常的类似，所以下面不妨演示一下如何利用云效去做Devops。这样一来可以让大家对阿里如何 Devops 有一个更加直观的理解，二来可以熟悉阿里内部的项目管理流程。再一个，如果同学们想要实现自己的 Devops 平台，也可以做一个参考。

[https://devops.aliyun.com/workbench?_userId=63f66a7156230889ff7ffb9a&timestamp=1677093610938&mode=redirect&sign=701626b5dd70c258124cf05a51ce48b5](https://devops.aliyun.com/workbench?_userId=63f66a7156230889ff7ffb9a&timestamp=1677093610938&mode=redirect&sign=701626b5dd70c258124cf05a51ce48b5)

在左侧是菜单栏，企业成员可以管理企业里面的人员，比方邀请人加入到自己的企业管理后台，可以管理企业的相关信息。基本信息里面可以把免费的企业转成收费企业，可以去删除企业。另外还有成员管理、权限管理、项目管理、应用管理、安全管理等等各种信息。比较简单，其他的几个菜单一会再来探讨。

总的来说， Aone 的玩法和云校基本上是一致的。而通过演示可以看到阿里内部的 Devops 玩法和课上的讲解也是非常一致的。只是阿里把相关的工具都做了整合，形成了一个超强的品牌机。开发人员可以傻瓜化的去实现Devops。同学们所在的团队如果有实力，也可以参考云效去研发公司自己的 Devops 一体化平台。

首先，使用自己研发的工具，就不需要让用户分别去学习各种工具如何使用了，从而降低学习成本，也就是你再也不需要用户分别学习 SonarQube、Jenkins、 Redmine 等等一系列工具了。不同的系统有不同的玩法。学习成本其实还是不小的。学习成本虽然均摊到每个人身上并不大，但是从整个团队，比方，团队有 200 号人，每个人都要花一天的时间去学习，就是 200 个工时，学习成本其实还是不低的。


第二，使用自研的平台。大家也不用适应不同工具之间迥异的使用习惯和使用体验，比较好理解。更重要的是，利用自己研发的工具，可以做很多定制性的工作。


比方在阿里内部有一个叫做阿里大脑的项目，它可以分析每个员工的产出情况，比方代码量，解决 bug 的数量，响应问题的及时程度，每千行代码的 BUG 率等等，这样可以作为绩效、年终奖晋升的一个依据。这些数据的主要来源就是 Aone。而如果我们使用的是 read man 加 Jenkins 加 SonarQube 这种组合，想要实现类似阿里大脑的功能就比较困难了，对吧？这是使用自研平台的好处。最后简单总结一下这几个，结合了阿里云效，探讨了阿里如何Devops，希望给同学们有一个良好的参考。这节课就到这里，谢谢大家。
