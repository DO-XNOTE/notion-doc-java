大家好，我是大木。这几个来探讨 Jenkins 的流水线。流水线是间隔在 2. 0 引入的，得到了广泛的关注和好评。来看看流水线怎么玩儿。首先点击新建item，输入任务名称，比方 Jenkins Pipeline， eureka server1。任务类型选择流水线，点击OK。最后也会弹出任务的配置界面。 general 构建触发器。和自由风格的项目基本一样。高级项目选项其实一点都不高级，只能指定一下任务的名称。关键是流水线这一栏，你可以选择 pipeline script，把流水线脚本填到输入框里面，也可以选择。 Pipeline script from scm 表示从 scm 里面读取流水线脚本。

```shell
pipeline {
    agent any

    stages {
        stage('Hello') {
            steps {
                echo 'Hello World'
            }
        }
    }
}
```

首先， pipeline 表示这是一个流水线。 agents 表示流水线在 Jenkins 的哪个环境里面运行。实际项目里面， Jenkins 可能是一个集群，你可以控制在某一个 Jenkins 实例上进行执行。 any 的话表示流水线允许在任意一台机器上运行。

stage，是一堆 stage 的组合。stage 表示一个阶段，它是一组任务的组合，这里定义了一个阶段叫做hello。 step 表示一个阶段里面需要执行的具体步骤。 step 可以认为是流水线的最小组成。


我们可以发现语法还是非常清晰的。它表示一个流水线里面可以包括多个阶段，每个阶段里面可以包括多个步骤，每个步骤可以做我们想要让他做的事情。另外你也可以点击 GitHub 加Maven，这样就会弹出一个比较典型的流水线脚本，你可以参考脚本编写自己的流水线。

```shell
pipeline {
    agent any

    stages {
        stage('Clone') {
            steps {
               git credentialsId: 'edffa585-9406-4962-bb0b-8392df7b31c4', url: 'http://192.168.13.224:3000/guojun/eureka-server.git'
            }
        }
         stage('Test') {
            steps {
              dir('eureka-server') {
                 sh "mvn test"
               }
            }
        }
         stage('Build') {
            dir('eureka-server') {
                sh "mvn clean install -DskipTests"
                sh "docker build -t my-eureka ."
            }
        }
          stage('Deploy') {
            dir('eureka-server') {
                sh "kubectl apply -f k8s/my-eureka-ha2.yaml"
            }
        }
    }
    
}
```

再来编写test，用 dir eureka server，表示将目录切换到 eureka server目录, 大括号执行 shell 脚本 sh mvn tests 执行单元测试, test stage 编写完成。下面编写 build 需要注意的是每个 stage 都是独立的，所以每个 stage 执行的命令默认的目录都是当前项目的根目录。所以在 build 里面，我们也需要用 dir eureka server 将目录切换到 eureka server目录。少了 step，我们把 step 整体拷过来粘贴，然后 shell 脚本写上 mvn clean install -Dskiptests，在执行 docker build -t my-eureka
