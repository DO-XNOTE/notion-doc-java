大家好，我是大木。前面我们虽然每次的构建结果都是 success，部署也都成功了，但是细心的同学会发现了，由于我们每次构建的时候使用的镜像 tag 都是latest，对吧？不管是直接编写的 K8S 脚本也好，还是 helm 的 chat 记上都是latest。所以不管是用 kubectl apply 还是用 helm，不管你去部署多少次，即使你的代码发生修改，并且用 Docker build 重新构建了镜像K8S，也不会重新去部署。
