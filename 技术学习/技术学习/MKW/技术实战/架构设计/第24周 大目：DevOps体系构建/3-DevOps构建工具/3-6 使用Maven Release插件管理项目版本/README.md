这节课来探讨项目版本如何管理。在项目升级版本的时候，不可避免需要修改泡面 x mail。如果手动修改版本号是比较麻烦的，因为在复杂的项目里面可能要修改 n 多处。比方这是一个多模块的内容，项目，有父项目，还有子项目，到处都要去修改。同时，手动修改版本也很容易出问题，比方有些地方漏改了或者改错了，都会导致版本发布出问题。所以项目建议使用 Maven 的 release 插件去管理版本，从而减少出错，提高效率，增加项目的可维护性。
