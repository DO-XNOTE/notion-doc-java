这几个来探讨如何用 Maven 的 Docker 插件去构建 Docker 镜像。目前业界主要有这么几款 Maven 插件可供选择 

1 是基本 maven plugin， 

2 是 favorite docker Maven plugin 

3 是 Spotify 的 docker Maven plugin。其中 Spotify 的 Docker Maven plugin 已经不维护了，替代品是 Spotify 的 Dockerfile Maven，但是 Dockerfile maven 也已经 2 年多没有维护了。



大致上是这么几个选择。不过对于现在的 Springboot 玩家来说，我个人已经不太建议用这些插件了。

原因主要有两点一是不管你使用哪一款插件，本身都会存在一定的学习成本，而且学习成本还不低。既然这样还折腾啥，索性直接用 Dockerfile 得了，对不对？


第二，对于 spring boot 玩家来说， spring boot maven plugin , 从 spring boot 2. 3 开始，已经具备了构建镜像的能力，你只需要用 Maven spring-boot:build-image 就可以构建镜像了
