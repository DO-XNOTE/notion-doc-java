第二是Maven。 Maven 是目前最流行的项目构建工具，它提供了项目构建、测试、发布等等一系列的项目管理功能，大家应该是非常熟悉的。第三是Gradle。 Gradle 是这个圈子的后起之秀，它基于脚本编写，使用起来更加的灵活，同时它的概念相对会多一点。另外，由于是基于脚本的，所以要去学习脚本如何编写它的学习曲线，相对 Maven 来说要高一点。从功能上来说，这几款工具都能够满足项目上的需求。

Maven 虽然是最流行的，不过已经有越来越多的企业选择用 Gradle 构建项目，甚至有越来越多的项目从 Maven 迁移到Gradle。比方大家熟悉的 spring frame mark，早期是Maven，现在已经改用 Gradle 了。可以看到这几个 Gradle 的相关文件。 spring boots 早期也是Maven，最近也已经改用了gradle。 okay 回到 b d，如果同学们想将你的 main 项目迁移到Gradle，只需要执行 Gradle init 杠杠 tab Pom 这个命令就可以了，命令会自动的把 Maven 项目迁移成 grade 项目还是比较简单的，多数情况下命令都能够很好地实现项目的转换。当然也不一定，有的时候还需要人工介入做一些调整才行。


目前国内的情况来看， web 方向的 Java 玩家多数还是使用的Maven， gradle 相对还是会少一点。所以咱们课上也是基于 Maven 去讲解的。并且下余时间课上我们不会去展开探讨 Gradle，但是结合国际上的趋势，同学们还是可以抽一点时间研究一下，毕竟技多不压身嘛。至于 Ant 可以不去了解。好，这节课就到这里，谢谢大家。
