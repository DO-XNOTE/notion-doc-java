大家好，我是大木，遇到了划重点的时候，经过这一章的学习，同学们要能够掌握。

**首先 Maven 私服的搭建与配置， Nexus 基本上是企业的必备工具**。

至于项目的版本管理，弹幕强烈建议大家按照课上讲解的方式去管理。不过你要是你们公司有自己的版本命名规划，或者你们团队喜欢手动修改版本也是 OK 的。

但不管怎么说，弹幕还是希望大家能够把**应用的版本管理给重视起来**。

3: 项目的发布，这也是一个必备技能，同学们至少**要能够掌握怎么样把应用发布到 Maven 私服**。至于发布到中央仓库，可以作为拓展知识不是必须的，毕竟不是每个人都会去做开源软件，对吧？


第四， Docker 的私服搭建。本质上这也是一个必备技能。不过你只需要掌握 Nexus 或者 Harbor，又或者其他 Docker 私服的任意一个就 OK 了。所以我也用斜体去表示了。


第5，编写dockfile。这是一个必备技能，重中之重。


第6，**编写Kubernetes。编排文件也是一个必备技能**。同时同学们还需要能够活学活用，这样才能够满足业务上各种部署的需求。


最后，**有关helm，我个人是强烈建议大家在企业里面去使用，但它也不是必须的，毕竟它只是一个简化 K8S 应用交付的工具而已**。OK，大概是这样。好，这节课就到这里，谢谢大家。
