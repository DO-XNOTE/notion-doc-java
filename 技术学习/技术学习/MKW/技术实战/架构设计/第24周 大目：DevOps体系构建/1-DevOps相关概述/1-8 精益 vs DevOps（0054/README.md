经过前面的讲解，我们可以知道，精益的本质是为了高质量、低成本的实现产品的价值交付。而 Devops 的核心原则也是快速交付可工作的软件，从而实现预期的价值收益。这也是为什么 Devops 它特别推崇敏捷开发的短周期迭代，以及激励打造持续交付的自动化部署能力。因为只有把软件部署到生产环境，才能够真正体现预期的价值交付，产生的价值也才能够真正的被用户看得到。我们可以把 Devops 看成是一种非常成功的精益管理实战，而精益思想就像是指引 Devops 落地的指路灯塔，这是我个人的看法。好，这节课就到这里，谢谢大家
