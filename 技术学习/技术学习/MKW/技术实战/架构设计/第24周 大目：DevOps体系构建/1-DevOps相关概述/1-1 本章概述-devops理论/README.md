而本章则是一个纯理论的章节，旨在普及 Devops 以及 Devops 相关的一些思想和理念。来看一下这一章里面会有什么。在这一章，


首先会探讨什么是Devops，让大家对 Devops 有一个最基础的认识。接着围绕 Devops 知识图谱来做一个 Devops 知识体系通览，让大家知道在 Devops 的世界里面都有哪些东西之后，来探讨敏捷理论体系。记敏捷的 12 个原则敏捷开发框架、 scrum 并发敏捷和 Devops 进行关联探讨，从而让大家对敏捷有一个必要的认识。再之后是两节精益相关的小节，最后在课上会普及一个 Devops 成熟度模型，让大家在企业里面对 Devops 的发展有据可依。


大致是这么个安排实事求是，这章的内容可能会稍微有一点枯燥，毕竟是纯理论章节，但是理论又确实非常的重要，是 Devops 实践时候的指导。不得不讲，如果同学们更加侧重于实战的话，那么不妨先囫囵吞枣，大致过一下这一章的内容，然后就直接进入后面的实战章节，等到 13 章节学完之后，再回过头来二刷一下本章的理论，这样应该会比较轻松一点。OK，下面我们正课见。
