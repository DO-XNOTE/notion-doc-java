我们可以把 Devops 看成是敏捷思想从开发端到系统运维端的延伸。这句话怎么理解？我们来分析一下。

敏捷，它解决了客户和 IT 部门之间的矛盾，从而降低开发成本和风险。而 Devops 则解决了 IT 部门内部开发和运维之间的矛盾，所以我们说 Devops 是敏捷思想在 IT 部门内部的延伸。使用 Devops 之后，也进一步的提高了软件的开发效率，让敏捷更加的敏捷，对吧？另外，敏捷和 Devops 从来不是对立的关系， Devops 也不会取代敏捷，而是敏捷的一个重要补充。


另外需要提一下的是，同学们也不能把敏捷和 Devops 混为一谈，两者不是一个东西。在这篇文章详细的对比了 Devops 和敏捷之间的区别，同学们可以看一看。
