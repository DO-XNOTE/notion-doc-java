不用我多说，大家应该都知道这是哪家公司了，财大气粗，这家公司，它急需开发一个简单的抢厕所业务系统，这个系统有两个业务目的，一方面帮助员工抢厕所，避免员工因为抢不到厕所拉在小便池里面的尴尬。第二方面，我要记录你每个员工上厕所的频率，那如果你上厕所时间太长了，这可不行，带薪拉屎这件事情在互联网行业不存在的。同学们，那我们把这个系统分为两个部分，非常的简单。第一个部分我们叫它 restroom service， restroom 这里可不是休息室的意思，那它其实是厕所的意思，比如同学们要到国外，想问个路，问厕所在哪里？

where is the restroom？


这个系统主要有两个功能，第一个功能就是查找可用的厕所，厕所在左边，这个是厕所。第二个功能抢占可用的厕所。第三个功能就是上完厕所之后，可以要清洁一下这个位子，控给需要它的人，这是 restroom service。第二个部分，我们叫它 employee service。我们互联网公司同学们都知道，虽然员工过得很惨，但是我们才总喊着员工第一的口号，那要为了员工服务，咱的这个 employee service 为了员工开发了两个功能，第一个功能就是上厕所，从 restroom service 这边查找可用的厕所。


我蹲进去，那如果没有抢到坑位怎么办？我们资本主义大小周的公司赋予了员工一项特别的权利，员员工有权行使在小便池里，大便的特权体现了特权阶级的荣誉感。这是它的第一个功能， employee service 这边的第二个功能就是戳厕所，上完了厕所之后打扫厕所，物归原主。这两个服务它后台都有一个独立的数据库，它是模拟了微服务架构，每个服务后台有自己独立掌管的数据语言，其他的服务无权直接访问它的底层数据。


这两个服务和咱的 Nacos 之间有什么关系？很简单，他们都会将自己注册到 Nacos 当中，并且上面的这个 employee service，它其实是作为一个消费者，它会去调用底层的 restroom service，这整个教学案例系统非常的简单，同学们在搭建这个系统的过程当中，我们就略过一些很基础的概念知识点，用最快的速度来去落地每一个 spring cloud 组件。所以对于基础相对薄弱的同学可能会稍稍吃力，那么没关系，同学们只要根据老师的节奏去把这个项目给它先搭建起来，这就成功了一半。然后我们针对自己搞不懂的点，可以课外去做一些涉猎，去学习对应的基础知识。
