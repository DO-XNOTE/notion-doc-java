那如果我们通过理解你这个源码背后的思想，然后尝试临摹一下，诶，**这个临摹不是说让同学们完全的把这个代码给它背下来，不是说抄，而是说你读完这个源码之后，我们尝试用自己的理解去写一个功能类似的组件，然后再把你这个 cosplay 的这个成品和前面的源码做一个比较，看一下孰优孰劣。**


**这里面还有哪些可以改进的地方？那这个就是深度源码阅读的一个途径。而且同学们很多的高手，他们读源码并不是只读，而是说在代码当中还会加入很多注释，尝试理清这个思路，比如像一期架构师的白老师就是这方面的顶尖高手**。好，那咱这里就不多扯了，我先带同学们去认识一下 ribbon 这里的体系架构。那在做介绍之前，我先**给 ribbon 这里做一个定性。那在我眼里 ribbon 是个什么样的形象？那渣男**，咱没错就是一个渣男，因为他简直就可以说是开源软件当中的海王，但身后傍着很多的小迷妹小迷弟，那不信我们来看它支持的这些组件都有多少？

那从服务治理组件 Eureka consul  nacos ，还有网关层组件 Gateway Zuul，以及服务间的调用FIN，还有容错组件 high strikes，那这些组件只要是 spring cloud 当中它粘到了负载均衡的边儿的这种组件，那它都会把 ribbon 作为默认的负载均衡器，集成了 ribbon 的依赖。
所以说 ribbon 在 spring Claude 的这个体系架构当中，是一个盘子非常大的组件，别看它简单，但是地盘很大，不过随着 Netflix 公司的项目逐渐进入维护期，那同学们应该也有所耳闻，那 spring Claude 这里推出了一个新的负载均衡组件 loadbalancer，那如果我们说这个老渣男 ribbon 诶，我不想用它了，我想换成小鲜肉。


那 **spring Claude 这里出的 load balancer 挺新鲜，我也想用一下，那咱能不能摆脱跟老渣男的这段旧感情，容不容易那老师告诉大家非常非常的容易。那这里得益于的就是什么呀？在 spring Claude 它底层的这种插件化的思想后面，我们就会带同学们去手把手的跟渣男 say goodbye。**

那首先我们这里就从一个用户的角度来讲发起了一个请求，那抵达了像 Nacos 这类的服务注册中心，那或者向网关层组件 Gateway，那这里后面会讲到，那这里作为一个本地的负载均衡组件，我需要去访问你右侧的这个服务器的集群，那这就像你翻牌子一样，我分哪个牌子？这个过程我就需要经过内置的负载均衡器组件 ribbon Jina 来去帮我做这个决定。


那 ribbon 这里的内部流程，首先那它会去访问本地配置的 i row 这个规则，那 i row 是什么意思？那它就是具体的负载均衡策略，它既可以是我们 ribbon 内置的非常丰富的负载均衡组件库，那也可以是我们自己定制的一个特殊的实现。那它的核心思想实际上就是借助于前面的 necklace 或者 Eureka council 这类服务，发现组件获取到我当前所有的注册服务，那它拿到注册服务是要做什么呢？那你翻牌子是不是得要先拿到花名册？那这个服务注册表就相当于所有服务集群的点名册。花名册我可以获知所有集群当中的节点，它的位置以及它的服务状态，比如说你当前是否是可用你的可用连接数等等。


那当然 ribbon 这里本地，它也有一些内置的负载均衡策略，会多做一些事情。比如说我对你请求的这个 response time RT，我给它做一个分析统计，那根据你的 RT 响应时间的长短，我做一个加权来去选择当前性能更好的机器。那 ribbon 这里就是通过这种本地设置的 error 规则，在一众服务器列表当中选择到目标服务器，然后发起一个访问，在这个过程当中其实并不是永远成功，它也有可能失败，那你渣男泡妞，你也有可能被拒，对吧？那这种情况下怎么办？这里就依赖另一个规则叫 Retry row，那 Retry 这里是一个类似包装器的模式，那我把 retry row 包装在你这些底层的 error 之上的一层。那当你的请求超时或者是返回失败的情况下，那我的 ribbon 会尝试根据你配置的次数发起一定数量的重试工作，那这个流程就是我们 ribbon 当中最核心的体系业务流程。


okay，那除了这部分以外， ribbon 这里还有一块比较重要的部分是它的炭火机制，我们也叫它 ipin 炭火。那什么意思？就是我这个渣男，我既然是渣男，我就要渣出新的高度，我主动出击去撩一下，我看当前服务有哪些服务器是可以做正常响应的，哪些不能。那把这个主动出击炭活的结果作为我负载均衡的一个考虑因素之一，那这个过程同学们想其实它这里也会产生额外的性能开销，那所以我们在默认情况下是使用的达敏品，达敏品的意思就是可以把它理解成我默认 ribbon 是不开启 i pin 的。那同学们看简单不啦？就像我前面说的，这是 spring Claude 当中最短小精悍的组件，抓住这两块 i row 以及 i pin，
