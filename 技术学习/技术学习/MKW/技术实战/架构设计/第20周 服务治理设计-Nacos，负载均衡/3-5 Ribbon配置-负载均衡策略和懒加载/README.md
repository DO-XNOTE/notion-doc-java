那接下来我再跟同学们一起了解一下什么是 ribbon 的懒加载？okay，那同学们在使用的时候可能隐约之间会感觉到第一次调用会非常的慢，对吧？是这个感觉，为什么是这样？因为是 ribbon 在做客户端负载均衡的时候，它的client，也就是 ribbon client 并不是在我们应用启动好的时候就已经加载完成了，而是说我在发起首次调用的时候，我才会实际的去创建这个client，那么因此你的第一次调用就很有可能去获得一个超时的错误，因为咱首次调用也会花费一定的时间创建 ribbon client，那么这样一来你就有很大的概率获得一个接口超时这样的一个错误。


那么这种情况如何来去避免？ ribbon 这里就给我们提供了一个特殊的配置项，可以来解决这个延迟加载的问题，那我们这里怎么来配置？
