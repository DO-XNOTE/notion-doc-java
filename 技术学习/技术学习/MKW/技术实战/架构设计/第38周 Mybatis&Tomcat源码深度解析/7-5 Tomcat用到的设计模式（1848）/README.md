接下来我们来看一下命令模式，它是指我们把请求封装成一个命令，将命令的发送者和命令接受者进行一个解耦，并且所有的命令对于客户端来说都有一个统一的调用接口，那么使用命令模式还可以支持命令的一个撤销操作，这在很多的这些 GY 程序中大量使用了这种模式。


当然命令模式我们知道，如果说大家了解 activity 的话， activity 里面实现了大量的命令模式，简单来说一下一个场景，让大家感受一下这个命令模式的过程。我们有时候可能会遇到一个接口的方法的参数特别多的情况，这样接口看起来非常复杂。对于一个方法参数特别多的话，我们一方面阅读不友好，使用的时候填那么多参数也是很容易出错。那么遇到这种情况的话，我们可以把这些参数进行一个包装，打包成一个参数对象接口，只需要一个参数对象，它就可以把我们整个的程序转起来。
但在具体的接口实现过程中，我们又要做一些条件判断，根据参数值的不同做出不同的响应操作，就像我们在做正常改查操作的时候，我们的parameter，也就是我们的参数都是一个对应的VO，可能 VO 我们需要标记一下它的类型，它句型是增删改查具体哪个类型的操作。
