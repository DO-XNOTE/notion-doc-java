好，这里有一个关键问题了，存储的数据该如何复制？尤其是如此大的数据，如此高带宽的情况下，不断地有什么我们的外部供应商传输我们的视频内容进到我们的在线存储，也同时不停地有数据归档到我们的概括，这个时候通常是采用企业级的什么光纤列入复制总线。好，我们看一下散网络是也是可以支持什么复制总线的，我们通常可以这样画，散网络，因为什么是一个特殊的协议，所以它这里面通常是没有额外的防火墙路由器的，它是通过什么光纤直通的方式在同城的两个机房很容易的实现数据的互通，那这个时候存储服务器这里主服务器就可以把数据实时的复制到我们的灾备服务器，可能只有什么半秒到一秒的时间延时。


同样道理，带库，我们的高端什么企业级带库都是具有复制功能的，也具有容灾恢复功能，所以只要主服务器在活跃的时候，它会把所有的什么归档数据的变化复制到灾备的带库，而灾备带库一旦发现长时间链路没有什么没有响应的时候，灾备带库也可以自身成为一个可激活的卷，被加载到小型机里面来对外提供灾备数据的访问。


好这样一套光纤链路，直通以太网直通，同时对外提供专有网络跟互联网网络的防卫的双数据中心，主备机房。这种模式既可以实现应用节点，可以实现 no SQL，可以实现对象存储，可以实现归档数据的完整的什么双火，这张图大家重新再去审视一下，是不是跟我们很多时候看到了企业系统部署图，
