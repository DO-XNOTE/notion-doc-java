我们把整个系统当成一个黑盒，看一下它跟哪些用户打交道，同时它又跟哪些外围系统进行对接的，它们中间传输的是什么样的内容，它是下一级就白盒系统用户案例用例，系统的前置条件，只有我们了解了我们的系统谁在用，大概怎么用，我们才会进一步去分析它具体有什么样的具体功能。


好，我们就来看一看这个媒资系统应该怎么样画它的系统上下文吧。那既然它是一个系统，那我们就用一个什么大大的圆圈来表示这个系统，那这个系统是黑盒，所以里面不画任何内容，只说一个名词，那它就是什么？它是媒体的媒体资料媒资，对吧？管理系统，这就是这个系统的全称。那这个系统要提供哪些功能？我们逐一去跟产品经理，去跟什么项目经理，去跟他客户的研发架构师、应用架构师聊，那聊下来以后我们会发现，是吧？这个系统它有一些专门的叫外部的内容供应商，就不是在这个点播平台内部，是跟它的这个合作伙伴，那通常外部系统用这样的，就是左右两边有两根双杠杠的这样的这个外方框来表示。


好，那有一个外部系统，我们叫它什么外部内容供应商。好，那这样一个外部内容供应商会什么？会有一个箭头的形式表示什么？我们跟他们之间是有一个互通的，通常什么他们把内容提供给我们，所以这样的箭头是可以这样画提供给我们的。上面的数据，我们可以拷一个这个标准的数据的内容框框过来，它是什么？它其实是个内容的上传，是不是啊？内容的上传。好，那除此以外还有什么样的用户或者外围系统跟我们打交道？会不会有一些外部的人？这些人比如像大家很熟悉的抖音，是吧？我们可以由什么用户直接传？UGC？ user generated、context，这是很热的一个话题，就是用户独立内容上传，那它有这样的功能，还有什么？像我们 b 站的 up 主是不是也能上传？像我们斗鱼的直播是不是也是一种内容上传？所有这些上传的过程当中，其实都是什么用户直接去跟我们这个点播平台的媒资系统打交道，而不是通过外部供应商了，他提供了这样一个 API 接口，那这个人我们要给他取个名称，取的名称叫什么？就叫做独立内容提供者。


好，这里独立内容提供者把它长，稍微好看一点，好美容。还有包括我们这个 PPT 胶片，这个整容，那不是菲亚老师的章节的主题内容，这整过程会尽量简短的，会把内容罗列出来，大家有时间可以进一步优化胶片。好，那除了这些外部的用户，大家想一想，当我们系统收到这些这个视频的时候，是不是要做一些处理？有哪些处理比较容易？想到了什么？首先我们什么系统会有一个系统管理员，那这个系统管理员能保证这个系统的稳定，是吧？当比如说有这种黑客攻击的时候，我们要及时进行应急处理。


系统管理员，好，必然是一个很核心的人物，那他呢？又是什么管理这个系统的基础架构和它的平台软件等，那这里文字我就不写了。好，那除此以外还有什么人？还有一个人是很关键的，就是当你的视频传上来的时候，谁都不知道你的视频内容是什么，所以必然有一个人要进行编目。那我们叫他什么没资管理员，所谓边目就是什么？对，一个视频我们会描述一下上传的人是谁，然后呢？他的这个视频的内容主要是什么？主要是一场战斗的视频，还是一个电影的视频。然后他上传的时间，那同时我们会给它加一些关键词，就把整个这个视频当中有很多的信息的丰富化，变成一个，大家可以从这个搜索引擎里面查到关键词，然后找到这个视频，这就是BM。


如果你一个视频没有边牧，它就是一个纯文本，它没有一些搜索能力，没有一些匹配能力，没有关键词，所以编牧什么是我们媒资管理员的一个核心功能。好，这里写一下每次过程要进行编目，对吧？除了编目以外，是不是你要去搜索一下？有些素材库的内容已经有了，我们要查查看是不是跟以前的冲突，对不对？有些搜索的功能除此以外还有什么获取？每一次管理员可以获取一个，每次进行详细的这个内容和审核和比对。那除了这个以外，大家想到既然说到获取什么，哪个系统也会获取这个信息内容的什么？版权和审核系统，这也是一个外部系统，这个系统虽然叫外部系统，其实也是在公司内部，这是一个什么？版权和内容审核系统，对吧？你这样一个系统是吧？主要做的功能是什么？就是看看有没有什么，比如说反黄、反暴力，对吧？这些内容，然后违反版权的要求的这些音频视频上传是拒绝的，是拒绝播放、拒绝存储的，对，它是什么？它有一个审核的这个职责在里面。


好，那除此以外，当我们内容可以编目了，也就是可以搜索了，同时我们也增加了一些审核，大量的这个外部供应商以及独立内容提供者都上传完了，内容系统管理员保持稳定以后，我们的媒资系统就可以对真正的用户提供内容了。但通常我们不是直接让用户提供或者说获取到这个内容，而是通过一些什么内容的发布代理。
