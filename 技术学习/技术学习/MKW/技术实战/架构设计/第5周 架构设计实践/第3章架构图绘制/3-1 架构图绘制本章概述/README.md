我将在 PPT 里面， Powerpoint 里面跟大家一起分享一下如何用手写的方法。直接就在我们的 Powerpoint 里面进行绘图。我们不一定需要用 VISO 欧、 Mini GRAPH 或者其他各种更专业的架构师的绘图软件，因为平时大家很多时候可能跟同事交流是在白板、黑板或者是纸上画图，那我们一样，我们拿 PPT 其实就可以做到纸上画图的感觉。


除此以外，我也会用 PPT 展现一下基本的除了图以外的一些主流工件，这些弓箭可能用 word 更加详细，但是我会用 PPT 把整个框架大概长什么样子、一个脉络给展示出来。好，那为什么要把这一章作为一个重点章节呢？因为什么？因为画图才是我们架构师范的老本行。如果你说项目管理，我们有什么项目管理专家？如果你说产品思维，我们有产品经理，如果你说团队管理，有没有研发总监，对吧？CTO、 CIO 需要架构是真正最主要的关键点，就是把需求到落地之间的桥梁给搭建起来，把一张图、一个弓箭给写出来，画出来，同时把最终的决策给定出来，这才是架构师最出挑的地方，所以我们也希望通过画图来体现我们怎样进行思路和决策，同时也要展现我画图是多个维度多个角度来展现我们架构师思维的广度。


好，那最后我会跟大家看一下，这章节跟其他章节很不同的地方在于，这章节全部是实战。在哪里？实战是在什么？ Powerpoint 里面，飞扬老师直接手画图的形式来跟大家分享，画图不一定精美，但是思想和思路，希望通过这样的实战再实战来给大家分享一下。

第一句是，嗯，我要用全新的热门技术，比如说分布式、 no CQ、对象存储，全部重刷我的原有的企业的所有的软件的资产，整个领域变成焕然一新。你听到这样一句话，你会感到不错，这个 CEO 居然什么懂技术，他懂分布式、懂 no SQL，也懂对象存储，这就是一个很前沿的CEO。


好，后面两句就是 CEO 老本行，他就不讲技术了，讲他的实际内容了，一看这句话就是 CEO 说的，我要建立一个数字化、网络化、自动化、高效率的节目制作、采编、波存管、发布一体系统。 CEO 说话一般现在每个字都有含义，其中的每一个顿号，每个字都有它的含义。好，我们这里先不详细看每个字的含义，我们层层递进，很快大家就知道怎样去分析一个领导，或者说是一个 c 什么 o 级别的人，他的那个思路和思想了。最后一句话就是一个比较官方的话了，我要打造一个全方位的音视频资料服务业务平台。所有这样的话，其实给飞扬老师一个这样的概念，就是什么？它是要打造一个视频的资料的平台，那这个平台这一直叫什么？叫做什么媒资平台？媒就是多媒体的意思，资就是资料，多媒体资料管理，它要存的它基本的管理转换以及什么一些编目，就是你能很方便的找到这个妹子，然后最后发布到我们的用户终端，这样就是它的一个核心化。这样一大段话其实理解起来你要先去映射一下我们领域里面，其实就是点播公司的一个媒资平台。


理解了这些以后，我们再看下一步具体该如何执行。因为 CEO 的话是什么？是战略性，所以我们通常把 CEO 的这些话记在一个弓箭里面，叫 business direction 战略文档，或者叫什么业务战略好光听了 CEO 的话还不知道该做什么。

那转码平台就是什么？我把老师的可能是 1080P 的编码变成 720P 编码，统一一个编码格式，方便手机快速浏览，这是转码。


那除此以外，迁移就是我要把这份数据放在多份，甚至于我有可能放在中国一份，欧洲一份，美国一份，然后在每个大洋都可以很快的接收到我慕课的这个课程。那这是一个迁移系统，那除此以为检索系统也很容易理解，当你要什么去查有没有什么好的架构师课程的时候，很容易就能查到。飞扬老师曾经讲过一段这个架构师设计实战的课程，那这就是后台我们的什么慕课的管理员人员会用到的。这家企业也一样，它也有收入转码、迁移编码这些基本功能，基于功能原来就有这些渠道，也就有它需要什么。 CTO 希望是把它再做一个整合好。第二步是在整合的过程当中实现这样几个功能，一个是存储的数字化存储，一个是BM，就是把所有资料它的索引都获取到，那方便用户来进行检索。除此以外，一些简单的素材转码要实现。另外把最终的素材发布了，因为点播平台要快速的推送到用户的终端，或者离用户终端很近的，比如像 CDN 网络缓存当中，那这就是它的系统的核心目标。


那最后也是一句官方话，什么我希望有一套业务流，把原来的工作模式、操作模式全给取代了，可以满足我业务很快的进行更新迭代的需求。那这样一套一套的话该如何理解？很多好人感觉是套话，因为 c 什么 o 嘛话都有点套话，但其实每个字都很关键。那飞扬老师就需要跟产品经理，跟我们的客户的这个架构师，以及它里面的一些这个研发的总监，我们一起来聊一聊，想一想。
