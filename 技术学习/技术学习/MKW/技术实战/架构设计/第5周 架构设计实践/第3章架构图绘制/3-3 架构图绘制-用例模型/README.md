写个 UC 编号， UC 编号就是 use case，那之前的那些所有的这个用例已经写完了。比如说上传下载系统管理，假设我们已经到了 13 号编号，那这里就是 14 号编号 use case 14。其中什么是分配一个我们的边目任务？那分配完了以后，什么具体的这个媒资管理员就来进行实际的执行。


在执行过程当中，我们假设是15，增加边目，具体去把这个边目增加，所谓增加边目就是把这个视频快速的浏览一遍，知道这个视频主要是什么，是一个，比如说体育类的还是一个电影类的，写一些这个特征，把这分类写进去。同时把你编牧的时间，你这个编牧过程当中看到的一些主角，比如明星的名字，或者是发现这个视频上传的时间，比如很老旧，我们要提一下这个视频可能已经是十年之前的视频，会提一下这个上传的时间，或者是我们在审核过程当中发现一些敏感内容需要进行额外处理，那在编用过程可以追加一些特殊的标记给其他的或者说剩余的媒资管理员来进行下一步的操作。


这些都是编布过程，等编布完成以后，很方便的通过一些搜索引擎查出具有特征或者特性的一些内容。比如说你是古天乐的粉丝，你就要查一下，嗯，古天乐视频 2019 - 2020，你很容易就能查到最近上传的一年之内上传的所有古天乐的视频相关的内容。这就是什么整架编牧的一个效果。


那除了增加边牧以外，我们在这个整个过程当中通常会顺便去执行一个什么内容审核，这跟后面的版权是跟内容审核平台不一样，那个是全自动的，那这个是人工的，在边牧过程当中通常会完成的内容审核好是4516，那 17 是什么呢？ 17 是我们的要关键的一个内容，就是对边牧进行不停的修改。


刚刚也提到了我们边牧过程当中一开始可能有些内容涉嫌扫黄打黑，你会编牧一个扫黄打黑这个字段，那这个字段会由下一级的这个媒资管理员搜索出来，然后进行进一步审核，如果它确实有问题，就编目成违规审核不通过。如果是审核通过，发现它只是整个屏幕当中的话，它用了黄色背景，是吧？跟人没有关系，那这个时候就会边目前审核通过，那等等这些边目的管理和具体的这个我们打 label 的这个标签的更替，都是在边部管理里面进行，那除此以外，我们前面也提到了我们复杂的这些视频制作，比如非线性编辑把几个视频揉起来进行特殊的特效处理，这些都不会在我们的媒资系统里面，因为我们做了什么小媒资。


但是有一些非常高频的操作，我们不需要每次都放到制作档。比如我们这里举个例子，比如一个很经典的那个例子，就是我们在嗯案例 18 里需要完成这样一个功能，就是视频的抽样。所谓视频抽样就是我们视频如果在整个传输过程当中是一个 g 很大，那这个视频如果是放到手机平台就不是很适合，那这个时候我们会希望能有一些这个抽样，我们截掉一些针，使得这个整个视频可能变成一两百兆，那容易发送到手机端，那同时我们也不会影响整个视频的观感。那这是一个很合理的，就是包含每一帧的这个细节以及帧数进行视频的抽样处理。


那除此以外，视频的转码也是很经典的一个案例。比如说你上传的是魔武格式，我们转成 MPG four，或者甚至你是一些传统的格式，我们把它转成一个比较经典的这个快速播放的格式。视频的转码这个都不需要什么，我们在非线性编辑的这个制作岛进行制作，快速的在我们的媒资平台，通过我们媒资管理员的这个点按键就可以完成。
