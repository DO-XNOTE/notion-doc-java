那其中一大类是方法资产，重点强调一个方法。另外一大类是弓箭资产是什么？是一些 ARTIFACT 已经制作过的弓箭，我们可以重复使用。首先来看一看什么是方法资产，那说到方法资产，必然会有一些我们架构师的基本原则，一些策略，一些规定。大家还记得吗？我们前面聊过哪些架构式的基本原则？是不是有一些像单一原则、开辟原则、依赖倒置原则、好莱坞原则这一些架构师的基本原则？那除此以外，如果你要选择微服，也许微服务要强调什么？把有状态变成无状态，要强调前端和后端的架构分离，强调分布式等等，这些都是什么？一些基本的策略和原则，除了这些原则指导性的意见以外，还有什么？还有一个很关键的内容叫模式，我们也可以认为它是一种模板或者是一种风格。


飞扬老师是不是跟大家讲过一些架构风格，比如像数据流风格里面有什么？有批处理的方式，有管道过滤器的方式，还有什么？还有调用返回风格，比如像主程序、子程序，我们的面向对象、层次结构等等。还有还有一些比如像独立构建风格，那其中牵涉到近程间的通讯以及事件驱动的系统，或者是虚拟机风格里面包含什么？包含一些基于规则的系统，包含一些解释器等等。

大家还记得什么框架吗？这一节我们讲的其实最主要的一个框架就是什么，就是架构风格， architectural thinking，架构师思维的这样一个框架。前面我们还聊过像 ABSD 基于架构的软件开发， DSSA 特定领域的软件开发等等框架，这些框架会告诉你一个方法论，先做一二，再做三四，逐步从需求到设计到研发，最后到落地，直到你一步一步的去把设计图画出来，一步一步的会去把产品给研发出来、部署上线。这些方法论总体来看有原则、模式、框架相对来说还是偏理论的，可能是一两页文档或者是一两张图来总体指引你。


同样道理，弓箭就是一个更加重要的一个资产了，它既能帮助你快速的进行代码的开发和设计，同时也可以减少你很多的什么忧愁。我们看一下弓箭的作用是什么？第一点就是你可以拿到很多现成的库，很多开源的源码，是不是少了很多开发的过程和难点？那除此以外还有一个很大的作用，就是我们的弓箭也包含一些比如像标准的 IDE 平台插件 CICD 的流程，或者是一些 CICD 的资产，这些资产可以简化我们的发布过程，简化我们什么代码的编写编译过程，那同时也最后简化我们的生产系统的部署过程，那除此以外还有一个工建资产也是很关键的，就是我们的参考架构和架构积木。
