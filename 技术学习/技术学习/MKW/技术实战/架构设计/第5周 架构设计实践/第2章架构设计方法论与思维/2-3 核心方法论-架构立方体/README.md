全国出现了第一个大象，你去摸它的时候，如果你摸到的是它的屁，那这个盲人就会说什么这个大象就像一根绳子，是不是？如果你摸到了是它腿，就会说它像一根柱子，如果你摸到了是它的鼻子，就认为它像一个花洒。那这不同的这个摸出来的结果不同的描述，其实都是一个一个的。
什么观点？英文叫view，这个观点不能说对与错，因为它是什么，站在某一个角度去看它是对的，但全局来看它又不对。所以我们这样的角度，我们在什么架构设计里面一个很关键的名词叫 Viewpoint 视角，我们所有的观点，也就是所有的架构图，所有的架构文档，其实都是为了描述这样一个大项，但是大象是立体的，是鲜活的，你必须要通过不同的视角进行切片才能够来描述它。

举一个实例，如果你要开发一个什么商品中心的这个 CRUD 功能，这是不是要找一个数据库来实现它？那不管你什么实现方法，最终你实现其实就是对商品信息的CRUD，这是一个功能，对吧？但如果你实现过程当中就希望什么它是分布式发布，甚至于跨不同的城市，跨不同的省，就是单元化的一个思路，它就是这个，就跟 where 跟 when 有关，它是一种运行的状态，它是描述一种运行的分布的形式，怎么样的形式、怎么样的时间、什么样的地点进行一个运行和管理，才能实现不同的非功能型需求，最终实现我们整个系统能够真真正正的运行起来，实现用户的所有的用例。那这就是功能和运行。
