比如像系统监控，还有跟质量跟我们的什么系统架构的多维度相关，比如可用性维度、扩展性维度、性能维度、安全维度等等，还有像容量规划这样一些基本的质量维度，那除此以外有没有其他的要关注的点？有你组织架构的不同，比如你是什么？你是全球化的开发，全球化的部署，是吧？多数据中心的多单元 set 化的部署，这种情况下你的架构你的部署都会有略有不同。除此以外，包含像什么服务熔断降级服务的加速缓存等等各种各样治理手段，也会导致我们运行架构不一样。那另外当你选用什么外包和采购的时候，你的部署情况会有不同，同时你的产品或软件包的选择不一样的时候，我们也会有不同的需求，不同的限制。比如我们选择公有云或者选择混合云和私有云，那这种部署情况会完全不同，我们在架构图里面能看到清晰的区别，那或者即使是我们选择私有云或者是机房环境，但是一旦我们采用像 CDN 这种什么全球化的网络缓存，那你的部署结构，你的用户体验也会很不一样。


所有功能性不能涵盖的内容，我们都在什么？都在运行性模型里逐一展现。那运行型模型整个模型过程分为两大环节，那一第一大环节就是什么呢？就是我们后面要看到的单元分解我们什么，把之前分解的模块微服务进行进一步的拆分。
