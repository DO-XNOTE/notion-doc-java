一块是组件相关的，我们叫它组件模型。另外一块是跟非功能相关的，我们叫它运行模型。那除此以外，我们会有三个话题作为引申话题，一个是资产复用，一个是架构验证，一个是设计误区。通过这样一套完整的流程，我们希望能把主要的弓箭都介绍给大家，如何进行这些架构设计和准备，以及完整的架构设计过程。
之所以这个章节是一个很重要的章节，因为架构师的功力其实很多，包含沟通，包含如何站在聚光灯底下来展现自我，包含项目管理，包含很多软技能，包含知识面的广度。但是所有这些可能都有其他角色具备，比如研发总监，也许也具备曝光能力，也具备我们的技术宽广性。我们的产品总监也许对项目、对于产品、对于业务更了解。


只有一个功能是只有架构师才具备的，那就是画实实在在的架构工件，制作大量实际的、能够展现需求，能够落地的架构图，这才是架构师什么立命之本，真正的一技之长。所以这个章节就是来给大家讲如何搭建起我们架构师的立命之本，真正的一届之长。在整个过程当中，理论分析会非常简短，简到为止。那么整个过程重点是聊流程上怎么样一步步地展现我们的需求，一步步地去细化我们的模块，最后变成落地的。在过程当中工具会简单介绍，但是更多的时候是用这种工具来进行实战，把真正的图，把真正的文档给做出来，那最后依然会有面试分享环节。
