你的架构设计如何体现公司的整体战略的？是不是一个很庞大的题目？四个词，整体战略 321 做的。那相信大家学过企业架构就会知道什么其实是有方法论的，什么方法论？ SCN 策略能力网络既然讲到企业的战略，就可以从它的战略策略逐步细化到需要提供什么样的能力，再细化到我们需要提供什么样的资源，什么样的 it 系统，那这些 it 系统就构成了我们的业务模型，构成了我们的业务功能 CBN 的业务模块这样一条线。同时我们通过我们的策略的要求，挑出重点的业务模块，进行重点的业务的 it 开发。


要拿到这道题的亮点加分项，就需要结合实际了，而真正从你们公司当时曾经出现过的战略转移，我们 it 重心的转移为实际的例子出发，聊一聊当时这个过程当中，你拿到了什么样的战略需求，你提供了怎么样的一个信息和技术的架构的设计，从而实现它的 it 的能力，从而提供足够的 it 资源。


所以这道题目其实考验的是我们整体的思路，从战略到 it 的一个思路，同时也考验我们实际到底做过多大的项目，考虑到什么样的因素。比如说你只考虑应用开发，那你可能你的视角就有点小。如果你考虑到了底层的中间件平台、 Devops 监控、所有技术环节，甚至于网络基础架构，那你的视野就有点高。同时你所有的功能都是为了前端的业务模型、 CBM 的核心模块提供服务的，而这些核心模块又为了企业战略服务，这就是真真正正的一个企业架构师的一个视角了。

然后那具体这个项目的整体思路3、做大，相信大家都有一些核心项目可以拿出来说一说，而这个思路应该从哪里开始？重大二字必然是影响整个公司战略，所以建议从公司的战略和策略角度来聊一聊差距，分析我公司希望在哪几个方面进行什么整体战略的提升，但是当前的价格要离他差距很大，比如当前只能达 1- 2 个点，而未来我们希望能达到 4- 5 个点，那这就是重大架构转型的意义所在。


所以对于这种项目，你就可以开始采取一些架构转型的规划的思路了，对不对？我们不希望在面试过程当中直接看到一个架构师，就聊很细很细的内容，我到底是采用哪种中间件，哪种数据库来实现我的这个重大架构转型？你整个过程当中没法描述出它的重大吗？首先要让面试官了解你的项目的重大程度，对于策略，对于现状的改变有多大，然后才进一步去展开具体的细节。那这道题的加分项必然还是要结合实际的，但是在结合实际过程当中，希望能够对比公司的策略和目标，一个重大的策略和目标的转型，以及当前我们现阶段的 it 能力的差距有多大。然后你具体怎么样去制定执行计划，怎么样去设定项目的优先级，怎么样去进行详细的架构设计，以及这些设计当中产生的好的结果，坏的结果都可以分享。


那重点是让你和面试官有一个心灵的互通，是吧？有一个这种共鸣的感觉。嗯，你也有曾经失败过的时候，你也有曾经辉煌过的时候，把这些内容讲实在，讲具体，就给面试官感觉你不是杜撰一个项目，而是真真正正参与过或主导过重大架构的转型。你的整体思路要从策略到实际的什么 it 系统，到最后的实际的转型的方方面面的步骤的考虑，以及最终的成与败的分享，这样一条主线来讲，就能够产生一部分心灵的互动和心灵的共鸣。聊完了三道面试题，我们来看一看 QQ 讨论题，也是一个很轻松的讨论题，把你们公司的整体架构梳理一下，
