那信息架构里面的各个模块，尤其是其中要重点进行什么架构设计和加强的核心模块，我们必然会在什么技术架构里面进行进一步的阐述和支持。


那除此以外，我们在技术架构非功能性，比如说可靠性、安全性、扩展性等等这些能力，那整个技术架构的三角形是由三条边组成的，那其中有一条边是聊的是应用和外部连接，那重点就是什么？怎么样通过我们的应用对外提供一个安全、可靠的、稳定的服务？那这是对外的这条线，另外一条线是内部线，重点关注的是内部的数据的落地，不管你是应用数据还是大数据，怎么样在内部形成数据的支撑。


那最后一条边就是我们的底层的基础架构的支撑，怎么样通过我们的中间件，通过我们的底层的服务器，通过我们的网络等等基础架构平台真正搭建起来我们一套能够支撑核心业务，能够保持数据落地的这样一个平台。好，我们废话不多说，就直接开始撸图吧。那这张图是技术架构的三角形，还是沿用前面的银行系统的案例，但是我也希望通过这样一张图的描述来给大家分享一下，一个正常的，或者说是一个专业的我们的企业架构师会具有哪些能力。
