还有食物资源，如果按步骤来讲，就是说我们要识别，然后获取。比如说我们要组建一个项目团队，我要前端的开发，后端的开发，还有UI、 UE 测试等等，这些都是识别资源。识别我这个项目需要哪些资源，那实物资源就是说用于项目所用的一些材料、设备或者设施等等。


项目资源管理的目标就是我们的项目团队和项目经理能在正确的时间和地点使用正确的资源。那整体的资源分为两部分，一个是团队资源，一个是项目资源。那团队资源就是我们项目团队成员、项目经理还有项目管理团队。那我们这个团队为了完成一个项目，可能技能需要不一样，像刚才说的前端、后端、QA、 DBA 等等，我们可能需要具备不同的技能，可以全职或者兼职。例如说某一个同学可能同时在做两三个项目，但是它并不是关键路径上的。而项目资源可能随着说我们项目的进度、投入度也会发生一些变化。


架构师起到的角色也是非常重要的，比如说整个架构的一个设计，也可能设了一些资源管理的一些事情，那项目经理不用讲了，那项目管理团队，也就是说我们碰到有一些问题可能需要升级处理的时候，无论是机会还是威胁，这种情况下可能需要升级到项目管理的一个团队里边，统一做出决策。


项目资源就是说除了人力资源，比如说成本、物品、信息，我们开发这个项目需要宽带，这些都算实物资源。所以整体的项目资源管理就分为这两部分，一部分是人，一部分是物。那对于物我们要关注说我们的要求、性能、数量等等，那无论是人力资源还是食物资源，我们项目资源管理主要是关注我们组织内的一个资源，那对于组织外，比如说我们要买一个软件，买一个系统，这种都算是项目采购管理方面领域的东西。

这个是一个责任分配矩阵， r 就代表对执行某项工作的一个责任， a 代表最终的一个责任。例如说问责的时候，我会找标识为 a 的这个人，那对于某项工作，我们只能出现一个 a C 代表咨询，可以对某项工作提出意见， i 代表通知了解即可。通常会有多个i。我们看一个例子，比如说某一个项目，对左边的活动，前边启动的时候，创建章程、收集需求、提交变更、制定测试计划，可以看到每一个活动，只有一个a，也就是说当这项活动出现问题的时候，问责的是这个人创建章程就是安这个人OK，所以在我们做项目的时候也应该分解出来 ICI 矩阵，它能说明说这项工作是谁负责，同时可以确保每一项工作只能有一个人负责。换句话讲，如果一项工作两个人负责的话，最后的结果可能不会太好，主要也是说避免甩锅的一个情况。
