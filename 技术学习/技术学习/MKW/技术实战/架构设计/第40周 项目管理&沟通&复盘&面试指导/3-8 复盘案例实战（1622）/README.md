第一点叫做行为经验，第二点叫做策略经验、行为经验和策略经验的提取。有什么要点需要我们注意？首先行为经验的提取，具体的行为要进行提炼，我们千万不要把前面的具体的行为直接复制或者是抄袭过来，要找到前面具体行为当中的要点和核心点，把这些要点和核心点进行相关的浓缩，然后变成我们的经验。另外我们也知道我们在项目的过程当中，很多的行为可能无法被直接复制或者是被直接借鉴，因此我们要把这些具体的行为背后的影响找到，这些影响就会变成我们提取经验当中的策略。


经验的提取要注意的要点是他要能够描述出完成上面这个具体的行为带来的结果，它的路径是什么就行了。而且另外我们有了策略，可能利用其他的一些行为，或者是其他的一些方法也能达到这个影响就足够了。因此通过这两点，我们帮他们去提取的行为经验和策略经验分别是什么？第一个行为的经验，高效的会议有五点，第一点是内部沟通业务逻辑，达成共识。第二点是定期开会，全员参与，包括 PM 的总监。第三点是会上回顾阶段性的目标，讨论模型问题并共识。第四点是会后指定人员解决对应的问题。


第五点是问题解决有反馈策略。经验是什么呢？细致的会前、会中、会后跟进，提高会议的效率。也就是说我们把具体的行为把它提炼成五个要点，这五个要点就是我们要得到的经验，我们把它们的具体的影响把它转换成了具体的策略。这个策略就是关于我们开会前、开会中和开会后我们要做一些事情，提高会议的效率，这个是达成高效会议的一个路径，或者是一个概述的方法，这个叫策略。

第一条集体制定开发策略。第二条，通过会议确认共识及认可策略。第三条，站会分角色共享模块进度及问题和风险做相关的记录。第四条及时调用 PM 或者是 QA 反馈问题，同样也是把他们的具体的行为核心要点提炼出来，变成四条，这四条就会变成我们的一个经验了，另外我们把他们的具体的影响也是变成了策略的一个经验。


这个策略的经验是什么呢？通过集体的行动建设保障措施，提前暴露风险来提高开发效率。也就是说提升开发效率有三个路径，分别是通过集体的行动建设保障措施和提前暴露风险。策略是路径是一个方向，那么这个策略内容具体是什么？对于不同的项目来说是不一样的，对于不同的团队来说可能也是不一样的，所以不要具体的去写了。我们提取的东西，对于经验来讲，这是一个大的方向，一个大的路径就 OK 了。

为什么十几二十条原因，我们只有 5 个总结教训，我们在总结教训的时候也讲了明确的要点，只找本质原因，对吗？我们只是把本质原因得到了一些负面的影响，变成教训写在这里。另外我们要找重要的教训、主要的教训，而那些次要的表面我们就不找了，因为次要的和表面它即使解决了也不能带来长久的好处。另外我们复盘是要提升组织能力，也是让我们的能力在未来有一个很大的进步。


这个时候你要找主要的一些矛盾，主要的一些要点，我们总结的 5 条是什么呢？第一个需要考虑合作团队之间的关系建立，需要考虑不同团队的任务导向，这是第一个。第二个是不能仅凭过去的个人或主管的经验判断项目合作方的意愿和配合度。第三个需要对人员安排的统筹和人力变化预估，并对此做出预案。第四个不能缺少对半暂停业务链条的关注。
