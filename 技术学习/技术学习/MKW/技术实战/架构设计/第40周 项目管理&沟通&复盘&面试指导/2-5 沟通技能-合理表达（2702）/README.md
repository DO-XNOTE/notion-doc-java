# 2-5 沟通技能-合理表达（2702）

下面一个语义规则的话叫做潜台词，或者是抽象化屏幕前的男同学，不知道你们有没有女朋友？


如果有女朋友的同学的话，你可能会经常受到你女朋友潜台词或者是抽象化表达的迫害。什么意思？如果有一天你的女朋友跟你讲，唉，我的闺蜜收到一份特别好的礼物，特别特别的好，是一个限量版的一个包包，我真的特别的喜欢。如果你是她的男朋友，你是一个直男的话，你会怎么办？很多同学都是我简单的回应几句，或者说，噢，好的，我知道了。然后女朋友就开始生气，就开始闹别扭了，怎么回事？因为女朋友在表达这句话的时候，她其实是有潜台词的。她的潜台词是什么呀？是我的闺蜜收到了这个限量款的包包，收到了礼物，我也想收到一个，我也很喜欢，你给我也买一个，对吧？只是他在表达的时候没有把他的潜台词表达清楚，因为他想让你去猜。


那就像我们刚才前面的那个案例一样，我的朋友跟她的老公说，你能不能这个少花点时间在工作上，其实它的潜台词是多花点时间陪陪我，只是没有说出来。那在生活当中或者在谈恋爱的时候，这些潜台词的表达，抽象化的表达也许是一种情绪，我们不能直接的去说说这个女生不好表达的不够具体，表达有问题，也不能说这个男生太笨了，或者是其他的一些原因。但是我们在职场当中，我还是建议大家尽量把你想要的需求要求说明白，说清楚，尽量减少潜台词和抽象化的表达，不然的话你很可能会跟你的合作伙伴一起，把一个非常简单的事情变得非常的复杂，甚至你们的沟通会脱离轨道，会去到一个你们都没有意义，对你们来讲都没有用的。

你在这喝茶，这句话其实绵阳是有一些潜台词的，他是希望强哥干嘛呀？赶紧去查询业务，查询故障，解决问题。但是强哥听到这句话，它是一个抽象化的，你到底要表达什么意思？你是指责我喝茶吗？可能强哥就会说，我喝茶怎么了？我不能喝茶吗？我不喝茶我喝什么呀？那这个时候绵羊可能又会回怼回去，说你就是不能喝茶呀？都已经故障了，你还在喝茶，那强哥可能还会再回怼回去。那两个人在这种冲突式的这种沟通的表达当中的话，他们本来的话题就会越来越远。


本来是为了解决问题的，解决故障吗？现在变成了什么？变成了开始讨论应不应该喝茶，对于工作会不会有影响，这毫无意义，对于解决故障没有任何的帮助。所以绵羊应该去降低这种潜台词和抽象化的表达，应该用另外一种方式。比如说什么呢？你应该做一个负责任的研发人员，这种表达方式是不是就稍微的具体一点了？但是这个负责任这个词它也是一个不太明确的，也是一个抽象化的。


什么叫负责任？每一个人对于负责任的这种理解的话都是不一样的，是五花八门的。所以我们可以再把它具体一点，你要观察系统的运行状态，这个就相对来讲比较具体了。但是如果在这句话上我们再加一些东西可能会更具体，比如说一些是时间限制、条件限制，你需要在活动高峰期查看系统的状态，这个就非常的明确了，我应该在什么时候干什么事情，这是对方的一个需求。但是如果在这个需求之上，我们呢再加一些评判的标准，或者是它的一些标准可能就会更好。比如说什么呢？你需要在活动高峰期查看系统状态并解决突发问题，来确保系统的正常，这个既有行为又有标准，还有时间的限制，就非常的准确了。所以这 5 句话的话从上到下它是越来越具体的表达，从下到上是越来越抽象的表达。

那沟通的话不仅仅是传递信息，也不仅仅是把沟通当成工具来去达到我们想要的目的或者是目标，沟通的话还有一个很重要的这样的一个作用，大家还记得吗？就是要促进双方的关系，至少是维持双方的关系，所以很多的沟通方式的话，对于两个人是对立的关系还是联盟的关系的话，影响就会非常的大。

最简单的话就是人称代词的使用，你们我们这样的陈述方式的话，其实对于沟通的氛围是有很大的影响的，而且也会对两个人的这样的一个沟通的冲突也会有很大的影响。比如说绵羊在表达的时候说你们的系统崩溃了，这是什么呀？这是一种论断，而且是一种肯定性的负向论断。


这句话其实让强哥的感觉是什么呢？是绵阳站在强哥的对立面在指责强哥，在发出抱怨，发出一些批判性的指责，隐含的是绵阳对于强哥的不满，你们的系统崩溃了，影响了业务，都是你们的问题，强哥当然会不爽了。那么强哥一旦不爽，他就会防卫，甚至的话会去抵触，去发生冲突，会怼回去，这个时候双方的沟通氛围，包括沟通的效率都会降低，那么实际上这种方式的话就叫做对立的这样的一种沟通的表达方式。对立的沟通表达方式的话，它其实对于沟通的氛围的影响，效率的影响都是很大的。有的时候我们去跟对方沟通的时候，我们表达的内容是对的，是正确的，但是由于我们用了你们你的这样的一些表述方式，对方可能也不会轻易的接受，比如说你们应该早点睡觉，不要熬夜，你们应该更加的努力，你们这个团队应该更加的细致，这些表达其实都是对的，都是为对方好。但是由于这样的表达方式，对方可能会就会很抵触，感到不开心，心里不舒服。


但如果绵阳换一种表达方式叫做我们，我们遇到了系统崩溃的问题，怎么样会让强哥觉得你跟我是同样一个团队的，我们是联盟的关系，我们是共同的在面对这个问题，我们需要一起去解决这个问题，双方的关系的话可能会更加的亲近，那两个人的合作也会更加的密切，这种凝聚性和向心力也会更加的突出。


当然了强哥的话他也有一些对立的表达方式，比如你把问题描述清楚，这个时候绵阳会觉得是被孤立的，强哥站在了我的对立面，那问题是我自己的，是我自己看到的，你让我把问题描述清楚，绵阳心里可能还会想，凭什么是我向你描述？应该你自己查对不对？但如果强哥换一种表达方式，叫做我们一起看看问题是什么，这个时候就会瞬间的拉近和绵阳的距离，那么两个人组建了一个团队，沟通的氛围的话会非常的好，我们的凝聚力会更强，我们一起来看看问题，一起排查问题，一起解决问题。


当然我们这种联盟关系的表达方式的话，也并不是在所有的地方，所有的场合都适用的，那比如说如果有一些问题的话，我们明确的知道这个问题的原因在于我们是因为我们的原因导致的，那这个时候我们就不要主动的优先使用我们这个词。比如说出现了一次故障，我明明知道这个故障是因为我的原因造成的，我跑到了产品那边去说，我们看一看我们遇到了一个问题，这个问题是什么？什么？什么？那么对方很可能会反问，那是你的问题，你为什么说是我们的问题？所以这个时候我们尽量不要优先的使用我们，而是先说我发现了一个什么什么问题，当然了我们尽量的少用你们怎么样了，你们应该怎么样这样的表述方式去建立一个对立的关系，我们尽可能的要建立联盟的关系。

我们说我们在表达的时候要尽量的先说结果，先把结论抛给对方，当别人知道了我们的结论以后，我们再去把这个结论得到的过程以及原因去进行解释和描述的时候，对方才能有一个更加清晰的框架，才能够清楚我们说的这些原因到底是什么。


另外如果我们没有说结论就说原因的话，我们可能表达的原因很多很杂，那对方听起来的话就会一头的雾水，因为他不知道你要表达的这些原因到底最终指向什么。那么如果我们先表达原因，最后说结论，即使对方知道了结论，他也很难再通过结论回过头来去梳理那些原因了，因为怎么样都已经忘记了，很杂乱，自己都记不清楚。所以在表达的时候，如果我们想把表达的内容更好的让对方理解，我们应该先说结论先行。


第二个问题，原因能不能被分类？如果我们表达的原因能够被分类的话，我们尽量的要分类去表达原因，把同一类的原因放在一起去表达。那如果你不这么做，你的原因表达一会儿是 a 类的原因，一会儿是 b 类的原因，一会儿是 c 类的原因，那么对方听起来也会非常的困难，到底要表达什么呀？一会是 a 类的，一会是 b 类的，一会是 c 类的，这三个之间没有关系，他们共同能推出来什么呢？这个会让对方理解起来非常的痛苦，沟通的效率和表达的效率也会非常的低。


同类原因的要素有没有顺序，那如果我们把原因进行了分类，我们在表达这个类别下的原因的时候，看一看有没有时间顺序，有没有重要性的顺序，有没有空间的顺序。如果我们能按照一定的顺序去表达，先什么再什么，后什么，是不是逻辑就很清楚了，那么对方听起来也会觉得很舒服，也会更加容易理解。

第一个特征的话叫做结论先行，也就是说不管我们的论证结构有多么的大，有多么的层次多，那么我们最终在表达给对方的时候，一定要把我们的结论先讲出来。比如说我们要跟别人论证一件事情，我们首先要先说结论，在这个结论之下我们再去说原因。1234，那这样的话别人先知道我要表达什么，我要说的是什么，我要讨论的话题是什么。那其次在结论先行的基础设施上，我们要以上统下，那就是任何一个层次上的思想都必须是其下一层次的思想的概括。那就比如说A1、A2、 A3 这三个思想的话概括成a，那么a， b seed 话又概括成g，那 g 的话就是最后的这样一个结论，这是从上到下的一个以上统下的思路，那这样的表达的逻辑的话会很清楚。


那第三个的特征的话叫做归类的分组，那就是说我们要把同一类别的原因的话放在同一个组别里，A2、 A3 都是同一类别的这样的一些原因，对吧？如果你把A1、B1、 C1 放在了一起，是不是又乱了？一会说的是 a 原因，一会说的是 b 原因，一会说的是 c 原因，那 ABC 又没有办法归类总结，得出一个总结性的这样的一个结论，那你表达出来以后，对方又会听不懂，一头雾水。


所以一定要注意把这些能够分类的原因的话放在一个类别里，那最后一个叫做逻辑递进，也就是说在表达的时候，这些能够分类在一起的原因的话，一定要按照一定的逻辑顺序去进行组织，比如说它的重要性的顺序，时间顺序，先发生什么后发生什么，最后发生什么，这样表达起来会更加的清晰。


那么这四个特征的话，就是结构化表达的一个重要的这样的一个特征了。大家一定要记住，你们不管是在表达还是在写文章，还是再去做分享，都可以用到这个结构化的思维这四个特征去写，对于你的这种输出的这样的有效性和你的输出的这样的一个逻辑性的话，会有非常大的帮助。
