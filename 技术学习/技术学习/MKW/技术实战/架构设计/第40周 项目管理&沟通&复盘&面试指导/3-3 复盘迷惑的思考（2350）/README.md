但是实际上我也发现大部分的复盘都得不到真正的效果，原因就是很多同学甚至是一些部门的管理者，对于复盘的结构化流程并不真正的了解，对于复盘还是存在很多的疑问的。

第一个问题，复盘执行的整体流程是什么呢？

[image](https://prod-files-secure.s3.us-west-2.amazonaws.com/28cd6f37-bc4c-49e6-8d26-8dc351a825af/a8ffc265-a5c3-40c4-958b-ca78c43344a9/Untitled.png?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Content-Sha256=UNSIGNED-PAYLOAD&X-Amz-Credential=ASIAZI2LB4663WTROR75%2F20260721%2Fus-west-2%2Fs3%2Faws4_request&X-Amz-Date=20260721T232304Z&X-Amz-Expires=3600&X-Amz-Security-Token=IQoJb3JpZ2luX2VjEP3%2F%2F%2F%2F%2F%2F%2F%2F%2F%2FwEaCXVzLXdlc3QtMiJGMEQCIE3Xt4%2FUORRz35tXFfDuwK%2F5G3CT4YgYGr0vH7rVXywsAiBOdVWVsvMKAttmlxg6ygGVVWK9ype8KDv4Q33MJRz9%2FCqIBAjG%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F8BEAAaDDYzNzQyMzE4MzgwNSIM9lWklNLCJG5F5UyLKtwDIC42DtlMRq80cftFZa00P4IIc6AH0DZ6yaRHEbDPBymQX8Xd4ppzxzaYcuw9Pr5Rv1FcKlaE1ABMb23%2Bt5Pxe%2FBT4ss0ZlEZDyaHOzTmEucwi36i2Znl7PjoOSVC7BdCYzwl6ElNcNgAh0IGH7DYvBNewdXQtXgqYqYV6n%2Ba%2FQ7cULQx%2BoQGJqJ917jst%2F3imsXAKsTTAsB69AExw6rbBixeCxMvZDYpqD95AJHPE1Evg75FEK59y4ZE%2FV6zUvUO6JyjM8f3myWbkPQFfst7aRhLmBrUW44ztLsrg0YJZ9xbUCgt7zYmxTOloM79dLuyX%2BFpTTKY4vfeIBog2P%2FaIDSgZhtdFqvkmPBsJgGxEvRGhANmTW34lovcW5ex%2FbzO%2B%2BP2NAQCXM9%2F2vlP8IFdWjJcEbqpLA3yDK7LziHHBouu7OYT%2BVUM71n1CYOZCp%2FNcajwf9OFpv1kPLDWhfIX0wd41%2BEzV4B7Lb%2F6EjtyV9VmiELnOqGYjWVu9e%2FyObPoFnG8TqxycqovXupmARVSmNzAuJFJWSNTSHz6Ocm8Kjc4SqIkbBT09EPYir4VY0wdxgjTH2r7%2Fp%2BmVq0hBO4ZEEsA%2Bcddxr05EDSEyzUenL%2BzC%2FGC9z2DRV2g5rgwpLf%2F0gY6pgFC%2By3R2QfnWXlExqUqa312IZDiSyJNb77FeHu8dJ%2FNr%2FjfZDnjN3k%2FB0m6%2FLqisduEg1u952BljclDj9WzrXjEFcBAFvQhcGpNn1jbNwfoKElykQtfYnCWsgNG4y2GcTfzVaJX41gaYHzEKattKHu%2B5JEt%2FIuKuOw%2B3S4UtaqEdzEkxOFB47T6JV0VK%2BTSvNmHxGjB3dNiBPgeSUjMekPnWAjhogu3&X-Amz-Signature=cd20006ea336a64018fddda8428ecbf52d6488d12709bd618d26fce769e4fbdc&X-Amz-SignedHeaders=host&x-amz-checksum-mode=ENABLED&x-id=GetObject)

其实很多团队在做复盘的过程当中，都没有认真的去经过复盘的准备，甚至是突发奇想，突然想到我们今天要复盘一下，来我们开个会，立刻复盘。那像类似于这样的一种急匆匆的没有准备的临时复盘，它是缺少计划和流程的，人员有的时候可能都不齐，所以复盘的效果当然也就不好了。那其实复盘的筹备和执行是需要一个完整的过程的，它主要包括四个阶段，分别是计划阶段、筹备阶段、执行阶段和跟进阶段。在计划阶段，我们是需要根据团队和企业的实际情况，选择合适的事件、活动或者是项目去进行复盘，因为我们只有确定了复盘的主题和范围，我们才能去设计复盘的操作形式，那去确定复盘的参与人员，因为我们前面也讲了，复盘的参与人员一定是实际亲身经历项目的人，所以我们只有把项目确定好了才能确定人。


此外，在复盘之前，我们其实是需要提前的去确定复盘的场地和时间的，并且去邀请复盘的参与者，给这些参与者一些准备的时间，那确保他们能够有意愿参与这样的一个复盘，并且在复盘之前能做好准备之后，我们还要选择复盘非常重要的两个角色，分别是主持人的角色和记录员的这样的一个角色。那这两个人对于复盘的成败其实是起到了非常关键的作用的，后面我们也会具体的去讲他们的作用到底是什么。


最后复盘之前我们还要去挑选合适的辅助工具来制定复盘的流程，用于帮助高效的复盘科学的推进。那第二个阶段的话叫做筹备的阶段，我们首先要先引导复盘的参与者提前的去回顾项目，对项目的过程进行回忆思考，做好复盘的准备。那这样的话不仅能够提高复盘的效率，也能帮助大家更好地更快地进入到复盘的状态当中，去得到更多的和复盘相关的信息。


主持人和记录员在我们选定之后，也要和他们去确定他们的角色，确定他们的职责，尤其是主持人一定要做好引导计划和引导技能的这样的一个练习，要确保复盘能够顺利地开展，顺利地推动。那作为发起复盘的组织者、举办方其实还要去筹备一些复盘的材料和物料，比如说这个项目在执行过程当中，执行结束以后的一些存档资料，其实应该梳理一下拿出来提前给参与复盘的人看一看，或或者是在复盘的现场作为一个辅助的资料来提供给大家，便于在复盘会上使用和查阅。那最后还需要提前对复盘过程当中可能会突发的一些情况做好应对的策略。比如有一次复盘过程当中，不同部门的成员开始去相互推诿责任，现场爆发了一些比较这种严重的冲突，那这个其实是需要我们复盘的组织者、举办方去提前地去预判的，并且主持人的话也要去进行相关的练习和准备，避免类似事情的发生。


那第三步是执行的阶段，复盘的执行的话其实也是非常的重要的，执行的好坏会直接影响复盘的结果，而复盘的现场的把控是执行的重中之重。首先，复盘一定要有一个非常轻松的全员参与的这样的一个氛围，因为如果复盘的氛围很紧张，大家就不愿意表达了，大家不愿意表达，那么项目回顾的信息就会很少，那么这个时候我们很难通过有限的信息能够很完整地、全面地去进行复盘。其次，常见的复盘的问题是在过程当中讨论的话题脱离了项目五花八门，越来越远，那这个时候其实是需要注意引导大家保持聚焦，聚焦在复盘的这个流程当中，聚焦在这个复盘的项目当中来提高效率，那第三复盘它其实是挖掘事实的真相，找到事实根本原因的这样的一个过程。


如果复盘的过程只是讨论一些表面的问题，不深挖，那其实复盘是无效的，而且这些表面问题的话，对于我们未来的学习和成长，以及组织智慧的提升的话，其实都是没有意义的。所以复盘的过程当中的话，一定要让主持人去引导大家深挖这个复盘背后的一些真相，找到问题的本质。


最后复盘的过程当中可能随时会迸发出一些思维的火花，那大家可能会提出一些创新的新颖的理论或者是策略，不遗漏的记录的话也就显得非常的重要了，所以复盘的记录员的这个角色的话，一定要时刻的关注场上的一个动态和变化，那么自己的这个记录的这种状态也一定要非常的集中。


那最后一步的话就是复盘的跟进了。复盘和总结是不一样的，复盘是动态的，它是为了未来的学习和组织的改变而做的这样的一个行为。所以复盘出来的东西的话，它是需要组织来达成共识的，并且通过这种共识的话来去推动复盘的结果的推进和执行。比如说我们复盘出来的一些制度，一些流程，一些新的解决策略，如果不执行、不落地，那么复盘其实就没有什么意义了，它跟总结的话其实也就没有什么太大的差别了。所以复盘结果的周知，共识、执行、落地都是必不可少的。


这个四个过程的话，其实就是整个复盘执行的整体流程了，那希望大家也能去详细的去了解，那不管你是复盘的举办方、组织者，还是复盘的参与人的话，你其实都要通过这样的一个流程来确保复盘的整体的效果和高效。

在复盘当中我们其实说大事阶段性复盘，小事及时复盘，意外的事情，重点复盘，意思就是说对于意义重大的项目，意想不到的成功或者是失败的项目以及创新的项目，我们都是要去进行及时的复盘的。首先第一个是 EE 重大的项目，那工作当中的话，能执行 EE 重大的项目的机会其实并不是很多，很多时候重大项目是关系到团队的发展，甚至是企业的走向的，那这种项目的话涉及的人员多，周期也很长，里边可能会包含了很多管理的、执行的，甚至是一些团队协作的冲突的一些具体的示例。那这些都是可以反映团队的整体能力和需要提升的方向的。


对对于这类的项目的话，进行阶段性的复盘，可以支撑项目的这种及时发现问题，确保顺利的推进。事后复盘也可以得到许多宝贵的经验，为以后的工作起到借鉴的作用。其次是意外成功的项目，或者是惨遭失败的项目，也是值得我们去复盘的。那很多公司认为只有失败的项目才需要去复盘，成功的项目就不需要了，这其实是一个非常大的误区，成功的事情的话有的时候更具有复盘的价值。成功的过程当中会有一些行为，有一些策略，有一些问题解决的方案，是值得我们去提炼和总结的，而且这样的提炼和总结很可能会成为未来的创新制度或者是创新流程的起点，这是具有很大的组织更新的价值和意义的。


第三是创新实践的项目，创新项目在过程当中本来就会有很多的这样的新的模式或者是新的尝试，那创新的项目不管是成功了还是失败了，它其实都是极具复盘的价值和意义的。那这个过程当中一定会遇到过去没有遇到的问题，过去没有发生的事情，以及过去没有去应对的这样的一些策略。


第三个是创新实践的项目，创新项目在过程当中它本来就会接触很多新的模式或者是新的尝试，那这个项目不管是成功了还是失败了，这些创新的模式、创新的尝试其实都是极具复盘的意义的。这个过程当中一定会遇到过去没有遇到的问题，也会产生过去没有产生的一些应对策略。这些提炼和总结以后，很有可能会对我们本来的组织有一个能力提升的、迭代的这样的一个作用。那最后一个是有助于提升组织智慧的项目，那这一点我需要去着重的强调一下。很多时候我们会发现团队整体上是有一种能力的短缺的，或者是某一个能力项是有一个短板的，例如沟通协调的能力、执行的能力等等。而有些项目的过程当中会出现一些问题，或者是出现一些亮点，跟这些能力是息息相关的，那这个时候我们完全就可以借助复盘这样的一个工具来去分析，那这样的一个能力背后，它具体的行为，具体的流程、具体的制度是什么？那把这个能力和组织的整体智慧的提升连接在一起，利用复盘的机会，让项目的成员能够关注到自己的能力短板，并且通过复盘这个本身来达到一个总结和提炼。


更加关键的是什么呢？是复盘它本身就是项目成员一起探讨，一起总结出来的一个结果，这个结果的出现的话是大家共同认可的，达成共识的，那么这个结果的话大家也更容易接受，在未来的具体的落地和实践当中也更容易去推进，所以我们都更相信自己的结论，更愿意去执行自己的总结方案，那这个时候用这种方式去提升组织的智慧是更加有效、更加高效的。


以上这 5 个适合复盘的项目的话，其实都是有一个共同的前提的，这个前提就是如果要复盘的话，就一定要及时，复盘的时间离项目的时间越近越好，因为只有这样大家对于项目的过程的记忆才更加的深刻，很多的事情才更有体会。如果过了很长的时间才去进行复盘，那么复盘的深度很可能就不够了。


第三个问题，复盘是一个人的表演吗？

答案显然是否定的，但是在很多企业，复盘确实是由领导一个人的表演，领导的一言堂，复盘变成了领导的演讲会，甚至是变成了领导对于下属的批斗会或者是表扬会，那这样的复盘其实是毫无意义的，其实也不叫复盘会了。复盘是一种集体学习的行为，集体学习的重点的话就是相互的启发，分享知识。参与者不同的学习风格可能会带来对于项目的不同的看法和不同的观点，那经过不同的这样的一个思想碰撞以后，集体的知识得以增长，那这才是复盘和集体学习的这样的一个最终的目的。所以复盘是集体学习思维的这样一个重要的体现，那真正的复盘的话，它不会是一个人的表演，而是需要至少三类人群共同参与，共同表达，共同完成的这样的一个协作类型的会议。那首先第一类人的话，叫做复盘的核心参与人员，他们是亲身经历项目的这样的一些参与者，他们应该是最了解项目的过程的，也是对项目的问题最具有发言权的。他们熟知项目的过程，参与了项目的执行，因此只需要他们提前准备项目的复盘内容，提前做好项目的执行的回顾，就一定可以为复盘输出很多的具体的信息，可以去对复盘进行很好的推动和结果的达成。


那第二个重要的参与者是我们刚才讲到的主持人，那主持人必须去熟悉复盘的流程和工具，能够带领大家按照复盘的流程一步一步的走下去，并在适当的时机引导大家运用合适的工具。同时复盘的过程是集体讨论和共同输出的过程，那么每一个人发表的观点的话就是基础。因此主持人还要去善于引导大家的发言和提问，掌握一定的教练技术。


那前面我也提到过，有些跨部门的沟通的复盘，很可能会因为气氛不好来导致现场的冲突，这也需要主持人善于观察现场的氛围，来把控现场的氛围，来将现场的氛围进行合理的控制和引导。那么基于以上这些内容，主持人他到底具体需要做什么呢？主持人需要在复盘开始之前做一个详细的开场，那这个开场的内容要包括介绍复盘的流程，引导大家的开放的心态，并强调复盘的目的是共同的学习和共同的成长，而不是推诿责任，彼此甩锅，这一个点是非常非常重要的。而且主持人还需要对现场的规则进行强调，例如不要随便打断别人说话，不要用指责性的这样的语言进行表达，这些规则不仅要说明，还要主持人在过程当中怎样要严格的执行和监督。


最后主持人在复盘的过程当中还要引导现场的发言，还要时刻的关注现场氛围的变化以及讨论内容的这样的一个方向，让复盘围绕项目的核心去进行，不要天马行空，最后浪费了时间，降低了效率。当复盘结束以后，主持人还要去进行复盘的总体的这样的一个总结，那总结得到的经验，那总结后续的行动计划，明确后续的这个行动方案等等。好的复盘收尾的话，其实对于复盘后续的推动是具有非常重要的意义的。第三个重要的人是复盘的这个记录员的职责的话，是记录现场的复盘内容，所以记录员必须熟知复盘画布的流程，对复盘的项目过程要非常的清楚，要能够理解大家表达的内容，并善于做梳理和总结的这样的一个工作，记录员一定要思维敏捷，注意力集中，这样才能不遗漏内容。


另外更为重要的是记录员一定要保持自己的开放学习的心态，要确保自己记录内容的真实性，不要出于自我保护这样的一个原因去遗漏和个人相关的问题的信息，让复盘造成一些内容的缺失。根据以上的内容我们可以看到复盘不是一个人的表演，而是一群人的学习。复盘是所有的参与成员积极地、务实地去投入到回顾项目当中，去达成复盘的一个共识的这样的一个过程。

复盘是一个结构化的流程，需要一步一步的进行，那这个过程是需要一定的现场氛围的支撑的，那这些氛围包括参与者要有开放的心态，要愿意去倾听别人，愿意从别人那里得到新的知识和信息，同时还需要对于参与复盘的人有包容和理解的这样一个心态，而不是以自我为中心，只顾自己的想法。那这样的话很可能就会陷入闭门造车的这样一个状态，不利于复盘的结果。


复盘是一个多人共同学习的过程，集思广益也是一个基本的原则，大家一起思考，一起讨论，共共同输出的话是很重要的，而且讨论的问题也需要实事求是，也不能捏造事实，用编造的内容进行敷衍，那样的话其实复盘就没有意义了。复盘的过程当中有时候会很耗时，时间很长，所以为了让大家能够保持积极的热情和积极性，复盘轻松活跃的氛围也是非常有必要的，这样的话才能让大家静下心来，能够投入到复盘当中去，来关注复盘，关注内容，共同讨论。那这些的话其实大家都很容易理解，但是以下有两个点我需要特别强调的就是刨根问底和复盘步骤的关联，复盘就是要找到成功或者是失败背后的真正的原因，这些真实的原因的话是需要深挖的，我看过很多复盘的结果，结论的话都是浮于表面的，不能得到真正的启发，这样的话其实就不是成功的复盘了。所以我们需要通过一些工具来彻底地去挖掘问题背后的本质。


那关于复盘的关联这个事情是很多的复盘的书籍或者是课程当中都没有提到的。复盘的五个步骤的话是设计出来的，每一个步骤之间的话，它一定是有相应的关系和联系的，那在复盘的过程当中，每走完一个步骤以后，也也需要和其他的步骤相互关联，相互看一看这个步骤的内容和其他步骤的内容之间有什么关系？相互之间有什么借鉴，通过这种关系和借鉴的引导，有可能会得到一些新的启发。
