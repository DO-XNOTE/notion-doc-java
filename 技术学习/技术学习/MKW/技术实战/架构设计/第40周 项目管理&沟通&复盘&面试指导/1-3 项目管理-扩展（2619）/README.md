知识技能就是你了解到的和我了解到的我们共同认知的知识和技能。那这里边工具与技术要简单说一下，那在项目管理里边，尤其 PM book 工具技术这两个词是经常一起出现的。比如说甘特图，它就是一种进度的控制工具。那例如说三点估算法，这个也是一种技术。所以工具与技术我们简单理解，就是说帮助我们更好的能对项目进行分析、量化和管理的一些手段和方法就可以了。


那整个项目管理知识体系里边主要分为五大过程组，分别是项目启动、项目规划、项目执行、项目监控，还有项目收尾，那这些我们后面都会拆开来给大家一一讲解，哪里是重点，哪里不是重点，也都会和大家讲一下。同时对于说我们应用在实际工作中非常频繁的，并且对工作有很大的促进意义的，我们都会重点来讲解。

我们用日常生活中大家都比较理解的一个字来总结，成本就是省，时间就是快，质量就是好，又省钱又快，质量又好。大多数发起人对这个项目的期待都是这样的，这三个因素都满足还是挺难的，满足一半儿都挺不容易了，而且这三个之间还互相影响。举个例子，不又快，质量又好，它的结果可能是贵，这个和成本的要求是冲突的，因为要求成本是便宜的，那如果又快又便宜，也就是说满足时间和成本的要求，这个质量可能不好，又便宜又快，质量就会比较粗糙。那如果想又便宜又好的话，那么时间可能会拉长，只能等了。

我这流量已经打过来了，我现在只有一台单台服务器，配置又不怎么好，这资源的问题，那就这个环境下怎么办？难上加难，没有事儿，大家莫慌，先讲个小故事，那如果以后你作为项目经理，又要多快好省，没有风险，并且资源又非常有限的情况下，可以回顾一下这小故事。


讲这些主要是想让大家更深刻的理解这件事情，因为它非常非常的重要，在后边我们各个章节也都会讲到这些，本身它们之间就是相互联系，又相互约束，不可能动了范围，我就影响成本、时间质量了，我把时间缩短可能就影响成本质量和范围了。所以要从深层次的思维层次去理解这件事情。


那接下来讲小故事，来自于论语先进子贡和孔子的一段互动。那子贡问，师与商也孰贤？子曰，师也，过商也不及。然后子公说，然则施欲与子曰，过犹不及。那这句话就非常重要了。过犹不及什么意思？子宫问老师和商这两个人谁更好一点？那这个诗就是专孙诗，也就是子张。然后这个商就是博，商就是子夏。那子张和子夏都比孔子小几十岁，属于比较年轻的学生。子贡是他们师兄，比他们大一些，他们在当时都是非常出色优秀的青年才俊。


那这个时候孔子就说了，师也过，商也不及，什么意思？纸张就是颛徇私，这个人有点过了，而这个子夏又不够，子夏还有些赶不上，然后子贡就问那是不是说专孙师更强一些？然后孔子的话过犹不及，他给我们中国人追求这个中庸的一个态度，找到了一个特别好的例子。

那我们不同公司的PMO，相信大家有过跳槽经历或者跟朋友同学沟通的时候，会发现会有一些不同，那从方法论上来讲， PMO 主要分为支持型、控制型和指令型。首先说一下知识型的PMO，就是担任一个顾问的角色，像这个项目提供一些模板比较好的知识wiki、实践培训，还有从其他项目可迁移过来的经验教训等等。就是说这个项目资源库这种类型的 PMO 对项目的控制程度比较低，影响程度也比较低。OK，然后说一下控制性，这种 PMO 不仅仅说给项目提供刚才说的那些支持，还会通过一些手段来要求项目服从，就说整个项目本身可能是按照它制定的一些流程来执行，包括参与项目的一些人力资源等等，它会制定或者采用一些项目管理的框架，然后进行一些特定的一个模板、格式工具进行一些治理。


控制型的 PMO 对项目的控制程度属于中等，那重点说一下，指令型 PMO 直接管理和控制项目，它对项目的控制程度是非常高的，你得听我的。整个这三种类型会通过一些方式和手段来对这个项目和参与项目的人员进行干预。主要的原因还是说为了管控，那根据公司的不同行业、不同类型，甚至同一个公司、同一个行业、同一个类型的不同阶段，那一个公司采用 PM 的类型也是会发生一些变化的，并不是说一成不变的，那前面我们也讲了说项目组合是服务于战略的不同阶段，本身这个项目组合也会有变化，强调适应变化，所以 PMO 的这几个类型同样会变化。


那 PMO 的具体形式，它是哪种类型，或者具体的职能做什么事情？还有这个结构主要于取决于说我当前所在组织的一个需要，也有组织战略本身的需要。甚至在一个公司针对不同的项目，也有可能有不同的类型在执行。那刚刚也说了说我们日常有时候会认为 PMO 就是项目经理，有一些公司是这样的，包括我认识的一些小伙伴，那其实这俩还是有本质区别的，当然大家可能都这么叫，只要有共识就可以，并不是说要纠正大家这个叫法，这无所谓的，大家都知道代表同一个意思就可以了。


那这里边就扩展讲一下 PMO 和项目经理的一个区别，主要几个方面。首先第一个方面他们追求不同的目标。那项目经理主要是说在项目的一些约束条件下来完成特定的项目成果性目标，那 PMO 具有特殊的授权的组织机构。这里边要记住它的结尾是office，它不是一个人，它是项目管理办公室，它是一个组织机构，它的工作目标经常是说会包含一些组织级的一些观点，那项目经理是关注于特定的项目目标，而 PMO 管理主要是说在实际的项目中，我怎么更好地吸收经验教训，整理知识沉淀，然后更好的为组织战略目标进行服务。甚至一些项目范围的一些变化，它也会进行一些干预。


那项目经理刚刚说了这两个一个大的一个目标不太一样，第二大点就是说他们的权限范围可能不太一样。例如说项目经理赋予他的可能是说关于这个项目的资源，我们如何利用这些特定的资源更好的实现项目目标。而项目管理办公室我可以说把所有项目之间的共享资源进行优化使用，进行协调，然后他们管理的范围可能也不太一样。
