那我们这个课程的目标主要是做项目管理的学习，并不是为了考 PMP 的一个证书，所以我们还是以实用为主。那和小伙伴们日常工作息息相关的，我们也会重点来讲，从思维层次来帮助大家更好的理解项目管理这门课。如果吸收好，如果小伙伴们还想考个 PMP 证书的话，那对大家考 PMP 证书绝对是有非常非常大的一个帮助。


好，那我们继续。我们先讲产品，例如一个汽车，这是一个产品，那我去构建一个配送和物流的服务，这也是一个项目。对于成果来讲，例如说港珠澳大桥、大兴机场，这个就是一个成果，所以从产品、服务和成果来理解项目的一个定义。然后我们再解释一下独特，那例如说不同时间、不同团队、不同国家、我使用的不同工具等等，做的这个项目都是独一无二的。同时对于产品、服务和成果这三点来讲，也能说明项目的独特性。假设同一个应用不同的团队来做，因为团队里边的成员构成是不一样的，然后大家的工作经验也是不一样的，大家的技能点也是不一样的，所以做出来的产品也会有一些差异，不可能完全一模一样，所以这里边也是强调独特的。那后边还有一个临时性，那一般我们实现目标就会结束，这个项目正常情况下都会有一个明确的周期，那临时性并不代表这个周期短，它只代表是临时性。然后我们看一下特性、独特性，这个刚刚讲了，临时性也讲了。还有一个是渐进明细性，那有的地方称之为不确定性，其实说的是一个，因为项目主要是围绕项目的目标范围计划，那这个渐进明细性怎么理解？首先从项目管理主要大块的一个范围，主要是围绕项目的目标范围和计划。那有的小伙伴们可能会问，项目不都是确定目标之后才开始做吗？那实际上确定项目目标也是项目管理的一部分。那后面我们会讲过程组的时候也会讲到这里。举个例子，例如我们打游戏，双方对决，目标就是摧毁敌方的总部，那在实现这个最终目标的过程中还会有很多小目标，例如在这局进行到多长时间之后，那我们需要摧毁一下敌人的防御塔，这个摧毁敌人的防御塔就是这个大项目目标下边的一个子项目。那这个可能就是随着项目进度发展渐进明细的子项目的一个目标，那范围和计划也是如此。这些细节后面我们都会讲。那刚刚举个例子我们来总结一下，间接明细主要是指我们随的信息越来越详细，具体估算也就越来越准确，会让我们可以随着项目发展对项目的工作更加了解，并且可以更为深入的进行项目管理。

只不过这个项目不同于我们日常做架构师、程序员去开发的一个项目，它本身也是项目具备的特点，它基本上都具备。举个例子，独一无二，不同的装修工人、不同的设计师、不同的材料，装出来的效果都是独一无二的，这个就是独特性，渐进、明细性也有体现。还有那些依赖关系，比如说我先要把墙里边的线先走好，无论水管还是电线，这样吧后边我才能装插座。我现在想组织一个全家的一个旅行，或者个人旅行也OK。


本身这个也是一个项目，我要做一个计划，然后进行一些搜索去预定，包括兑换钱、证件的一些准备等等，这个也是我们生活中的一个项目，那从项目管理的历史讲到或讲的超大型的一个项目，再到我们生活中的一些项目的就是想让小伙伴们更能感同身受的理解。项目是无处不在的历史中的项目，当今正在发展的项目，还有我们身边的项目，所以我们学的这个知识对于我们软技能的提高是尤为重要的。吸收好之后，我们可以用项目管理的一个思维来分析这个问题，那在分析和解决问题的时候，也更容易形成结构化的一个思维。那我们看一下本节的总结，我们学习了项目定义，学习了项目特性，还带大家一起过了一下项目管理的一个发展史，总体来讲，希望项目管理这份软技能的课程能帮助到大家，我相信只要小伙伴们跟着我认真学习，一定会收获颇丰的。我是Gitly，下节再见。
