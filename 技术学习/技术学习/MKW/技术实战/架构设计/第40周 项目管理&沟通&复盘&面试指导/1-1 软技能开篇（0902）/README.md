那其实我们在日常的工作不知不觉中已经在使用软技能，只不过我们对于软技能里边有一些知识，可以让它更有体系，更系统。简单理解，如果把软技能比喻成一个软件执行的一个流程，那么软技能也是有流程的，而且软技能之间也是可以相互配合和融合。那软技能和硬技能从名字上来看是对立的，软硬是对立的，但是从他们的目标上来看又是统一的，他们的目标都是服务于我，做工作、做项目，战略落地或者技术架构落地。


所以软硬技能不应该存在这种排斥的这种现象，尤其对于说我们架构师，更多的不仅仅是说，唉，软件架构设计，那软件架构设计拿到之后，我怎么才能把握这套架构更好的落地？那落地也不能强制落地，可能你的架构还有一些问题，我们也是有很多软技能的一些技巧，当然这些软技能都是正向的，所以它能服务于我们工作，同时在个人能力提升上，它对我们的帮助也是非常大的。从最开始你的面试进行了一些沟通，这些本身就是很多软技能需要的一些点。


软技能对于我们架构师来讲是非常重要的，因为架构师、研发经理、研发总监、项目owner、技术负责人、项目组 r 这些职能在日常工作中都会使用软技能，只不过我们大家没有聚焦在软技能上，所以对于软技能的一个感受可能并不是像硬技能。例如说我们学习某一个开发框架，或者某一种架构思维的设计，并不像硬技能感受那么强烈。

再往上有架构师、经理、主管，包括执行官，那这个图中的技术天花板基本上可能是因为缺乏软技能所致，那当然这些知识可以通过我们的学习，系统化的学习、有体系化的学习来尽量弥补，所以这一点不用担心。


那对于软技能这个天花板之上的更高的职位来讲，技术确实是非常重要的，但是软技能，比如说沟通这一点上也是非常重要的，那从技术专家的角度提出的解决方案可能是纯技术型的，但是如果这个解决方案足够完美，能否实际的落地，这是另外一码事情。因为在这个技术方案落地的过程中，可能要满足方方面面的一些需求，还有对于公司战略项目的一些制约，运输的一些理解，可能会比单纯的写软件的工程师理解的更深厚一点。


那一般的技术型公司，像互联网或者 it 企业，有一些管理岗位，基本上都是从之前技术比较好的同学，因为他的影响力比较好，从这些人群里面提拔上来的，拥有着比较好的技术背景，因为这些人掌握的技术可以起到主导的作用。但是走上管理岗位之后，可能因为一些原因又从管理岗位上换下来，这个时候一定要注意，这肯定不是因为你的技术能力不行，大概率可能是因为软技能方面的一些水平。


从技术岗位走上来也有很强的优势，例如说我们熟悉专业技术，不至于犯技术上的低级错误，包括指导下属的一个工作。同时对于在技术人群当中，这类同学也很容易建立一些微信，影响力也是比较大的。但是做上技术管理之后，可能是完全不一样的一种方式，所以从技术带头人到技术经理、主管、加固师，这个是一个比较大的转变。有很多事情不能仅仅依靠技术人员的心态去处理这些问题，而要和管理和技术之间建立一定的联系，并且了解他们之间的区别。

拿最简单的一个例子，那技术带头人做上管理之后，有时候看团队的成员做事情慢，不太满意，自己亲自上，这个是一个非常好的事情。但是对于团队来讲，大家是需要成长的，是不是前期的成本需要给到，时间成本需要给到。如果第一次、第二次、第三次、第四次、第五次都是我们自己上的话，首先眼光还是要放在整个团队的这个角度，那同时如果每次都是我们亲自上的话，那么对于这个同学的成就感可能也会比较差。


所以这里边有很多平衡的一些思考，包括一些思维方式，这些可能对于技术带头人升上来之后是有比较大的一个提高空间的，这也是说软技能凸显它价值的地方所在，所以说软技能从我们职业生涯来讲，尤其到中后期是非常重要的，在没有建立有体系的软技能之前，我们也在使用，那建立了之后我们可能使用的更好。


OK，那作为架构师或者技术经理主管，我们要对产品的商业认知要比较了解，并且还要把这些转化成技术实现，同时还要把这些东西怎么系统的传达给项目团队，要有比较好的方式，并且能带领大家朝这个方向和目标努力，和整个项目成员密切合作。那还有一个比较重要的一点就是架构师我们还要帮助大家在整个产品项目的研发过程，大家包括对他的交付成果能达成共识，所以促进团队达成共识，也是技术经理、主管还有架构师要完成的一项任务。如果没有这个共识，这个团队成员做这个项目就会碰到各种各样的问题，这也算一种预防手段。那我们这个，那我们这张主要是从以下几个方面来帮助大家提高软技能。

个人的英雄主业还是要放一放，我们很多事情的完成是需要合作的，大家一起合作来做这个项目，所以对项目管理有一套自己的思考体系是非常重要的。里边的很多事情要怎么管理，怎么控制，怎么监控，也能锻炼自己的一套思维框架。比如我们去做一个项目，可以按五大过程组进行拆分、启动、规划、执行、监控和收尾，每个过程都要重视哪些？这个也是非常重要的。然后就是沟通，那架构师沟通是非常必要的技能和工具，为什么呢？因为架构师有时候没有对其他项目或者工作人员，项目人员的直接管辖权，有可能是一个虚线的汇报关系，而有一些项目还是跨职能部门的，或者跨整个业务单元的。那在这种情况下，如果说能促成这个事情落地，唯一的手段就只有影响力了，就这个人靠谱，那我就把这件事情做了。但是只依靠影响力还是不够的，包括向上管理、同级和向下管理。


向上管理你要和老板沟通，我这个架构很厉害，比现有的强，那强在哪里？怎么去沟通？所以沟通这件事情是非常关键的一个技能，所以对于我们软技能来讲，也会重点讲一下沟通，因为沟通这个技能在我们日常工作中使用的也是最高频的，时间占比也是最大的，同时沟通也是解决问题的一个手段。举个例子说，我通过沟通识别出来风险，然后阻止这个风险发生，甚至寻找问题，解决问题。有很多事情都是需要沟通的，所以沟通在我们软技能里边是一个比较重要的一个权重。
