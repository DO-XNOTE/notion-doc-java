那么非对称密钥它实际上是有一对密钥对，为什么叫密钥？对？那它有一对是public，还有 private 公钥私钥。那我这边作为一个签发方，我签发了这对公私钥之后，我这里可以用其中的一个公钥来去对我的报文进行一个加密，然后我的解密方使用一个私钥来对这个报文进行解密。那就这样的一个流程，它的安全性是比对称密钥要高得很多的，因为对称密钥你的密文，你的加密的 key 一旦泄露了，那相当于别人就获取到你这个加密的秘密了，对吧？所以说这个非对称密钥，它通过这两个密钥对来避免了这个 k 泄露的一种情况。


但是这个两个密钥对实际上同学们对他们的认知可能会有点误区，一个叫 public key 叫公钥，一个叫私钥，实际上公钥并不是公共的密钥，这里同学们要认清一下，公钥它不是公共的密钥，它这里会颁发给谁？颁发给授权授信的，你的这个信息的加密方不是说颁布给社会公众，而大家随便都可以拿这个公钥，它其实也是密钥，对当中的一个密钥也是需要保密存储的，所以大家不要看到它是公钥，就以为它是一个公开的，谁拿到都无所谓的，其实不是这样的。OK，那么我们接下来就到 Intellij 里面跟同学们来去落地一个非对称密钥的小案例。好，同学们，咱在落地这个案例之前呐，先生成一个密钥好不啦？
