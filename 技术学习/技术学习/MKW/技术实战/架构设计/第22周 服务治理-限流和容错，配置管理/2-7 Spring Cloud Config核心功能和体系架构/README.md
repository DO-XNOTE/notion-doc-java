第一个，环境隔离，不用说，作为一个配置中心，它其中一个很重要的功能就是把各个的开发环境通过某种机制把它隔离出来，那 production 环境或者集成测试环境或者异常日常的开发环境等等，这之间各个环境参数诶不能串。那在后面的小节里，我们也会详细跟同学们介绍如何利用咱的 config 组件的环境隔离，来去隔离它的环境参数。OK，这是一点。


另外一点，属性加密这一块儿，实际上它是要优于 neck 的，如果大家要在 neck 当中应用属性加密，其实还是要费不少功夫的，而要做一些改进改造，那么在我们的 config Server 这里，实际我们本身就已经支持了这种对称密钥加密，或者说可以使用非对称密钥加密。


这里还有个错别字，非对称密钥，这个非对称密钥它的安全程度会比对称密钥更高，那么我们在后面的小节里也会跟同学们一起来去用本地的 Java 命令生成一个非对称的密钥队，那么通过这个密钥对来完成它的属性加密操作。不过这里要注意的一点是 config Server 它实际上是一个二选一，也就是说你要么用对称密钥吗？要么用非对称密钥，那么我们这里建议大家在生产环境当中尽量还是使用这样的一个非对称密钥加密，这样的话可以保证它的安全性。okay，那么这是它的属性加密这一块儿。那么对于持久化，这里作为 config Server，它自己其实是不担任这个持久化任务的，那么它的持久化要把这个属性保存在像本地文件，这是它默认的 native 的方式，可以通过本地文件形式的这种方式来去存储，还有我们可以使用很多的版本控制工具， git s win 等等。


那么咱在日常的开发当中使用 Git 是一个最常见的一个操作， config 和 Git 搭配是一个绝配，你可以利用 Git 的这个版本控制功能来完美的去做各种回滚历史版本的搜索。那么 config 它本身并不像我们的 necklace 这里它有非常详细的版本控制功能，它是一个内置功能。但是 next 这里你如果使用像 Git 这种技术选型，那么你实际上是依赖于 Git Hub 它自己的这种版本控制的能力来做。那我们除了版本控制工具以外，还可以选择把自己的属性文件给它保存在数据库当中，这都是可以配置的，咱可以在自己的 config Server 这里指定你想要做持久化的这种类型。


OK，那这是三个比较基本的功能，还有一个就是作为配置中心一个附送的一个功能点，动态刷新你的应用，对吧？它在运行期间我要刷新某种属性，那这种方式我可以采用你的 bus 组件加上 MQ 组件做这种批量的动态刷新。那么我们在后面小节里也将跟同学们来去落地一个案例，使用 Sprint Claude bus 再加上 Rabi m q 来达成这个集群环境下的这种刷新。


那我们如果不使用 bus 加MQ，实际上也是可以从单机的角度来去触发刷新的，只不过在我们的线上环境往往是一个大型的集群，那么你肯定需要这样一种集群刷新的方案，那 bus 加 MQ 是一个非常天然的一个组合搭配。OK，这是一点。还有一个，我这边如果想把某些属性，我不想推送到整个集群，我想推送到一些具有特征量的机器上面，我先做一下验证，你这个属性它都正确了，我再推送给整个集群，对吧？那这种情况下，你可以采用它的定点推送功能，这一个功能点实际上用起来会比 neck 要方便一些。
