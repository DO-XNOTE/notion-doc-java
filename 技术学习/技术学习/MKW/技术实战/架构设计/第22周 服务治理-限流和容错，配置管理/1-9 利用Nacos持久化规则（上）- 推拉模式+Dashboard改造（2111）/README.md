第一种就是叫拉模式，第二种叫推模式，那这个拉模式，推模式这里说得很清楚，那对于拉模式来说，它就是从客户端主动发起的，客户端向某个规则管理中心定期轮询来拉取规则。注意这里客户端一个关键字，它是主动方，第二个定期轮询，那么这种方式它的缺点这里也说到了，无法及时获取到相应的变革，但是它实现起来非常的简单。那么我们这里 Sentinel 支持的这种数据源扩展，对于咱的拉模式来说就是第一个动态文件数据源，第二个 concile 或者是Eureka。
