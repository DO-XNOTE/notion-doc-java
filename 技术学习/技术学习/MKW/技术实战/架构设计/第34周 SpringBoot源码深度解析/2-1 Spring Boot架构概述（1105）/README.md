首先 spring boot 是什么？我们为什么要选择 spring boot？怎样用好学好 spring boot。首先我们解释 spring boot 是由 Privatee 团队提供的全新框架，其设计目的就是为了简化 spring 应用初始搭建以及开发过程的一些投入。这个框架它使用了一些特定的方式进行配置，也就是我们的智能装备，从而简化开发人员使用 spring 框架那些模板化的配置，大家最熟悉的应该就是数据源。


data source 的配置几乎每个业务系统都会有，如果我们通过streamboot，它可以自动装配，也就我们减少了这个 data source bin 的配置。站在技术使用角度去理解 spring boot，它并没有提供像 spring Mac AOP 这一类的具体功能。 spring boot 它主要是简化了 spring 框架的应用开发，通过扫量代码就能创建一个独立的 spring 应用，同时为 swim 平台以及第三方库提供了一些开箱即用的默认配置。


通常我们能想到的常用框架子分布的都集成了，那么我们再看为什么选择 spring boot。其实你不得不选择 spring boot，目前它已经是微服务的事实标准跟开发带来很大的便利。在 Java 场景下， spring boot 逐渐成为一个必备的技术，尤其是在 Java 面试过程中， spring boot 也是必考题。至于怎样学好 spring boot，我觉得对于大家来说不是太难的事儿。 string boot 本身就比较容易上手，我们的目标是深入理解 string 步骤的设计原理，从源码角度理解 string 步骤的实现思路。

我们如何管理好这些配置文件，避免配置定义不明确带来的干扰，也是开发的一大痛点。 string boot，它默认加载的是 application 点 YML 配置文件， YML 相对于 property 属性文件具有更好的结构，一方面它可以简化配置，另一方面对具有相同前缀的配置 key 进行一个强制归类。


spring boot，通过优化设计，它把所有的配置项都集中在这个文件，并且每一个大类型的配置都设计了合理的前缀，也能做到了一些见名字义。另外，文件配置和必应属性它具有一个自动映射的关系，这个功能很方便，其实也比较容易理解。 spin boot，它通过反射的方式把配置的 k 值和变属性相同的项进行一些数据赋值。当然这里面还有许多细节需要处理，比如说类型转换、合法性校验等等。对于多级配置的入口进行合理的顺序支持，这个怎么理解？我们可以把配置放到 application 的 YML 里面。当然其实在系统运行过程中存在着各种复杂的业务。 string 部的同时也支持其他方式的参数传递，比如说最直接的也就是命令行启动参数，如果这些参数存在相同的 k 值，那个覆盖逻辑和优先级的顺序应该如何判断？它肯定是需要一些哲学思想的，一般的原则是配置越早的优先级越低，在这里面我们可以明显地感觉到我们命令行的参数，它的配置优先级应该要高于我们。
