# 3-2 软件安全-操作系统安全

这个机不是机器的机，而是个基础、基准和基石。那它包含像安全内核这种非常小巧而且安全的内核的关键内容，以及这个内核所在的什么安全边界，通过边界向用户态、向工具向什么第三方的应用软件来提供服务。如果要把可信这套思路化成洋葱圈模型，就是 0 圈、一圈、二圈、三圈，最核心的 0 圈就是里面的安全内核，但是飞扬老师这次换一个化了方法。化成什么？化成印度的种性金字塔，大家还记得吗？最下面是手陀螺，相当于是我们的应用软件。好，那第二级叫废舍，相当于是操作系统提供了各种各样的什么系统工具。那第三层叫什么？叫差地力，相当于是操作系统本身，它其中的特别是用户态的部分，那它的内核态在哪里呢？当然就是婆罗门了，对吧？从上面到下面，是吧？是级别是什么？内核最高，婆罗门最高级。


普通的用户的应用软件最低级，首脱落好，如果是这种性的金字塔结构情况下，那安全模型会告诉我们什么呢？请遵守两大基本准则，第一条准则叫访问控制准则，这个准则的模型叫做贝拉模型。他怎么说啊？下等人不允许偷窥上等人，所谓下等人就是应用软件不能什么获取工具类的里面的内容，工具不能去偷取什么操作系统里面的内容，操作系统的用户态不能去偷取内核态里面的内容。只能是什么上等人调用下等人，不能下等人去偷取上等人，如果上等人要对下等人提供一些功能，会提供一些接口，或者是什么 API 的形式呢。要调用指定接口，而就要提供指定身份的验证，才能够去了解上证人的少数信息，大部分上证人的信息是不会对下证人进行暴露的。这就是被拉原则，不允许偷窥上等。


那除这个以外还有一大原则叫什么完整性原则，我们叫它比 8 模型，不是芭比娃娃的芭比，反过来比 8 模型怎么说呢？上等人，不要随便相信下等人。那什么概念呢？我们婆罗门系统的什么内核态？它从什么用户态这里拿到一些信息，它能够直接使用吗？不能。要通过层层过滤筛减才能知道什么我们差地利给的信息是真实的，那这种情况下才能信任和使用，所以所有内核的内容是最关键的，他们之间是可以互相相信的，你不能随便去相信，是吧？相信用户态的内容，同样操作系统不能去相信任何工具，也不能相信第三方的各种各样的软件。


通过这样一个模型，其实我们把印度的种性完全划分成四个不同的等级，是吧？你既不能偷窥上等人，了解上人的行为，上等人也不能够随便从下等人那里拿到的话或者内容，相信他来使用，他这样就进行了什么？进行了印度种性的风格，那同样道理，我们在什么？我们在安全内核里面实现了安全内核的层次，完整的切分。


这到底有什么样的操作系统完全符合这套安全模型呢？可以说几乎没有处方式，除非是军方的系统。那通常我们都会有什么？留出足够的空间，让不同的形态之间存在一些调用，存在一些互通，存在一些信任。所以安全模型是一个理想模型，也是一个军方的超级计算机内部系统的模型。

我们有前几年的SARS，我们也有现在的新冠病毒，什么样的特征啊？病毒复制很快，对吧？它很快的在什么？在人群中复制自己，然后它隐藏着每个病毒，不是什么一附身，然后立马发作，它隐藏一段时间。同时它具有一定的破坏力，是吧？比如以新冠为例，是吧？如果有 50 个人当中随机可能就要挂一个人，这就是病毒的破坏力。


好，那除了病毒以外还有什么样的威胁？蠕虫，唉，什么叫蠕虫？就相当于是你身体里的一个小的什么小细菌，这个细菌是坏的，它不是病毒，也就是说它不会对你产生致命威胁，但是它确实实实在在潜藏在你身体里面。而且它的复制它的克隆更快，它没有破坏性，所以它更容易生存下来。
复制和克隆我们很经典的如从是什么某一国的科学家和黑客们发明了一个叫什么西门子电气蠕虫，然后通过蠕虫控制了伊朗的核射时，破坏了伊朗的某些的油产品。那这种蠕虫是吧？其实它的影响面什么比破坏一个计算机系统是不是更大？它控制的是整套什么自动化核设备系统，那这是一个经典的蠕虫的案例。


好，我们除此以外还有什么？大家是不是记得特洛伊木马这个名字经常在我们的黑客工坊里面出现，所谓特洛伊木马就是什么披着羊皮的狼，他 p 什么皮？ Linux 里面他最常 p 的什么皮？ if config 当你跑 if config 命令的时候，请注意，也许你是在跑一个黑客的什么执行脚本啊？这部 Windows 里面什么样的 p 最常见？ note Pad 记事本当你打开记事本读取一段内容的时候，也许这段内容正在向什么黑客进行发送，这些都是披着狼皮的，披着羊皮的狼啊。


好，那除此以外，跟特洛伊木马很接近的一个叫后门叫 root Kit，它跟特洛伊木秒区别什么？ root kid 是一套组件，这套组件是黑客。当攻破你的操作系统内核或者是攻破你的操作系统用户态以后，它强行在你的操作系统里存一套组件，通过这套组件它能够实现什么？一套完整的用户是提权就把它的权限提升，然后进行什么数据偷取的功能。


这套组件相当于是在你家开了一个后门，通过这套组件它可以植入不同的木马，不植入不同的病毒，进行不同的操作，也就是什么千军万马通过后门都能进来，不光是木马里面的 5 个人、 10 个人，而是千军万马的战士从你们家里面经过，是不是很恐怖啊？如的 kid 确实是什么？整个黑客攻防里面对操作系统安全影响最大、破坏力最大的一套攻击手段。


好，除此以外还有什么？还有很常见的僵尸网络，所谓僵尸网络就是什么？就是我们的一些法师，它会召唤很多的傀儡。这些傀儡其实就是我们 it 机的肉机，它通过肉机可以用迪多斯来攻击你，甚至于它还可以把你的那个操作系统，你的这个电脑变成它的一台肉机来去攻击别人。


这就是僵尸网络，一个法师掌控成千上万的僵尸，进行什么？进行强大的破坏和狩猎好？另外我们还经常看到的一个问题就是零日攻击，尤其是在微软的操作系统里面，因为微软是时时刻刻发现一些安全隐患，它都会公布出来。什么？什么情况存在隐患？它的威胁性多大？如何利用它也会告诉大家，然后通常是周二或者周四，微软就会及时的发布一个什么补丁，在隐患告知和补丁发布，或者是在隐患告知和你们企业打补丁这段时间之内，黑客就可以利用这个什么隐患快速的开发一个攻击的这种病毒蠕虫，或者是 root kid 后洞后门，然后破坏你的操作系统，提升他的什么用户权限，偷取相关的数据信息，这就是灵日攻击，也就是在你没有来得及反应的时候进行黑客攻击。


好，那除此以外还有一些类似于 root Kit 的东西，比如叫 Web shell，那它也是一个小工具，它是类似于什么？类似于一个 shell 脚本跑在你的机器上，但它提供一个对外的一个外部服务。也许是一个 API 服务，所以当黑客想来获取你的数据的时候，它只要对你的服务器发起一个 API 请求，就会触发这个 Web shell 来进行数据的偷取或者什么数据的破坏。

对不对？兵来将挡水来吐液。对，杀毒软件确实是对付操作系统安全的第一要务，或者说是第一利器。那除了杀毒软件以外还有什么升级补丁，对吧？及时的什么进行版本更新，这些都是防止零日工具的主要手段。那除此以外，我们应该合理的限制。怎么样限制哪些用户能够访问？什么样操作系统需要限制每台服务器？或者每个操作系统连什么样的网络需要限制另外当一台操作系统里面起一个服务的时候，这个服务占用的端口一旦是出现一些奇怪的端口，我们应该进行限制。另外这个程序的名称，这个程序的功能都要进行限制，很常见的一种手段是什么？通过安全基准。
