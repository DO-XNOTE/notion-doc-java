我们要重点通过这一章节来聊三个话题，一个是操作系统安全，一个是数据库安全，一个是 Web 应用安全。好，之所以要把软件安全也作为什么一大维度，原因是什么？因为有人的地方就有江湖一样的道理，有软件的地方就有风险。所以我们要想一个方法来了解软件到底存在哪些风险，以及在我们的实际的开发部署过程当中去规避这些风险，这才是我们这个章节主要的目的和内容，那整个章节依然是理论结合、案例分享，我们会重点聊一聊不同的层次、不同的维度的一些安全的风险和规避手段，那这是理论。

那第一个点是什么？是操作系统安全，其中包含像病毒、蠕虫、木马、灵日攻击补丁这样一些概念，如何防御？他们如何保证操作系统的安全？好，第二个点，数据库安全，我们会聊一聊最让大家担心的 CQ 注入，以及有时候会出现的推理攻击等等，怎么样来防这些安全漏洞？如何来保证数据库是安全的、有效的、正常运行的？那最后一块 Web 应用安全，我们经常能听到 XS 攻击， CSF 攻击。嗯，耳熟能详，但是你真的理解他们的攻击原理吗？还有他们之间的区别吗？还有他的很多的衍生和变种吗？那我们就来实际的看一看，他们到底是怎么样攻击的？我们应该如何防御好？废话不多说，让我们从操作系统安全开始起步，真正了解软件的安全，大家敬请期待。
