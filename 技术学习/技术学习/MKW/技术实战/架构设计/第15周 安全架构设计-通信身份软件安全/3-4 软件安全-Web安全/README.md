那第二种攻击方法不是攻击用户了，而是攻击一个服务器。这个服务器通常是比如像博客、论坛、互动社区这样一些服务器，它这里面会有数据库，也可能是文件系统。不管怎么样，本来应该成一个什么值的地方，我尝试把一个整个脚本塞进去，当用户浏览器跟博客，跟我们的社区跟互动平台进行沟通交互的时候，会有一个互动过程，那这个互动过程就会触发这段值，那这个值可是一个 JS 的代码，它一旦被执行就会什么？作为用户尝试在浏览器里面多去做的 cookie session local storage，从而什么变相的攻破了用户的浏览器？那这个攻击方法有好处？怎么好处啊？当你攻破一个用户浏览器的时候，你只是攻破了一个用户浏览器，但是如果你是存储性XS，攻破了一家厂商的数据库的时候，所有访问这个论坛，对吧？这个社区的用户都会被你挟持他的数据，所以存储性 XSS 破坏性更大，严重性更大。
