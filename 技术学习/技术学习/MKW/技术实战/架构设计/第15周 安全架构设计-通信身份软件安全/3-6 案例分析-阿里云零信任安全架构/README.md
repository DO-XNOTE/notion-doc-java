相信这张图是不是贵企业或者是你们所在的行业里面很多企业用了一个整体架构，采用DOC，对吧？进行镜像的管理，然后采用库布内提斯的 yaml 来进行什么资源的配置，通过 DOC 和 yaml 的配合来实现最终容器的发布，这也是阿里云了十年以前的架构，那前两年这套架构已经被淘汰了。换成什么了？换成一套通过 DOC 镜像加 Helm chart 的统一管理平台，那这套管理平台能实现多个库布能提斯 yaml 的什么联合管理，中间的 value 数值的什么更替，以及相关的应用的依赖以及配置的统一管理。也就是说比 doc 加 K 8 s t s 的简单描述最终状态描述要更进一步了，它有一个整体包的感觉，应用包的感觉，但这已经不是什么阿里云最当前、最火热的安全交付平台了。


那当前怎么交付呢？唉，有个新的概念叫什么？ CNAB 云原生应用包大家应该听说过很多小的 room 小的传言吧。比如说 Kubernetes 不支持刀客等等。在 CNAB 里面是符合云基础架构的标准协议，不管你是用 Kubernetes 卡了 foundry methods Swarm。底层你，不管你是 container d， Docker run c，不管你是什么容器方式，不管你是什么发布方式，只要符合云原生协议的基本标准，你就可以打包成这样一个应用包。这个应用包里面除了有你应用的镜像，也有你的配置文件，也有你的应用的依赖关系的描述以外，还能加入很多安全的内容，比如说安全断言，比如说用户的签名，比如说服务账号的 token 签名等等。


统一打包最终变成什么不可变基础架构的一种发布方式，是不是酷炫方便，而且向未来看齐，不管未来是库能提斯的天下，还是 doc 的天下，还是 CNAB 的天下，都必然是云原生统一发布统一管理的天下，对不对？好，那这样一套什么应用发布方式，既把原来的流程简化了，同时也加入了很多安全内容，但光发布是不够的，应用最关键的是应用本身的运行和隔离。

难道还不够吗？当然不够。因为 content 和 Pod 有很多的什么黑客技术是可以攻破的，可以通过一个container，或者通过一个 Pod 可以去横向去攻击其他的容器哦。


那什么样的技术更安全呢？这里阿里云采用了一种叫喀塔或者叫喀塔容器的技术，所谓喀塔容器，我们可以理解成什么一个比虚拟机小一点，比古尼迪斯特炮的大一点的虚拟单位。这个单位可以号称叫安全沙箱，也可以号称叫卡塔容器，或者叫做安全炮的等等，我们有不同的叫法，本质上它对内核的隔离更严谨，它的内存的访问控制更完善，同时它比我们库布内提的普通容器再大一点，它一个里面可以放多个容器，可以放多个 Pod 它，但它又比虚拟机比服务器要小一个单位，依然能够很轻量的在一个操纵系统上起多个，那对应着卡塔容器，我们在整个操作系统层，也就是我们的物理机或者虚拟机的操作系统层。


我们又提供了一种叫 GYZ 的技术，这种技术对每一个进程进一步虚拟化，大家其实玩过容器很清楚，容器里面跑的东西，最终在操作系统里都是以进程的形式来体现的，怎么样让进程和进程之间不光是在用户态，而且在内核态都进行很好的隔离？ GYZ 提供了一些思路，同时它对内核进行了很好的增强，它是属于符合安全内核架构的一种新的体验，新的行为方式。


Kata 容器加 Gyzer 合并起来就是什么？双键合并形成了隔离共享按需分配的一种总体策略，使得所有应用和应用之间、容器和容器之间既隔离还能共享资源，同时还能按需分配的调配内核、 CPU 以及内存，这是不是很酷炫啊？应用做得很好，那数据又该如何保障零安全信任里面阿里云提供了这样一套思路，叫做数据全链路加密。

好，这个全链路跟我们平时说的全链路还不一样，它既包含传输链路，在传输链路上全部采用HTTPS，而且推荐什么以 still 双向认证，也就是前面跟大家分享过的，既要服务器去验证什么客户端的证书，客户端也要验证服务器的证书，通过 Excel 服务网格透明的进行双向认证。


除此以外，证书管理维持什么？每月更新公私药证书通过公司要来掌控对称、密钥等标准的证书管理协议。另外用户全部建议采用 VPN 接入，这种安全的基于 IP 赛克的标准接入方式好，除了传输加密以外，数据还有什么？还有数据计算加密在计算过程当中通过快速计算来实现加解密，通过很强大的权限管理和分级管理来保证数据采用不同的级别、不同的防控措施，不同的用户角色访问，那最终在落盘的时候依然是全数据加密。


跟谷歌云一样，所有落盘的数据，不管你落在对象存储数据库、文件系统里面，全盘加密，然后加密的密钥通过什么信封加密， HSM 密钥生命周期管理来实现密钥的过期，公私钥的什么解密以及什么对称密钥的额外的处理工作，通过这种信封加密的对称密钥实现最后的落盘全加密，以及在整个过程当中还能提供像数据脱敏等等功能。


除此以外，标准的清零销毁也是什么阿里云数据全链路加密的一个环节。从这里可以感受到，从传输到计算到存储数据怎么在全盘的处于一种加密安全可控的状态，这就是阿里的计算应用和数据的什么安全零星人模型。好，聊完了谷歌的安全模型，聊完了阿里云的零星人模型，大家是不是能理解大厂真正关心的是什么？安全比安全更重要的还是安全，对不对？好，我们下一节就是什么，我有一个小节，来聊一聊我们所有聊到的这些内容的核心点以及面试环节，大家敬请期待。
