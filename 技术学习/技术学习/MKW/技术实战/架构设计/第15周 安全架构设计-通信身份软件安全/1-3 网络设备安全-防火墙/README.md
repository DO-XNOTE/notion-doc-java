一个军事战区的时候，首先要进行一个安全审查，这个安全审查是放在安全隔离区里面进行。同样道理，我们数据中心有一道门，有一道墙，来防止内部的数据随便被外部获取。这道门，这道墙里面有 4 个主要的产品，那这个产品分别可以对应我们南天门的四大将啊。


我来跟大家娓娓道来哪两产品最关键，邮件跟 DNS 夫妻他们分别对于南天门的千里也很顺丰，当一个网站要跟其他网站沟通的时候，是不是要互相了解？之间的域名，互相信任之间的域名解析，这就是 DMZ 的 DS 服务器干的事情，同样道理，当人跟网站进行沟通的时候，很多时候是采用邮件的方式，不管你是接收外部的邮件还是向外部用户发送邮件，所有这些都通过 DMZ 区的什么顺丰儿邮件服务器来进行沟通。


除此以外，我们南天门还有一位大佬，大家还记得吗？是谁？托塔李天王，同样的道理，我们的数据中心里面的大佬是谁？ Web 服务器，注意不是应用服务器，是 Web 服务器，通常 Web 服务器部署的是阿帕奇 Nginx 这样的反向代理。同时我们大量的什么静态文件，js， c s 照片文件全部直接部署在外部服务器上。


Web 服务器有什么样的特点？一 Web 服务器的访问量非常大二通常用户体验就来自于 Web 服务器， Web 服务器反应越快，延时越少越小，这样用户体验就越好。另外， Web 服务器上放的都是能够对外公开的内容，所以不管你是什么数据泄露也好，公摊也好，都没有大问题，只要我们能支撑很多个外部服务器并行运行就可以了。所以通常外部服务器什么离客户更近，哪一个东西离客户更近啊？当然是 DMZ 隔离区。它是什么？整个数据中心最外层的一道什么？一道系统或者一个区域。所以大部分的阿帕奇应用 engines 应用不放在后台，直接放在 DMZ 去对外提供服务。


好，那又有谁来保护我们的李天王？保护我们的顺丰儿和保护我们的千里眼，那就是我们的二郎神兄弟了，二郎神和他的哮天犬承担了整个数据中心防火墙的责任，他既要防御天庭，保护玉皇大帝和皇母娘娘，也要保护我们 DMZ 区的另外三位大将。这三位大将战斗力可不怎么样，完全得靠我们二郎神上去，跟什么？跟我们的孙悟空、猪八戒来大打一场。好，那既然说到了防火墙，二郎神非常厉害，他有什么招数呢？怎么样来克制孙悟空跟猪八戒呢？嗯，二郎生的招数可很多，

不管你是孙悟空来了，还是猪八戒来了，不管你变身成小白兔还是小飞鸟，我都呢查一查你的包头。那包头里面在 IP 层有一个什么 IP 地址，在 TCP 层或者 UDP 层有一个什么？有一个 TCP 的端口或者 UDP 的端口。我检查你的原端口，原地址，目标端口、目标地址，一旦发现违规我就进行抛弃，一旦是符合要求，我就进行允许。这就是以一套规则来应变各种变化，不管你变成小白兔，我就用猛虎来应对。你要变成小飞鸟，我就用苍鹰来进行应对。


七十二变能够应付猪八戒，也能够应付孙悟空，各种各样的黑客都能够怎么通过我的包过滤进行合理的处理，所有违反规则的一律抛弃，这样就保护了南天门，也保护了我们的天庭。但是有时候孙悟空还不光有几遍，他还有什么？还有前生今世跟来生很多套路可以组合起来应付杨戬，那杨戬应该如何应对这种复杂情况呢？杨戬说，不管你是前身金石还是来生，不管你召唤的是六耳猕猴还是其他什么猴，我用一招以克制什么开天眼。

所谓开天眼就是状态检测，我不光检测你当前数据包的 IP 地址和端口，我还检测什么？还检测你曾经的历史，如果你有黑历史，我就把它查出来。我们去赌场玩过，应该很熟悉，在赌场的那个入口处有个安全探头，他探测的是什么内容啊？探测你曾经是不是在这个赌场里出过老千，获利过，或者是你曾经在这个赌场里大赚特赚过，一旦这种人来了以后，保安会把你拒之门外，一样的道理。


杨戬这个状态检测的不是你当前的 IP 地址和端口，而是你曾经的行为。比如说你曾经发起过一个叫think，尝试进行什么 TCP 连接，但是你这次再来以后又发了think，嗯，不行，照理你应该回个 a think，但是你还是发think。我认为你是什么？你是一个黑客。同样的道理，上次你来的时候，你说我是太白金星的亲戚，好让你进了，对吧？这次你还是用同样的连接，但是你指向了目标地址，坏了。你这是说我又是玉皇大帝的亲戚杨戬，可是有天眼头一看就知道什么。你前世是太白金星的亲戚，这次你来生又变成了玉皇大帝的亲戚，一定有奖，肯定是孙猴子变的，就把你给抓出来了。好，通过这种方式什么状态检测很容易抓出黑客的行为变化，从而根据历史信息捕捉出什么异常数据包，这是杨戬的第二大招。那有没有第三大招呢？有。

杨戬的终极大招是什么？就是放狗咬，放出哮天犬。它的狗是什么？就是应用级代理，我们看一看应用级代理怎么样进行处理的？左边就是外网终端，也许是普通的天兵天下，也许是孙猴子。不管是谁来了以后，杨景说我要放大招了，我放一个什么我的哮天犬在中间，他来跟你沟通，不允许你直接去跟太白晶晶或者玉皇大帝进行沟通。你先跟我的哮天犬沟通，沟通完了以后哮天犬代为转达你所有要说的话，这种情况下你即使翻脸，一棍子打死的也是哮天犬而已。所以放狗咬不过，狗死了也就死了。


我们同样的道理，防火墙，一旦你采取了代理防火墙模式，也就意味着你可以牺牲这个代理，从而保证内部应用的安全。所有的 TCP 连接，所有的什么我们的这种什么宏泛攻击、泛红攻击，你只能攻击到我们的代理防火墙为止。你没法通过这种连接攻击，或者是什么特殊的黑客攻击手段直接攻击到内网，通过这种方法来实现我们的内网和外网 DMZ 区的内部和外部的用户之间的隔离。


好，大家知道了杨戬的强大，他的三大招七十二变、开天眼、放狗咬固然很厉害，但是真正要守护好南天门和天庭，必须要给他下达合适的指令，对不对？一样的道理，我们作为架构师，固然防火墙能力很强，但是我们必须要设定好合适的防火墙规则。有三条规则，我想跟大家分享一下。那第一条规则就是默认禁止规则所有和外网的连接，不管你是 Ingress 就是从外网访问内网还是 Egress 从内网访问外网，都应该是默认禁止的。

只有什么玉皇大帝写出的白名单，或者是在玉皇大帝的口令里面允许的人和物才能够进出我们的南天门。不能随便让天庭的官员下凡间，也不能随便让凡间的妖魔进天庭，这是第一条策略，那第二条策略是适合内部的。说的是什么？内部的不同的官员之间怎么进行沟通啊？叫高安全域优先。谁是高安全域啊？王母娘娘就是高安全域。那谁是低安全域啊？像太白金星此类就是低安全域，也就是王母娘娘可以随时传唤太白金星，但是太白金星不能随时去找王母娘娘，防止出现什么太白金星勾搭王母娘娘的情况，对不对？一样的道理，我们数据中心，我们的支付域是可以调用后台数据库来查找订单的，但是订单系统、交易系统不能够随便去支付域去查每一个用户的信用卡信息，这是违规的。


那另外我们的交易系统是可以什么通过我们 DNMZ 的 DNS 去跟外网进行沟通，但是 DMZ 的 DNS 不能够随便向我们的什么交易服务器发起一个数据库查询请求，这也是什么违规的？这很有可能是一个黑客攻击。先打探了 DMZ 区的 DNS 服务器，然后尝试去攻击后台的支付系统，所以高安全域优先。这是什么？整个数据中心，整个生产中心的核心规则，有了上面的默认禁止和高安全区优先，基本上问题都防住了。


但是总归有一些例外，比如说在玉皇大帝闭关期间，一旦孙猴子打上天庭，应该允许太白金星及时向王母娘娘进行汇报，对不对？这条规则该怎么设呢？嗯，这条规则是一条具体规则，应该设好原地址、原端口、目标地址、目标端口除了这个 SOCKET 四件套之外，还应该写清楚通讯协议到底你用的是 TCP 还是 UDP 抑或是 ICMP 等其他协议。另外方向也很关键，你这个通讯如果是从外网到内网，应该表明是Ingress，如果是从内网到外网，也要写清楚是Ingress。这样的 6 元组很适合在云上，同时云上还支撑。有些时候不用写原地址和目标地址，直接可以用服务器或者设备的标签来标记这个地址。这样我们的 6 元组就能够通过标签端口、通讯协议和方向，很好的指定整个数据流量。


所有的具体规则都可以，什么盖过默认规则和高优现役规则。也就是说具体规则的优先级更高，因为它更紧急、更特殊，通过这样的优先级的调整，通过默认策略高、安全优先以及具体规则的调整，就能真真正正搭建起一套完全符合通信要求，同时又能够防住黑客攻击的天庭规则了。
