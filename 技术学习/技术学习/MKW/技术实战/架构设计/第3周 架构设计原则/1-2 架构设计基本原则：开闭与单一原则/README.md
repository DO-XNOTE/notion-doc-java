先从这个开放封闭原则来讲好了，那开放封闭原则它有着两层含义，分别对应开放和封闭。听起来好像是一句废话，那我们看具体开放的是什么？封闭的是什么？开放是指对扩展开放，而封闭是指对修改封闭。那什么是扩展？什么又是修改？很简单，扩展就是对你已有的这些业务、已有的对象进行抽象和封装上的变化，那修改修改就是它的字面意思，也就是说更改你的业务逻辑。那这个开放封闭原则的精髓就在于对于一些新的功能、新的需求的变化，我们使用扩展的形式来完成，而不是说对现有代码的修改。那具体表现在你的微观代码架构层面。
