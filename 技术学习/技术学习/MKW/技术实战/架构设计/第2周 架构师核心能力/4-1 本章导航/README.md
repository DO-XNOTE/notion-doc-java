第一个，既然咱说要了解业务，那必然最终是想怎么样，想让自己的技术架构方案服务于业务，更好的支持业务。所以在第一节当中，我会跟大家去介绍一下技术助力业务都有哪些角度，怎么来做。那到了第二节里，老师这里通过一个大型的案例，一个非常生动形象的一线大厂的案例，跟同学们一起来探讨一下如何去培养技术团队的业务素质。作为一个架构师，如何把自己的视角从纯技术领域转向到从业务视角来出发，那这里我们将挑选的是阿里系绩效最好的业务部门，那多管齐下，从各个方面提升业务素质，那同学们看了之后就会感叹，哇，技术团队还可以这样玩的好，那到这里我们探讨的都是如何来培养技术团队的业务素质。


那培养业务素质是为了干什么呀？是为了让你的架构方案，你的技术发展的路线可以更好地围绕业务来展开。这里老师也会跟大家去分享一下目前阿里系里面非常流行的这样一种中台化思想，它为什么要这样做？它的缘起是出自哪里？事出必有因，这平地起高楼的中台服务战略是如何来围绕业务来构建的？okay，并且老师这里跟大家再抛出一个思考题，去跟大家探讨一个目前的现象级的APP，抖音，它的视频审核业务围绕这块业务如何来构建技术发展路线。
