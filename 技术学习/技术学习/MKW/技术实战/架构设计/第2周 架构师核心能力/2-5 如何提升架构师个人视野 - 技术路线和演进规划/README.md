我们先来看一看安卓应用和 iOS 应用，它们的传统 native 版本的发版是怎样的。首先对于安卓应用来说，它有很多的应用市场，比如我们常用的豌豆荚，包括腾讯、华为、小米都有自己的应用平台。如果我们要发布一个安卓的系统变更，需要把自己的应用发布到各大的应用市场，那对于 iOS 这个情况好了很多，因为 iOS 只有一个市场。但是问题是你的发送的变更都需要去有一定的审核周期才能上架，可以看出它俩发布周期都非常长，那如果发生了很严重的bug，你不得不去立即修复。我举一个例子，比如说某一个 P8 包小三，他被开除了，一时气不过，诶，怎么自己老板包小三没被开，他被开了？那一气之下在 APP 里面埋下了一个罗基炸弹，让你的 APP 在 520 那天所有品类全部都卖绿帽子。


我这只是打一个比方，同学们，在这种情况下，你除了强推补丁做强制升级，完全没有任何办法。但是一来强制升级并不是保证 100% 升级，二来这个有损用户体验。所以这就是 native 的局限性，你无法做到快速响应，你有了致命bug，无法做到快速修复，那这是你绕不开的。


native 版本的局限性，那大家知道 H5 带来什么变革？同**学们应该都知道 H5 自己有非常丰富的页面表现形式，那它改变了 APP 的业务生态。比方说通过 H5 构建的系统，你可以做到快速发布，随机应变。这个在电商行业，这个特性非常重要，尤其在各个互联网 APP 中，这几乎成了一种趋势，因为它并不要求你做 native 发版，而你的内容都是从服务器中动态获取过来的。**

**
那为什么现在所有的大厂都在应用H5？很简单，比方说你现在在一个电商行业，你在这种双十一双 12 的大促活动中，你的竞争对手应用了H5，它可以瞬息万变中随时的更换营销文案。但是你没有使用H5，你的发布要一周发一个版本，你说谁会抢占市场先机？那这种趋势随处可见。**
