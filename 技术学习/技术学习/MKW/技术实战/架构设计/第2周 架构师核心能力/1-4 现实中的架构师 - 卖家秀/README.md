这是架构师的脸面，是我们的门面。比如说做POC， PO seed，全称是 proof of concept，是概念验证，这个我们又叫做技术原型，或者说是在技术选型的过程当中，我们去做一个可行性方案的快速验证的这样一种能力，所以它也常伴随着另外一个词叫 hands on，那也就是说动手能力。


所以说架构师基础，架构师吃饭的本领不能丢，这口饭说白了就是你的技术功底，你的知识面的深度还有广度，没有这些底子，你在技术选型上有很大可能不能识别出风险点，或者说你做出的选集并不适合你当前的这个业务。[另外一项，我把它叫做知识迁移能力。什么意思？再牛的人我们不可能把所有东西都学会，对吗？所以我如何把以前自己的技术经验或者是过去的知识迁移到新的领域中来？这个能力是非常非常重要的。咱工作这么多年积累下来的工作经验，如果不能总结沉淀出一套行之有效的方法论，那对于我们来说换一个技术领域，那就相当于是从头做起，像头条的面试，其实就非常看重知识迁移这一点能力，那他会问你一个完全未知领域的问题，看你如何基于自己以往的经验来把这个问题拆解，以大画小，最终解决。这是头条考察一个架构师基本能力的一个侧重点。](/d7cbf4fb862c4c41ab73f7062bd87fc8)


好，聊完了技术，我们再来看一看业务。作为一个技术人员，现在不光是要专注于自己的技术，更要求你有这样一个跨界能力，去了解自己的产品，了解自己的业务线，对你当前的业务模式有所洞察，懂得你业务模式的花式的玩法。前面提到了技术是服务于业务的，那所以你的技术架构，你的技术选型也要紧密的结合你当前的业务的特点来做分析。还有现在在一些互联网大厂当中，我们经常就要强调这样一种能力，就是技术人员在业务层面上要有平地起高楼的能力。


什么意思？发掘新的业务点？那作为一个架构师，我们不再是像以前的一个一线的开发人员一样，作为一个单纯的执行者，你要在深刻理解当前业务的基础上，去发掘一些新的业务点。那同学们可能会奇怪了，诶，这不是产品经理要做的货吗？对不对？这里我要对这个想法打一个小叉。在咱的互联网行业里，一个很重要的思维方式就是要打破界限，不要在各个角色之中人为的做一些很清晰的界限。那很多产品其实并不知道技术层面的可行性，那对我们而言，那些产品团队是什么呀？对不会魔法的马褂对不对？咱是会黑魔法的人，所以很多产品根本无法料想到技术行业的黑魔法可以给业务带来什么样的帮助。


OK，那所以这就需要我们从技术人员的视角来看，如何通过技术手段来挖掘一些新的业务点。在后面我会跟大家讲几个非常接地气的例子，让大家去了解一下一线大厂的架构师是怎么从自己的视角来挖掘业务，发掘新的业务增长点的。那最后一点就是业务效率这一项和前面一项新业务点的发现其实是非常相似的，都是说我们从技术人员的眼光和角度出发，来思考如何去助力当前业务更好的发展，提高自己业务的效率。好，下一个点就是大家在游戏行业当中经常去听到的一个词叫什么大局观，大部分的技术人员是一个非常较真的性格，因为不可避免你做技术就要长久的把自己的精力沉浸在一件事上，久而久之就养成了自己一种非常钻细节，非常较真的这样一个性格。但是作为一个架构师，你不再是一个可以一头钻进去细节中无法自拔的一线的开发人员。


你要拥有一个非常全局的视野，去从技术、从资源、从产品角度去规划你的产品，梳理上下游所有业务这之间的联系和界限，梳理你当前业务中的主链路，和上下游业务一起去规划可用性方案，这些都要求你把自己的视野提高到一个全局的层面。还有一个近年来非常推崇的架构方向叫什么啊？叫面向未来架构，高瞻远瞩，具有前沿视野。架构师是一个要一直向前看的这样一个角色，那我必须要清楚的知道你当前的架构能支撑、能承载的系统水位在哪里？在你的脑中应该有一个小人一直在敲一个井中警钟长鸣，一直提醒自己这样一个问题，我的业务量如果翻了两倍，那这套是架构，还能不能撑得住？应用承载到了什么水位？我必须要换架构，如果要换架构，我接下来的演进方向应该是什么？如果我们往更大范围讲，那面向未来又可以指你的技术开放路线，还有生态圈的搭建路线，这两点。


后面我们会用独立的小结跟大家详细展示一线大厂的企业级架构师是如何来规划自己的生态演进路线的，帮助大家从上帝视角来观察，来分析一线互联网公司的架构师是如何做这些技术路线规划的。OK，那最后一个话题我们就要回到软技能上了，它叫左右逢源，左右风园，是形容一个人和其他人之间的关系。我相信这个对于技术人员实际上是最难的一项。但是同学们发现你越往上发展，你就会发现技术硬实力的作用在逐渐的弱化，而你个人的情商、个人软实力和影响力的重要性会逐渐的放大，那别人会因为你的技术厉害而认可你，但是未必会给你办事情，对不对？这种事情在任何公司都会发生，我认为是一件对的事情，别人可不买账。

首先，一个架构师一定要是一个非常优秀的技术人员，记住，一定要技术过硬，才能从技术团队中脱颖而出。OK，除了技术以外，作为一个架构师，你也要是一个优秀的产品激励。只有我们去深挖业务、懂业务，对业务有更深层次的理解，才能保证我们做出良好的架构。比如说你如何运用自己的业务抽象能力，把自己的业务玩法抽象成可以落地的技术方案，尤其在一些具有复杂业务场景，比如支付领域、金融领域、银行间系统等等，这些场景更需要你的架构人员具有非常优秀、非常敏锐的业务视角、业务眼光，还有前面提到的如何挖掘新的业务点、新的业务场景，以及提高业务效率，这些都要求你对业务具有更深刻的认识，否则在你领导下的技术团队就会成为一支被业务团队倒逼走得非常非常辛苦的团队。


好，我们要掌握主动权，一定要去自己身先士卒的了解业务。好，那最后一个角色是项目经理，架构师已经不再是一个出一张架构图就可以拍拍屁股走人的甩手大掌柜的角色。在你整个架构方案的落地过程当中，如何去协调业务团队和技术团队，设置里程碑逐步交付，并且在整个过程当中对业务进行调整，这些方方面面都要求你具备一个项目经理的协调能力，这个趋势非常的明显。比如说在阿里面，如果你去面试一个高 p 项目，协调能力，跨团队的沟通能力其实是一个非常着重考察的方面，而且你要在面试当中去举出一些可以足够证明你自己协调能力的案例例子。

在一二线的互联网大厂当中，不管你是程序员还是架构师，其实你所要做的工作只有一个，就是 get things down，把事情搞定。所以作为架构师，你也会承担远大于自己工作职责以外的责任范围。所以在阿里里面有这样一句非常出名的话，叫做为过程鼓掌，为结果买单，这就是结果导向。那这个结果指的是什么呢？落地还是落地？它就是指把业务落地的能。所以即便是作为一个架构师，我们要打破自己职位的界限，身先士卒。所以我这里讲到每一个将军都是士兵。注意，这里并不是说每个将军都曾经是士兵，而是说每个将军现在就是一个士兵。所以作为一个架构师，你不再是一个驻足于幕后去输出技术方案的这样一个角色，而是需要冲到台前，打破职位界限，去承担更大的责任，要有一股憧憬和拼劲。

那三维的第二个方面自然就是业务面，今年业务这里有一个听起来很简单，但是实践起来非常难的准则，就是转变业务视角。那咱的技术团队和业务团队其实就像两个机枪阵地一样，业务视角自然有它自己的盲区，同时技术视角也是一样的。不过对于业务人员来说，如果让他们了解技术行业其实是非常非常困难的，但是对于我们技术人员来说，熟悉业务就简单太多了，那作为一个架构师，如果我们可以立足技术面，同时去触类旁通，经研业务思考问题的时候既可以从技术视角也可以从业务视角出发。这样的话就像两个机枪阵地形成了一个交叉火力，这样一来就可以去挖掘业务，精挖深挖，找到可以无中生有的业务点。


平地起高楼，实际上只是关注自己一亩三分地的技术是很难在公司当中得到晋升的，尤其是在互联网公司好那三维的最后一个方面就是软实力。我这里一直在强调一个走到聚光灯下这样的一个概念，鼓励大家不要做幕后英雄，其实这是一个很残酷的现实，因为我们鼓励而且欣赏幕后的英雄，但是掌声是献给上台演出的人，所以怎么在职场上表达自己，那这是一个值得大家深思的问题。酒香不怕巷子深，这个理念在个人发展上其实是没有益处的，所以我们在后面课程当中设有专门的章节，去教大家如何去提高个人的软实力，从而打开你职业发展的天花板。除此之外还有一个个人综合实力的体现，那就是落地能力，推进一个大型项目从规划到实施到落地的整个环节，是你个人综合能力最好的一个体现。那我们从以上三点做一个总结，塑造三维，第一，去拓展深挖自己的技术面。第二，转变业务视角，去挖掘自己的业务场景、业务玩法。最后全面发展，着眼自己技术以外的软实力。OK，同学们，那这一节就到这里结束了，在下一节当中，老师跟大家分享一下那些年作为一个初级架构师，我所走过的一些弯路。好，同学们，我们下一小节再见。
