赚得又快又稳，赚钱也分快钱和慢钱。那如果你可以傍上像 BAT 这样的大金主，就相当于傍上了王阿姨，后面就不用努力了。但是走技术助力于业务的这样一个务实路线，我们还是要脚踏实地从两个点来出发。第一个就是开源，说白了就是开拓新业务，对于运营团队来讲，就是获客，去获拿多的客户资源，然后丰富你自己的玩法，让这些客户离不开你的业务，增加用户粘性。那对技术团队来说，无非就是满足运营团队、业务团队的各种新的功能，这是新业务的开发，这部分的话语权在大部分公司里是掌握在运营和业务团队里的。但是技术团队并非是一直被压在身下的，这青年老二他也可以去掌握主动权。等后面的小杰里娜老师会跟大家去用实际的案例去讲技术团队如何在业务当中占据主动。


好，那这是开源的路线，那后面与开源相对应的就是什么呀？节流，那节流从技术的角度来说，就是利用你当前的技术手段，新技术或者具有一些颠覆性的技术，比如说人工智能等等，去降低你的业务成本，比如提高业务自动化的比例，降低人工成本、用工成本，或者让业务的差错率降低、正确率提升。比如说利用人工智能机器学习建立的风控系统，分析客户的行为，阻拦风险交易等等，这都是利用技术手段来让业务成本降低的一种方式。


咱这一节的主题是从技术团队的视角出发，如何让技术架构紧密围绕着业务来展开，所以我们会分两个视角来展开，第一个是如何提高技术团队的业务知识，那第二个就是如何利用技术手段来增强业务的运行效率，那这两个主线方向都要求技术团队具有一定的业务话语权，咱知道业务是公司赚钱的根本，那你让一个技术团队去染指业务，那你要问一问这个大房、二房这些业务部门、运营部门他同不同意了？而通常在传统公司，往往这之间的分工是非常明确的，

**技术团队基本是被排除在决策层之外的，也就是说运营团队、业务团队、产品团队，拿这些团队来共同决定你业务的规划，还有制定业务的需求，然后给到技术团队来实现。**


所以在整个食物链当中，同学们可以看到技术团队基本上是处在最下游的，所以咱对技术团队也有另外一个别样的称呼，叫 cost center。什么意思？就是花钱的主，技术团队只会花钱，但是不会直接带来业务，其实我蛮不喜欢这个称呼的，所以哪里有压迫，哪里就有反抗。但是每当你需要去染指业务的时候，你总会被业务团队说这样几个字，不懂业务多么痛的领悟，这一句话就把我们怼回去了。所以我们如何掌握话语权很简单，你不懂什么就去了解什么。

而要去了解业务，首先就要学会像业务团队一样去思考如何从技术角度、技术层面，从开源和节流这两个方向来深度介入业务的决策，让自己的架构可以更好地符合当前业务的特点，符合公司的业务发展的方向。所以这是一个实践性非常强的话题，并不太适合用过于理论的方式跟大家去来讲述。


因此这一节我把它规划成了一个故事会，我们将**用阿里系的实践案例向同学们讲述技术团队如何培养业务视野，而且这不单单只是大厂案例，这里面还有整个阿里系绩效最顶尖的技术部门，绩效 4. 0 的技术部门**，他们的技术团队是如何反客为主去了解业务的？好同学们，那就搬好小板凳，瓜子花生矿泉水哪好准备听故事了，那我们下一小节再见。
