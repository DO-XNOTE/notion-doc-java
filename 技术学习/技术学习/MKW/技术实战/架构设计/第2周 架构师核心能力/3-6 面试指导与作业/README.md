往往技术面试不只是关于技术点知识点的考察，通常也需要借助一些问题来考察这个面试者是否具备比较好的综合素质。因此通过这类开放性的问题给到你发挥，让你去讲你在自己最熟悉的项目当中解决的一些问题，不仅可以考察到你对技术的钻研，还有你对业务的理解，以及你的综合的表达能力等等。

*
那么碰到这类问题其实有一个非常标准的错误答案，不能再标准的一个错误答案了，就是我觉得我个人技术能力非常强，那我到现在还没有碰到所谓比较具有挑战性的技术问题，那如果这样回答只有两种可能性，那要么你自己比较吹牛啊，要么就是你这个项目 low 到爆，没有任何挑战性 CRUD 的项目。*


那么通常这类问题我们记住一定要强调一个强技术导向的难题，不要讲你在这个项目推进、项目落地过程当中，你如何运用自己推拉弹唱去协调各个部门把项目落地，不要过分的渲染和技术无关的内容。碰到这类问题，从面试官的角度来说，我希望是考察你从这个技术点深入下去的一些思考，那你和面试官可以就一个小点的技术问题向下探讨到很深的层次，来考察你对这整个难题的理解。那么通常来讲，**这一个问题连带着还会带出第二个问题，就是你是怎么解决这个技术难题的？你是怎么解决它的？咱前面课程内容当中跟同学们讲了一个万金油的方法论，是什么呀？三段论**。

大部分的面试者碰到这个问题就像说大白话一样，以一篇记叙文从头到尾讲一下这个问题怎么发现的，怎么解决的。那么如果你作为一个面试者，你**抛出了一个三段论，它是一个有你自己的思考，有一个结构性的解决问题的框架的一种方法论。那么作为面试官来说，我有一个很深的印象，就是你对这个问题有自己的思考，而且你碰到这类问题有自己的打法，知道该怎么来去拆解这类问题，那我就会潜意识的认为这个同学碰到一些技术问题，他不仅会把它解决掉，而且还会把其中一部分经验提炼成自己的认知，这就是非常看重的知识迁移能力以及复盘能力。**


那么这类问题通常我们可以通过简单的介绍业务背景以及你当前的技术痛点，并且套用三段论去跟面试官去沟通，探讨你如何把这个问题拆解，然后以大化小，最终把它解决掉的。最后别忘了要强调一句，你从这个过程当中学到了什么？记住前面老师在面试环节当中跟同学们讲过的每一个问题，回答完要有一个偏默点滴，那么这里我说学到了什么，是什么含义，让面试官认为你是一个有思考，有想法，有打法的人。OK，那这就是这一节的两道小面试题的分享。


那么最后在作业环节当中，同学们去回顾自己过去做过的项目，从中挑选出一些比较具有代表性的例子，比如说你如何攻克了某些技术上的难点，那这是其一，其二呢？

我们再从业务视角来出发，看一下你之前做过的项目当中遇到了哪些业务上的难点，然后你是如何去把业务难点最终解决掉的，多准备几个这样的例子，那么你在面试过程当中就会游刃有余，侧重技术难点，同时又连带着自己对业务上的理解。OK，第二个小作业就是大家去复盘一下之前自己在复杂的项目当中去攻克的一些技术上的难点，**是否可以套用三段论，将这个解决问题的过程做一个升华。**OK，那这里呢，跟同学们就分享到这里，我们下一小节再见。
