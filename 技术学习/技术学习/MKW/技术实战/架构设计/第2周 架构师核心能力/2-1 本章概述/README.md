了解了这些内容，我们就清楚了自己的职业规划以及对应的技能加点。那同学们就跟我一道来看一下这一章的课程目录都有哪些。好在第一节里，我们给同学们介绍一个二转的路线，什么叫二转？如何从程序员转职成为架构师？那都有哪些相应的可以选择的方向？比如说应用域的，一线域的架构室，还有业务架构室以及更高阶的系统企业级架构室。在这一节里我会和同学们一起去探讨不同的发展方向，它背后对架构师的能力要求都有哪些不一样，以及对应的技能加点应该点在哪些技能上面？好，那再往后我们来通过一个大厂的案例来跟同学们解读一下技术路线和演进规划是怎样来做的，这是帮助同学们提升架构师事业非常有用的。


安利教学部分，那相信大部分同学都是在一线比较优秀的开发人员，那当我们把思考问题做架构的这样一个视角，从一线的开发人员转换为架构师视角，你会发现整个世界豁然不同，那这个不同会非常明显的体现在你的视野和格局上，比如接下来的这个话题如何去规划你的技术生态的发展？那我们一道从这个问题出发，看一下提升了自己视野的高度，看待问题的方式会不会有所不同。


那再往后我会跟大家通过一系列的一线大厂的真实案例去来探讨如何来做技术生态的拓张，这些技术生态拓张与他们本身自身的业务之间又有什么样的关联？

正所谓事出必有因，没有平白无故拍脑袋就上的技术架构方案，我们就要去探求它背后技术与业务之间有哪些的关系，一线的这些大厂又是怎样围绕自己业务来构建自己的生态拓张的方案的？那在这一章的最后，我在跟同学们一起去探讨一个架构师成长过程当中必不可少的一个软实力，那就是如何来推进你的架构方案的执行和落地，那落地两个词是现在互联网公司当中出镜率非常高的一个词，那正所谓白猫黑猫抓到老鼠才是好猫。
