hello，幕后网的各位同学们，大家好，我是姚半仙。咱同学们经过前面几个小节的学习，相信大家对架构师这个职业，它的职责范围，还有在项目当中应当承担的责任都有了比较清晰深刻的认识。那么这一节里我们做一个总结性而又拔高的一个话题，就是如何从架构师的角度制定一个技术发展的路线图。


同学们在平时工作中应该就有所发现，架构师他可不是像一线工作人员一样整天埋头写代码，那他总要去做一些总结规划性的东西，或者说在 PPT 上的东西和领导层做规划和汇报，小到你的项目规划，大到你公司级别的战略产品的规划，那我们就从这个角度，一个比较宏观的角度来出发，一起来探讨一下作为架构师去制定技术发展路线图的过程当中应该注意哪些内容。那我们来看一下本章的课程目录都有哪些非常的精简。


第一点就是如何来规划一个短中长期的技术路线图，所谓短中长，这每一个不同的期限，它其实侧重的点是不一样的。那在这三个时间范围的路线图中，我们怎么样体现你的业务战略，如何来紧紧地围绕业务来做你的技术架构规划？这是在第一小节里老师要跟大家一起去探讨的，往后我们如何在做技术路线图规划的时候，体现一个面向未来架构这样的一个思想。那什么是未来？未来就是你的业务全面奔小康的日子，那业务规模、容量规划变得越来越大，并且对你底层的技术架构有着更高的稳定性、可用性的要求，那有了上面这几点，我们就形成了一个从技术上和业务上都比较连贯的一个路线图。但是咱知道外界的环境是在不断变化的，那以不变应万变，这个道理在互联网行业是不存在的，所以我们要经常去调整自己的技术路线图，去纠正在路线图执行过程当中的一些偏差，那有哪些因素会引起这种偏差？第一个竞对业务和市场环境的变化，那第二点从技术层面我们要做一些定期的评估，去调整技术方案。
