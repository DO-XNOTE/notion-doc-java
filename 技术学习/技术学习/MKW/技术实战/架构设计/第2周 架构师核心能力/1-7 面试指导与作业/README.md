那么我们这里就可以应用前面跟同学们去提到的几个点，比如说你的高可用吞吐量，还有如何设计你系统的可扩展性，每一个技术选型如何考虑它的横向扩展集群能力，以及你的技术选型是否可以去支持业务的快速变更。


那当然这里还要去平衡技术以外的一些话题，比方说你的成本考量，还有系统的安全性的要素，那么每一个小点的问题，我个人建议把它的时间控制在 2 分钟左右。那面试过程当中问到这些比较宽泛的问题，它往往不是为了考察你的技术能力，而是考察你是不是有这样一个大局观。那我们尽可能的速战速决，在这个问题上不用纠缠太长时间，把面试的时间留给后面真刀实枪的一些具体的技术场景的设计，OK，还有一些问题，比如像后面的架构师面试环节，还有你的直属老板面试环节。


喜欢问到的就是说你如何来规划你的职业生涯，那或者说你当前订立的这些职业生涯成长的目标不错，那么接下来的 2- 3 年内，你打算如何来付诸行动，达成你这个目标？那我这里比较建议同学们根据自己的自身年限，工作的年限做不同的问答。比如说你在职业生涯的初期，那你尽量把这个职业目标定为强调你的技术路线，那即便你是一个非常有野心，想要管理团队，身为技术主管，那么在职业生涯初期尽可能的把这个野心稍微隐藏一下。比如说你面试一个 1- 3 年的技术人员，那我个人比较希望听到的是他想在自己的技术路线层面做更深的巩固，而不是说一个 1- 3 年的初级技术人员，他好高骛远，思考着以后想要当技术主管。因为其实你的回答最重要的不是表达出你个人的性格，而是表达出你自己的能力，你自己的期望和当前岗位的匹配程度。那对于 1- 3 年的技术人员的招聘，我们需要的就是希望你是一个专注于技术的这样一个角色。okay，那当你工作年限积累到了一定年限之后，那么你在回答问题的时候，可以去强调一下远期的一个发展路线。比如说你远期希望可以带领一个团队，那会着重培养自己的软实力，因为对于高级的技术人员来说，我们不光其实是招聘一个码农，更多的情况下我们也想给团队作为一些技术储备干部。


比如说你是一个阿里的 P7 的员工，那尽管 P7 还是处于一个一线大头兵的这样一个角色，但是在这个职级上面，我们可以适当的透露出自己的一些远大的前景的目标规划。那么这里就可以从前面课程当中提到的几个点，比如说你作为一个架构式的技术调研能力、产品能力、大局观，以及你如何构建自己的影响力，那把自己的培养目标逐渐的从一个纯粹的技术路线，向管理路线向这些软实力方向来匹配。那么同时在这个过程当中一定要强调业务的重要性，那咱就可以拿阿里系的招聘来说，像阿里系这种重业务的公司，强调一个技术人员的业务视角是非常重要的，那你强调你自己有这个业务的方面的这种思考，那其实对你的面试过程也是非常加分的。
