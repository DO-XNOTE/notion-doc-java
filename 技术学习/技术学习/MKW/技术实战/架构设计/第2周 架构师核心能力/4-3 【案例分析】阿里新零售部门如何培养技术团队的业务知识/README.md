因为新零售它拼的就是你对业务的理解，它是一个从头到尾完全是由业务来强驱动的这样一种模式，那新零售业务的每一个旁支分支，它都有很深的业务深度，比如说我们采购团队对商品类的甄选，以及供应商的竞价等等。还有像质量检测部门对应的采购货品，它以什么样的标准来进行检测，不同品类它的标准完全是不一样的，那各种国标也会每年发生变化。


那再往后前场卖场，因为新零售是一个具有线下卖场的这样一种实体经济，因此它有像大型超市一样的前场卖场，这里就涉及到线下零售业的管理。还有还有后场仓，那后场仓是前场仓的一个弹药库，是它的库存部分，那既然涉及到库存，我就有补货计划，那每天的分时补货计划以及天周月的补货计划，再有新零售强调一个线上业务、线下配送这样一个概念，所以我们在后台还有分拣打包的团队，会把线上订单到前场仓进行分拣，然后打包出来。


那既然打包就少不了什么呢？物流履约，那这是传统的线下业务所没有的一个模块。那说到传统零售，那新零售跟他们的区别是什么？线上业务对不对？它的评效之所以非常的高，是因为它有线上业务的流量的加成，这个还不是新零售业务的全貌，它只是其中的几个重要的部分。那同学们从这里可以看出新零售业务它本质是什么？本质其实没有什么新东西，它无非就是一些现有业态的结合，线上业务像线下门店二合一，很像以前的 o to o 的概念，但是对于 it 团队来说，这种业态在我们架构设计 it 系统的时候难度非常高，因为什么呢？我们看这个前场卖场、后场仓，它涉及到一个场地，对于我们这些在电商行业摸爬滚打的人来说，没有线下的经验对不对？因此它的业务复杂度是呈几何级数式的增长。

这就是我们前面说的，如何把技术团队在业务面前的这种被动化为主动的一种非常有效的手段，对不对？去体验你的产品，将一些微小的功能的改进，由技术团队来进行驱动，而不是仅仅依靠于业务团队，这必定是可以大大增加你技术团队在业务面前的话语权。OK，那好，最后一个点就是业务部门直接对接研发好，
