我们叫一线的开发人员的一个定时思维，就是他希望你所有的系统，所有的需求可以一次搞定，原始的需求给过来，尽管非常复杂，但是我们都可以设计成一个非常精妙的系统。不过问题出在哪？非常抵触这样的系统变更。


这个想必大家在工作中都经常遇到，当你碰到产品提出的新需求，第一的反应是什么？推辞对不对？这个功能不太好做，这个功能可能会引起一系列的连锁反应等等，这是一线员工的思维，大部分一线的开发人员其实都是这么一个风格，在像互联网公司也并不例外。但是我们作为架构师来说，第一点要转变的思维是什么？没有什么是不可以的，所以我们怎么样从一线员工的思维转变成架构式的这种思维？这里面我就教大家一些很小巧的很精妙的心理游戏，给大家的大脑做一个清醒的催眠，
