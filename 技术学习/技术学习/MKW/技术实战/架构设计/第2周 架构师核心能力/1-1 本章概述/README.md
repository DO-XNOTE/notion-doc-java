OK，那接下来跟同学们聊这样一个话题，架构师的九九八十一难，成为一个架构师难不难？难，非常难。那这些难究竟难在哪儿？我们这里要跟同学们去用一些实际的案例来探讨架构师的工作内容中的复杂度来源都有哪些？那这不是干巴巴的理论，老师会使用一些阿里系的黑科技，跟大家去展示一下这些复杂度来源的背后，这些头部的互联网公司怎么解决这些复杂度来源呢？那再往后跟同学们一起看一看现实中的架构式，它的技能数加点都有哪些？它的角色的定位又是什么？那聊完这三个问题，相信大家对架构师这个角色的理解又更加的具象化了，那在之后咱再去聊一聊，成为这样的架构师，那你容易在这个过程当中犯的错误都有哪些？最后从咱技术人员的视角出发，看一下一线的开发人员在通往架构师的这个过程当中，需要弥补哪些欠缺的能力，并且如何将你的架构紧密着围绕着业务来展开，去开始关注于你自身的公司业务？还有一个成为架构师必不可少的一点就是要学会影响力的扩张。那我们的课程目录介绍就到这里，欢迎同学们走进架构师养成计划之第一季对架构师的种种误解。好好，那各位同学咱走起。
