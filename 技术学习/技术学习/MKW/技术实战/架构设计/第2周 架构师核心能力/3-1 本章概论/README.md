hello，幕后网的各位同学们，大家好，在前面的两个章节里，我们主要是讨论了架构师的职业发展的能力、维度上的要求以及它的发展路线。那么这一节里，咱循序渐进地进入一个新的话题领域，架构师养成计划之第三季，如何画技术为生产力，去解决企业碰到的实际的技术问题？技术是为了解决实际问题而存在的，那再复杂再困难的问题通常都是可以通过以大化小，应用一些理论来把它最终解决掉的。那接下来的这一章，我们就做这样的一个理论和技术相结合的章节，和同学们一道，通过几个真实的例子来体验这种将一个复杂问题以大化小最终解决的这样的一个过程。好，我们一起来看一下本章内容的课程安排目录都有哪些。第一节里我们就把一个实际的困难问题先摆在这，跟同学们一起去看一下当年的天猫线赫赫有名的迈巴赫团队碰到了什么样的生产级问题。

那所谓化技术为生产力，那在生产出了问题就要用技术手段来破解，那我们破解问题的过程绝不只是说盲人摸象或者是摸着石头过河，它一定会遵循着某些方法论去拆解问题。所以在这第二节里，我跟同学们一起去探讨如何来拆解技术瓶颈这些难点问题。


我们来看一个一招鲜的分析方式，也就是三段论，那理论咱有了，如何把它应用到实践当中？老师这里就通过另外一个例子，通过分布式事物的例子来跟同学们去体验一下如何将三段论的理论应用在实际的问题当中，并且把它解决，而这里我只是点到即止，我们不去过多的深入分布式事物的底层原理，这个在后面的课程当中老师再跟大家详细来介绍。


那最后咱再回到前面提到的这个天猫线迈巴赫团队碰到的生产问题，来看一下如何通过三段论来把这个实际的生产问题解决掉，主要是想让同学们去思考体会这个问题解决的方向和思路，注重它的这个解题的过程。并且这里再跟大家分享一下这个问题最终的一个解决方案，那就是一个广泛应用在集团层面，包括交易链路等核心链路当中的一个非常厉害的中间件叫双引擎回归测试框架。


okay，那老师这里希望通过上面的理论还有实际的案例，让同学们碰到一个实际的问题的时候，不管它是复杂的问题，简单的问题至少有打法可以知道从哪开始下手去分析问题。好，那在这一章的最后，老师跟大家玩一个游戏。什么游戏？
