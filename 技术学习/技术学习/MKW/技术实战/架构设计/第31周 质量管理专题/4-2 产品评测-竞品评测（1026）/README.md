# 4-2 产品评测-竞品评测（1026）

hello，慕课晚的各位来架构师们，欢迎回来。在上一小节里，我们为大家开启了一个新的章节，产品评测章节。在这个章节里我们将为大家讲解关于了解自己以及了解竞对，了解友商这样的一个过程里我们都有哪些可以去做？那在这一小节里，我们就先去聊一聊我们怎么去评测一下我们自身和友商或者竞对相关的产品，来看一看到底谁哪里做得好，哪里做得不够，我们就怎么样子能够以彼之长补己之短好。


说起产品评测，我们首先还是要针对其基础进行相关描述去分析，这也意味着我们先要很接地气的去看一看各家产品到底是什么样，

所以这也意味着我们首先要进入到行业里来看一看这个行业背景是什么样？制作的产品它的领域定位又是什么呢？到底定位在哪里？比如说美团外卖，它就是一个基于餐饮这样一个大行业下，而聚焦在怎么样子将非堂食的这种外卖送好，那这就是它的领域的一个点。进一步就是它的市占分析，比如它占了市场的60%，那它就这就是它的市占的一个分析。通过这些描述可以知道一个大背景，来明确这个产品它所处的行业，所深耕的领域，以及在这个行业这个领域中它所处的地位。


描述行业之后我们就要去细看它的产品描述。这里第一位就是用户的定位，它是面向于哪一类用户去做的这个产品，以及它面向于这些用户它的商业模式是什么样子？他是向用户收钱，亦或向商家收钱，还是向广告主收钱？还是说他目前就是为了扩张用户，所有的钱都是投资人或者是公司自身的，那这就是它的商业模式以及它当前的一个发展现状，这些就是产品本身的一个基础描述。


并且还有一个点就是我们在做这个评测的时候，我们要注意本次行为有一个描述，第一就是我们评测的环境，我们属于一个什么样的环境？是电脑还是手机？我们评测的它的版本是什么样子？它是最新版吗？它是哪一个版本？因为不同的版本功能和定位可能都有细微的差别，所以我们要针对我们需要评测的版本做好树立一个说明，以及我们的评测方式。


这里就包括我们是针对用户做相关调研，来看一看在用户眼里的各个不同的竞品它们是什么样子的。还是说我们直接去体验这个产品，明确处它具体的细节功能两边做相关的对比，还是说我们使用一些更透明的这种盲测，让一个用户他不知道你什么样的一个目的情况下去做对应的测试，从而来获取其最客观的一个反馈。那这些就是我们做竞品批次中最基础的这些基础描述。

那这里就会有很多的点了，比如说产品的定位，他讲了一个什么样的一个故事？在这个故事里它又针对的是哪样的一个目标市场？那这就是我们在做产品定位这块儿时候，要去分析不同的竞品之间的这样的一个点。


第二个就是整体的产品框架是什么样子的？它的功能框架是什么样子的？为用户带来了什么样的一个能力？这个能力通过怎么样的一个框架，有几块的东西，每一块里面都有什么功能？然后这就形成了整个地理一个模块分类，它的几个模块，几个板块，有首页是吧？有分类，还有我的订单，还有我的有设置，这些可能就是它的模块分类。


通过这些去细看一下在整个产品设计过程中是否实现的是否完整，并且是否有一些突破和创新的点，然后去做好对应的对，比如说两个产品之间它们的合理性是什么样子的？在功能的设计上它们的设计性又是什么样子？谁更合理一点，谁的交互设计、用户界面设计更好一些，通过这些细节的对比，明确出在细节点上谁更好。


而从战略分析上则更多的还是看看它的商业模式和它本身想要成长的一个打法。模式是一个套路，这个套路是指它本身要一直朝这个方向去做，而打法则是在贯彻这个模式的时候，它每一个阶段怎么样子去拿到更好的一个结果，拿到一个更好的成长，更好的收益这样的一个方法。而特色则也围绕着产品框架以及它本身的定位和它本身特有的一些合理性与设计性来看一看它整体的设计风格，和它特有的一些卖点，因为不同产品可能都有自己的一些卖点，这些卖点可能是其他人不具备的，那这些也是我们额外要关注的点，同时也要去关注在商业模式下，在这些特有的功能之中，它的本身的盈利模式是什么样子的？和它的盈利能力是什么样子的？它用什么方式盈利它，他的能力又怎么样？他是否能够大幅度的挣钱，还是说他现在只能勉强的活下来，这都是我们要在盈利对比中需要分析的。然后去看看他的核心用户，他的用户细分是什么样的，他的用户画像是什么，他通过特色功能，通通过它的这些核心用户打造出来的产品壁垒是什么样子，它有哪些差异化的能力，以及它通过的技术是否建设出了一些技术壁垒？而这些壁垒是其他人其他的产品不容易抄袭或者不容易模仿。


并且最终来看一看它在整个扩散对比上，它的扩散能力是怎么样子？它的扩散收益又是什么样子？它通过增加一个用户，它能带来多少额外的收益？那这些就是在扩散过程中去看。通过这些我们就能够从它的整体，从它的战略，从它的定位，到它本身的细节和功能，到功能之间的差异化，到它的盈利模式，到它用户它本身的一些壁垒和特色的功能，以及它在分享扩散过程中本身的表现和其具备的能力和收益。通过这样子我们基本上能对一个产品有了一个相对完整的分析了，在除了对功能和战略之类的分析之外，我们还要更多的去看什么呢？


去看它本身的一些特点，和它与其他人相比其优势和劣势等等。那这记住了我们的 SWOT 分析。

SWOT 的分析什么是SWOT？ s 是指其优势， w 是指其劣势， o 的话是指其存在的机会， t 则是其目前面临的威胁。通过这些我们就能从它的正向和它的负向的两个方向共同去分析整个产品当前的一个情况。那优势来说，我们更多还是看产品的优势是什么，这些优势它领先的程度是什么样子？它是领先一年还是领先半年，一会是领先三个月，这也意味着竞对要花多久时间才有可能追上它当前的这样的一个优势？而这对劣势来说，我们重点要看这个劣势可能会带来哪些影响，它是否会将我们优势产生，那些地位拖下去？还是说这个劣势其实是从其他的方面对整体产品产生影响，而针对机会来看，产品发展机会是什么？是行业发展带来的机会吗？还是自身的一些突破带来的机会，抑或是一些新技术的一些落地，新技术的一些突破带来的一些发展的机会？通过机会的分析，我们能看它面临的这个机会是大家共有的还是独有的，从而去判断其竞争力和在危机之中它本身的一个生命力。


最后就是一个危险，产品面临的危险是什么？是行业趋势在下变化，还是本身的生命力开始下降？亦或是有个强有力的竞对或者合数班？比如说像淘宝，它现在面临的最大的威胁是谁？是拼多？那这是行业趋势吗？可能并不是。生命力吗？也不是，只是出现了一个非常强有力，而且打法和竞争力都非常强的这样的一个对手，而这个对手其实有的时候正面战场的面对面的硬刚反而会让你觉得更加的难受。通过这样的 SWOT 分析，就可以帮助我们从全面的正向和负面的这种逆向来共同去看整体产品当前的一个情况。

为什么还要回来强调这几个点，就是我们不管这个产品是什么，不管他们发展什么样子，我们最终的评测还是要有我们本身的出发点和我们的目的的，我们也需要去抱着改善我们自身的产品，对于各个产品的战略定位、用户功能和交互做明细的一个分析与评测，最终还是要将我们下一步要做的什么事儿这个点明确出来。


不管是整体的一个基础基本面是什么样子，行业是什么样子，产品本身是什么样子，以及各个产品它本身的这种定位、框架、战略特资、功能、盈利等等，这些都不是我们最终的评测需要得来的结论。我们评测的结论还是通过这些的分析，最终来看我们自身的产品哪些是要调整，哪些是要升级，哪些是要做改变，那这就是我们要做的这样一个点。也是最终回到了这句话，大胆假设，小心求证。以笔之长，改善自身，最终的目的还是改善自身好，这就是竞品评测中我们需要去做的，以及我们可以去做的，和我们最终要做到什么样的一个效果这样的一个内容。好，在下一小节里我们将为大家介绍一下 Badcase 挖掘，看一看这个概念，它到底是什么？它能够帮助我们做什么样的一个事情，带来什么样的一个价值？各位未来架构师们，我们下一节课不见不散。
