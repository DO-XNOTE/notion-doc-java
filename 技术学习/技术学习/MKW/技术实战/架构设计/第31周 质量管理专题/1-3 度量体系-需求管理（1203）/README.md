那当然这里最直接的一个指标就是需求的数量，那针对需求数量，我们要能够明确出我们围绕着这些需求我们要花多少时间去完成它，那这里就也意味着我们会有需求的完成周期。其次我们还要明确出对这个事情我们有一个明确的一个评估，那这就是我们针对整个需求数量的一个评定。针对这个需求，那他肯定在明确了之后，他要分派到不同的人，由对应的人来进行相关的实现，那这里也意味着每个需求最终都能落到对应的研发同学的人身上。这里就是要需求到人，在这里就能明确出第二个指标，叫做需求周期。也就是说每一个人他在做这个需求的时候，他花了多长时间？当然单一需求去看这个指标没有任何价值，因为不同的需求其周期肯定是不一样的，一个小需求一两天就做完了，大需求两个月可能就开发不完，但是从平均上去看，这个周期其实就存在了相关的价值。


如果一个人员需求周期都是两个月以上，那肯定是有问题。如果一个人的需求周期它都是两天以内，那也肯定有问题，所以通过这个我们能够很明确的去发现那些明显偏离的情况。所以这里更多的是一个对人的一个评估，从需求的数量、需求的相关的周期到需求拆解到人每个人身上有多少需求，以及到人的需求周期，这样的话，对事对人两个层面上的一个评估，我们就能够进入延展出来的我们的需求大盘，它就是一个针对于所有的需求和人我们整体的完成情况，而这里就会是一个综合性的评估。


所以我们从这三个维度上的评估，就是我们进行需求管理的三个主体的一个视角，在这三个视角下，它能够帮助我们去做好我们需求的这样一个全生命周期的管理。

因为需求同事很多，每一个产品都在拼命的提需求，有客户那边的需求，有老板这边觉得我们需要去做的面向于公司战略规划的需求。还有一个是我们比如说面向于这种架构升级、技术升级这样的技术内部的需求等等，这些需求都会揉在一起，共同构建出这样需求的规划。


而在这样需求规划中，我们就要去明确我们怎么样子能够更好地实现这些需求，那也其一就是作为这个资源的利用效率，资源有很多种，除了我们平时说的这种电脑、机器，然后各种各样的钱之类的也是资源，对吧？然后还有相信我们使用的中间件属于一些计算资源，网带宽等等，这些都是。当然还有最重要的一个点就是人人也是我们那个资源，我们怎么样子来提升个体人的资源的效率利用最大化，这也是我们在需求管理中需要去考虑的。


其第二就是业务支撑，结果我们做了需求肯定是要想要去支撑业务的，不能说需求做了就白做了，那这个东西真的就是废球了。所以我们也要说在规划完需求之后，我们要考虑我们这些需求最终能够支撑起什么样的业务结果，在规划之后我们肯定就要进行需求的拆解到人，然后进行需求的执行。


在需求的执行过程里，我们要明确出需求整体的这个进度的一个管理需求，它一定是从需求的分析、需求的评审，然后到需求的实现，那再到这需求的验收这样的一整个过程。那在这个过程中我们要明确出需求在每个时间点它是否能够按照其预期设定的这样的一个时间去执行，那这里又意味着很多需求可能会被延期，有很多种可能，一种可能是需求本身的复杂度超过了体育期。


第二种就是需求被额外的需求给打扰了，本身说我做的一个需求做的好好的，你突然说来了个紧急需求，我必须马上去做这个事，那我这个事肯定拖到后面去了。那今天差一个，明天差一个，我手里这个需求可能就延期了。那这也是一个很常见的一个延期情况，当然偶尔还有提前完成的，比如说我本身来说，我估计这个要一周，结果我发现有一块代码可以复用，这下好了，一下子省了 3 天时间，我这个需求我两天就做完了。那这也是有可能。那命令这种提前完成的是好事吗？当然是好事，这也意味着我们可以将更多的需求纳入进来考虑。


第三就是需求的跟进。需求的跟进是指我们要明确需求它最终转化出来的业务，结果我们做了这样的一个事儿，它是否产生了价值？也许做了一个事儿，它最后什么价值也没产生，甚至是说上线的都没人用，那这个东西就纯粹的是个资源的浪费。那在这里也意味着我们要对需求的价值进行复盘，并且根据需求转化的情况做好试错方向的调整。因为我们很多时候会做一些需求，这些需求可能是不确定的，就是我们不能确定其最终是否能产生价值。但是我们要明确这是一个试错，也许它就是个爆款，这也许它最后变得了无价值，都是有可能的。那在这里呢，我们就用最短的时间内去验证这个试错，看看我们这个需求它是否能够产生转化，是否能产生价值。如果明确它能产生价值，只是在当前阶段它还需要一些时间，那我们就往里投更多的时间。但是如果你有明确出这个东西做了就是没有用的，那我们就借涨止损开始调整我们的方向。所以我们要能够在需求的之后对需求做好相关的复盘，包括需求的合理性、需求的完成度和需求的价值。

需求它的缘起是什么，它是由什么样的一个情况下才提出这样的需求，然后重新从源头来去看一看整体需求是否合理。其次就是需资源消耗的合理性，这个需求它本身是存在价值，但是它消耗了 20 个人一个月的时间，那这时候这个需求还是否合理？这可能就需要变动的去看一看了，也许它消耗资源所消耗带来的成本可能已经远远的大于它带来的价值。第三就是优先级的一个合理性。公司永远是在缺人的，那在这种状态下，我们要将所有每一个人都用在刀刃上，也意味着我们要明确出最有价值的这些需求放在我们的最高优先级上。最后就是投入时间的合理性。我们在整个需求的这个过程里，我们到底投入了多少时间？这些时间是它也是有成本的，那这些成本是否带来了其应有的和价值回报？第二就是针对需求完整度的我们的回顾主要是围绕着整体的一个完成度，我们是否把它真的做完了？是否达成了我们原来彻底的用一批？第二，这个时间需求周期是否是符合我们的预期了？它大概是个什么样的情况？第三，需求是否有延期到多久？最后最终实现出来的这个需求和原来我们的预期是否存在着偏差？最后我们来看看需求的价值，需求价值就是需求它实际产生了的价值，我们要对齐进行回顾，来看一看这个需求最终它是否的的确确产生了相关价值。


这第一就是需求带来了什么样的效果？比如我们需求是一个用户流量增加这样的一个需求，那它最终有否是否带来了流量的增加？如果这个需求它本身是一个想要提高用户转化的需求，那它最终是否带来了用户的转化的提升？这就是我们要去回顾的它本身带来的效果，而这个效果最终是否产生了价值？比如说它的确带来流量的提升，但是我们的成交一点都没有变化，那这里单纯的流量增加是我们希望的价值，并且其最终是否产生了实际上的价值转换，这是我们要分析、要回顾的。


还有就是看这整体的一个投入产出比，这就包括我们之前 the 那刚才合理性中资源消耗的合理性，比如说我们投入时间的合理性，我们不整理的一个偏差，这些都要考虑进来，最终来看看这个投入产出比是否是符合预期的。
