# 4-5 产品评测-用户研究（1126）

hello，慕课网的各位未来教课师们，欢迎回来。在上一小节里，我们重点为大家介绍的是舆情分析。舆情其实是在当前这种信息快速传递，快速膨胀的一个互联网环境中非常常见的一现象，也是在这种状态下，我们需要去关注，需要去协助于公司的公关团队和管理团队，去帮助控制相关的舆情，保障公司的形象、品牌与产品不受影响。那也意味着我们技术需要去用我们的方法去帮助他们做好这个事情。


那在这一小节点我们为大家讲的是用户研究。用户永远是我们的立身之本，我们只有尽可能地支撑到我们的用户，才能够更好地发展我们的产品，那也意味着针对用户研究是我们一定要做好做透的，只有这样子我们才能更好地针对他们做好相关的产品和相关的活动。

这些属性比如说他的性别、他的年龄、他的职业，这些是属于他本身的一些属性是客观存在的，我们则需要通过这种方法去发现这些属性，无论是他的个人资料也好，或者根据他网上的相关的行为也好，去判断出它的属性。然后根据这些属性，我们就可以细分好对应的用户群体，从而明确出一些用户的诉求。所以从属性去挖掘出用户的本质诉求，这个就是用户分析的重要做的第一步。


怎么样子？比如说男性会偏向于什么？女性会偏向于什么？如果我们的用户大部分都是男性，你就给他推化妆品，可能就没有那么效果那么好。但如果都是女性，你给她推电子产品可能效果也没有那么好。所以针对相关的属性，我们可以衍生出来相关的用户诉求，相关的用户需求。然后从用户需求我们其实可以细化到一些用户的场景，因为为什么说要从需求到场景，而不是从场景到需求？因为我们这是从属性里去挖掘出来的需求，这些需求是跟属性紧相关，但是它就会比较泛、比较大的一个需求，而这时候我们就需要通过这些需求和我们的相关场景进行匹配，这里就可以哎呀出一个完整的一个场景，这样的话就不是一些零散的需求了，然后再重新去场景去形成我们的核心功能，以场景为基础建立好我们的核心功能。所以这里的场景其实对应的也是我们的场景需求。


整体的思考链路就是从用户属性到用户需求，再从用户需求完整的建立成用户场景，衍生出场景需求，再从场景需求去建立好我们的核心功能，最终用功能去支撑好我们的核心用户。

那第一点肯定是核心用户，因为只有核心用户才是我们真正需要去服务好的用户，也是这些用户支撑起来我们产品和我们公司。那这里我们需要关注的就是第一核心用户的留存粘性怎么样？我们可以针对这些高留存的用户做好对应针对性的营销和活动，保护我们这些核心用户，并且让他们持续为我们产生价值。第二就是这种付费的用户，凡是付费的用户，那都绝对是核心用户，因为他愿意为你付费，说明他绝对的相信你，所以针对这类用户，我们也要单独的进行相关的营销活动。第三就是具备的相关资源和扩散的用户，他手里有一些资源，他能够利用他的资源帮助我们的产品从更好的方向发展，或者它本身就是一个超级扩散单元，它能够帮助我们把我们的产品分享给 n 个人，当然这些 n 个人很可能还会再有 n 个人就像病毒一样快速的扩散，那这些都是可以帮助我们快速地推动产品成长，推动公司营收，为公司带来利润的相关用户。这些就是我们的立身之本，所以这些用户是我们必须单独区分，并且单独的去针对其做好相关的活动。


营销是指功能设计。第二类就是我们除了核心用户之外，要去分析我们整体的用户结构，包括我们的性别分布、年龄分布，他们的收入的情况，他们的职业的一些分布，还有他们的爱好和他们在互联网或者在当前行业、当前领域的相关经验，那这些就是相当于客观的一个用户的一个划分。通过这些划分，我们可以有针对性地去挖掘他们的本身的一个诉求，从而细化成我们相关的用户画像，而这里就可以针对于我们的核心用户和我们细化好的具备用户画像的用户结构，去做相关的用户运营，包括他们本身的行为分析，包括差异化的用户的一个分层运营，以及我们可以针对一个群体的目标用户做好我们的全员触达。从而当我们有一些针对性的运营活动的时候，可以快速的触达到他们，并且最大程度上提高这些活动它本身的一些效率。


综合起来看就是以数据分析为依据，针对用户结构进行分层精细化的运营，从而最大化的提升运营效率，这个就是我们用户细分所要做的事情，以及它能为我们带来的价值，


同时我们也要更进一步的去看用户的行为，因为我们用户行为可能会分为两类，第一类是探索性，但通常我们针对这类用户需要去提升我们用户体验。

哪一类用户是我们最常见的探索性？我们的新用户他要来试验我们这是怎么玩儿的，他要从注册开始，或者如果我们不需要注册，他可能就是首页开始。如果我们的用户体验不能够给他们带来一些新鲜感，或者满足他们探索的欲望，那本身来说他们可能很快就会关掉APP，然后去干别的事情了。


第二就是我们针对这类用户要做的就是促进留存，尽可能的让他们仍然保持着对我们产品的相信，让他们愿意留下来继续的探索整个产品，所以新用户、新功能用户是个逐步去发现、去探索并且了解产品的行为，这一类行为其实是我们要特别小心谨慎且仔细去打磨的，因为这个里做不好，我们新用户就进不来，进来了他也留不住我们后续的所有的思考，所有的大故事、大框架都无从谈起。


还有第二类就是习惯性的，通常是说一个用户他已经习惯了用我们的产品了，我们目的就是让他们保持 ROM 的习惯，并且不断地给他们新鲜的一些事情，让他们保持对我们的新鲜感，保持对我们的好奇心，并且进一步的促进他们的一些转化。比如说从非付费用户向付费用户的转化，比如说从一个低频用户向一个高频用户的转化，因为他的习惯也作为他基本上是老用户，那用户习惯性的这种使用功能，但是他其实并不需要一个过多的引导，以它也不再是一个需要说更多有细节的探索，甚至连更多的思考也没有了。但是我们要做的就是怎么样子他插这种习惯性的操作过程中，尽可能的让他们用得更好，用得更爽，甚至说燃起要付费的欲望。


当然针对这两类用户行为，我们都要去做好对应的行为分析，比如说热点分析，他们更愿意去点大力漏斗分析，每一个界面往里跳，甚至说每一个弹窗都可能会有对应漏斗的一个形成，那我们要看看到底。而其次就是路径的分析，我们要去看看用户他本身在使用我们产品的时候，它的路径是什么样子的？怎么样子去保证，甚至说尽可能的去增加他的一个用户留存。


其次就是路径的优化，用户他可能会有一个任务，比如说用户要下单，那用户下单要经过多少个步骤？他要先搜索进入详情页查看商品，点击添加购物车到购物车调整数量，添加地址，点击下单去跳到支付等等。这样的一个使用路径其实非常长的，我们是否还有可以优化的空间，从而让它可以更快的完成下单？比如说原来我要 5 步，现在我能不能 3 步就完成下单？这样的话机制每一步的流失不变，整体的流失也在变少。当然还有退出分析用户在什么时候离开了我们的PP，他在哪个页面离开了我们APP，他因为什么原因离开了APP？通过这些分析可以一方面挖掘一些badcase，另一方面也能看就是说在他离开的那个页面上，我们是否可以做好相关的引导，让他能够留下来进一步的使用我们的产品。


这些都可以是帮助我们更好的去分析我们的用户，掌握他们的用行为趋势，行为分布，行为路径，从而来确认用户的情绪态度，然后明确我们的产品优化方向，从而让用户可以用得越来越爽，让我们的本身的产品的效益会越来越好。


当然无论是前面的所有的我们的目的还是形成用户画像，因为用户的画像最终才是我们可以很明确的针对这类用户做好相关运营的一个基础。
