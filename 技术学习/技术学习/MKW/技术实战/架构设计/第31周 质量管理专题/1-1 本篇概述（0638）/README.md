很多内容可能在之前，比如说在安全生产，比如说整个发布体系，这些大家都是有了解的。那么在这一篇当中，我们将从管理的视角重新出发，去看一看，如果我们作为一个管理人员，我们应该关注哪些点，我们应该怎么样子把这些事情做到更好里？我们就要转换一个思维，从执行的思维转换为管理的思维，从管理的角度出发，重新去看待这些事物。其次要明确我们做这件事情的目标，因为管理它更多就是怎么样逼近我们想要达成的目标，这样的一个过程当然不可避免的还要抓好我们的执行，并且针对其中发现的问题做好相关的治理，最终保障我们的日常工作可以顺利的执行，那这就是我们管理所需要的事情，但管理除了这些之外，我们还需要做好内外的沟通，去保障我们所有的事情相关的资源，并且去推动整个过程的落地。


然后可以把所有的事项都能够协调到位，并且建立相关的流程体系，去推动整体的事情向更好的方向去发展，最终通过总结复盘，不断的迭代自我，从而让整个事情变得越来越好，这就是整个篇章我们为大家主体讲的内容。我们的目的其实很明确，就是推动事项整体有序，让这个事情能够更好的、更有序的、更高质量的完成，同时将研发质量、维持质量和提高质量的活动构成一个一体的一种有效的这种体系，我们就把它叫做质量管理。在这个篇章中最重要的一个点是我们先要明确出我们的目标，

因为只有了目标，我们才知道我们要朝哪个方向去前进，然后去抓好我们的执行，最终去拿我们的结果。所以定目标、抓制性、拿结果就是在质量管理中的我们最关键的三个点，也是这三个点指导的我们在这个领域内去做好。


当然目标这里我们要求是目标明确，并且能够围绕着数据驱动我们的目标，最终根据数据带来的这些反应和外界行业的变化，我们要主动的去拥抱变化，来明确我们的目标，怎样是一个合适的目标，怎样是一个有效的目标，怎么样是一个能够激励大家向前进军的这样一个目标。


而在执行过程中，我们要做到小步快跑，不断的调节我们自身的方向，通过快速试错来明确我们这个做的事情是不是对的，我的目标是不是需要调整，我们能否拿到其相关的结果。当然在这些前提下，我们都是稳定要大于一切，我们不能够为了不标不顾一切，过于激进，那样反而不为我们带来一些意料之外的灾难。而在结果中，质量架构和质量中台，这都是我们需要去跟他探讨的。最终能够做好相关的工作，让我们的团队，让我们的整体的一个工作变得越来越好，这意味着我们需要稳扎稳打，更体系化的去思考与落地，将所有之前学习的知识在这一章融会贯通，形成一个能够复用在多个团队、在多个人群中都能够拿到结果的这样一个方法论。而在质量管理中，我们也会贯彻这样的一个主思路。

第二就是持续交付体系，持续交付体系是为了能够让我们持续迭代，想步快跑，通过持续交付能够最大程度上提高我们的研发效率，降低我们的迭代周期，从而能够更好地探索出我们的项目应该朝哪个方向去走，我们应该怎么样去做好我们的执行，从而拿到我们的结果。第三就是产品评测，我们也要去了解其他人的产品，来明确他们哪里做的好了，他们哪里做的不好了，他们做的好了，我们要学习他们做的不好的，我们也要去避免我们犯同样的错误，所谓他深知识可以公育也是这样的道理。而针对灰度发度来说，这是一个大家已经很熟悉的体系了，因为讲过好多次了，但我们灰度发布的目的就是为了快速试错，最小化损失情况下，发现我们发布的程序中是否存在着问题，从而能够快速止损，降低损失。而质量中台它是我们在这些知识之上去思考我们是否能够建设出这样一个灵活、快速的去应对变化的可复用的职场架构，通过中台进行相关的支撑，从而能够帮助我们带来更好的、更复用的效率更高的这样的一个体系架构，从而提高我们的一个质量效率。而通过这些最终的目的还是打造自身的一个质量管理体系，只有适合自己才是最好的。
