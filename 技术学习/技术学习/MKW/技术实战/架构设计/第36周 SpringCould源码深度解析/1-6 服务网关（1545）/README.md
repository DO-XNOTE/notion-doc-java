API 网关的流行是源于近几年来移动应用于互联网系的一兴起，像移动应用互联网 4 的后台服务支持的对象从以前这些单一的 Web 应用扩展到多种使用场景，且每种使用场景对后台服务的要求都不尽相同，这不仅增加了后台服务的现有量，还增加了后台服务的复杂性。


随着这些微服务架构的提出， API 网关它基本上成了微服架构的一个标配的主线。简单来说， API 网关它是一个服务的门面，是系统对外的一个唯一的入口。 a API 网关它封装了系统内部的架构，为每个客户提供定制的API，所有的客户端和消费端都通过统一的网关接入微服务网关层，处理这些所有的非业务功能。


说了这么多，网关有哪些好处？我们这样去比较。对于这些服务数量比较多、复杂度高、规模大的这些业务来说，引入 API 网关还是很有必要的。比如说它可以聚合接口，使得服务对调用的透明，客户端和后端的耦合度可以降低。同时它可以聚合后端的服务，可以主导节省流量、提高性能，提升用户体验。同时它可以提供像安全、流控、过滤、缓存、计费监控等 API 管理的这些功能。
