# 1-5 服务远程调用（1309）

同学们大家好，这一章节我们来介绍服务远程调用，还是分三部分来介绍。首先带大家一起了解一下什么是服务远程调用，然后给大家介绍服务远程调用的一些常用的技术实现框架。最后来给大家演示一下基于 spring Claude open phone 实现的服务远程调用的实例。那么我们先看一下什么是远程服务调用。服务远程调用，也就是 remote procedure core，简称是RPC，在计算机科学中已经存在了很多年时间，但在今天仍然有同学对 RPC 是什么，做什么用存在一些疑问。我们经常在一些技术网站上遇见什么是RPC，如何评价某个 RPC 技术这样的一些问题。现在技术更新比较频繁，隔一段时间就会有一个新的不同形状的 RPC 轮子被发明治疗出来。


大家对 RPC 关注度这么高，一方面是由于微服务风潮带来的热度，另一方面也不得不承认，确实有不少开发者对 RPC 本身它用来解决什么问题，或如何解决这些问题，为什么要解决这样的问题，或多或少存在一些认知的模糊。简单来说， RPC 出现的最初目的就是为了让计算机能够像调用本地方法一样去调用远程方法，不用程序员显示的去编码这些远程调用的一些细节，即无论是调用本地的一些接口函数，还是调用一些远程的接口函数，本质上都是编写调用代码，它的基本逻辑是相同的。比如说两台服务器， a 和 b 一个应用部署在 a 上面，要想调用 b 服务器，让应用提供的函数和方法，他们由于不在同样一个内存空间，是不能直接调用的，这时需要通过服务远程调用应用 RPC 框架的实现来解决。我们在这里面去可以看到，通常一个 RPC 框架它的实现是通过我们的服务方 provider 和消费方 consumer 来关联起来的。单纯说 RPC 的话是不需要注册中心的，通过消费方的 consumer 直接直连掉服务方的provider。


那么这种情况下为什么需要注册中心的实现呢？大家已经清楚了，其实我们在比较复杂的微服务架构下面如何去做这些服务的治理，它通常是需要我们通过这注册中心来完成的，我们现有的这些 RPC 框架，它已经把注册中心紧密的关联起来了。所以说这里面我们通过消费方 consumer 调 provider 的这个过程中，它这里面会涉及到一些代理远程通讯序列化、反序列化的一些技术点，不同的 RPC 实现它的方案会稍有一些不同。


总结来说， RPC 它会隐藏底层的一些通信细节，不需要我们直接处理 SOCKET 通信或者 ICP 的通信。 RPC 它是一个请求响应式模型，客户端发起请求服务端的响应请求，它类似于 RTP 的工作方式。 RPC 在使用形式上，它像调用本地方法一样去调用远程的这些方法。现在我们了解了什么是服务远程调用？下面我们看一下服务远程调用的技术实现，这里面我们主要关注 Java PC 的一些常用技术实现。


在国内 double 使用的是比较多的。 double 它是阿里巴巴公司开源出一个高性能加分布斯服务框架，目前它已经是阿巴奇的一个顶级项目，它是捐献给了 R87 double，它可以通过高性能 RPC 实现服务的输入和输出处，支持一些服务治理，提供了一些控制台界面可以独立使用，也可以和 string 框架进行一些无缝的集成。我们大多情况下使用都是基于跟 string 框架集成起来的。


double 的设计过程中，它采用了微内核架构，基于加 SPI 机制实现一些扩展 double 对分布式服务的调用核心功能都开放出了它。一些扩展点，包括服务调用的负载均衡策略、序列化协议、传输协议等，使用者都可以添加一些自定义的实现。 double 在国内曾经拥有很高的人气，是微服务架构的首选。后来随着 string close 的流行，社区一路停更，最近 double 设计又开始重新活跃。更新后的double，他也发布了 3. 0 的一些版本，并且宣布未来的版本中更多的一些支持值得期待。同时， spring Claude 阿里巴巴里面也把 double 作为我们的一个重要选项集成进来了。


另外一个是GRPC， GRPC 是谷歌开发的一个高性能通用的开源 GRP 框架。 GRPC 它使用 Pod buffer 来定义服务，那么 prod buffer 它也是谷歌开发的一种数据序列化协议，这个协议性能词比较高的，它的压缩和传输效率都非常高，语法也比较简单。另外 GRPC 支持多语言，并且能够基于语言自动生成客户端和服务端的一些功能库。


下面我们看一下shift，它起源于Facebook，和 double 一样，后来被提交给阿帕奇的基金会，将 shift 作为一个开源项目。 Facebook 创造 sift 的目的是为了解决 Facebook 各系统之间大数据上传输的一些通信，以及系统间语言环境不同需要的一些跨平台的问题。 shift 支持的多种编程语言，比如像常用的Java， C + +，Python，PMP， Ruby 等，它可以在多种不同的语言之间进行通讯，像 safe 的需要在一个语言无关的 IDL 文件里面定义数据类型和服务接口，生成用来构建 IPC 客户和服务器所需要的一些代码。
