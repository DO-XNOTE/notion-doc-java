# 1-1 本章概述（0150）

同学们大家好，这章节我们开始学习加库斯源码解析的 spring day 的部分。先介绍一下 spring day 的课程大纲，然后开始 spring day 的框架概述的部分。 spring day 的课程我们也划分了 5 部分，分别是 spring day 的架构设计解析及 spring day 的核心源码解析、 spring day 的英文技巧解析以及 spring date 二次改造。


最后一章就是 spring date 面试体的深度解析，通过这五部分来去深入了解 spring date 源码的一个各个部分。我们这里面会想一下 spring date 究竟是什么呢？spring，对的，它出现的目的是基于 spring 来实现一套数据访问的抽象编程模型，这个抽象的编程模型可以对各种数据库的存储引擎提供相似的一个操作方式，同时仍能保留底层数据存储引擎搁置的特性。比如说我们通过 MySQL 作为 boss 去存储数据，这是我们的关系性数据库，这里面我们可以通过 GP 的方式进行数据库操作。对于 Redis 或依赖色词这种情况下，我们非关心型的数据库，我们这里面也是通过对应的 spring date 依赖色子和 spring day 的 Redis 的操作方式进行它相同方式的操作。 spring date 它使我们的数据访问变得非常简单，这里面包括关系性数据库，非关联数据库，甚至是 map reduce 或基于云的一些数据库都可以进行一些适配解决。


spring day 等，它包括是一系列的项目模块，这里面大多数是 spring day 的官方出品包装的，也有一部分是通过社区的开发者贡献出来的。我们重点学习理解 spring day 的思想如何逐步成为数据访问成统一实现的这个过程。 spring day 的本章概述的内容我们就先介绍到这里，接下来我们一块去学习一下 spring day 的架构设计解析的 spring day 的框架概述部分。
