什么危险？叫窃密瘫痪提权险。所谓窃密就是我这次不光是要提钱，我要偷偷的去查一查银行账号里面我的这个邻居，他到底家里面有多少钱，这就是窃密瘫痪呢？就是我感觉不爽，银行，你老是怪我偷钱，我把你银行搞瘫了，当你们两三天之内没法正常营业，这就是用 DOS 或 dedos 进行瘫痪攻击，提权限也很容易理解。当我尝试什么跟你做交易的时候，我只能查我的内容，但我把权限提升一下，我提升成 root 或者 admin 的权限，这样我整个银行里面任何用户我可以查，可以改，是不是很夸张啊？所以窃密瘫痪提权限也是威胁的主要手段，这上联跟下联合起来，基本上涵盖了所有的威胁。


感受一下到底什么是威胁？其实威胁来源于脆弱性，也就是你有vulnerability，你的系统有脆弱，所以外面的客户或者是什么潜在的黑客，他就可以通过一些威胁手段来攻击你，但没有真正攻击，也许他说，嗯，我可以攻击你，我把攻击透露告诉你，然后问你要 100 万，说你要给我 100 万我就封口费了，我就不攻击你了，我也不把攻击你的手段了公告于天下。这是黑客经常做的事情，所以威胁是没有发生的攻击，但是确确实实它是已经可以攻击你了。


我们攻击树其实就是要形成一棵攻击的树，看一看到底，如果他真的要发起攻击该如何攻击？这样我们在什么还没有发生攻击之前，就通过威胁模型去保护我们的脆弱的这些应用环节、这些系统，这就是攻击术来帮助我们形成威胁模型，威胁模型来防止我们的脆弱性被暴露，我们来看一看怎么来玩转攻击术，要玩转攻击术，要玩转威胁呢。辅彦老师比较喜欢用的是威胁资产脆弱性评估这样一个方法论，这样一个建模方，那这个建模方法的本质是什么？

是从资产开始着手资产，一个企业的资产太多了，我们假设这个企业的 CEO 的邮件就是一个核心资产，因为最近这个企业有一次并购的交易。


所以如果有谁能查到邮件里面 CEO 到底写出了什么样的内容，就能够很好地去预估未来 5 天股市的这个涨跌情况，然后就能够用做空，是吧？或者是什么？或者是大量买进这个股票，这个 CEO 的邮件怎样才能获取到呢？那这就是资产了，要获取这个资产呢？是要有一些特殊的什么信任评级的，比如你是CEO，是不是你就能获取到这个邮件了？还有谁能获取到邮件系统的管理员？甚至于我们后台的什么普通的系统服务器管理员，他们都有可能可以接触到 CEO 的邮件，这就是系统评级，谁能访问这个资产？一旦有了系统评级以后，我们就要考虑了这些人或者说这些相关的角色，他们通过什么样的渠道来访问这个资产，这就是端点识别 CEO 和什么通过手机笔记本这两个渠道来访问他的邮件。那同时它有可能用了一个 163 的什么 POP3MF 的代理，或者是什么火狐 Firefox 的一些 POP3 IMAP 的做邮箱的信息中信。那这个时候它会利用到 POP3 和 IMAP 服务器做中继，这也是一个获取邮件或处理邮件的渠道。

这个矩阵里面我们来看一下左边是什么，列出了很多渠道，顶上是列出了很多角色，而每一个角色对不同渠道有一些处理方法。我们拿出一支笔，好，我们看看 CEO 最长怎么样？他通过手机这个渠道来进行增、删、改、查增很简单，发个邮件删、删除邮件查是查看邮件改，怎么改，通传邮件系统都有一个 recall 功能，也就是回收自己邮件，然后发一封新的 CEO 可以 recall 一下之前这个，比如说交易的这个详细内容，改成一句新的话，然后这个内容会重新发到所有的接收方，然后把原来的邮件自动给覆盖掉，除非你已经看过之前邮件，否则让你直接会收到一封新的邮件，那之前的痕迹就会被抹除。所以在邮箱系统里其实是一样可以进行增删改查。
