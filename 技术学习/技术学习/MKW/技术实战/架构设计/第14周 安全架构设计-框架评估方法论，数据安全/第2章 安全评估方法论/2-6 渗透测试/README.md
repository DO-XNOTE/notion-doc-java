原因是因为我们的渗透测试章节不是用来培养黑客的，而是要来培养真正的白帽。所谓白帽必须什么？有我们的 SEESOL 首席信息官以及 CTO 的合法授权，我们也通常称之为免死金牌。有了这张金牌，你才能够在企业内部进行渗透测试，否则你是非法的，而且是什么是违规的，一旦有了免死金牌，是不是能够任意妄为啊？也不是，我们要时刻记住我们是什么，我们是白帽，不是黑帽，我们要控制我们所有行为的影响。你可以尝试渗透一个系统，但你不能尝试破坏这个系统，或者去偷取一些最核心的机遇。同时我们还要保证什么？我们所有的行为要及时的什么进行报告，因为我们的攻击不是为了攻击，也不是为了快乐，而是为了什么？很好的防御，一旦你成功地渗透到一个系统里面，你应该及时的向上级领导层进行汇报，一旦你成功的植入一个木马，你还是应该向上级汇报，一旦你破坏了一个系统呢？哈哈，不行，你已经变成黑猫了，不能随意破坏细胞。


你可以植入马木马，然后从中去尝试去进行普通的用户提升到什么 root 用户的权限升级，但是你到了 root 用户以后，不能因为如此用户去获取全面的所有信息，也不能用它去获取最机密的信息以及进行系统破坏。这是什么渗透测试基本原则？他聊完原则以后，我相信大家不会成为真正的黑帽，还是保持我们白帽的这个心态，有这样的心态我们就可以教大家套路了。套路其实很简单，是吧？基本套路就三条。

第一条什么先做信息收集？所谓信息收集，通常我们的白帽或者是作为渗透测试的专家，都不是企业的内部人员，很有可能是外面聘请的专家，或者临时雇佣的一些渗透的专家，这个时候你可以采用社会工程，跟企业的安全部门的人聊天、吃饭、打球的过程当中去渗透一些信息，获取一些信息，这些都是允许的，通过社会工程可以携有信息收起。然后你通过一些基本的扫描工具扫描一下这个端口是不是在谷歌上查一查这个公司的域名，然后看一看常用的一些邮件服务器的一些特征，猜测内网的 IP 地址，以及通过一些这种渗透测试的工具来扫描这个企业对外提供的服务和他的服务目录和他的用户目录，从而完成前期的信息收集。


信息收集完以后，下一步就是什么漏洞理由，所有的收集其实就是为了更好地找出漏洞。通常我们所有的漏洞都会在有官方的有一个 CVE list 通用漏洞披露列表里面可以找到最新的，不管是操作系统还是各样各种各样的工具中间件，甚至是一些开发语言，中间的漏洞都会在上面暴露出来。一旦有这些漏洞以后，我们挑0日、1日和 2 日，因为什么？通常每个企业每周都会打补丁的，各种软件也会打补丁，但是前几天可能没来得及打补丁，这个时候我们拿这种漏洞去扫这家企业，然后利用它的零日、一日、二日没有来得及打补丁，用这种快速的生成的漏洞的分析或者是反吸解析的工具进行渗透测试的一些准备工作。一旦准备完成以后，我们就进入了后渗透工具。


当你能用一个很普通的用户登录到客户的系统，或者能够尝试调用他的 API 的时候，我们可以做一些进一步的信息的收起，甚至于我们可以想一些方法来把自己的权限从一个普通用户提升到高级用户，再从高级用户提升到 admin 或者 root 用户，做权限提升，提升完了以后可以安心地植入很多的木马了，这个时候你就可以通过木马很轻松的去获取到整个企业内部的这些信息。然后通常我们为什么是有红蓝两方？我们白帽是作为红方攻击方，那企业内部的安全人员是作为蓝方防御方，所以要我防止防御方找出你的，怎么样呢？你要把所有的这个渗透的日志给清理干净，这就是扫尾，这些行为都是合法的，只要你不要去偷取公司机密信息以及破坏数据信息，这些都是属于白帽的标准行为，做到这些以后，基本上渗透测试就大功告成了。
