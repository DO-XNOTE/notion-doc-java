所以每个角色的都用什么 t 开始？ T1 就是第一个角色， T2 就是第二个角色，讲师就是第一个角色，副言老师就是了。我可以通过我个人的账号，也可以通过我内部的一个令牌来访问我自己的课程。学生们也可以通过什么大家的用户名密码来访问你们已经购买的课程。除此以外，我们这次课程的总指导是张老师，可以飞扬老师本家张指导他可以通过他的用户名、密码或者是令牌来访问他所管理的所有课程内容。同样道理，我们的视频网站都会有什么？有个视频编辑可以登录非线性编辑工具，对视频进行再加工、制作、合成等等。他会用他的用户名密码来登录着整个非线性编辑的平台和工具。除此以外，大部分的这个系统我们都是采用什么媒资库，而且是数据库。不像菲亚老师上次什么架构设计章节聊的是对象存储这种超前的、高级的、新型的这个梅子库，我们慕课假设还是采用传统的数据库来进行一个不管是媒体资产还是后台的元数据的统一管理，这个时候数据库的 DBA 就可以这么任意处理所有课程了吧。

也就是最关键的资产，是不是就是我们的课程视频啊？这个资产老师、学生教学指导、视频编辑及 BA 其实都有权访问，根据大家权限的不同，比如飞扬老师只能访问自己相关的视频，不能访问其他课程的视频，但是我们的视频编辑级别就不一样了。


第二个不同的资产，是吧？是图文MD，图文 m d 也是大家都能访问，但是视频编辑是不会去处理图文 MD 的，所以他不关心图文MD，还有数据媒体的元数据，也就是这个视频的录制时间，谁传上来的，以及是不是已经经过视频边界的非线性处理，有没有边默等等这些信息谁能看到教学指导视频编辑和 DBA 辅研老师看不到，跟大家一样，我也看不到这些元数据。


第四个内容是只有 DBA 才知道的，就是什么数据库的连接串，只有它才能够访问数据库，才能连接数据库。除此以外，跟我们这个视频相关的很多的课程信息，比如到底跟哪个用户对接？跟哪个订单对接，什么用户关联了什么订单才能够访问这个该视频？这样所有的数据库的关联信息是不是只有 DBA 才能够获得？这里是罗列了几个跟媒资有关的这个资产，我们呢？除了这些资产，除了刚刚聊到的一些什么基本的用户权限以外，还有一个很关键的点就是端点识别，所谓端点识别就是访问渠道，但是因为端点的英文单词是Endpoint，所以这个渠道就以 e 开头。

EE 就是第一种渠道，通过外网 https 的形式来访问资源，谁用的比较多啊？飞扬老师还有小伙伴以及教学指导，他们都会用外网形式来访问我们的视频资源。内网谁用的多呢？教学指导和视频编辑是什么？内网访问数据的主力？除此以外， DBA 怎么访问？ DBA 两种方法，一种是直接 SSH 登录数据库服务器，然后跑 MySQL 命令，还有一种是拿一些什么小型的 SQL 登录的或操作的这个工具，然后连接数据库采用什么 TCP 连接的方式登录到数据库里面进行实际的操作，这些都可以，这就是常用的连接方式和端点识别。
