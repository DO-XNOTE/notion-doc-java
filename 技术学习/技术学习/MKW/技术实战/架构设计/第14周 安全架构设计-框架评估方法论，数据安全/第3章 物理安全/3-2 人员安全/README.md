这一章节我们就来聊人员安全，最关键的一个安全吧。说到人员安全有个潜规则，就是什么人的人身安全是最重要的。举个例子，假设一个数据中心发生了火灾，这个时候你应该是什么关闭设备，还是应该把所有的硬盘数据恢复，或者是磁带数据带出机房呢？都不是，应该是人员首先撤离，你不用去灭火，你也不用去抢救设备，你也不用去抢救数据。人员撤离是所有安全的第一要务。


同样道理，我们的数据中心是有很多道屏蔽门，很多道安全检测设备的，但是一旦发生火灾，几乎所有的数据中心首先第一件事就是关闭所有的安全设备的功能，也就是说蚁人如果想逃出数据中心，是吧？所有设备一路对你敞开，这个时候虽然有黑客、有小偷可以潜入我们的数据中心，但是没有关系，因为什么人员安全才是最重要？这是一条基本原则，那在基本原则之上，其实人员安全还有很多其他的衍生的内容。


那我们跟大家来聊一聊怎么样从员工管理权责分离的角度来实现员工和人员的安全。

首先我们大部分的企业里面都实现什么权责分离、三权分离，比如说你是DBA，对吧？数据管理员，但是你并不是很容易去偷取数据的，为什么？因为你的所有的操作是在一台我们的数据库服务器上，或者是连接到数据库服务器的专有设备上，而网络管理员会限制你的网络流量，防止你把其中的核心数据泄露到外面去。同样道理，我们还有系统管理员，还有开发人员等等，大家三权分立，防止共谋。


除了这个职责分离以外，我们通常还会做一些叫岗位轮转，岗位轮转是验证大家是不是螺丝钉的一个关键方法。比如我们是支付架构师，你在支付这里做了两三年了，已经很熟悉支付系统。也许你在里面埋了一些后门，也许你在里面埋了一些钉子，怎么样防止你利用这个岗位做一些这个人的利益的事情呢？很简单，到了第三年他把你转成了什么交易架构师，这个时候你的接班的小伙伴来承载支付架构师的职责，他会看一看我的前一任到底做了哪些功绩，还有哪些坑留给我。


当他发现你在里面埋后门、埋木马，或者是埋一些安全隐患的时候，你的所有行为很容易就是被审计出来了，得通过岗位轮换，是比较容易审计出前一个同一岗位的这个人员是不是在里面做了一些特殊的手段。除了职责分离、岗位轮换还有一个令大家很羡慕的一个限制方法，叫什么？叫强制休假。有人说 996 每天都在这个忙，每周都在忙，哪里有休假，还强制休假？飞扬老师可经历过有强制休假的公司强制售价是这样说的，他说什么？这是一种企业的福利，比如说有的企业说你工作五年满五年以后，我就给你一个放假一个月的机会，那这个一个月不算在带薪假期，也不算在病假里面，是完完全全独立的一个月，但其实它是一种变相的，对吧？强制休假，在休假期间你是什么会被收回所有权限呢？你不能登录生产系统，你也不能登录开发测试环境，你的账号临时会被关闭。通过这种方法，企业可以排查一下你这个员工是不是利用你的岗位做了一些什么非法违规的事情。


通过一个月的这个关闭期间，如果大家的功能还能正常轮转，证明你的作用是吧？没有发挥到特别大，也没有影响到整个业务的流转。同时也能证明什么，你没有在系统里面埋后门、埋木马、埋只有你才能处理的实际的钉子，通过这种方式能保证大家确实都是螺丝钉。同时也给员工一个感受，真不错，每五年还有多一个月的休假时间，大家是喜欢到这种有强制休假的企业，还是继续在九月的企业里干呢？这个叫什么？大家每个人的想法可不一样，员工管理里面的什么？员工人身安全为先、职责分离、岗位轮换和强制休假以外，其实每个企业还有更多的方法来限制了人工的，或者员工的设备如何接入我们的企业系统？哪些设备可以接入呢？

比较一个经典的例子是，很多企业都说我是很开放的，我同意员工把它的自有设备，也就是 BYOD bring your own device，你自己家的，比如说电脑、手机、 Pad 连入公司的网络，可以收发邮件，可以聊天，可以在钉钉里面互通有无。


也可以什么连到我们的开发测试环境进行开发测试，这个时候有个关键点就是所有权，设备的所有权依然是你，但是上面所有数据的所有权通常就宣称成公司了。另外你的设备必须每周打补丁，装反病毒软件，以及这个设备里面的所有隐私信息有可能都通过一纸铭文的形式归属到公司了。这样你才能把你的BYOD，你的自带设备连入公司的网络，连入公司的数据中心。而通常我们还有企业会做一个更高级的限制，就是数据中心里面的生产中心，大部分时候你的自带设备是无法连接的，只有什么样的设备才能连接到数据中心的生产中心通常是公司的电脑。


公司电脑首先会在 Bison 和底层硬盘来做一次加密，也就是说如果你偷出硬盘做什么磁介质恢复，或者是偷出电脑尝试用各种各样的方法破解它的那个操作系统，都是没有用的，因为在底层 Bison 和什么硬盘层已经做了底层加密，你没有这个密码，你是没法真正恢复出其中的数据的。甚至有些笔记本还具有自毁原则，当 Bison 密码尝试三次、五次以后，它会自毁，防止里面的数据的泄露。


除了这种笔记本电脑的基本加密以外，还有一些高级功能，比如说防止网盘。一旦你在电脑上装一个网盘，它就形成告警，发个邮件给你的经理，或者防微信、防 u 盘等等操作，总体来看就是防止数据的泄露。除了这种自带设备和笔记本电脑以外，还有更夸张的假设，你是做一些核心的通信设备，比如像华为，比如像中兴的这个核心的 5G6G 的通讯设备，或者是什么苹果的新一代手机，这个时候你拿到的不是一台公司电脑，也不允许你把自带设备接入公司网络。你拿到的是什么？右边这种叫做机顶盒，机顶盒上面可以通过 USB 接口去接鼠标、键盘和显示器，但是不好意思，这个机顶盒不是个电脑，它只是个远程的什么Modem，一只猫，它把你的所有的数据流发给了一台什么虚拟机，那个虚拟机才是你的电脑，但是那个虚拟机在哪里啊？在数据中心里面你所使用的只是一个远程终端，也就是说所有你开发的代码，所有你调试出来的弓箭，以及所有你发布的软件，全部都在数据中心内部流转。当你离职或者什么偷偷的把这个猫带走的时候，并没有带走任何一个数据。


大家看到这里有个蓝颜色的 USB 口，当你把 USB 盘或者是一个硬盘连接上这个 USB 口，会有什么样的结果呢？不好意思，因为它是一个Modem，而不是一个实际的服务器，所以你没法从这里直接获取到任何的有效数据。所有的数据依然在远端的硬盘，远端的服务器、远端的数据中心里面，这就是 show 客户机形式来保护数据安全。


我们聊完了电脑、聊完了 show 客户机，聊完了设备安全、人身安全和人员的权责分离等方式。相信大家对什么人员管控、人员安全这一块已经非常了解，下一节就让我们进入另外一个话题，数据中心的安防，来看一看如何实现访问控制和入侵检测的，大家敬请期待。
