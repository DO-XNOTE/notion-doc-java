这里的设备主要是指什么？终端设备？跟人员相关的设备？321，请作答。那自然大家会什么？想一想，前面聊到了人员管理、设备安全管理这些章节，要聊一些具体的什么技术，以及具体的故事。比如说哪些技术权责分离在打仗的时候的虎符，以及我们单位里面，比如说研发岗 SRE 什么运维岗，对吧？我们的 Devops 的，是吧？发布流程岗，不同的岗位之间的什么权责分离也是一种安全手段。除此以外，岗位轮转有时候你会发现自己经常在不同的岗位之间调整，比如有时候是工程师，有时候是架构师，有时候又是产品经理。原因是什么？因为自己太强大了，未必要，有些时候是企业岗位轮换的需要。那除此以外，某些企业令大家非常羡慕的强制休假，突然就多出了一个月的假期。这种强制休假其实也是企业的安全手段。


好，除了这些人员的安全手段以外，我们也要什么？在面试的时候跟我们的面试官详细去聊一聊设备安全，我相信这一块才是面试官更想听到的，比如说我们什么在企业里面允许自己有自带设备，像是自己的手机等等，要 bring your n device BROBYOD，那这些设备该如何进行安全设备扫描？如果设备丢失以后，如何防止其他用户能够登录我们的内部系统？等等，这里面都有很多安全的细节。


除此以外，当公司给你配置资产的时候，你要可以看一看在这个资产里面是不是有很多的什么合规性、扫描性、强迫升级等等的管理要求，这些都是远程接入的设备安全的关键点。好，聊完了两道面试题，其实我们大部分的 Java 工程师未必会碰到此类题目，但是当你向架构师转型的时候，为了考验大家架构师的广度和知识面，有时候不经意之间面试官就会问一问此类问题，那通常希望是什么？不是大家能对答如流，而是能够聊出一些故事，聊出一些笑话，能跟什么我们的面试官产生一部分互动。
