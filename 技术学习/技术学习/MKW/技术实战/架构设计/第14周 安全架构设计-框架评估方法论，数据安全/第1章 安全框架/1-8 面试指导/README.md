你曾经做过一些类似的案例吗？ 32 一起做的。相信作为一个架构市场，你不光是在考虑什么功能性需要，你必然会考虑一些限制，那这些限制有的来自于国家的法律，也有的来自于行业的规范和合规性的要求。这个时候怎么样把一些你知道的法律？怎么样把一些你知道的合规讲清楚，以及聊一下怎么样你去改你的应用，或者说通过你的应用来适应它？不管你是适应网络安全法国内的主要法律，还是像适应 GDP 这种什么欧洲的用户的隐私保护条例，以至于一些行业要求，比如像健康行业要求、支付行业要求等等，他们都有一些具体的条例，你对你的这个实际落地提出一些具体要求，你如何来改造你的落地架构，来实现这些基本的法律合规性要求？从这个层次的横向的来感知一下，我们作为架构师到底是不是足够考虑了限制、考虑了质量、考虑了非功能要求。
