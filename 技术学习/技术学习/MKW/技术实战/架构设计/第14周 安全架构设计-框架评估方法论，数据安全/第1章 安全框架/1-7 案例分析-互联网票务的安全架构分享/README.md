这家企业怎么样规定我们所有的这个网络通信采用白名单，也就是不在我们上面列到的TCP、UDP、ICNP，还有VPN、常用的 Ipsec 的协议范围内的所有其他通讯，全部一律拒绝。那通过这种方式防止了各种五花八门的网络攻击。


好，那第二个网络加强的点是什么？所有的网络访问一律采用token，不能采用普通的用户名密码。所有的这些 API 都是用token，比如像 JOT 这种 token 的防护来进行用户或者服务的授权。那这个时候 token 是有过期时间的，一旦 token 过期以后要采用 refresh token 来重新去刷新token，那这样的 token 才是真正能进行实际 API 访问的。


那通过这种复杂的什么 token 管理机制来实现访问的安全性，那除此以外，每一个服务器，不管你是物理机、虚拟机还是容器，只要你是服务器级别的，这个，嗯，设备它是没有公网地址的，也就没法直接让黑客攻击进来，或者通过公网地址对外进行数据暴露。那如何访问这些服务呢？一律采用负载均衡器，在负载均衡器上我们进一步进行什么？ WAF 安全防御，进行这个流量 DDoS 的安全防控等等。当服务要访问，就像 GCR 这种谷歌的容器的这个仓库的时候，我们什么允许所有的服务器通过 net 地址转换去访问外网？也就是说我依然没有公网地址，我会通过。 net 转换成公司的一个唯一IP。在转换过程当中，我有安全审计，有流量监控，这个时候我再访问公网我就不会了，不经意之间把敏感数据暴露出去，那这就是这叫票务系的网络安全的部分。

这个内容我们会在后面通讯安全环节详细聊一聊，比如说怎么样通过 token 来进行安全保护？怎么样通过HTTPS，怎么样通过 Ipsec 以及负载均衡？ net 等等很多安全的内容，我们会在通讯安全章节详细跟大家聊好，还有什么方法来保护一个票务系统啊？所谓的服务资源是不是要很好地保护起来？比如说我们的什么操作系统或者是容器的基础镜像，这些镜像都要进行安全加固，不能直接从官网上去下一个镜像就可以了，所有镜像都要安全加固，然后一旦这个镜像发布出去，也就是说我们的容器，或者说是我们的操作系统被你们安装完成以后，它就是不可编辑数架构，我们的系统管理员或者是任何我们的其他的这个人员不能够随便在什么系统里面进行配置的管理的修改。一旦你修改完的配置，你必须重新发布成一个独立的新的节点或者一个新的容器。


好，这是操作系统层面的，那除了操作系统层面以外，所有资源和用户访问层面我们要求什么？MFA，所谓 MFA 就是多因素认证，不光认证你知道什么知识，比如说你知道什么密码，也不光认证你拥有什么，也是你拥有一个令牌，或者是硬令牌或者软令牌。也不光验证你是什么，比如你的指纹，你的虹膜，我们需要采用多因素认证，好几个点在综合认证你到底是谁，认证完了以后我们会采用什么 RBAC 角色访问控制来进行相关角色的授权？当所有的这些网络和操作系统镜像都加固完以后，我们进一步会把整个环境进行切分，不管是授权还是操作系统还是网络，会切分成了生产环境跟办公环境，不光是我们内部办公所用，还包含什么？像我们的客服系统，像我们的后台管理系统等等，这些都属于办公环境，它完全独立于单独的生产环境之外，从而保证什么生产环境几乎是很难被变化，很难被破坏，是一个完全独立稳定的系统。


那这里面提到的很多知识点，我们会依次在后续的软件安全章节聊一聊怎么样做镜像加固，怎么样做这个操作系统防病毒、防黑客攻击。以及会在身份安全章节聊一聊什么样是用户认证，什么样是多因素认证，什么样是角色访问控制。那除了这样的技术保护以外，嗯，要做票务就必然要做什么数据的相关的保护，那怎么样去保护数据呢？它整个公司的定义什么？密钥管理作为整个企业最核心数据，比这个机密数据还要机密数据的是密钥，那怎样对密钥进行管理呢？所有密钥都有生命周期，同时对称密钥一律用信封加密保护，在一个信封里面用公私钥进行进一步的管理，就是密钥的保护。

那除了密钥以外，实际的业务数据也分为四层，那最关键的一层叫什么？叫机密层，那在机密层里面包含跟我们整个企业财务相关的核心数据，以及跟用户相关的所有支付卡信息的数据。那在第二层比较重要的层次，我们叫隐私层，在隐私层里面包含 GDPR 里面提到的个人隐私，以及跟卖家、跟买家相关的所有票的相关的票子信息啊。票里面的编码信息、二维码信息，我们也作为隐私在什么相对机密的这个数据存储里面进行保存和保护。


除此以外，数据还有全盘加密或者全链路加密，既加密把数据的传输通道，也加密数据的存储介质，同时不同层次它的加密复杂度还不一样哦。具体的对称加密、非对称加密、信封加密，我们会在什么后续的数据安全章节跟大家聊一聊这些技术是如何实现，如何使用的？好，除了这三个技术以外，要把一个企业的安全加固能力提升，
