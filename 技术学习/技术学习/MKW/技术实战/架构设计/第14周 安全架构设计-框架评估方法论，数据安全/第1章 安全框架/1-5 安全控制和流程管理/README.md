比如说管理可能是一个总控的思路，对不对？威慑的是什么？一旦有这种违反控制行为，比如说有一种非法入侵，这个入侵可能是网络入侵，可能是对机房的非法的强制入侵来偷取我们内部的内容的时候，才是一些威慑手段阻止他。


补偿是指什么？一旦出现危险的或者是风险发生的时候，我们尽量去补偿这个问题，使得风险产生的影响比较小。比如说水灾、火灾，这时候我们进行什么数据中心的切换，切到灾备中心来补偿这个问题，从而实现数据仍然能够流转起来。预防就是什么防患于未来。检测就是最快速度找出问题，然后纠正就是修正问题，恢复是保证业务重新回到正确的轨道上。


这七大自控理论基本上什么制约了我们整个的 it 行为的发展，同时也推动了安全的控制。你一旦 it 行为什么违背了这什么七大控制理论，以后就会产生很多风险，所以他说可以说是一种职业，那他所谓的争抢和帮助就是他保证你的 it 是什么，不光有功能性，而且有非功能上有足够的安全性、稳定性、可靠性等等。所以这个控制理论不光是安全的控制，也是什么可靠性的控制、扩展性的控制、性能的控制。

这个缓解手段通常是什么样？是把这个服务账号给 disable 掉，停用掉，同时把溅出来的虚机或者是什么物理机给快速删除掉。那这缓解完了以后并没有解决实际问题，所以我们首先要在公司内部进行一个通告。所以这个时候你就会发现有一个文件发给所有跟安全相关的部门，包含我们各个什么各个领域的架构师应该都能收到这样一份安全通告。
