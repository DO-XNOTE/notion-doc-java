像你用 excess 一样，你下载一个官方的默认版本，你下载来之后，这样版你也可以使用它，对不对？这个时候它核心功能就是它最小功能集，它就体现在一个东西上面，对吧？然后从核心系统里面讲，你还要注意的就是说我们的一个决策分离可以这么定义，就说如果说你要开发一个系统，但你首先得明白说哪些东西是你最关心的，在核心系统里面要放的，对吧？一般来讲的个人看法是可能一些通用的流程或者定义什么之类的，可能这个是在核心系上面比较正常。


其他的关注点是不是要分离一下，就比如具体的功能、使用细节，这些东西到底是在核心还是在插件？我想不言而喻。更多的更重要的来讲它核心，它实际上这个是一种规范定义，是由它来控制，但是可能更具体的一些业务逻辑或者呈现方式什么的，可能有插件来完成的。这点我相信大家应该比较有体会的。就是说你使用一种插件系统，比如像 clips 一样默认的 Eclipse 就可以跑起来，但是你可以额外的去加载好的东西，对吧？它提供了一些更具体的功能，它作为核心里面的职责，提供一个框架，让你插件能够在上面运营起来。它也定义来说，你要做这种事情，有些什么样的流程要遵循，这才是我们的核心与这个插件的一个值得一个分工，各自的侧重点是不一样的。
