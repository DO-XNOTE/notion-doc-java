当然大家这个见智，操作系统就从本质上来讲就是符合我们对这个 video 的架构的定义之一，对资源的封装。不严格的说，大家甚至可以把操作系统当作一个隔离层，一组API，这些 API 的主要作用就是封装了大部分的硬件层的功能，你可以认为是操作系统提供了一种硬件的抽象，但这种说法不是特别严谨，但这意思的差不多就是这个意思，因为我们也不需要一个严谨的定义在这个地方，毕竟我们只是来学习 video 和架构的，你体会一下这种思想才是最重要的。因为我们并不是去专念细节的，毕竟对不对。


比如作为这个软件的开发者，也就是说 Windows Lydics 上面的一个个的软件，在这种架构下，我们认为它就是插件对不对？最好是通过这些抽象之后的接口来访问硬件所提供的功能或者服务，如果你非得要绕开它也不是不行，但是可能会有一些预期之外的后果。此从这个角度来划分， CPU 就将不同的软件运行在不同的模式下面。当然我们这里是把操作系统也当成了一种软件。一般来讲，操作系统就是将 kernel 的代码运行在 kernel model，也就是在核心模式下，普通的程序就运在 user model，就是我们的用户模式之下，其实所谓的这个红内核或者微内核最大的一个区别就是你定义一下，把哪些代码它是运行在核心模式下，哪些代码是运行在用户模式下，就是这么简单。当然我是简化了它们的区别，我们理解的是它的本质，但是它们的本质区别说白了就是这样子的。


所谓的红内核就相当于说你运行在这个 hero model 下面的这个程序或者代码特别多，如果你是微内核，相对说你这个操作系统它提供的功能就是说核心功能就相对较简化一点，也就运行的这个 kernel model 下面的这个遵循少一些，所以说一个叫红，一个叫灰，就是这个区别，这就是他们的本质。


当然因为我们这个课程它也并不是转这个仓库系统，它，对吧？这里我们就不一一详述这个红内核、微粒核的各自的含有些什么东西，什么之类，这个大家如果感兴趣的可以各自去搜索一下，然后深入地了解一下这个到底是什么，我们这里就不详细的讲解了，对吧？我们重点是通过这种了解操作系统开放式设计来说带来的好处，因为不管是这个红内核还是这个微内核，这种资源封装、隔离和开放接入的设计才是我们本章要关注的重点。
