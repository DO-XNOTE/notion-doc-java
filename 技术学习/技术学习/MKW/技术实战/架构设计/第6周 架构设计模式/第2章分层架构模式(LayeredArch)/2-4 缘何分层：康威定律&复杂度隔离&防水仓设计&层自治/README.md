# 2-4 缘何分层：康威定律&复杂度隔离&防水仓设计&层自治

现在我们已经知道什么是分层架构模式，但是不知道大家有没有想去问一下，为什么我们需要一个分层架构？而且为什么是从古至今我们就有这样分层的一种做法的原因是什么？在开始回答问题之前，我会相信在座的来上课的同学，大部分的应该都是在公司工作的，你有没有了解过自己公司的这个组织架构图？如果说对公司全貌不了解，但你所在的部门有没有去看过？为什么问这个问题？那就是因为这关系到我们的恰当讲的一个定律叫康谓定律。

讲的就是说在任何一个组织，它都有各种原因会形成一种结构，这就是组织的结构。而如果这样的一个组织，它要去开发一个系统，当然系统也会有自己的架构称谓定准描述的就是这样一个事实，系统的架构是受制于组织的沟通结构，简单来说就是组织里面的结构是什么样的，那么该组织开发出来的系统就会有一个类似的架构，也可以说是系统架构，是组织结构在信息世界的投影。

这样一个职责和这样一个软件架构是不是印证了康危定律？我们开篇中讲到的古人对九天的描述和想象， 9 天中的每一天都是有自己的特点和职责分工的，各自处理不同的问题，重新系统来看就是我们定义了不同的层，他们各自要提供的功能就是不一样的。
而从工程的角度来看，我们的系统都是要解决实际的需求，但是业务需求的变化和我们系统的变化速度肯定是不一样的，因为需求总是在改变，我相信在座的诸位作为开发应该深有体会，而系统实现的变化肯定是要花更多的时间，所以通过分层架构，我们可以将业务复杂度和技术实现的复杂度隔离开来。


因为当业务发生改变的时候，和业务离得近的层改变就频繁，而相对隔得远一点的层改变就会少一些，这样整体的开发效率会提升很多，同时因为频繁更改而引起的质量问题也会相对减少。同时在从技术本身来看，在系统被分成以后，因为层与层之间是隔绝的，仅通过接口进行交流，那么理论上每一层要采用什么样的技术栈是完全自由的，虽然会受一些工程原因的制约，但是每一层要选择什么样的实现方案，

什么样的技术栈是完全可以只参考当前层的需要，而无需考虑其他层的实现技术也不受取。在每个层独立选择不同的技术栈之后，也会有另外一柄双刃剑，那就是每一层所选用的技术可能进化的速度是不一样的。


我们举个直观而简单的例子，以移动应用为例，底层通信协议从2G、3G、 4G 再升级到现在， 5G 已经升级换代好几代了，按上层的应用，比如h、t、p、 HTL 等等，它的升级换代反而没有那么频繁。可是你看我们的移动应用一样是飞速发展，为什么？这就是分层架构的功劳，因为分层以后，每一层可以独立变化，理论上最多影响自己的直接上层，而隔得越远影响越小。从 OSI 七层协议我们可以知道应用层和底层物理层隔得有多么的遥远。已升级一个基站，从 4G 到 5G 亏多少？影响我们的移动应用？嫌答案是否定的。也就是说，只有分层技术才可以将系统拥有如此清晰的界限，

简单来说就是分层架构它可以防止错误传播。这句话什么意思？就是如果系统不分层，整个系统是杂糅在一起的，那么任何一个点出现一个错误，就有可能会导致整个系统宕机。而通过分层架构之后，因为层与层之间做的逻辑上的或者硬件上的切割和隔离，那么错误自然就被限制在了错误最初发生的地方，或者顶多在影响上面一层的几个功能，这样一来就不会因为一个小小的错误而导致整体系统宕机。


接着说，假如我们有一个很重要的股票交易系统，它最重要的功能当然是撮合股票交易，当然可能系统中有些微不足道的功能，比如说用户登录时候显示一些欢迎语，假如今天有一个很小的改变，导致这个欢迎语出现了问题，那如果说我们不做这种分层隔离，那有可能这个小小的登录欢迎语就会引起整体的交易系统的崩溃。


但如果我们采用了分层架构，那么显然这种细微的分支功能和最核心的交易系统不会在同一个层里面，无论如何都不会让一个小的功能错误，这会导致核心系统崩溃。也正是因为分层之后，我们降低了每个错误的影响力，将其限制在本层次，这样分支功能手上的层错误是注定无法传播到核心系统所在的层，这就是通过分层架构降低的错误影响力，从而拯救了整体系统。


当然我们这里谈论都是简单的场景，其实后面我会了解到各种什么降级、分流这些技术手段和实现，但是这些具体实现都是基于分层架构而来的，不分层这些技术都是无法派上用场的，也只有将系统分成之后，他们才能实现最初的设计目的。所以大家有没有一点点感觉，就是平时我们听到那些高大上的技术分享，其实也是建立在这些基础的价格模式之上。

根据康危机理，分层也运输了组织架构，那么在每一层都会有负责人的，当需要做出任何决定的时候，这个负责人是否需要被限制其做决定的权限，在大多数的情况下显然是不会的，因为无论是从组织的行政结构还是系统的专业性来讲，这都是不太可能会发生的。比如说 DBA 会对前端 UI 设计做出任何要求，或者产品经理会直接去解决 set issue 吗？这都不太可能会发生，所以每一层都是可以按照自己的意愿和专业来做出独立决定的。最后，陈字还有一个好处，就是陈列的组件只需要知道自己层的知识。这样作为两个好处，第一是从安全的角度来看，当黑客要进攻系统时，会穿过层层阻碍，从不同的地方获取到自己想要的攻击信息。


现在假定整个系统是一个杂糅在一起的系统，那么黑客只要攻破一个点，是不是所有的信息都全部拿到？但是如果我们将系统分层，这样信息就散落在各个层之间，每个层都只拥有部分知识，要利用每一层中获得的信息去工具别的层也是比较困难，因为每个层的信息是非常有限，而且仅限于当前层，这样攻击者无法获得系统全貌，自然也就降低了整体系统信息泄密的风险。此外因为只拥有相对有限的知识，对外界的依赖也就相对减少了，这样自身被影响的可能性也是下降了。同时成自制就是前面讲到这个各自的内倾和它各自的决策性，
