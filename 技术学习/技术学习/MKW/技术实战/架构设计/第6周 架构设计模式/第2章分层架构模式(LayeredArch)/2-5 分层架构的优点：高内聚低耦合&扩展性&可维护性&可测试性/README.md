# 2-5 分层架构的优点：高内聚低耦合&扩展性&可维护性&可测试性

前面我们了解了这个分层架构的各种特征和一些底层技术知识，现在我们来了解一下分层架构的优缺点，其实相信新的同学已经发现了，在前面我们讲了那么多特点之后，市场人文总结出传承架构的这两个主要的优点之一，高内聚、低耦合。

我们先谈谈高内聚。首先什么是高内聚？内聚性就是指在一个系统和组件当中所有的功能都是为了统一的目的而而分配在一起。高乐距就是在这个系统中所有的功能都是相关的，所有的无关功能都不会存在于这个系统当中。那在我们这个分层架构模式当中，你看它每一层的任何变化，它最多影响自身和自身的上一层，显然它是不会干涉到别的地方，那相对来说它们自身的功能特点都是比较接近的。


第二点还是专注于自身的功能，其他层的影响都会被屏蔽，层与层之间的调用也是通过分层隔离，它只会依赖于直接下一层，更下层的东西全都被屏蔽掉了。那这个也是符合 solid 原则中的一个 single responsibility，也就是说每一层或者每一个组件里面，它的职责都是相对单一单调的。

那在每一层里面，我们通过分层把不同的功能聚合在一起，形成一个高位置的系统，这是分层架构的一个重要的优点。


另外一个优点就是  低耦合 解了这个六级系统，看看这个低耦合，所谓的耦合度，就是说两个系统之间有多依赖于对方，如果依赖程度是很高，那表示耦合度高。如果依赖度低，耦合度就是低。


依赖度，适用于衡量系统但灵活性和脆弱度的指标。比如说我们在系统中要做出一些改变是否方便？同时是不是我修改了一个地方，那其他的地方也要做出修改才能适应这种改变。在一个低耦合的系统中，任何的变更都可以很好的被处理，因为其他系统依赖东西很少，所以当被依赖者做出改变时，对它的影响也就很小。这就是低耦合的优点，它更利于系统的各种变更。

所谓这样子，当新的需求变化，只要知道这个点它是发生在哪一层，那我们做出相应的层的改变就可以了，实现了这样一个新的需求，那同时音域它这个是整个层内聚，呈现内聚的好处就是说它利于横向扩展。


什么横向扩展？就是说因为你这一点的功能相对较单一，那如果说我要把它复制几份部署到不同的地方去，这样子也是非常容易的，这就是说易于横向扩展。从我们这个系统的 scalability 来讲，当然同时另外一种就是纵向扩展，就是说所谓的更强更好的这样一些机器来支持你的独立部署。比如说你这个机器上部署的这个应用是我们的某一个层，它的作用是收发信息什么之类的，它可能对内存和 CP 的要求不一样。另外一个可能是说对计算密集型的，那所谓 CPU 要好一点，就是这样一个意思。


最后因为我们每一层是通过接口来互相交流的，所以说它成与成之间第一有一个很清晰的边界，第二它是很好的一个 Gecko 沟通的模式，所以说实现细节的修改理论上不但会影响到 Gecko 之间的这个通信和信息交换，这也是符合我们缩略的原则的一个开闭原则。
