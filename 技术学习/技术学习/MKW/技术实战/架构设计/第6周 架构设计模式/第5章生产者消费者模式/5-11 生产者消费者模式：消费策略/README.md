# 5-11 生产者消费者模式：消费策略

核心问题之后，我们再聚焦另外一个问题，就是消费者他如何来消费这个数据，这个问题相对就比较复杂一点，如果从实现层面来看，不同的这个设计它有不同的处理手法，这个我们在后续的分析中可以进一步探讨。这里我们主要从比较通用的角度来看一下应该从哪几个方面来解决问题。就是这个地方我们可能只是点到为止，在后面我们讲如何来做的时候可能会讲得更多一点，这个消费策略就相当于什么呢？我们从几号来讲来看一下。

第一，首先这个消费策略的意思是决定这个消费者了，他如何去消费数据，这个你乍一听好像是个太简单了，但是前面你记得还提过的可能平衡性什么的其实都是特别有关系的，你让一个消费者如何去消费出去是你在中间件层，或者在你的这个容器层你可以做到控制的，对吧？保证它不会浪费太多，保证它也不会闲着，就这样子，很多东西你都要考虑的。


那所以说一般可能会从前几个角度来看问题，第一就是消费的方式，第一就有push，什么push？相当于是从咱们这个容器端或者中间件端把这个消息推送到所有定了这个消息的人，或者这个系统pool，就是相当于说客户端，就是 consumer 自己不停地来问你有没有新数据，有没有新消息，有没有新东西，我要来拿，我要来拿。


这就是两种不同的消费方式，这两者其实行为模式差异很大，而且在系统的资源层面都是各有优缺点的，后面我们详细探讨一下，这里我们只是点到为止，同学们可以在这个过程中思考一下，推和拉它各自有什么好处和坏处，作为一个思考题一样。另外就是这个怎么讲的分发方式就是谈过的，主要你有两种模式，一种是基于topic，这就是一种 PUB SUB 相当于一条消息可以发给多个人去消费，这是一种。还有一种是q，至于 q 的这种就相当于说是一条消息，是一个像点对点一样的，只有一个人能有用，对吧？还有就社会事实上可能还有一种比较复杂的问题或者技术，它要实现起来比较难的事务性能体，这个讲起来就长篇大论，唧唧歪歪的，我们不会作为重点，但是你也要知道这个事情在现阶段很多比较先进的东西，意见当中筛选的链接，我指的是都是可以支持事物的，这就保证你从发送到接收消费这个过程它是可以保证一定的原则性的。


为什么我对你加个字呢？一定就其实你也没影响它，那么一定还是会一定给予失败的，然后做到在分布式系统下面做到一个完全的事务性，其实挺难的，对吧？这个时候我们只是提一下这种事情，所以消费策略就决定这个消费者当怎么去消费这个 q 里面的数值，
