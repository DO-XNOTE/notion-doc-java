# 5-2 本章概述

这个生产者消费者模式它主要用于解决一个什么样的问题？就是在一个体系里面有一种东西，当然这个东西它是一种代称，代指可能是物品，可能是钱，可能是系统里的某一个数值。这个东西它是生产者要拼命的增加它一直往上加，但是消费者拼命的减少它，那么什么样的情况下整个体系才能够取得一个较好的平衡？那么就算是解决了生产者消费者的问题，而同时这种生产东西也消耗东西的授权方案，就引申请一种架构模式，就是生产者消费者模式，也就是我们平常要研究的主题。

下面我们就来看看我们将来大概有哪些主要的内容。首先就是我们前面讲的核心问题，就是一个人或者一系统拼命的增加，另外一个拼命的花这个增加出来的东西，怎么在这二者之间达到一个微妙的平衡，让整个体系运作的比较平稳，既不浪费资源也没有闲置，就这样子，对吧？这就是我们的核心要解决的问题。然后我们主要是从各个方面上来看一下，生产者消费者这个模式它是怎么样组成的，有哪些部件，如果要运作的时候，它怎么才能运作得起来？主要是了解一下在这个生产有效过程中参与的几个角色，每一个角色大概的定义是什么呀？然后他们是怎么在一起工作来产生了这样一个生产者消费者模式的化学反应的。


同时我们经过这个前面这个理论的熏陶，主要是给大家讲一讲怎么用生产者消费者模式来设计一个系统，这一点是我们这个架构师课程一直强调的一点，就是要学以致用，尽量把这些解决问题的能力和这个设计模式能够结合在一起。各位听课的同学们能够理论联系实际，同时和自己身边的一些课题，或者说一些曾经遇到过系统难题，看能不能结合在一起得出一个答案。当然就是说我们也会分享很多经典的这个应用的架构模式，跟这个生产者、消费者模式之间这个关联性，有些可能是关联很紧密的，你可能一眼就看出来，有时候相似度非常高，有些人就相对于说你要仔细的用心去分析，才会发现原来它是这个模式，这些作用除了这种就是经典的这个公开市场上案例之外，我们也会找一些就是相当于说在一些公司里面的真实的这个案例。


这个可能就是很多东西都是不对外的，唉，在这个地方咱们也一起分享，来研究一下他用这种方案的好处坏处，为什么要应用它解决什么的一个问题。这种真实案例投射在课程里面，比刚才讲的那种经典应用的更贴地气，因为经典应用一般来讲都是被捧上神坛的，那我们这种接地气的是跟靠近你的真实的工作场景的，所以希望大家都去用心体会一下。
