心有灵犀一点通就是说我们只需要一个眼神，就明白了彼此的衷肠，互相了解了自己和对方想要什么，这个是不是就有一点哎，一点就通的意思？什么意思？我告诉你一点很微小的信息，然后你马上知道我的需求，然后你也知道自己应该采取什么样的行动。


大家是不是追女孩子的时候有这个很特别的体会，给你一个眼神，你自己去体会一下，对不对？这个就是相对说，我告诉你一点点小小的信息，你不要管我今天是什么心情，我想要什么呀？我是为什么生气你就不要去管，但是你就要拿出一个对的答案来让我感到满意，这就是什么？这就是我们今天要讲的主题，事件驱动架构。

就是 event driven Architecture，而是一种全新的架构模式，跟我们的分成架构模式是完全不同的。从这个字面上大家可以大概看出来它的意思是什么，由事件来驱动了整个系统的运转，至于说事件是什么意思，它怎么又来驱动了整个系统的运转，那么这就是这个市井系统架构最重要的养成含义，今天我们会在后面的章节中一一把它进行分析，掰开了、揉碎了和大家来一一的阐述一下各种的运作原理，他朋友的设计思想。希望大家通过这个实验学生架构的各种细节和例子，能够把实验学生架构想要表达的中心思想、核心理念理解清楚，并且能够运到你的工作当中去。

所以我们的第一部分就是去了解什么是实践与系统架构，学习它的一些理论知识和一些初步的认知，也就是前面讲的它的基础原理、基本内容，这就是第一部分。而第二部分我们去公司中小一些实际中的问题，或者EDA，它本身这个架构被设计出来支出，他为什么要设计？就他要解决什么样的一个问题。在对这个申请学生交给了初步认知以后，这才是我们第二步要做的，就是找到他要解决的问题。也就是说如果在实际的工作中，什么情况下我们能够使用它？因为大家都知道模式也罢，架构一半，它简单地说就是一些套路，但是你怎么去套用这个套路？是什么场景才能让你去使用这些套路？这个就是非常重要的。


这个可能就是我们第2章想要讲究的一个问题，满足系统的一些功能新需求或者非功能新需求，通过某一种特定的架构模式，然后大家都知道任何东西可能是要从多角度的一个方式思考和自己去了解它。因为说如果说你不加深从不同的角度来理解一个新的事物，那么你是不知道它的全面性的。这样一个问题就是说在任何技术带来好处的同时，肯定也会有它的副作用。那么如果我们不知道采用这句技术会带来什么可能的后果的时候，那么你可能就会遇到意料之外的问题，那甚至这个问题可能是非常严重的一个问题。


所以在学习这个新技术的时候，我们除了要去了解它的优点，它更要关注它的缺点是进去的架构，当然也毫不例外，因此我们需要更加全面的去了解一下四种虚空架构的各个方方面面，以便我们在日后的工作中能够运用自如。最后我们学习了这么多理论，在脑子里面对事件群的架构有了大概的印象，那么是不是就意味着我们就已经掌握了？那显然不是这样子的，我们会用一些更多的实际的应用场景，这些场景都不是凭空从脑子里面杜撰出来的，都是我们的真实案例。就是说可能是广大的老师从不同的厂商那里的经验总结、浓缩和这个抽象简化而来的，但是对我们理解任何一个模式或者架构的场景都是非常有帮助的。


所以希望大家能够通过这些典型的案例，能够把这些知识如何被应用到的，它是为什么会被套用到这个场景下面这些架构模式是为什么？这个都是很重要的，希望大家能够通过这些场景的实战，能够结合理论与实际这一块打，把它理解得更深入，而且能够灵活的运用。前面我们就先说这么多，下面我们就要开始进入技术主题和一些细节。
