这个前提一定是我们其实已经在做了一些面对于这些场景的演练了，在演练中我们发现我们某些处理过程其实做得并不好，我们的处理时间比较慢，我们怎么样子去提升。所以这里其实目的就是为了能够更加快速的恢复。


这里其第一就是在紧急应急组织，我们是不是能够做得更加高效，第二就是在问题快速定位问题的恢复时间的缩短，以及在整个的爆炸范围的控制上去进行做好。这里额外增加了一个爆炸范围的控制，是为了我们尽可能的将爆炸的影响控制在一个小范围内。这样子我们恢复也只有恢复，我们需要去恢复，而针对相关影响的团队就不需要再做相关的恢复了。当然还有整个系统的止血恢复，能够在最终止血了之后做到完整的一个恢复。这些都是我们需要去做到的，也意味着我们需要做到争分夺秒，竞争与有效，同时熟练的轻巧。这也是我们为什么需要去做好我们面向于触点时间提升的这样的一个故障演练。而在这个过程里，其实我们要考虑几个点。


第一个点就是多样化，我们要尽可能的去覆盖我们多样化的故障场景，不断的积累我们对于这些多样化场景的熟练度。

经历的越多，我们越熟练，也意味着我们越快，同时还要有我们的准确度，我们可以一下子就找到这个问题，同时快速的修复它。所以多样性、熟练度和准确度三个方面共同的帮助我们提升我们的演练时间。其实关键的词就是熟能生巧，尽可能的让这些经验转转化为下一次执行的效率。


怎么样子熟能生巧呢？第一就是重复，不断的重复。正如慢油翁说的我一乌它为首数耳。通过了不断的重复，我们就能够形成这种肌肉级。当真的线上出现问题的时候，我们就可以快速的去直接有效的直达要害，解决问题。同时也要针对性的去做练习，因为你光无效的重复也是没有意义的，你一定是练习发现问题，解决问题，再进一步的练习，巩固我们解决的效果，最终让我们真的能达成熟能生巧这样一个效果。

通过每一次演练都比上一次更好，来提升我们的演练时间。这也意味着我们要在效果评估中去看看我们做到好的，针对于做到好的去进行复盘记录，将这些信息记录下来，我不看哪些做得好，我们当时为什么做得好，是怎么样子做，所以它才做得好。将它作为一个经验沉淀下来，并且在下一次的这样的一个故障演练中，能够不断的巩固夯实。


而针对我们做的不足的这些地方，我们要去做好相关的复盘分析，来看一看到底我们哪里做的不好，是我们一些没有做对的，比如监控报警没有做好，还是我们的故障的一个分析策略不对，亦或是我们在整个故障恢复过程中没有做好，还是预案没有覆盖全等等，都针对于这些做好朋友的复盘分析进行针对性的提升总结。


这样子在下一次的场景模拟中，我们就能够做到更好。而这些都是为了能够在现场真的出现故障的时候，我们可以有条不紊地快速精准完成故障的恢复。那说了这么多，会发现，无论是面向于知识故障的故障演练，还是面于时间提升的故障演练，我们都只是简单的说了个皮儿，并没有提到真正的一些故障恢复可能故障演练的一些方法方式。这就是我们在下一小节将会为大家引入的一个新的大章节，叫做混沌工程。通过混沌工程来帮助我们做好更多的这样的故障注入，并围绕着故章注入和我们这一大章节为大家讲的恢复的这些手段，执行的手段，最终来做好我们的故障演练。到这里，整个的一个过档大章节就被大家介绍完了，我希望各位同学能够在这中吸收到我们怎么样子快速的完成故障的定位，止血恢复，并且通过故障演练来提升自己的技能。各位未来加工师们，我们就下一节课不见不散了。
