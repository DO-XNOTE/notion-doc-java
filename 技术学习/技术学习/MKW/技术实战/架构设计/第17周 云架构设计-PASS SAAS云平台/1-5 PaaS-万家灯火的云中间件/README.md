首先进入哪里？进入 slb ， slb 负载均衡器也有一道缓存好。再之后可能进入了应用节点或者是反向代理节点。这些前置应用节点或者反向代理节点，本身也有一道缓存的，做了动静分离。后面真正应用的时候，包括我们提供 API 的这些应用，可能在堆内或者堆外都有什么内存的缓存。再之后是什么是独立的缓存服务器，比如像Redis，Memcache， d couch， base 等等。这些是做什么？这些通常是用来保留一些session，一些会话，或者是做一些应用的集中式的缓存的处理。


最后，数据库前面还可以装一层什么数据库缓存，把什么把数据库的读跟写分离开来，而大部分的数据库读都去命中数据库的缓存。只有当什么数据库要进行写的时候才是。什么才把数据库更新，同时把缓存过期，让它重新去刷到最新的数据库内容等等。有很多种不同的实现，很多种不同的方法。总体来看，至少包含以下至 7 层不同的缓存。所以飞扬老师更喜欢把缓存认为是中间件，因为每一道缓存的目的都是为了什么，为了尽量减少最后达到数据库的压力。好，我们就以缓存服务器这一个为例，看一看云上能提供什么样酷炫的套路。以阿里云为例，阿里云提供了tear，非常知名的什么阿里内部的淘系缓存系统也提供了Redis、Memcache、 d 等等。相对来说，按照当前新颖的技术栈以及使用量来看，其实 Redis 是一个很好的什么初期选择。当你 Redis 无法承载特别大的缓存压力的时候，你可以考虑去转型到tail。 Redis 本身它是有什么缓存的落地能力的，持久化能力的。同时 Redis 的并发性也还是可以的。它也支持什么？支持那种哨兵的监测模式，主备模式，也支持集群模式。这些都可以。在什么在语音平台上所见即所得，点一点就能创建出来。

不用大家去了解这个算法了是吧？神经网络不用你去建数据的抽取，也不用你去管理，连数据的建模都不用去考虑，你可以直接拿到一些什么已经建模好的基本的API，直接采用他的什么预测他的分析报告的能力。比如像语音识别，语音合成，很多已经建模好了。如果你说一个很奇怪的语言，比如你说维族语，也许他没建模好，他汉语，什么英语，一些常用语言可能都已经建模好了。这个时候你就需要改造一下它的模型了，对吧？来个一段维族语的输入，你要改造它的模型，可能还是用什么长短时效模型，但是稍微调整一下它的里面的一些基本参数，调整一下前面的什么数据工程的输入，最终我们还能建模出一种适合新型语言的什么语义识别，语言分析能力。
