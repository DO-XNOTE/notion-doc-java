也是最知名的私有云数据中心平台的解决方案供应商。不管是计算资源、存储资源、数据库资源以及很多的解决方案和咨询，基本上都是什么以华为天下或者是以华为老大的这样一套竞争场景。在这种场景里面，华为不管是价格、解决方案的先进性，以及具体技术栈的落地的整个支持程度，都是 number one 首屈一指的。


说到上云，其他的云平台可能都是以云平台的软件对吧？云平台，酷炫的 pass 跟 SaaS 功能为亮点，而华为云另辟蹊径。华为云特点是吧，能跟一个企业很好的集成当前的系统。不管你当前是 on premised car 的私有云，还是很传统的物理机都可以，它可以把你的系统采用不同的解决方案。上云这里列出了 6 大解决方案，我们拿出一支笔，分别来跟大家分享一下。


好，从下往上分别是retire、日落下线。所谓的 retire 就是你原来的系统，其实已经没有意义了。比如原来是一个从纸质的 CRM 到电子化和 CRM 的改造系统。改造系统只是为了当时改造用，改造完了以后，业务就会已经转成一个电子化的系统，你当时的那种过渡系统就作废了，或者成为一种叫 Tech debt 技术债了，这种系统就会下线。所以 retire 就是要勇于把一些老旧的套路给下线。


还有一些系统可能就没法改造，比如你们企业内部用了一套什么嵌入式安全隔离系统，这里面隔离的是某一个，是特殊的东西，甚至于是什么核弹发射系统。这些系统可能不太适合上云是吧？那怎么办？那就 return 仍然放在云下，作为什么保留曲目好。这两种情况其实跟云没有太大关系。所以我们变成黑色跟灰色，我们不看，我们往上看。


上云有 4 个主要的思路。一个思路就是这里叫什么基础设施上云，英文名叫Rehost。还有一个很常见的名称叫 lift and shift。抬起来再放下去，放到哪里去？是从原来的数据中心抬起来，然后放到云上，一模一样的架构完全不变。原来你是物理机，现在可能变成虚拟机。里面的所有应用几乎不改造数据库。如果是 Oracle 还是Oracle，如果是 DB2，还尝试用 DB2 这样的一种。摆放完以后好处是什么？好处是你不用运维机房了。


机房的运维变成由华为云来提供服务。这是一种比较简单的状态。比较推荐的状态是上面 3 种。以 Paas 为例 replatform 我们要用华为云提供的，不管是虚拟机技术容器化技术，或者是中间件技术数据库技术人工智能 AI 技术等，用这些技术替代原有的传统技术，传统工件，传统的物理设备，这样的情况下真正能用好云的功能。所以它是云使能 cloud enable 的一个重要一环。


再往上什么？再往上，就是我们之前聊到的，要把应用变成云原生，变完了以后，你就可以真真正正去什么，在云上发挥它的强大的弹性伸缩能力、解耦能力， CICD 的快速发布能力等等。这里面用红颜色框框标出来的，是付元老师最推荐的，也是华为上云最推荐的一个。最终状态就是把应用重构、组织架构重构、管理流程重构，以及整个部署环境偏容器化、偏解耦化的微服务发布。这是云原生里面最推荐的，叫 RE architect，也是我们架构师应该做的事情。


除此以外，还有一些新服务，比如你是传统的银行，银行里面推出手机银行，这是一个完全新的服务，原来没有，现在有了。这个时候你就可以直接在云上搭。甚至于去买一个 SaaS 服务的一个账号，tenant，一个租赁的账号，你直接用它的服务就好了，你就不用搭，只需要把一部分业务数据放上去，这就是replace。重新摆放，重新建设一个完全新的，直接在云上建出来。


这样的场景其实未必那么多，大部分情况下就是红框框着的，叫 re architect。这只是 6 种场景。华为上云的理论里面是有不同场景、不同目标，不同的解决方案。一部、二部、三部、四部，不同的文档，不同的部门的组织结构，以及应该怎么样去逐步的上云中间的整个项目规划，它全有一套标准的套路的。所以如果你是一个传统企业，你希望是通过原因来改造你，使得你企业更有竞争力。


华为云是一个很好的合作伙伴，他不光是给你一套平台，给你一个 support 服务。很多的上云，很多的咨询，都可以从华为云那里拿到一手的资料，一手的信息。好，聊完了。刚刚说上云的方法论。

从开始项目开始，首先要判断一下你是迁移还是新建。如果你是新建一个项目，你就要看一看你是不是要完全掌控。所谓完全掌控，就是什么制台机器上面所有的，比如任务调度应用，还有一些什么按软件安装一些依赖库，完全是自己可控的，也就是 lift and shift。传统套路如果是这种传统套路，我建议就用 e c s，类似于阿里跟谷歌的 e c s，或者是类似于 AWS 的叫 e c two。一个完全标准的虚机。如果你是另外一个场景，你是要高性能计算，你的任务是瞬时的，但是可能要跑几百几千个，你并不需要这几百几千个机器独自去一个一个的进行处理，去安装一个包，或者做一些基本的网卡配置。


你不希望做这些复杂的流程，你可以用HPC，它是一个什么批处理高并发同步任务的一种操作手段。如果恰好你的应用不一样，你的应用已经完成了微服务改造，是一些 spring boot 的 code 或者 node JS 的一些小code。这个时候你可以选择叫 service stage，这样一个比较偏什么比较偏服务化、容器化的一套解决方案。如果你的应用甚至于都不光是服务化老师，而是无服务气化service。这种情况下，你是被事件驱动，来一个事件，你就回答一个服务，或者完成一个 API 的操作。


这个时候你就可以在华为云里面选 function graph，类似于 AWS 的叫Landa，后面我们会在 AWS 的章节详细来聊。也类似于像微软或者是谷歌的 function 或者是functions。除此以外，有些应用可能是需要有一些什么编排的功能。是有一些容器的管理功能，这个时候就进行选CCE。如果你只是一个很简单的容器，你可以选c、i。这两个分别是类似于 Docker 和 Kubernetes 的容器功能。好，这些是什么新建的服务？如果你是一个迁移的服务，这种情况下，一般情况会考虑一下是否使用容器。如果也是一样，如果需要编排，就是什么c、c、e。类似于 Kubernetes 的。如果不需要编排标准的 Docker 容器，就是c、c、i。当某些场景情况下，其实你并不需要容器化，比如我的整个环境是中间件或者数据库，容器可能并不能带来很多的优势。这个时候你就选择普通的物理机或者虚拟机。


ECS。也有可能。我应用本身就是一个 service 的应用，就是一个小代码，我甚至希望我是完全托管在云平台的 SARS 或者是 pass 之上。我不要去理解那些容器的概念了，我就是个应用，让我帮它跑起来就可以了。这个时候选 service stage 也是一个解决方案。
华为的很多产品是吧，它都会组织成一些标准的这种流图，告诉你我应该如何选择。我选它有什么样的好处。不管是计算存储网络，还是各种各样的互联手段，或者是数据加密手段等等，华为都有相对应的。这些叫选型图，很方便去选择不同的套件，不同的工具来实现或者搭出你的整体架构。是不是感觉我们架构师的工作减轻了很多？这是华为云什么？很多的工程师很多的架构是反复推敲，希望能提供的一种便捷的，同时也是希望能吸引更多的小伙伴们来喝，或者来吃华为什么冰淇淋自助餐。
