我是张飞扬。上一章节我们说了说云发布，这一章节我们来聊一聊云监控。说到监控大概有哪几块？大家心里面想一想，还记得吗？三个关键内容，一个是什么根据时间汇聚的性能信息，APM，还有一个是根据什么用户的 API 调用，或者是交易信息的，什么链路追踪，tracing。最后一个是出错的或者是正确信息或者 debug 信息的，什么日志告警、监控、 login。


这三者还有很多组合对吧？可以是按照时间维度的交易汇聚，可以是按照什么出错维度的什么性能和日志汇聚，也可以是按照交易维度的 tracing 和日志汇聚，亦或是好几个功能结合起来，形成报告，形成告警等等。我们会在什么后面的监控系统设计，由大木老师跟大家详细来交流。

我们看一看它能做什么，它能根据当前的性能和压力，给出一些容量规划的建议。这样我们架构师就不需要花很多时间再怎么做性能测试了。我们根据生产性能可以粗估一下，我们开发测试性能，测试环境应该怎么样调配资源，以及生产和灾备未来应该怎么样调整。


除此以外，有些什么比较资深的架构师是能够根据各种各样的报告进行性能分析的。现在即使你不资深也没有关系，因为什么我们资深的这种 a p m 工具就会给出很多的性能调优、性能问题的解释。他感觉出来说延时应该是什么？某个容器的某一个里面的 CPU 内存分配不够导致的。现在我们比较酷炫的 IPM 工具都已经能给出这些分析报告，减少了很多人工的要求。

一套云监控平台。其实你要搭得像阿里云那么好，很困难，所以我们应该怎么样？我们应该多利用公有云的监控。比如我们是混合云环境，我们是应该在自己的数据机房里搭一套去管理公有云，还是索性用公有云的资源去管理数据机房？大部分选择都是利用公有云现成的监控平台。废话不多说，我们下一节就来实战一下，看看阿里云的监控怎么样，帮助我们来管理容器世界，大家敬请期待。
