很多种不同的套路，比如人工智能算法里面可能是像什么视频或者是什么图片所用的卷积成功神经网络，也有可能是什么我们前面说到文字里面的什么循环的神经网络，亦或是什么长短期的记忆模型等等。这不同的模型有不同的效果，不同的流处理的思路。storm、阿帕奇 streams 有不同的效果，不同的大数据分析引擎，规则引擎也有不同的能力。把这些都能够侃侃而谈。尤其是什么，有一两个内容要聊得很深，一旦有面试，就像这一两个靠对吧，总归能靠过去，以后就能发挥大家的口才了。聊完了，第二道面试题，我们来看一看第三道 minster 容器云该如何选型，如何设计？ 321 请作答。相信看到容器云，脑子里应该蹦出了是吧？不是 Kubernetes 和 Docker 了。如果还是蹦出这两个单词，那 fair 老师这节课就很失败。应该能蹦出好多个SWOT。各种各样的技术栈，各种各样的编排引擎，它的优劣，它的机会，它的风险。跟传统的那些老模型，比如像 Docker 这种传统的老的容器模型，甚至于和虚拟机模型，和什么物理机模型去比较，你能感觉出什么样的场景更适合你的某种业务。
