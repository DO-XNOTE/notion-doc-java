这些容器技术其实大同小异。我们首先就从 Docker 分析起来。我们不看 Docker 的具体实现，我们来看一看 Docker 到底处于什么样的一种地位，以及它能带来什么。 Docker 我们按照架构式套路，用 SWOT 进行分析优势劣势，机会威胁。左上角的优势。里面很明显，它是一个很强大的生态圈。


几乎大部分的小伙伴玩容器，首先就学的是Docker。圣诞圈很完整，环境准备相对简单，一两分钟就能准备好， Docker 引擎就可以跑一个 hello world。同时 Docker 里面推出了一个很强大的镜像管理机制，通过 Docker file 对吧。

我们能制作镜像，同时我们也可以有很多的什么仓库，不管是公有云仓库还是私有云仓库，各种各样的供电仓库都支持把镜像以 image 的方式保存在那里，然后通过 Docker 的命令或者是其他容器编排的命令来拉取这些镜像运行容器。所以镜像胚胎的管理非常强大。另外业界也非常知名，这是 Docker 的优势。当然劣势也很明显，它不是跟某个业务相关的，所以对业务的DIY，对业务的关联性相对较弱。


另外， Docker 本身对于存储、对于网络提出了一些思路，但是其实没有实现，或者没有完完全全做到自动化。这些其实是 Docker 作为一个容器引擎，还有待遇强大的地方，他把这些需要强大的内容放到了编排引擎里面。同时一样的道理， Docker 只是做轻量级的什么容器管理，所以对那些非功能性要求安全性、扩展性、可用性，它其实没有太多的设计。它有什么样的机会。

当前其实我们大概可以看是 7 三开，七成的容器其实都是Docker，所以有可能未来是 9 一开，甚至于是 100 开，江山一筒大有可能。还有一种情况是什么更多的 pass 平台，不光是Kubernetes，还有什么？还有各种各样其他编排引擎，还有各种各样的。这种我们的软件平台，比如像 Ranch 这样的平台都支持Docker，这样的时候 Docker 可能就真正做到什么江山一统了。但是我们也存在什么？右下角危险，因为风口一直在变。


大家是不是已经听说了 Kubernetes 跟 Docker 什么已经决裂了是吧？未来 Kubernetes 的新版本可能不默认支持Docker，而再过半年，一年，也许可不能提斯就不支持 Docker 了。都有这个可能性。所以这种风口的变化，导致很多的机会，也导致很大的威胁。同时我们公有云平台很多的pass，其实是什么？它是有定价权的。


我选 Docker 还是选其他的容器引擎是吧？都可以，我选了以后我就定下来了，定完以后我就什么我就长期的对外推广我这个技术。最后可能 Docker 就会被这些技术所取代。同时新型的技术也可以，什么 copy Docker 的思路，但是不要遵循 Docker 的机制，可以采用一些新型的机制，把一些非功能性增强，所以更多的芳心为爱的技术完全可能取代dog。所以大家不要什么过于钻研 Docker 的细节，还有什么了解容器的生态。
