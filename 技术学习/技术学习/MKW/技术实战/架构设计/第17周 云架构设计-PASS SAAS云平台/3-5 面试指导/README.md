第一道题云平台如何实现应用发布，重点是云平台和发布。 321 请做的。相信大家很容易想到什么，发布过程当中是不是有蓝绿发布，金丝雀发布，还有一些功能开关，通常要结合云平台，我们要可以聊一聊云原生的一些基本的思路，对吧？原则同时在CICD里面我们也要求什么。大部分情况下是采用一些能伸缩的容器化的方式进行发布。这个时候容器化是有很多不同的编排工具的。


你如何对接不同的编排工具，选择什么样的云平台，选择什么样的 CICD 软件？如何实现连续的集成，持续的发布以及持续的交付？这里面发布跟交付还有一点点区别。发布就是什么把代码发布出去，把负载压上去就要发布。交付是压上去以后还有一段时间去观察。通常 AB testing， feature toggle，功能开关，这些都属于持续交付，不是属于持续发布。所以集成发布到交付是一个完整的链路。


根据什么知识回想，根源分析，重新反馈回前面的持续集成。这就是一套完整的发布思路。云平台不出其用，重点是要聊一聊怎么利用好云平台的容器平台，以及云上也有一些 Devops 小工具来帮助我们实现云发布。好。

第二道面试题如何实现并行开发的时候功能的发布和回退？所谓并行开发，一个团队里面有很多个研发人员，甚至于多个团队在开发同一个什么代码功能，这个时候如何处理发布和回退的问题？ 321 请作答。我相信大家都会聊一些代码分支管理，代码的发布管理，还有一些功能开关，这些都是很主流的功能。不管是前面我们聊了云原生，还是这章节我们聊了云发布，其实都会涉及到一些这些内容。但要拿高分，一定要结合实力讲清楚。比如你们公司是什么？是多个分支进行开发，一个分支进行发布，还是一个分支进行开发，多个分支进行发布，它们各有优劣的，不同的场景，不同的敏捷性，有不同的要求，所以大家会选择不同的解决方法。同时发布过程当中一旦出现回退，我们是采用像功能开关这种回退方式，还是不得已必须要让代码回退。是前向回退还是后向回退，也有很多种不同的处理手段。


要结合具体的案例，结合你们公司的实际情况来聊一聊当时你们怎么管理的，你们是怎么思考的。同时你要把你架构师的思想在面试环境当中展现出来。好。

第三道面试题如何实现云平台监控也是一个很宽泛的题目。 321 请做的塑造监控老三样性能的 APM 监控链路的追踪对吧？ tracing 日志监控 login 对吧？老三样。这老三样其实每次都能说出新话筒，你可以结合一个实际的应用案例，聊一聊当时你是怎么样进行技术栈的选型，你是选什么样的语音平台，中间有没有什么定制化？通常有哪几块需要定制化？一个是什么告警和过滤规则，这也是需要定制化的。另外一个就是日志收集的过程当中一些处理的套路。比如我们在数据汇聚的时候，就会什么把几周之前的数据，什么把它清洗一下，把最新的数据留在那里，每周是有一些数据的迭代，甚至于还有数据生命周期的处理，把一些新的数据保存在当前。比如是 open search 里面，把老数据扔到对象存储上，对接 list search，实现老旧数据的历史查询等等。


有很多套路可以玩。这些套路能讲清楚为什么这样做，有什么样的好处，我在当中碰到什么样的坑，这就能体现架构是对监控对于问题调研的一种能力。 QQ 讨论题分享一个曾经碰到过的云平台的问题，我们碰到了一个issue，一个问题，当时是怎么样通过监控工具来进行问题排查的，最后是怎么样把根源找出来的。这个重点是要交流一下，看看大家排查问题的思路。这对于面试也很重要。面试经常会考官要考一下，看看我们选手是不是够聪明，他的思路是不是好。做事情的思路就可以通过排查问题来了解。同时根据你排查问题的思路，也知道你对云平台有多了解，你对各种监控工具有多了解，你对生产系统到底有多了解。好聊完了 3 道面试题和 QQ 讨论题，这一章就到此截止了。下一章开始我们会横向的去比较各个云平台，辅媛老师会带着大家尝试玩更多的平台，大家敬请期待。
