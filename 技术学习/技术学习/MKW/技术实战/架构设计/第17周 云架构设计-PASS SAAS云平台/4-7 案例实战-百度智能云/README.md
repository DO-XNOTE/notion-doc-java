我们点击这里训练模型，我们尝试去创建一个模型，把它训练出来进行预测。点击立即使用。或者你要点击具体的类型，我们做图像分类，来一个图像的识别，我们点击进去，好在左侧有一个具体内容，我们放大看一下。通常任何模型都是怎么样？先去建一个空的模型，我们可以再点击这里。建模型，取上名称。如果是公司，要填公司信息，如果是个人，就是一个人信息。填完以后，你要对这个模型进行训练。所谓训练就是什么针对模型传入大量的图片。比如我们传个 100 张图片，我们认为分类是植物，再传 100 个图片的分类是什么？假的花朵，再传 100 个图片，我们分类是动物等等。你要传大量的图片，我们把不同的分类结果告诉他，他可以有监督的。这种形式。也就是你的分类是你告诉他了，所以他这个结果是可预知的。根据这些可预知的结果进行了模型的训练。等训练完了以后，你就可以把传一组不告诉他答案的内容传上去，让他去把答案算出来。你再把答案告诉他，让他进行比对。这两个答案进行比对以后，他就会成为这样一个结果。它为什么？它会根据你不同的图片生成什么？生成一个result，它到底是一种植物还是一种假的净物，或者是一个动物？根据 result 跟真正的人工识别出来的结果进行匹配，最终实现一个打分准确率是 99. 9 还是 99. 99。


到底有多大的概率能够正确的识别分类？这就是整个模型里面，从创建到训练到校验，最后把这个模型发布出去，对外提供一个 API 服务， API 服务甚至还可以变成离线的，或者是云端协同服务。你甚至还可以把它发布到一个 AI 市场，额外去赚一笔费用。这就是 easy deal 很容易上手的地方。当然了， ECTR 不光有图像，有文本、语音、OCR、视频等等，甚至于还有一些什么特殊的行业解决方案，比如这里列出的零售业解决方案。

这个传感器也许是可以接收空调信息，接收你家里面噪音信息。同时它也许可以控制，比如空调的能力，可以控制你家里面的什么湿度控制器，增湿器等等。通过各种各样的套件来组合，来处理所有的感知，以及具体的物联的操作。通过这样的方式，你可以什么进行一些简单的配置，以及进行智能边缘设备的统一管理。通过这样的形式管理完以后，你还可以进行一个物可视化，也就是把你所有资源的信息，物联网信息打包起来，做成一个统一解决方案，甚至做成一个自动化解决方案。


比较经典的物联网解决方案就是你在电梯上面进行电梯声音监控。电梯不是扶梯，是指高层的电梯。当你听到电梯里面有滋滋的声音的时候，而这个字声音在逐步增强，连续 10 秒有增强的时候，直接触发一个什么小区物业报警，通知电梯维修公司进行电梯维修，另外发一个信号，把电梯停在某个楼层彻底停运，同时把电梯门打开。这样一套系统完全可以通过百度智能云进行全物联网及横向控制，从而完成一个整个社区或者是整个物业的整体把控。
