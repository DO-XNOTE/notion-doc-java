是耦合性由低到高，非直接耦合是最不相关的，一直到内容耦合是紧密相关。就是说你改一处代码，你这个模块改了，那模块必须也得改。为什么？因为它们耦合在一起了，它们像是什么大泥团一样粘在一起了，分不开了。那正是因为这个耦合性由低到高，所有它的模块的独立性也是由强到弱。所谓独立性，就是我可以独立开发、独立发布、独立的什么人员管理。如果是几个模块内容耦合，很可能这几个模块是由同一个部门来管理的，否则我这个部门要做一次改动，我要通知其他所有部门。你们要知道，我改了以后，我有可能对你们业务人造成影响，是非常糟糕的一种情况。
