多线程讲的是这一台系统里面，你在一个进程里面，你如何实现多个不同的线程，他们共享内存，但他们能实现互相之间的 CPU 调度来实现。我们在单节点里面尽量提高并发性，尽量提高吞吐量。所以多线程也是一种什么实现高并发的手段，但它更微观一些。好，那这道题有什么加分项？当然有，碰到这种开放性题，我们必然是希望你能挑一个具体的技术进行深入的展开，比如我们挑多线程，我们可以聊一聊多线程的这个线程池里面到底是怎样配置，得多大才能够比较适合我们高性能、高并发的压力需求。那同时我们可以针对不同的场景，比如说计算密集型，比如说 IO 密集型，那如果是 CPU 是吗？我们有 n 和的时候，是不是很多时候我们通常计算密集型为了减少什么我们的线程的上下文之间的这种调度，那可能是采用 n 加一的方法来配置线程池的大小。一旦碰到 i o 密集型，既然很多线程的等 i o，我们完全可以采用 RNN 这种方式来配置。


我们线程池的大小大家相信在自己的生产环境都有不同的调优参数，也有不同的性能测试的结果。你可以结合自己的实战来聊一聊线程池到底多大更合适，如何高并发更好。一样的道理，你也可以聊一聊分布式和高并发这个两个话题怎么样你实现 x 轴和 z 轴的分布式，顺便的能实现更强的高并发和高性能的。
