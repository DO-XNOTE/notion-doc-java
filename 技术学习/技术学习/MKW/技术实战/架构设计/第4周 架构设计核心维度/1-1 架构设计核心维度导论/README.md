最大的区别就在这里，就在非功能性要求里面的质量。这个要求对于不同的系统，我们会有什么不同的功能性要求，所以很难横向比较两个架构师他的能力，他的这个功力。那怎么样才能看出自身架构师和初级架构师区别呢？通常都是什么？看他怎么样利用当前的企业能够达到的资源，或者说能够找到的这些资源来实现最高的质量，来满足业务的需求。而质量通常就那么几个维度，在这几个维度上常用的手段也是有限的，如何合理的组合这些手段，最后实现高质量满足客户需求，这很难体现架构式的功力。


我们在讲质量维度之前，我们会先聊一聊系统的边际和耦合。之所以要聊这个是因为什么？因为通常一个系统去整体看它的质量太庞大了，我们通常建议把系统先拆分成子系统或者模块，让这些模块和子系统相对来说是高内聚低耦合的。然后对于每一个模块，我们可以对它提出不同的维度上的质量要求，从而实现这些模块最后拼装在一起，而形成我们的整个业务系统。


那具体在聊这个五个核心维度的过程当中，我们会埋一些伏笔，为之后的像分布式、微幅响应式，还有网格化等等架构设计的内容做好铺垫。我们就来详细看一看我们是怎么样去聊耦合边界，怎么样去聊这五大核心维度的。
