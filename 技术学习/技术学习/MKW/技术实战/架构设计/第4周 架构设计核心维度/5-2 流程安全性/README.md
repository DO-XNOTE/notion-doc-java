那这一节我们来看一看如何从流程上来保证安全的。我们会以非常快的节奏来带大家了解一些关键词。首先要看一下安全的基本原则，三大原则。第一大原则是可用性，是不是令大家很惊讶？这里的可用性不是描述你本身系统要能防止单点故障，它的可用性是指什么？当用户被 d DOS 攻击的时候，当你的系统被什么 d DOS 进行破坏的时候，你要能保证你的业务仍然是可达的，而不是完全被打瘫掉，这才是安全上的可有性。


那除此以外，安全上还有一个内容叫做完整性，当用户访问我们网站的时候，有可能黑客采用中间人攻击，它可以截获什么用户的数据包，然后进行篡改，然后发给我们的网站，从而去偷取用户的信息。那这个时候我们要验证所有的内容全部来自于客户，而且数据是中间没有被篡改，是完整的，这就是什么安全的完整性的要求？那除此一位最关键的机密性，前面已经通过各大案例反复给大家分享了，一旦一个企业的核心数据暴露到公网上，暴露在社区里面，会导致如何的惨淡的结果，对不对？这三大姓我们取首字母CIA，所以通常我们面试的时候会问安全的 CRA 特性，能给我们讲解讲解吗？这些 CIA 不是什么中央情报局，就是机密性、完整性和可用性。
