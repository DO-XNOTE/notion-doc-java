那我们份组件它底层是如何来发起远程调用的？它这里其实是通过一个动态代理的机制，代理一个接口，由它来执行一个远程的调用。那在这一步当中，其实我们就是为了去构造一个接口的动态代理对象，这样一来咱就可以通过调用这个动态代理的对象，由它在底层帮我们实际的去发起一个远程的方法调用，那在这个动态对象的构造过程当中，实际上如果同学们去看代码，它这底层也是应用了 builder 这种构造器的设计模式。


好，那再往下我们动态代理之下其实还有一个非常重要的一个点，叫做contract，顾名思义 contract 什么？就是协议，就像我们发起网络请求是用 HTTP 协议还是 RPC 协议一样，问它这里底层也有一个问自己的协议，我这套协议规范它解析的实际上不是 HTTP 请求，而是说我们上一层这里构造的一个动态代理的类，我通过去解析这种动态代理接口 contract 的协议，我会构造出非常复杂的原对象，那原对象这里是什么呀？ Meta data，但它包含了你所发起一个网络调用所需要的全部的接口特征，你的请求参数，你的访问的 target URL 地址等等这些所有发起服务请求所需要的元素。后面老师还会带同学们一起去了解一下这个 contract 协议的三级结构，那顶层是contract，再下面 base contract 以及 spring MVC contract。
