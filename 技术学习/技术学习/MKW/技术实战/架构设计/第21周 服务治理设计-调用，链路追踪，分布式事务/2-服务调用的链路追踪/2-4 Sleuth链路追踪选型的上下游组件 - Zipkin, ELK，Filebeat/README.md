那另一个层面来说，我们可以利用这个功能来去做一些性能上面的分析，比如说某次调用可能会卡在某个环节，那么它究竟卡在这个很长的调用链路当中的哪个环节？我们就可以使用 zibkin 来去直接定位，那通过这个柱状图的时间信息来找到花费时间最长的节点，来去进一步的往下去探究问题的解决方案。
