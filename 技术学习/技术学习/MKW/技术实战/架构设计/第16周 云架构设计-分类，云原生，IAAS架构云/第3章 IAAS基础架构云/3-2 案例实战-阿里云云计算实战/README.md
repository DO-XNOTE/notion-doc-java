# 3-2 案例实战-阿里云云计算实战

架起需求到落地的桥梁，构建 it 新蓝图。我是张飞扬，上一章节我们聊了聊云计算的几个关键点，虚拟化、弹性伸缩、负载均衡以及如何采用一些自动化的手段，类似于terraform、Cloudformation、 ROS 等等方式来实现自动化的资源部署。那这一章节我们首先来看一看在阿里云的控制台，如何手工地来完成资源的管理。好，废话不多说，我们切换到阿里云控制台的界面。这是登录阿里云控制台首先看到的一个主界面，这界地面里其实没太多的内容，我们重点内容在左侧，这里列出了大家耳熟能详的所有阿里云最主流的功能。比如 ECS 就是我们的计算服务器、虚拟机，或者叫实例，这个 RDS 是后面我们会介绍的什么云上的关系型数据库。 VPC 就是我们最核心的专有网络，云上的网络系统， OSS 就是什么云平台里面非常火热的对象存储系统。负载均衡就是我们前面介绍过的 SLB CDN，就实现了什么反向代理，实现了内容加速，也实现了 WAF 功能，也是一个非常知名的组件。


这些组件合在一起基本上可以什么？搭建起一个小型的 Iaas 系统了？好，我们这一节重点来看一看 ECS 计算资源，点击 ECS 进到里面，你可以看到什么？飞扬老师其实已经有了四台资源，这四台资源是之前在直播课的时候，为了准备给大家看一看服务网格这样一些亮的酷炫的功能，提前准备的，那具体机器我们可以点到这里实例与镜像当中的实例去看一看他们有很哪几台机器，有一个叫 Docker 的机器用来跑什么？跑客户端的压力测试的，还有一个 Kubernetes 集群，包含主节点和普通节点 NODE 节点。
那这样几个节点合在一起可以实现很多的功能，可以做很多的演示，但是不幸的是都被停止了，为什么？当一台机器停止以后，可能什么每个小时只花几角钱或者几分钱？但是它运行起来以后，那菲亚老师的钱就滚得很快了，每天都有几十块的出入账，所以价格就会比较贵。
对，大部分时候大家在做演示，跟着飞阳老师在做实测，学完立马停掉，等到下次需要再重新启动，这样是省钱模式。好，具体怎么省钱，我后面还有很多心得来跟大家分享。这里看一看，我们可以创建一个实例，让大家感受一下当我创建一个实例的时候，我应该怎样去掌控它的安全，掌控它的这个高可用，同时省钱。这里最关键的点，一键购买是什么？是默认模式不省钱，所以我们要用自定义购买来省钱。下面有三大模式，包年包月。好贵啊，每个月就要 300 块钱，不需要按量付费，是不是便宜很多啊？每小时来，当你需要的时候，你就一小时付个 6 毛钱，还是有点贵。
通常在实验环节都采用抢占式，是不是很便宜？一小时只要两角钱或者一角钱。抢占式什么概念？抢占式就是在什么在市场里面去跟别人 PK 价格，当我的这个报价比你便宜的时候，我就能买到这台机器。当我的报价偏高了以后，我就不得不释放我的机器，所以所有抢占式的实力是没有高可用可言的，也就是说通常我们在一周或者一个月的左右时间，有可能这台机器会被别人抢掉的时候，它也不是会把你的数据暴露给它，只是你的机器会被格式化，你的机器会被shutdown，然后被其他人把它的 CPU 内存给抢占掉而已。
所以抢占是没有安全问题，但是有可用性问题。我们在做测试的时候可以选择它生产系统，选择按量付费。如果是性能压测平台或者是真正的业务生产系统，那当然是包年包月可能更加合适了。好，我们这里也是用抢占式，然后选位置，选飞扬老师就近的上海高可用区随机分配。高可用区什么概念？就是一个一个的机房，每个高可用区要求是独立供电、独立制冷、独立安保，也就是我用社会工程就买通安保的人员，我可以偷偷地进入这个可用去记。但是我没法买通了另外一个可用区的安保人员，因为他们互相不认识。对，这就是从管理上，从他的这个系统架构上，从他的供电、供水等等的，这是基本能力上全部进行隔离。所以光上海这个地区有这么多的可用区，可见语音平台对于可用性考虑得有多周到。
好，下面实例里面我们可以选择具体的类型了，通常就选什么，选所有在就可以了，然后在。里面选X86，这个就是我介绍过的什么裸金属服务器，就是物理机，非常昂贵。我们看一看，点完以后看到每小时价格是不是明显提升了？提升百倍，所以大部分时候我们不要用裸金属，用X86，然后通用型里面选一个低端型，两颗 CPU 垃圾内存足够跑普通应用了。好，再往下是一些什么细节的内容？我们滚一滚，这里面设置什么时长，设置什么单价，基本上都可以挑过。

有哪些镜像可选呢？有很多种镜像可选，甚至于还有镜像市场，我们大部分时候选用共镜像，这里最推荐就是阿里的Linux，因为阿里云天生 Linux 就是一个加固版的Centos，如果大家本身就很熟悉Centos，同时希望有阿里云的安全加固，就选这个，然后挑选一个默认的版本，旁边勾上安全加固字样就可以了。
那磁盘这里也有很多选项，要便宜就选什么高效云盘，那这个是最便宜的模式。那贵一点我们也许可以选个 ESSD 盘，后面我们在磁盘章节会跟大家详细聊所有这种什么 ESSD 都是什么。以 SSD 盘为基础，后台实现了分布式，对前端实现展现成快照设备，或者展现成快设备，这是一种展现而已，我们后面会再详细的磁盘章节跟大家具体来聊一下不同的选型，以及它后台的一些机制。这里重点要提一下这个点系统盘和数据盘必须隔离，那数据盘选什么样的数据盘最合适呢？大家回顾一下前面的什么云原生章节选什么样的数据盘没有数据盘最合适？对，因为什么云原生要求我们把所有的数据扔到远端的对象存储扔到远端的Nosql，扔到远端的 SQL 数据库，而不要在本地放任何数据盘。
但是如果不信，比如你跑的是卡夫卡这种应用，它通常是有一个本地数据盘的，它不是一个什么云原生的应用，它是一个中间件系统，那这个时候千万不要和系统盘混在一起，因为什么系统盘的运行可能会影响你的数据的运行，又影响系统的稳定，所以系统跟数据必须要用两个不同的盘，这个时候你可以买一个更大一点的硬盘来跑卡夫卡的日志等等，我们演示就不需要了。
另外 NAS 是一个最糟的选择， NAS 虽然看上去好像是一个外部的文件系统，但是它是一个共享文件系统，就导致你可能出现一些文件锁，导致你从一个什么分布式系统回源到一个集群式系统，所以 NAS 不是一个好选择，我们大部分时候不要选NAS，需要文件系统类型的，这种什么非关系型数据，我们建议统一采用对象存储，扔在远端，每个人可以随意地访问对象存储的 API 来获取对象存储。好快照服务，这是磁盘的套路，我们后面会在磁盘章节跟大家详细来解读。
点击下一步，通常如果小伙伴在点击下一步过程当中出现问题，大部分概率。什么是支付问题？我们需要在个人账号里面存值至少在 100 元以上才能够。什么点到网络这个章节？也就是说在 ECS 的一开始创建虚拟机的时候，它没有限制你，但是第一个页面填完一点，下一步它就会确认你的账户里面有没有 100 块以上的余额。只有了 100 块以上的余额才能够进行下一步的操作哦。
好，这里要注意上阿里云还是得花钱的，那其他云也一样，任何云都要在里面做一些小小的前期投资网络章节，因为后面会在网络里面详细跟大家聊。这里我只聊几个基本概念，每一台机器都必须要有网络，所以这里专有网络 VPC 必须要选一个可以用默认的。通常一个网络底下还有很多的交换机，我们可以随意地选一个交换机，也可以选其中的任何一个啊。选好以后我们看一看公网地址，千万不要。前面我们跟大家已经聊过正常的什么应用访问是什么？所有后台节点是没有公网的，通过 SLB 来访问这些节点，所以尽量把公网地址给勾掉。我们不需要公网地址。好，安全组也是一个网络概念，我们会在后续网络章节来详细跟大家讨论。网卡就是在当前的交换机上面连一个自动分配 IP 地址的网卡，这样就可以实现什么同一个 VPC 里面所有机器的互通了。 IPV6 大部分时候是不需要的，点击下一步，这一步是做一些系统的简单配置。

好，下面 RAM 是吧？是用户访问策略。那这个内容我们可以结合后面对象存储来跟大家聊一下怎么样让资源合理地给某些人才能够访问。这里我们先跳过，我们看下一个分组什么主要是用作计费。通常有什么分组啊？部门分组、什么团队分组、技术分组、账务分组啊。我们这里简单一点， key 就是ENV，那 value 就是什么？就是 DEV 这个环境是个 DEV 环境，是个测试环境，其他的一些资源组部署集我们都跳过啊。
你点击下一步，这里就有所有账单的内容都在这里了，你是抢占式的。然后它的这个镜像是什么？可用区如何？磁盘如何？以及我们刚刚选的网络配置如何？系统的密码告诉你是用了密码模式，但不会把你的密码显示在这里，然后机器的名称以及你的什么这个计费标签等等，最后点击同意云服务的条款，点击创建，就可以在什么 1 ~ 5 分钟左右的时间完成这台机器的创建。
我们回到控制台看一看，应该是有两台机器在创建中了，什么时候能创建完呢？已经创建完成，已经在启动了。是不是很快啊？因为本质上什么？它不是帮你安装操作系统，它是拿你原有的镜像，你指定的那个我们的阿里云 Linux 镜像，然后快速地克隆出两个磁盘，然后通过这两个磁盘作为系统磁盘进行启动就可以了，大概在 1 ~ 5 分钟左右就可以启动完成。然后就可以采用各种方式来登录，因为我们默认是没有公网地址的，所以可能登录方式略有不同，我们后面再跟大家来详细分享。
