我们通过三个特点来跟大家描述。第一个特点叫虚拟化，基本上所有的云平台，它的计算资源主要都是用虚拟的方式给大家提供。它的底层可能是Xen，可能是 KVM 这样一些开源的虚拟化方案，当用户购买资源的时候，你其实是买的是一个虚机，你只要指定内存、CPU、网络带宽就可以了。
你拿到的不是物理机，当然也有例外，比如以阿里云为例，当你需要高性能计算，你需要一些特殊的物理节点的时候，你可以去租赁神龙裸金属，这就是物理机。但是多数时候你租赁的什么都是一个一个的实例，这些实例全是虚拟机，他们通过后面会介绍的 VPN 进行网络互联，同时他们什么不直接暴露在公网上，他们会只有一个内网 IP 地址，通过 VPN 进行串联，当外部服务要访问它的时候，必须要通过负载均衡器。
SLB 可能是 3 层 4 层的网络负载均衡器，也可能是 7 层的 HTTP 负载均衡器，在负载均衡器外面还有可以什么？有 WAF 防御，有 DDoS 防御，有 CDN 网络加速等等等等，但是跟计算资源直接相关的恰下就是SLB，同时 SLB 不是和一台机器相关，而是什么跟一个组相关，通过这个组就能实现自动扩展、弹性伸缩。
我们看看右边这张图里面有什么？有一个组有三台机器，这三台机器还恰好是在什么阿里云的不同的高可用区。所谓高可用区可能认为是一个机房。好了，那三个高可用区也许是在同一个城市，所以它们算成一个什么region，但是本质上来看，当什么出现电的故障、水的故障、地震的时候，通常只会影响一个机房，所以只影响一个高可用区。
我们一组服务通常要架在多个高可用区，同时这组服务可以是少量的节点开始启动，比如这里三个节点，但是当压力变大的时候，我们可能是 30 个节点， 300 个节点，但是对外它们统一称为一个组、一个伸缩组或者一个弹性组。这个组里面有很多个节点，这个组才和负载均衡器息息相关。负载均衡器其实不要去指定某一个节点、某两个节点，而应该和某一个组进行关联。那这个时候组里面的成员一旦进行了动态伸缩，那我负载均衡器的轮询机制，或者说它的这个负载的这个压力机制就会自动地按照一定的规则压到更多的节点。当某几个节点出现崩溃。
出现死亡的时候，负载均衡器可以通过什么健康检查？ health check 发现这个节点崩溃了，然后它的什么寻址机制，或者说它的负载机制会把这个节点从什么轮询列表里进行拿除，或者说从什么 round Robin 或者是 list connection 的列表里进行拿除，从而把压力压在剩余的活跃节点上。这就是虚拟化，这就是负载均衡以及负载均衡和虚拟化配合起来实现的弹性伸缩组。
我们后面就会实战一下，看看怎么样去创建一个组，怎么样去分布一些计算节点，同时把 SLB 跟它关联起来，实现弹性伸缩。那这个实战过程我们会采用图形化界面跟大家边聊边看一看如何去决策性能和价格的折中方案，当然实际的生产环境我们会用图形化界面吗？肯定不会。是不是用命令行啊？

它的特点是什么？易于使用？基本上一个小白如果是登录一个云平台，花个一两周时间就可以很精通如何来创建自动化的脚本，从而实现资源的创建，所以非常容易上手。
另外整个资源的创建是终态声明，也就是你不用记录以前有多少个节点，以前已经创建了多少颗CPU，不用管，只要告诉云平台。我希望是 400 个节点， 1, 000 颗CPU，它就会自动地根据你的最终目标来进行创建，而且你这个目标是吧？是一段小代码，描述性的可能是JSON、 yaml 或者是其他格式的代码。你把你希望最终得到的基础架构用这种描述性语言写清楚就可以了。那整个过程属于叫不可变基础架构的范围，也就是通常基础架构里面资源不会改动，而是通过删除创建最终达到你要的要求。在这个过程当中，因为不同的云环境、不同的项目、不同的什么虚拟化的资源组的组织形式不同，可能会有一些变量，怎么办？我们的基础架构代码都支持传参，也就是你先写一个模板里面有一些什么变量，这个变量在最后可以在实际部署过程当中用传参来实现。同时好几个自动化的基础架构的套路都支持乐高编程。
所谓乐高编程就是拖拉式的，你不用写一些这个很复杂的 yaml 文件，或者是很复杂的 if else 语句，只需要什么在界面里面进行拖拉，它会自动形成技术架构代码，是不是很酷炫？是不是很方便啊？好，当前最主流的三大自动化平台 number one 就是Terraform，几乎所有的云平台都支持 Terraform 自动化。还有另外一种说法就是当你的云平台不支持Terraform，我们就不认为它是一个真正的云。哇，好夸张啊，那飞扬老师其实也是最熟悉的就是Terraform，那另外全球第一大云平台厂商 AWS 主推的叫Cloudformation，它是一套自建的自动化套路。那国内的第一大云平台厂商阿里云推了一套叫RO。 OS Resource Orchestra. Kubernetes 资源编排调度服务类似于Cloudformation，也支持什么？全自动化了所有资源的管理和匹配，三国鼎立就有他们自己的套路了。
