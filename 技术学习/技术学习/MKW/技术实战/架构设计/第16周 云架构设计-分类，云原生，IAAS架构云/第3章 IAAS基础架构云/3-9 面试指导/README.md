我们可以从这几个角度来思考，一个是自动化，比如像我前面说到的，什么好几种编排工具，好几种自动化的这个自动化运维、自动化管理的工具，这些工具怎么帮助我来实现 is 的？另外在网络章节是不是听得好累啊，好难啊？所以网络互通本来就是个难点，安全的隔离又是一个难点。
除此以外，数据的持久化保存也有多种思路，是不是也有一定的难度啊？我们什么越是不懂的，其实就是什么越难的。也就是越要课后花时间，不管是跟小伙伴沟通，不管是查资料，还是跟飞扬老师之间来个问答互动，要搞清楚，面试官问你难点的时候，希望听到的不要是很肤浅的内容，要真的有难度。


我们平时说什么共情，就是我要把我的心情共享给你。难点在我心里面曾经造成过很大的影响，让我好几天吃不好饭，这种难点我攻克了，以后我在面试环节去讲这些难点的时候，我能讲得绘声绘色，就能让面试官共情。哇，是一个什么是肯钻研的架构师，所以一定要有实际案例，讲得很深。比如说自动化。我们要讲出自动化当中的注意点，一些不容易处理的点，比如传参确实有难度，比如说我们在自动化的过程当中容易出现什么删件，容易把数据节点的资源给删除掉，导致一些问题。或者是虚拟机在迁移的时候，我们采用什么自动化？比如像是 terraform cloudformation，有可能会存在一些坑、一些bug，讲得越深入越细，这种宽泛的题目越能拿高分。
