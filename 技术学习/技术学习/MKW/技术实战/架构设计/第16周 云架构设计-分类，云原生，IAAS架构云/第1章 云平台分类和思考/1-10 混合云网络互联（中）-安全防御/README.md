给大家感受一下。好，拿出一支笔，大家首先看一看，通常是不是有一些云安全的基本组件，这些是什么？是很基础的一些功能服务，可以是免费的，也可以是收费的。除此以外，像身份管理数据的安全、访问业务的安全以及一些增值服务都是什么云平台的基本功能？那除了这些基本功能以外，通常云平台还有两大利器，以队列的形式的给大家来展出，上面就是大数据分析，什么防御潜在危险，不管你是什么样的业务形态，不管你是什么样的攻击手段，都有一些特征和模式。那通过大数据分析能够批量的或者说是什么能够以历史为导向来找出所有可能存在的潜在的攻击的隐患。


那跟大数学对应的是什么？是人工智能，对吧？ AI machine learning，机器学习，通过一些建模，一些预测，你也能够防御什么内容的安全，所以云平台基于强大的大数据给予强大的人工智能，能够做更多的是整个系统的态势，整个网络的这种行为习惯、行为变化的一些检测，从而实现云平台的安全。

除此以外，病毒查杀也是一个关键点，既查杀操作系统的病毒，也查杀软件里面什么内容、文档相关的这些病毒的这个攻击。除此以外一些漏洞它能够自动地、智能地进行修复。还有就是如果我们对外提供的是一些网页服务，这些网页服务有可能被黑客进行篡改，所以网页防篡改也是什么系统层的一个基本能力。最后基线是一个概念，这个概念什么意思啊？就是你把一些常规的形态告知这个云平台，以及一些基本的规则告知云平台。云平台会形成一条机械，违反这个常态，违反这些基本模式的，他就认为是有可能是出现一些安全隐患，所以基于这样一条基线可以这么一键进行查找。
跟我们之前很稳定、很安全的这个状态进行比对，从而发现一些安全上面的可能存在的什么 vulnerability 潜在的风险，那根据这些潜在风险我们可以会形成一些什么日志告警，这些告警可能是发邮件，也可能是钉钉等等方式来告知我们潜在的安全隐患，这就是系统层安全，重点在于操作系统在于什么内容？在于我们应用部署的环境的安全。那除了这种系统层安全以外，还有什么安全？网络层安全。

左边一张图也是以阿里云为例，普通用户是左边的这种绿颜色的，当什么黑客或者是什么，或者是僵尸木手，它是操纵大量的肉鸡尝试对我们进行 DDoS 攻击的时候，数据流经过了什么？ CDN 的 DNS 解析，首先打到哪里啊？不打到后台的数据中心，先去打到一个什么阿里云的 DDoS 高防中心，就是高级防御，超强的底料磁防御系统，这套系统把大部分的黑客攻击直接给顶掉，它有大量的这种服务器，成千上万、成万上百万的服务器来顶我们的这个黑客压力。


当实际的真正的用户的业务流转进来的时候，它根据我们的内部规则的指向，或者是指向我们什么数据中心，或者是指向阿里云，或者是指向其他的竞争对手的云平台都可以。所以这样一套高防系统整体来看有什么样的策略呢？首先是什么？ dedos 的防御要全面，不管你是 TCP 红范、 UDP 红范、 ICMP 红范，各种各样的套路都能够防住同时很多的重放攻击，对吧？我们有大量的重放攻击，以及什么同样的包反复重放来进行攻击也能防御。还有一些什么机型包、一些内滴包等等奇奇怪怪的拼装包都能够进行正确的抵挡。


另外 d DOS 的 d 是什么？分布式客户的压力越大，或者说是受击的分布式能力越大。我们对迪道斯高防系统本身要求的服务器数量就要求越多，所以应对大型企业要买什么大量的迪劳斯防压测能力，我们可能是从买几 GB 的能力变成几十GB、几百GB，随着你的付费越多，你能拿到的云平台的 DEDOS 的高防能力也就越强。
