那如果上了云以后，那通常最先上的云叫什么？叫 Iaas 云，这个云其实上起来最简单，为什么呢？因为上面的这些内容依然保持它是什么，把下面的一部分经常出问题的内容统一进行了管理。也许这个 IS 是什么？是我们公司内部的一个团队来支撑的，也许是第三方的公司，比如像百度，比如像阿里，比如像腾讯，它们的云平台来支撑。大家可以选择不同的企业或者不同的环境的is，但是所有 is 都能实现网络存储服务器以及虚拟化的所有的自动化处理以及相关的SLA。我们的服务等级的合同都提供给你，你只要买这个合同，买这个服务器下面所有的虚拟化，所有的服务器，所有的存储和网络，你不需要再特别关注了，你可以更多的时间关注于什么，不同的操作系统的选择，不同的中间件的选择，不同的运行时环境上面跑不同的数据和应用，那这就是IAS，那么基础架构云带给我们的好处在这个 IAS 之上有没有更近一版的能够提供更大的便利性的方案呢？那就是什么 platform as a service 平台云，所谓平台云就是提供了基础架构云的基本功能以外，还多提供三个功能，一个是什么操作系统不需要你管，同时如果上面有容器也不需要你管，统一由它来管。另外所有的中间件包含SQL、no、 SQL 这些什么跟数据有关的集群节点也不需要你来管。
同时我们的运行时环境，比如像 NODE js、Python、 Java 的运行时环境，所有的什么基础的镜像这些内容也都不用你管，你只需要管设计好我表格里面每一个实体、每一个关联，以及如何把数据展现给用户，如何提供服务，如何，什么提供不同的展现界面，如何提高用户的体验？那这些才是什么企业里面核心要开发的内容？所以这是一叫分水岭，分水岭上面是跟什么？跟业务相关的，是跟核心数据相关的，是跟企业的竞争力相关的内容，而下面是更多是通用性的内容，所以 Paas 未来应该说是比 is 是更火，是吧？更容易被大家采纳了一个产品，又或者说是一个云平台，但是很多企业从左边尝试什么一步跨越 10 万年，对吧？直接到上PS，那这里面有很大的风险，那我就建议，首先你要把企业的虚拟化、标准化、自动化和什么基础架构、运营平台做好，先尝试走一走is，然后再什么经过有一点点的精炼，有一定的精力以后再逐步转型成paas，这未尝是吧？不是一个很好的方案。
那还有一些企业很激进，他把所有的服务全部外包给了一家云平台服务商，那这个云平台服务商通常就是 SAAS 软件及服务，那它就不光是提供底层的什么基础架构，提供它的平台，甚至于把它的数据和应用全部都给你做好，比如说我不想管支付相关的事情，那支付太麻烦，我把我所有的支付业务全部扔给一个支付网关，不光是我的用户的支付信息，还是中间的交易相关的这种支付的这种扣款形式。
以及所有跟什么用户、跟第三方、跟审计所有的相关的事情全部交由这个第三方这个 SaaS 平台来做。那我只需要搞我的电商什么营销，对吧？这是我的强项支付，不是我的强项，就完全可以把整套，是吧？整个词域、整个边界上下文完全剥离给一家第三方来做，可以是定制化的，也可以是买它的标准服务。那这种时候其实我们就采用了SaaS。但是一个企业不可能所有的内容都用SaaS，那如果所有的内容都用SaaS，那就是一个咨询公司了，它的企业里面其实就是一堆销售和咨询，所有的内容全是别的企业在做，它相当于是把别人的企业包装一下，卖一个虚假的解决方案。
这样企业也有，但他们它的存活期不会特别长，它本身其实没有核心竞争力，所以飞老师推荐我，如果我们上瘾，最终目标一个以 Paas 为主体，把跟业务无关的，把跟我们企业无关的，没有竞争力的内容用 SaaS 来提供，比如说用户注册系统，比如说是一些这种非核心业务，比如什么财务系统，如果它不是你的核心业务， HR 系统不是你的核心业务，不妨用 SaaS 来替换。
对于最核心的那些，甚至于是某些中间件，你希望主控的你也可以尝试买 IS 服务，然后掌控其中的中间件部分，尤其是指我们有很强大的系统架构师，我们有很强大的平台团队的公司可以尝试在 IS 和 Paas 之间做一些这个混编和处理。好聊完了，技术的分层，对吧？有IS、 Paas 和SS。后续我们也有三个大的章节，就是围绕如何在 IS 里面实现计算存储网络，如何在 Paas 里面完成中间件以及在用户认证等等的一些基本功能，以及在 SaaS 里面有些什么样的经典的场景，有些什么样经典的行业的解决方案，我们会以这样三条路线，很详细的有单独的章节来和大家聊。

不在讲什么技术分类为主，我们的真正重点在这里，除了技术分类以外，很关键一个点什么呢？是场景分类，哪三种场景呢？一种叫私有运 private cloud，一种叫公有云 public cloud，这两个应该大家很容易理解，私有云就相当于什么？相当于我自建一套我的自己的楼房，我可以去造一个别墅，我也可以造一个小屋，那这个楼房我自己建，所以我有自己的安保系统，还同时我要自己实现什么供水、供电，这是什么？这是老外，特别是很多富豪都喜欢自建，但是像马斯克这样的富豪就不是，对不对？他更喜欢用公有云买一些什么豪华酒店的行政套房或者总统套房，在那里享受酒店的服务。
诶，这是两种极端，但是更多的富豪怎么样呢？富豪选择的叫混合云，怎么样？叫混合云，就是平时它可能是在一个酒店，而这酒店市中心，所以他去什么？去我们的华尔街上班非常方便，但是到了什么？到了几个大了假期的时候，他冬天就飞什么？飞这个迈阿蜜，对吧？夏威夷去这个避寒了，那夏天就可能去五大湖避暑了。那这个时候他会在什么？迈阿密、夏威夷、武大湖都去买几套别墅。对，专门在不同的时候享用不同的感觉。


那这就是很常见的一种思路，就是你的资产既有公有云的，也有私有云的，在不同的时候采用不同的流向实现一个什么数据的连通。但是这里面有很大的问题，不光是什么网络要连你的数据，要同时你的应用谁对外提供服务，谁对外提供什么样的服务？相信大家在前序章节学过了我们的分层架构，对吧？学过了微服务，也学过了单元化，其实心里面是有点感觉的，是吧？不同的层，不同的单元，不同的纵向切割。横向切割其实是可以在两者之间进行一定的混编，但是有哪些难点要注意？同时混合云又能带给我们什么？这就是我希望在后续的好几节里面，几个小节里面能跟大家来仔细的交流的。混合云的详尽内容即将展开，大家敬请期待。
