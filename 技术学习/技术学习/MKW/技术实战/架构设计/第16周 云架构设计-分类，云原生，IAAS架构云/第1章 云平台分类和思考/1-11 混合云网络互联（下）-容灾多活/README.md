给我们引入了这样几个概念， VPC 叫什么？ Virtual private center，也就是云上的网络，或者说是云上的数据中心的一个总体，就叫做一个VPC。
在这样一个云上的网络和云上的数据中心里面还有什么 available zone、高可用区，所谓的高可用区可以这样理解，云平台说白了后台也是数据中心，对吧？那正常我们建一个数据中心只能在一个机房里面，但是在云平台上不一样，我们云平台上建的数据中心就是一个VPC。

这个数据中心的后台必然是在两个不同的机房，一个 available zone 就是一个机房。比如以我们亚马逊的美国中部为例，它有可能是怎么在芝加哥同一个城市，在东部和西部各有一个机房，它就叫什么美国中部 available zone 一和美国中部 available zone 2，这两个机房共同可以创建一些什么子网。
在这些子网里面可以架一些服务器，但这些服务器共同形成了一个什么VPC，我们每次创建 VPC 的时候，通常默认都建议把所有的机器跨越多个不同的后台的机房，实现一个新型的数据中心，这个数据中心能实现很强的高可用。当芝加哥东部的一栋楼整体楼出现了什么崩溃，地震、火灾、断电，不会影响西部楼的，是吧？ VPC 里面所有的节点，所以这样一套虚拟数据中心本身就是一个很强的高可用。

除此以外，虚拟数据中心还可以和我们传统的比如像洛杉矶、芝加哥、纽约和迈艾米的数据中心实现什么网络互联，在网络的三层结构实现 BGP 的互通，也就是你这里的所有的路由的规则全部可以通过一套网络连接方式连到其他的数据中心，实现路由信息的全互通，那这样的互通的关键是什么？是中间那根线，那有好几种方法，一种是什么？打开裸光线自建网络连接，还有一种是租赁比较强大的一些什么网络租赁服务公司的网络。
另外还有一个方法也是很常见的，也或者说是什么中小企业最常见的方法就是租赁电信运营商，比如像国内的电信、移动、联通的普通网络，但是在上面什么搭建VPN，上面安全章节是不是跟大家详细聊过？尤其是什么基于 Ipsec 的 VPN 通过隧道技术能够很好地实现什么点到点或者是什么一堆服务器到一堆服务器之间的加密传输。
那我们也一样在大部分的云平台的混合云实践当中，都是自有数据中心通过 VPN 连到什么公有云，然后把这网络的所有路由协议在 BGP 传输方式进行广播，两边的所有路由策略完全一致，这个时候两个网络相当于是一个大网络了，可以很方便地进行互通。好在网络层已经基本上聊清楚诶，通过这样的网络层，我们就可以实现什么容灾多火，怎么样容灾呢。我们把所有的数据流，比如说都是指向公有云的对外提供服务，但是同时我们可以把这里面的数据通过 VPN 复制一份到洛杉矶。那这个时候当公有云崩溃的时候，是不是洛杉矶还可以对外提供服务啊？那一样的道理？所谓多活，就是什么不光有洛杉矶在火，芝加哥在火，纽约也在火，我们的云上也在火。我们甚至于可以把数据分流比较均衡的打在这四个数据中心，当其中任何一个数据中心崩溃的时候，不影响其他三项中有数据中心的功能。
还有就是单元化，单元化其实前面有单独的章节了，详细聊一聊单元化的什么自循环、单元化的切割，我们可以业务形式态来实现单元化，比如说我们把淘宝放这里，然后我们把天猫放这里，我们把这个蚂蚁欺负的功能放这里，那这是一种思路。还有可以就近原则，所有美国西部的用户数据流全部流向这个洛杉矶的数据中心，所有这个美国中部的全部流向芝加哥，东部的流向纽约，这样也是一种单元化的思路。那还有一个思路也是单元化的一种体现，就是海外扩展，你可以看到这四个数据中心是不是都是属于美国本土啊？怎么样向美国以外的地方进行扩展呢？对于一个美国企业来说，通常推荐就是上 AWS 的什么公有云，然后把公有云和私有云通过刚刚说的VPN，通过刚刚说的 BGP 什么路由协议的互通来实现一个什么海外的扩展。
好，聊完了这样一个网络的基本能力，大家对什么内部系统之间的互通？服务器和服务器之间的互通已经聊得很清楚了，但是容灾和多火不光是服务器，更关键的是什么？是数据库系统，当我云上的数据库系统的数据流怎么样去流转到云下，从而实现数据的双份保障呢？好，我们来看一看下面一张图，基于前面的亚马逊的这套整体概念，阿里、腾讯、华为、百度都是什么如法炮制，实现了云上的 VPC 和云下的 VPN 互通。在此基础上，每一家又提出了自己的数据同步和容灾保护机制。
我们以阿里云为例，阿里云当前用的比较热门的是这种叫 HDM 混合云的数据管理器，这是一个控制器，它可以管理一个叫 DB 的 Gateway 的这个主控节点。下面什么其实是管理多个真正的数据库的网关节点，这些数据库网关节点可以跟中间件系统，可以跟我们应用服务器互通，我们假设我们的所有应用节点全部在这里，我们还在传统的数据库、数据中心里面，传统的私有云里面。然后它可以连接，比如说我们的 DB 网关，通过中间件连接或者是直连都可以通过这个网关以后就可以访问本地，像MySQL、no、SQL，比如像MongoDB、 Redis 等等。
但是如何把这些数据合理的复制到公有云平台？比如说是公有云的RDS，或者是在公有云上自建的数据库，以及如何实现业务的切换呢。好，我们通过下面一张详细的例子，还是基于这张图，我们给出一个更详细的例子来感受一下数据是如何在两个数据中心之间流转的。我们的前提假设是什么？在云平台主控整个数据流，然后当前的所有的业务，不管你是应用还是什么存储，全部在私有的云平台数据中心里面。
