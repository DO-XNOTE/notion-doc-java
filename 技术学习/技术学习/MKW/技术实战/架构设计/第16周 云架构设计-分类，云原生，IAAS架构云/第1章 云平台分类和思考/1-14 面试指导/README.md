思路，就必须要有一个套路，对不对？套路也是一样。先来个战略目标的界定，然后我们可以什么把这个目标细化到一个 SLA 的模板？或者是一套模板之后，我们就开始做架构设计了，那不建议自己凭空来设计，找一些成熟的云平台方案，找些参考架构改没改，改完以后采取一些步骤，比如说是以基础架构为主体走向 SaaS 或者pass，或者是是以什么以私有的环境为主体，逐步向公有云、混合云转型等等。
