架起需求到落地的桥梁，构建 it 新蓝图。我是张飞扬。好了，终于到了飞扬老师的最终章云云，大家是不是很熟悉啊？人云亦云，别人上云，我也上云，对不对？到底什么才是你？我们应该如何上云？你就让飞扬老师来跟大家好好的砍一砍，练一练，站一站吧。好，这个章节整体目标如下，首先大家相信已经学了前面很多的理论性知识，各种各样的方法论，不管是架构设计、维度原则，一些设计模式评估，以及很多的经典的架构设计框架，分布式框架、微服务、网格化、单元化领域、驱动安全等等这么多的方法论，这么多的框架都是指导性的，但什么才是实践性的呢？那付杨老师希望通过云这个章节跟大家实实在在把一些思想，把一些理念落地到真正的应用系统，落地到真正的什么部署环境中。
好，那这就是这个章节的这个棋手势，之后我们会在整个章节里面希望达到这样几个要求，一个要求是有实战，而且实战偏多，整个云章节也是什么飞扬老师所有课程里面实战最多的章节，希望跟大家手把手一点一点来攻克不同的云平台，不同的场景下该如何搭建，如何去设计，应该考量哪些要素？这些其实也是，不管你是在云上进行架构设计还是什么 on premise，就是在自身的系统里面进行这种传统的服务器应用部署、网络打通都需要考量的很多的关键点。


那这些关键点我们尝试在云平台章节里面一起跟大家来讲述，那除此以外，既然说到云，那还有一个很经典的三个字，什么云原生、 cloud native，这 cloud native 有好多公司都很推崇，比如像什么？比如像Pivotal，比如像 Heraco 等等，如何用这样的原理，用这样的思路来把我们的 x a s x 是好几个什么字母的替代品？ AAS 就是 as a service，把什么什么东西当成服务？这是什么？这是云平台里最常见的四个英文字母，好 XAAS 这样云平台来真正落地和实战的，除此以外还有一个目标，就是我们希望云平台、云架构这个章节讲完以后，当你去看后续章节，不管是技术的选型、服务的治理、运维质量保证，都能在云平台找到这些理论和思路的引子。
好有这样一个宏大的云平台的目标，那我们这两周的课程又该如何展开？我们看一看具体的展开的这个思路。首先要聊一聊云平台的定义、价值和分类。当你也在说云，我也在说云的时候，其实我们说的可能不是一样的东西。所以一定要把云平台的定义讲清楚，同时在不同的场景下使用不同的语音平台的分类该如何选择？当我们互相交流的时候如何界定我们俩说的是同一套云，这是一个很关键的作为架构师沟通的点。

什么样的平台呢？语音的认证平台？语音的数据库平台？语音的 no SQL 平台？语音的中间件平台，这些都是什么平台化的服务？把这些服务提供给第三方，提供给客户，提供给我们自己的应用团队，那这都是一套很关键的内容。那除此以外，我们还有最后一个大型实战，就是SAAS，那这个 SAAS 呢？相对来说实战难度有点高，所以我们会以交流为主，顺便给大家感受一下当你实战的时候应该怎么样去考虑。其中会挑几个很经典的场景，也是在云平台上经常出现的场景，那之后我们还会找几个经典的应用，这些应用之所以要在云平台上，就是要利用云平台的一些价值和优势。这个我们会逐一跟大家来交流和沟通，当把三个不同的云平台的分类都实战完，都讲清楚以后，我们会隆重推出一套叫云运维。
大家要说了，我这是架构师课程，是 Java 架构师课程，可不是 SRE 运维工程师课程啊。那我们要跟大家聊一聊在云平台里面其实未必需要什么，很多的运维工程师，大部分情况下底层的运维什么交给云平台本身来完成？谁在运维？云平台当然是我们的开发工程师，你既是开发工程师，也是测试工程师，也是运维工程师，也是向客户经理，向我们真正的客户提供服务保障的所有的一系列的操作人员。
在这样的这种思路下，在 Devops 这种场景下，我们的每一个domain、每一个领域的架构师就成为了云运维、云开发、云测试以及云服务的整体的供应商、整体的负责人。所以我们依然要对云运维有一个很深刻的理解。好，聊完上面这些所有的实战，最后我们会什么？希望通过总体的思路来跟大家分享一下。在云这个章节，我们不光要有一些思考，也不光要有些实战，还要准备好自己的小金库，希望我们这个课程在这个整个播放的时候，大家在看到这个课程的时候，离过年时间并不远，所以我们手头的压岁钱还在，我们需要什么把很大一笔钱投入到某一个云平台。傅阳老师这里举的例子是阿里云，你可以跟着傅扬老师在阿里云上进行实战，当然也可以什么在像什么金山云、百度云、腾讯云、华为云、亚马逊云上面进行施展，都是类似的。那同时付杨老师在所有的云的内容讲完以后，也会带着大家在其他几个云平台一一实战一些经典的内容，那希望有这样的一个课程，跟着大家一起有实战，有理论，只要准备好小金库，只要准备好压岁钱，就能够真真正正在这两周有一个很大的提高。好，废话不多说，就让我们进入后面的具体内容，云平台的思考和分类吧。
