咱这一章节当中的第一小节里，给大家画出了一个巨型项目，它的网关层构建应该有哪些部分？那我们这里去遵循一个从上到下，从一个请求发到DNS，然后再到最终落地到一个微服务这里的整个链路。上面来说，从头到尾可以跟面试官介绍首先 DNS 这里的负载均衡策略，以及我们 DNS 后面指向的这个 IP 它的虚 IP 概念。那虚 IP 这里实际上很多场景下我们都可以使用多级 LVS 加上 Nginx 集群来去做这个搭载，那么这里一个典型的错误，就是说我们认为网关层只是一层， Nginx 负载均衡，一提到这个网关层，我们就说 Nginx 接到外部流量导到内部服务器，实际上在大型应用当中肯定不止一层的。这里通常会搭载多级LVS、 giant Jax，每一级都有自己它独特的使命。

好，那再往后这里才会到我们的微服务网关。



那微服务网关这里的特性我就不再多说了，咱就是前面一个很重要的特点，可以利用服务发现机制来去到咱的服务的注册中心这里去，拉到花名册，拉到所有服务的列表，并且可以随着你的服务注册、服务销毁等等一系列生命周期的变化，把这种服务上下线的操作，这种变化体现在你的网关层面，因此你去添加服务，去添加新的机器，就不需要去修改网关层的任何的代码。


那么它是一个动态变化的服务列表，这个是微服务网关最大的一个作用。但是这两者之间实际上有一个差别，就是说我的微服网关并不是那种可以承载海量应用请求的这样一个网关，它不是一个高性的网关，那这一点相比较 nginx 来说，这个还是要嫩一些的。因此我们直接对外，不要把微服务网关直接对公众开放，我们对公众开放的应该是这样的，多级 LVS 加上 Nginx 的集群可以去承接这些海量流量。


那再到后面这里，我们可以跟面试官去来探讨一下网络分区和单元化的内容，那么网络分区就是我的微服务网关，这里实际上我会分为很多的不同的机房，比如说当前是一个中心机房，下面是一个单元机房等等，那么这些单元机房、中心机房它们之间如何做快速的故障转移？那么这是一点，还有前面章节里杂且有一个单独的单元化的章节，那么在这个章节里学到的应用点都可以去部署实现。


在网关层面，那像蚂蚁金服这里的单元化实践当中，实际上它是这个单元化的，这个重要的这个节点谁来负责单元化？蚂蚁金服这边直接把它应用在了网关层面，在网关层面我的这个请求不出当前单元，OK，那这都是可以跟面试官探讨的，每一个点都可以去往纵深上面给他探讨，这样一来你的这些知识点就由点及面一下给面试官铺开了。OK，**满分回答。**

你在三高应用当中还有哪些应用点？那实际上这个题目咱就在前面的课程当中都给同学们讲过了哪些应用点。第一个限流，这里不光可以在我们具体的业务服务上面，也可以在网关层这里应用限流还有鉴权很多服务，它首先你的请求打到了一个网关，对吧？打到网关之后，我要去对它的当前的这个请求的内容做一个验签，这里就是采用公钥私钥的一种验签。那如果是像国内，如果你做金融系统，那金融系统对这个签名验签的加密规则有详细的限制，比方说你必须用国密算法，这个名字非常好听，这个国密算法叫 ，你要用这个算法来去对你的当前的这个请求进行公私钥的验签。那同时一些网关层这里不光应用了公钥私钥，还使用了 GWT token 这样的一种鉴权的方式。


OK，那这是也可以应用在网关层面的，还有 fill over 故障转移，那么故障转移这里我们就可以体现出一个点， high strikes，这里的降级熔断。OK，这三个应用都是可以作用在网关层面的，但是它跟作用在服务点具体的微服务层面有什么不同？它相当于把这个控制点前置，对吧？网关层是挡在前面的，对吧？天塌下来网关层顶着，那么背后才是这些具体的服务，那么我们这里把控制前置在你的架构设计当中，实际上是一个资源节省的方案，那么你的所有的限流健全 fill over，如果都可以在网关层这里做到，那么就可以保证你的任何请求。在这一层，如果它不符合你的应用规范，鉴权没过，限流被限制了或者熔断降级了，那么它没有任何流量会打到后台的服务。因此我们在架构层面上来说，把当前的这些服务里面的具体功能控制，把把它前置，这个是节省资源的一个方案。


那么我们在探讨架构设计的时候有这样一点，在面试过程当中也不能只说一个方案的好，那么任何架构设计实际上都是一个权衡的过程，你有优点，你一定也有缺点，对吧？就像火影忍者里面这个宇智波又说过一句话，没有缺点的人数，任何人数都存在着一个缺点，那么这里你把控制前置的一个缺点是什么呢？你的控制力度降低了你这边的限流规则，你的鉴权，你的feelover，你无法做到在服务里面做这种非常精细力度的这种控制。那么这样一来你就要做一个balance，哪些比较通用的，比较粗粒度的这种控制权，我可以把它前置到网关层，哪些需要细粒度做控制的？比方说你具体的 fill over，它具体的这些降级熔断的业务代码，那么这些具体的部分，细致的部分我还是会放在具体的服务这里的。OK，那么这就是一个资源节省，还有控制力度的这样一个权衡。
