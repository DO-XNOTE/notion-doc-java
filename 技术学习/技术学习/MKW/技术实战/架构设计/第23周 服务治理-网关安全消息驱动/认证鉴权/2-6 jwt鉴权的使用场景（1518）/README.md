那 subject 不用说主题对吧？audience，当前这个 JWT 我想发给谁？是我的受众用户。后面的EXP，这个是我们 JWT 的过期时间，这个属性很重要， GWT token 一旦发放，它在过期时间到达之前是会一直有效的。如果你说在这段发放的过程当中，你这个 JWT 的令牌，它的加密的 secret 不幸丢失了怎么办？那我想把以前发放的 JWT 全部作废，可以不？不可以的同学们，所以这就是 JWT 的一个特征，也就是说我在 EXPIRATION date 到来之前，我就持续生效。君子一言，驷马难追，不带反悔。


好，那下面这个 NBF 是什么意思？它跟前面的 expiration 其实非常相似，这边是说我在什么时间点之前， GWT 处于一个不可用的状态。OK，那这两个控制 GWT 的生效。下面的一个IAT，它是 JWT 当前的先发实践。这里只是给同学们介绍了一些比较常用的注册声明的字段，我们这里还有其他字段，大家感兴趣的可以去网上搜索一些这相关的资料，比方说JWT，还有他自己的唯一身份标识等等，这都包含在注册声明里。
