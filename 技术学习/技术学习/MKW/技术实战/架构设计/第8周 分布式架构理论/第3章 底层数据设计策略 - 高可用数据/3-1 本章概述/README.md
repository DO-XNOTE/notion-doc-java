在第一节跟同学们探讨一个数据高可用的基石，如何去设计数据备份以及失效转译。失效转移又叫feelover，也就是说你当前数据库在失效的状态下，如何去转移到备库，或者是把从库转成主库。再往后我们来探讨一个数据高可用中一个非常重要的话题，那就是数据迁移。首先这里跟大家去看一下 MySQL 当中基于 binlog 的数据迁移的方案。其实不光MySQL，像 Oracle 之类的数据库也是使用 binlog 做数据同步的。那咱这里了解了 binlog 之后，再跟同学们去看一款阿里系的数据同步专用的黑科技Cornell，使用Canal，我们可以去做数据镜像备份，以及在数据库和缓存之间去做一个数据同步，还可以让你的应用层去消费数据变更，盯着数据库的一举一动作出相应的业务响应。
