冷备是什么呀？咱在前面讲过，是在某一天的某一个时间点做一个冷备份，那么这通常是一个定时的 job 在做的一件事情。那如果我们恢复数据的话，从当前的时间恢复到上一个冷备，这有可能会丢失掉一部分的数据，因此冷备是一个兜底的方案。OK，它不是一个常规操作。好，那除了冷备之外，我们还有一个热备。那热备，是啊，可以通过咱前面学习的 Canal 中间件做一个实时的数据同步做，搭建一个数据镜像，那这都是在主从这种形式之外的一些备份的手段。
