最后这一板斧是关于热点数据的部分，热点数据是做这些高并发的业务场景不得不关注的一件事情，它往往也是我们这些互联网巨头踩坑最多的地方，经常因为热点数据把自己的系统打崩。


那么这里首先跟大家讲一下这个热点数据它有哪些场景，为什么这个数据会被称为热点数据？不是所有的数据都被称作热点，什么数据才是热点？这是我们这一节探讨的内容。那再往后我们跟大家去讨论一下热点数据的隔离方案，把这个害群之马从你的业务场景当中单独隔离出来，单独处理，那具体的隔离方案如何来设计？再往后这里咱的热点数据，它不是生来就是热点，有些平淡无奇的数据有可能在你的业务执行过程当中变成一个突发热点。那我们如何在这个业务执行过程里侦听这种变化？那在这个普通数据变身成为热点的这个变身的过程当中，就把你给封印住，这是我们最后这一节要跟大家聊的内容。好，那这就是我们的三板斧大纲。同学们，我们在正课过程当中再见。
