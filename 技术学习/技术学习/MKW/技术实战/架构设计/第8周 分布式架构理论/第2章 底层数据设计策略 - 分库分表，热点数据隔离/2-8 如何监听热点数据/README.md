比如说对于一些秒杀类的这种商品，我们预先知道它会成为一个热点，或者说有很大可能成为一个热点，那因此可以做很多的准备工作，比如说缓存预热，把你的热点数据预先去加载到你的缓存结构当中。还有我们可以做访问单元的隔离，OK。访问单元隔离是什么概念？比方说我们来看一个电商平台，有时候它的一些大促的会场，或者说是特殊的秒杀的会场，它的网址是不同的，同学们有没有注意到？那么你的用户请求就会被导向一个隔离的单元，单独为你的秒杀活动所设计的这样的一个分区。这样做一个最主要的目的是你热点上的访问压力不要传导到其他正常的节点。


套OK。除此之外，我们最常用的还有一个数据隔离，那前面有提到过，你可以对你的缓存和后台的数据库做一个热点库。okay，用热点库来承接这种海量的读请求。好，那除了这些以外，我们也可以在接口层面做一个隔离访问，控制对这种热点数据的来访请求做一些特殊的业务逻辑。
OK，那这是隔离的几个不同的维度到下方的性能优化，其实说白了就是缓解这种高频的热点访问的压力。好k，最容易想到的是缓存，那这部分内容同学们可以去回顾前面的章节。除了缓存以外，我们还有非常通用的一个手段，降级和熔断。那当你的接口遇到一些访问异常或者超时的情况，我可以对它做降级处理，那引导到一个降级逻辑当中，假如在一段时间以内，你的失败请求的比例占你请求数量的一定百分比，超过这个百分比之后，我可以直接把你打到熔断开关开启的状态。那么后面的来方请求在一定时间内都会导向到降级流程，那我们叫这个过程 vast fail，就是快速失败，可以去迅速的减轻你的访问的压力。


OK，那在后面我们会对降级和熔断做更详细的讲解。好，除了这两个以外，还有一个最有效的手段，在源头就掐灭这个访问压力的手段，那就是限流，根据你接口的 QPS 做分布式的限流等等。OK，那从这个角度来看，其实隔离热点和性能优化并不是最难的部分，那这里都有很多成熟的技术方案可供大家选择，那真正难的是什么呢？哎，是这一步，识别热点，咱前面说了，秒杀场景其实是一个已知的热点数据。OK，对于一些未知的热点数据，我们怎么能在它成为一个热点，或者即将成为一个热点的时候发出一个预警？那让我们上下游的系统都可以把它当做是一个热点数据处理。OK，怎么来做？我们来看业界通行的识别热点的方案是什么？OK，那老师这里依然拿一个下单场景做笔画，但是咱不拿这种秒杀场景，我们可以这样想象，有一款商品，它的卖家非常有钱，给它买了很高的竞价排名，那他突然间就从一个默默无闻的商品打造成了一个爆款。

OK，在这个过程当中自然会有很多的用户来下单，那你的订单中心就会承接很多的来访请求，那在你下单的过程当中，你还会去调用商品中心，因获取商品的信息，那调用营销平台计算优惠以及调用库存中心去减库存、锁定库存。好在这个过程当中，我们如何针对热点数据做一个区别对待？同学们想，我如果想真听到这个热点怎么办？谁可以来告诉你这个信息？这里最好的方式，同学们要记住咱的这个互联网场景，尤其是这种电商类的场景，它是一个漏斗模型。什么是漏斗模型？就是说你这个流量的入口处，它像一个漏斗的瓶口，是最宽的，越往下越窄，到了订单这里流量其实就会变得很窄了，所谓入口处最宽敞，这个入口是哪里，页就是这里搜索页和商品详情页，他们这两个是导流的入口。OK，它这里的流量会导到哪里？它的后台是我们的 RPC 接口，发起远程服务调用的接口，以及 Gateway 咱的网关。


好，那从我们这个入口处的 RPC 和Gateway，同学们想我怎么来对它做手脚？找到一个热点商品。什么是热点商品？访问量大的就是热点商品诶，那我们就可以把这个问题转化出来了，谁调用你的 RPC 接口查询某一个商品，查询的频率越高，它就有可能成为一个热点。好，既然这个方向确定了，我们怎么来做 RPC 接口？怎么来告诉我这个系统当前哪个商品访问最多？谁来告诉你？[同学们在线上查问题要靠什么日志系统？OK，我们这里可以抓取网关层的日志，或者是 RPC 接口打印出来的日志。抓取到日志之后，通过一些 stream 流技术去做一个聚合，分析出来哪些商品的访问频次高](/1ebd41256ba04b8bb9e7d60dab2f79bc)，然后针对这些商品，我们去发布一个热点通知。


OK，热点通知发布给谁，谁需要获得这个通知，谁来订阅它。订单中心下面的商品中心、营销平台、库存中心都可以去订阅这个热点消息。通过这种方式，你就可以在业务流程当中获知哪一个商品是热点商品，从而对它做出不同的业务逻辑。这是一个正向的过程，同学们可以反着推，如果这个商品它从一个热点商品逐渐的热度褪去，变成了一个普通商品，同样的，这里的热点发布就会把它做一个反向的发布，那就是删除热点。


OK，同学们，这就是我们如何去识别热点的一个通用的方案。那我们这一节的内容就到这里结束了。那加上前面几节的内容，我们带同学们熟悉了高并发应用当中非常常见的一个热点数据的问题，以及如何来对症下药解决热点数据。下一章的内容我们依然从数据作为一个切入点，那咱的这个话题将要转向一个更加高大上的话题，那就是[数据的高可用](/39d3da4ac01f4aa0966b70d7cc2b0c10)。OK，同学们，那我们下一小节再见。
