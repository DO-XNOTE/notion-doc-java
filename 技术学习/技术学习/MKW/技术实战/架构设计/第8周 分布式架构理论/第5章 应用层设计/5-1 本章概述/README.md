那咱这里主要探讨缓存在业务层的两个应用场景，一个是全量同步的场景，这里我们去复习一下 Canal 以及布隆过滤器。OK，那接下来是一个时效性缓存，时效缓存，这才是缓存应用最多的一个地方。在业务层设置一个会定时失效过期的缓存，这样的一个时效性缓存，它就会必然牵扯到竞争。比如说当两个线程尝试同时更改一个缓存，这就是一个最简单的竞争方式。


OK，那么在这一节里，我们将针对缓存的时效性场景当中的一些读写过程中的竞争场景跟同学们做一个分析。那最后一节我们这里有一点小代码动手环节，哎，跟同学们一起来看复杂的业务状态如何来设计。什么是复杂业务状态？比如说你买一个订单，从下单到已发货到收货到评价，这就是一个最简单的订单状态的流转。那么我们这里在订单业务的后台可远不止咱看到的这几个状态，它的订单流转的生命周期是个非常复杂的有限状态机。所以我们这里首先跟同学们去讲解一下有限状态机的状态流转的概念，接下来再通过一个实践的Demo，基于 spring 的 state machine 实现一个轻量级的有限状态机的小sample。OK，这就是我们这一章节的课程目录了。同学们，我们正课环节再见。
