它的组成非常的简单，我们来看布隆过滤器是一段 0101 的二进制码来组成的，这个马里面的 0 代表着当前位没有数据，一代表着当前位置是有数据的，它是怎么来应用的？它是如何将一个数据匹配到这一长条数字上面的具体的位置的，这就要靠到散列函数。


我们打个比方，当前我做的这个布隆过滤器，它有三个不同的散类函数，那我们所要查询的数据，它对应的 key 经过这些散列函数，就会通过计算得到一个具体的位数。打个比方，我现在要买一个书包，那这个书包经过三个散列函数之后，它对应的这个 key 会被散列到不同的三个位数。这里我们把它对应到第一位 0 以及第二位一和12345，第五位也是1。当所有位数都为一的时候，那么说明这个商品在缓存当中是存在的，那如果其中有任意一位是0，那么则说明这个商品不存在。


再打一个比方，我又买了一个生日礼物，你看这第二个商品，它所散列到的三个位置都是一，说明这个商品在缓存当中有可能是存在的。细心的同学可能注意到了老师说的一个词，就是说这个商品可能存在，这个可能是什么意思？因为咱的散列函数它其实是一个哈希函数，哈希它有一个特征，对于不同的传入值，那么它有可能会产生同一个哈希，因此两个不同的商品所定义的 key 有一定的概率会散列成同一个数字组合。


那么也就是说布隆过滤器它有一定的误判率，这个误判率是由于哈希函数的特性所天然存在的，但是我们有一定的方法手段去降低它的误判，比如说我让布隆过滤器的存储空间尽可能的大，这样一来你所对应的散列数值就会尽可能的分散，降低你的重复率。


第二个手段，我可以去增加散列函数，我现在有 3 个散列函数，我把它增加到 5 个，这也是可以降低误判率的。但是同学们，考虑到你的散列函数越多，那么你的缓存查询效率相应的也就会越低，这是一个平衡的一个过程。从以上的特性我们可以看出，布隆过滤器是一个全量同步的缓存，也就是说你的所有在数据库当中存在的记录，在缓存中存在的记录，那你都要经过计算，把它预加载到布隆过滤器，那么这里就有一个坑了。同学们，咱既然是全量同步，如果我想删除一个商品，同学们想一想，布隆过滤器可不可以做这个操作？删除操作其实不能，为什么呀？


比如说我当前页面上的书包和第二个商品礼物，它们都共同的散列到了其中 12345 第五个数字一，那如果我想在布隆廓律器当中删除第一个书包这个商品，当我把书包对应的散列位数从一改成 0 的时候，那么我们要考虑到有可能其他商品也会散列到相同的位置，这样一来你的这个删除操作就会影响到其他商品，那原先应该被判定为存在在布隆过滤器的商品，有可能因为这个删除动作被判定为不存在缓存当中，那因此这种一般的布隆过滤器是不允许删除操作的。

同学们，看咱这个布隆过滤器这里只有一层结构， 0101 的一层结构，那我们不妨把它做一个优化，再下面再添加一层，我们来看添加什么内容。135247，这是什么东西？这些数字代表着你当前位置总共被设置了多少次？比如说咱的第二个数字一，它代表着在当前位数有一件商品已经被加载了进来。那这第三个数字一，下面的这个 3 代表着当前位置有三个商品都被加载了进来。同理，后面的每一个数字代表着当前位数被加载过的次数。
