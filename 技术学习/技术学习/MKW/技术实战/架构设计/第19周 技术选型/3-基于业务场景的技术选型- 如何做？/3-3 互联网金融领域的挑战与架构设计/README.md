再来看一看政策上的挑战。

由于目前国家有关互联网金融政策也是石油调整的，所以也带来了互联网金融的不确定性。比方最近闹得沸沸扬扬的蚂蚁金服取消上市事件，对吧？第二类挑战，合规监管的挑战。互联网金融领域，它常常具备跨企业、跨部门业务加上性强等等特性。互联网金融企业的经营范围可能同时包含了银行、证券以及保险业务，也就是所谓的混业经营。而国内目前对金融业采取了分业监管的模式，也就是银行的业务由银行监管部门专门监管，而证券类业务由证券类监管部门去监管。以此类推。这样一来，难免存在九龙治水的现象，这也是互联网行业的一个巨大挑战。

二来，对于研发、测试以及运维人员来说，相比于传统金融的架构模式，采用互联网方式架构的系统，它的复杂度会成倍的增长。因为在传统金融领域，很多的业务流程是线下进行的，所以业务系统相对会简单一些，而互联网金融把绝大多数流程都线上化了，增加的复杂度就可想而知了。

所谓的境界就是把资料准备好之后，提交给贷款公司或者是银行的系统里面。


好，我们来过一下。首先是派单。派单是根据一定的规则分配客户经理去追踪贷款进件。常用的派单规则。比方根据客户所在城市的网点进行派单，可以随机派单顺序派单，或者根据客户经理的忙闲程度派单等等。公安系统核查。这步会调用公安系统的接口对用户信息进行核实检验。之后进入准入阶段。在这个阶段会对用户资质进行一个评分，确认用户是不是有资质进行贷款。评分的标准有很多，比方个人信用、征信、内部评分等等。这步会根据风险的策略，对每一个评分标准设置一个相应的权重，从而得到一个完整的评分模型。


最后再计算出一个评分结果，看用户是不是有资质进行贷款之后合而根据用户的个人信息，比方个人信誉、经济实力等等指标，计算出允许贷款的额度之后提交，把进件信息进行存储，通过接口或者是 MQ 触发后续的流程，比方审核进件等等。

简单过一下反欺诈功能，会从申请反欺诈行为、反欺诈设备反欺诈等等多种维度去制定反欺诈规则，确保能够及时侦测和处理可疑警告。并构建黑白名单。其中白名单可以建立数据模型，利用积极学习算法等等挖掘优质用户。而黑名单则往往针对第三方征信公司的接口获得。


用户是不是有逾期，是不是担任过破产公司的法人等等，把用户加入到黑名单里面。规则组可以用来对不同的客户制定不同的贷款门槛。比如根据消费的次数、注册年限、消费总额等等，设置一个准入门槛，以便于后期能够针对不同的人群设置不同的风控策略。对于支付宝，针对不同的人群，他的花金额就是不一样的，芝麻信用的分值不同，享受到的权益也不一样。


贷后检测：试着对信贷用户进行日常的贷后监测，从而实施发现风险信号。对于触发风险预警的客户，采取一定的措施，比方电话核实、提前收回贷款等等。其他的比较简单，大家可以去看一看。在构建风控系统的时候，可以从贷前、贷中以及贷后这三个方向去考虑，并且可以结合传统业务的风控模型以及互联网用户的行为数据，针对资金以及资产进行风险等级划分，这是风控。

典型的互联网消费金融，它的账务核算流程大概是这样的首先是一个还款计划的试算，计算不同还款方式的花费。例如采用等额本金每个月花费多少，总计花费多少，等额本息每个月花费多少，总计花费多少等等。等额本金每一期的本金相同，而等额本息是每一期还款额相同。


接着由用户选择一个还款方式之后，之后系统会根据用户选择的还款方式计算每一期的还款额，到期的时候会自动扣款并还款。如款到期没有办法归还，用户可以申请展期。所谓的展期，就是经过批准能够延长归还的时间的一个手续。在展期流程里面，会看用户是不是满足展期的要求，如果满足就可以展期，从而延后还款。之后计算减免，很好理解，根据当前的国家政策去减免利息，减免手续费等等。此外，如果有账户上的问题，或者是会计政策有变动，还就要做调账。所谓的调账，顾名思义就是调账目的意思。

简单过一下记账核算，可以承担这么几个职责。首先是记账，很好理解。第二是资金内转。所谓的资金内转，是指在营业部柜台系统这样的一个相对封闭的环境里面，进行客户资金的一个调拨，也就是将资金从营业部 a 资金账户里面直接划到营业部 b 资金账户里面去。还可以做计提，也就是计算和提取，把尚未发生但是将要发生的支出计算出来，列入到费用里面去。还可以做核销。



对账服务。对账是为了保证在每一个账期里面，互联网消费金融平台和银行以及第三方支付系统它的交易的一个资金证券型柜面服务，包含了和柜面相关的所有业务。比方柜面提前还款相关的业务，你可以做申请提前还款审批提前还款咨询提前还款等等，可以做撤销放款，撤销还款，还可以做贷款签收等等。总而言之，就是在柜台可以办的一些业务。


贷后变更可以做，比如贷款变更、期限变更、卡号变更、利息减免等等。凭证归集比方可以做，比如还款本期凭证、结清证明、还款清灯等等。凭证的归集以及各种凭证的打印。相信经过本节课的讲解，同学们已经能够了解互联网金融领域的各种挑战、核心流程以及大致的架构组成。
