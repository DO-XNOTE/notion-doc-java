在这个阶段该怎么样去做技术选型？大部分认为在这个阶段你做技术选型应该带着两个目的，一是要选择合适的技术实现项目的落地，这是最基本的对吧？第二要为项目未来的发展方向定好一个基调，也很好理解。这个阶段做出的技术选型会影响到后面的决策。建议大家在做技术选型的时候，


要考虑清楚几个问题。

首先是运行平台的一个选择。你的系统平台是 windows 还是 Linux 或者是 Unix 语言，选择 Java 还是 PHP 要考虑清楚。


第二，考虑清楚是选择商业解决方案还是开源解决方案，是买一套系统还是自己开发。如果买，是买一整套系统，在购买的系统之上二次开发，还是只买商用的技术方案或者是某一个组件，比方只买 Oracle 数据库，其他的还是自己开发，都得考虑清楚。当然了，选择商业解决方案或者开源解决方案这件事情其实是贯穿项目的整个生命周期的。只是弹幕觉得在这个阶段，这件事情特别的重要，毕竟这是一个起步阶段，所以特别列出来强调一下。第三，要确定好项目的研发模式，比方要不要前后端分离，又比如你的开发以及运维是怎么样协作的。第四，才能确定具体要使用的基础。比方项目所使用的具体框架，MVC 用什么框架， 持久层用什么框架，用什么数据库等等。


这个时候大家可以按照前面选用基础时需要考虑的因素，并去结合团队以及项目的特点去做选择。

这个时候就可以采用商用解决方案，比如直接从外面购买整套系统，再去做二次开发，这样可以节省非常多的工作量，同时能够更快的上线。再比如，如果团队技术实力比较好，工期不是很紧张，这个阶段就可以做一些技术上的投资，提升技术的天花板，选用一些比较新颖的技术去解决问题。


再比如，团队精富力非常充足，但是在特定领域的沉淀还比较的少，可以在局部领域采用商业解决方案。比如团队的数据库，技术沉淀比较的少，但是可以预见的是，未来的数据体量是非常大的，而且也需要非常好的稳定性。这个时候就可以买一套Oracle，这样 Oracle 相关的问题都会有商业支持，出了问题都可以找官方寻求帮助。


大家不要认为购买商业系统是一个很 low 的事情，可以告诉大家的是，阿里内部有非常浓厚的购买文化，比方我们的门禁系统、视联网系统、停车系统等等，最初都是买的，经过快速验证，认为模式可靠之后，才逐步用自己研发的版本替换掉了购买的版本。甚至有同学可能知道，淘宝的第一个版本也是买回来的，稍后我们会探讨到。


总之，在这个阶段做技术选型的时候，定好基调以及几个原则就 OK 了，一般不会翻大车。
