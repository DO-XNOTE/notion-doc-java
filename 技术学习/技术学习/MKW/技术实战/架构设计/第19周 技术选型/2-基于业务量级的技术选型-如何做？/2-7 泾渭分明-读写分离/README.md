这个时候我们很容易想到实现数据库的一个读写分离，对吧？读写分离一般会用数据库的主节点承载写入，而从节点承载读取，从节点会同步主节点的数据。通过这种方式可以提升数据库读取的能力，并且能够实现数据的容灾。


有一天主节点挂了，会通过选举等等方式把从节点晋升成为主节点，对吧？事实上，在数据库层面有很多的玩法，读写分离只是其中的一种。比方 MySQL 要想扩展数据库读的能力，可以用pxc、mmm、mgr 等等多种方案，这其实是一个非常大的话题。


在技术选题阶段，同学们只要知道有这么一回事就 OK 了。具体的内容大部将会在数据库架构这一阶段为大家详细展开，敬请期待。此外，不仅仅数据库能够实现读写分离，一些缓存系统、启蒙搜索引擎等等也都是支持读写分离的。思想也是类似的，通过部署多个重节点，从而扩展读的能力。同学们要能够触类旁通。好，这节课就到这里，谢谢大家。
