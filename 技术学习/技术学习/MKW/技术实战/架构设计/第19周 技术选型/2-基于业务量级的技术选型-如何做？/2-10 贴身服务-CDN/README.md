CDN 它是一种组合技术。最简单的 CDN 网络由一个 DNS 服务器和几台缓存服务器组成。这里我们缓存服务器只画了一台。工作的过程大概是这样的用户点击比方某一个网站上面的URL，这里的客户端我们假设它是一个浏览器，用户点击某一个网页上面的URL。

DNS 会根据用户请求的来源，比方客户端所处的城市，或者是客户端所处的线路，是联通还是电信等等，请求转到离客户端最近或者是离它最快的缓存服务器。比方把南京电信的用户请求转向到南京电信机房里面的缓存服务器。缓存服务器用来为用户提供站点的资源。缓存服务器可以由一个或者多个服务器组成。如果缓存服务器里面已经缓存了，想要访问的资源就直接返回了。如果没有缓存的话，那么这一套缓存服务器会向其他临近的缓存服务器或者是直接向源站获取内容并返回。所以我们只要把原站内容给它分发到最靠近用户的缓存服务器，用户就可以就近取得所需要的内容。这样就可以提升访问的速度以及访问的成功率。

DNS 的解析过程大概是这样的客户端，比方浏览器，它要访问某个域名，比方这里的 www.zdns.cn整个请求的过程大概是这样的首先，客户单会发起一个 DNS 解析请求，如果递归域名服务器本地缓存里面找不到，就向根权威服务器发起查询。根权威服务器记录的是各个顶级域名所在的服务器的地址。比方向根服务器请求  www.zdns.cn、  的时候，就会返回点 cn 传位服务器的地址信息。递归服务器拿到点 cn 全服务器地址信息之后，会向 cn 权威服务器请求 www.zdns.cn 的地址， CDN 全屋服务器会返回z、d、n、 CDN 的全屋服务器地址。接着递归服务器，再向 zdns.cn 权威服务器去查询 www.zdns.cn 的地址。终于拿到了这个域名对应的服务器 ip ,当然了，DNS也不是每一次解析都要做这么一个递归的，一旦递归域名服务器拿到了解析记录之后，就在本地缓存下一次客户端请求域名的时候，直接从缓存里面获取并返回。从这个图我们可以知道， DNS 服务器可以分成两种递归域名服务器和权威服务器。这个递归域名服务器也被称作是 local DNS 本地DNS。OK，这是 DNS 解析的过程，

在引入 CDN 之后，请求的理论过程大概是这样的首先，客户端通过 DNS 找到最近的 CDN 节点， CDN 节点返回数据给客户端。但问题这是理论上的一个过程，实际情况不是这么简单的。由于国内多数城市的 SP 业务基本上被电信、联通以及联动给垄断了，而各个运营商之间又会存在网间费用结算，所以运营商他会想办法把用户的请求再自己的网内解决掉。这其实是各大运营商图省钱的一种方式，但是这样带来了很多的问题。

举个例子，我们现在广州联通的宽带用户发起了一次请求，尽管广通电信它的 CDN 已经有用户访问的资源了，而且访问速度也是非常快的。但是联通为了避免要跨网访问，于是联通的 DNS 它返回了北京联通 CDN 的一个IP，于是客户端就得去访问北京的一个 CDN 节点。而从广州到北京，网络延迟已经比较大了，访问速度可能并不理想。

另外有一些 sp 它更加的过分。它为了节省网间流量，还会针对一些CDN 文件额外的做一层 CDN 缓存。这一艘缓存大概是通过 DNS 劫持的一个技术。所谓的 DNS 劫持就是修改 local DNS 的记录，把它指向到一个非法的网站，比方指向到了一个他自己网内的非法 CDN 缓存服务器，而不是我们这里的缓存服务器。这样做相当于它帮助我们也搞了一个缓存服务器。

但问题是，运营商的缓存服务器很多情况下并不能和我们的 CDN 节点同步更新，于是就会导致使用 SP 的客户端会出现各种问题，比方本应该能够访问的资源返回了一个404，或者是某一个已经删掉的资源还能够访问。考虑到我没有办法用图表示，大家可以领会一下我话里的意思。

第二种 DNS 优化机制是 302 跳转。简单来说， CDN 厂商，它会维护一个 CDN 域名 ip 地址库。 DNS 解析之后得到的其实是 ip 地址库服务器的地址。之后再由 CDN ip 地址库根据用户的请求 IP 以及 CDN 节点的状态等等各种信息去选择一个合适的 CDN 节点，同时发送一个 302 状态码，并把用户的请求跳转到 CDN 节点。目前来说，市面上的商用 CDN 都是支持 302 跳转的。这是 DNS 层面的优化技术。下面来看看其他的加速优化技术。


HTTP 二点零加速 http2相当于现在最流行的 http 一点。一来说做了非常多的优化，比方头部压缩、二进制传输、支持服务单推送等等。目前国内还没有普遍支持 HTTP 2.0，但是 HTTP 2. 0 作为下一代 HTTP 协议是板上钉钉的事情，并且现在 http 2. 0 正在被越来越多的企业采用，所以在挑选 CDN 的时候，还是可以为将来打算优先考虑支持 http 2. 0 的CDN。


第三，压缩。很好理解，当浏览器访问静态资源的时候，由 CDN 压缩之后再发送给浏览器，最后由浏览器解压，从而减少文件传输的流量和时间。目前来说，各种 CDN 普遍支持 GZIP 的压缩，有些 CDN 还可以支持更加新一些的 Brotli 算法，甚至还有一些 CDN 能够支持智能压缩，自动的选择压缩算法。
