**假设你现在的机器服务器集群爆出了一个线上内存泄漏的异常，或者说是一个线上线程数过高的一些异常，那如果你拿到这个 case 之后，你如何来下手？你怎么来去排查这个问题？它可能发生在哪些点上？这个就是实打实的考察同学们在线上问题排查当中的实践经验的。**


这个问题可以一分为二的来看。第一也就是问题的定位，这里可以去参考前面的微夫的一个章节，当中的链路追踪，可以利用线上异常得到它的 global trace ID，然后找出上下游的调用栈，在这个站上面梳理问题可能是出现在哪一环，但是这种情况也不能保证是一定能找到问题，比如说你的内存泄漏有可能爆出问题的这个服务，它并不是罪魁祸首，你的大部分内存可能是被其他服务给吃掉了。那这种情况下我们就要上什么 JVM 的 dump file 去来做 JVM dumpfile 分析，以及 g seed 快照分析等等手段，然后从无效引用最多的那个类去做反向溯源，最终找到它的源代码，然后再做相应的分析。
