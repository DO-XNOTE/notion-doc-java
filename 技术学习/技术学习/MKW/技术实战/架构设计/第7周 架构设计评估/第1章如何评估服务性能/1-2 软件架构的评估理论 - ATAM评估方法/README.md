在大规模团队当中部署一个战略任务，沟通架构、方案与需求是一个怎样的流程。好表述部分通常有三个角色将要出马，第一个角色，中单节奏的带领者，我们叫他PMO， PMO 的角色更像是一个组织者，他负责把控你的整体的项目进度，那么这里由他作为你当前会议agenda，也就是你会议议题的起草者。那么他在这里的角色首先跟大家沟通到你召集会议的目的，以及会议将要讨论的内容，并且他负责把控整场的节奏，控场，不至于让这个会议拖沓，或者被某些钻牛角尖儿的技术人员把这个话题带跑偏。OK，那么第二个角色，产品，那么产品在这里出场主要是沟通两个方面，第一个，我当前要做的项目是什么？项目目标以及当前项目业务上下文和相关的重要的功能点的表述，那么这里尽量不要牵扯过多的讨论，是有产品陈述为主。这样的一个会议如果在一开始陷入讨论，那将是一个很麻烦的境地，必定会拖堂并且变得无效率好。


那么产品沟通完了之后，接下来第三个角色，架构师，这里将对你产品的相关功能点做一个简单的架构层的表述，仅靠业务来表述我们的架构方案以及各个组件流程图之间的梳理，包括上下游兄弟团队，你想从他们团队当中获得什么样的支持，需要他们提供什么样的业务能力。
OK，那从这三点同学们可以看出，它不是在会议的过程当中讨论出来的结果。表述部分开局，在我们跟大规模团队沟通项目之前，以上的这三个人，他们内部会做一个预先的预热，把当前要做的项目，从项目的整体目标再到项目的产品、业务目标以及技术目标，这三个角色之间要做一个对齐，把目标对齐，OK，那开局开好了之后，我们接下来就进入到了兑现期。
