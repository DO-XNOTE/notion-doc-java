那跟同学们一起来看一下推到重来的新零售商品中心，那这个商品中心为什么会推倒重来？那重建之后和重建之前他们有哪里发生了变化？如果我们在设计之初多考虑了哪些点，那就有可能去避免这种推倒重来的窘境。OK，好，那这个小故事之后，我们跟同学们讲一个大故事，这个大故事时代背景非常的宏大，那就是淘系技术演进三部曲，上中下三部曲。那在我们的上篇，我们要跟同学们一起来了解淘系应用。淘宝网在创立之初，它是如何由一个从美的买办的一个成套的软件，逐步走向了自主研发的这个道路，那就像从包办婚姻变成了自由恋爱，那这里面发生了哪些有趣的故事？我们在这个上篇当中跟同学们来唠叨一下。那中篇进入了这个风起云涌的章节，随着淘系应用用户量、交易量还有商品数量的增加，那逐渐的对系统的各个环节都产生了非常大的冲击，那淘系的技术团队如何顶住这个冲击？在这个风起云涌的时代背景之下，他们都做了哪些的改变？OK，那么下篇就是近些年来发生了一个非常具有代表性的一个趋势，那就是大中台的诞生，那阿里系作为大中台的一个很好的演艺者和这种价值观的推动者，我们来看一下大中台这种概念是在什么样的背景下面产生的，它产生了哪些显而易见的效益，是不是所有的应用都适合中台？我们就在这个章节当中跟同学们来探讨。那希望大家通过这样的一个长篇小说的上中下篇，可以从淘系技术演进的过程当中悟出一些门道。


什么是面向未来的架构设计，又如何来做一个具有前瞻性的架构？从这个故事当中我们也可以看出来，如今被捧上神坛的可以扛住双 11 洪峰冲击的淘系统，它是如何从一开始的一穷二白逐渐走到现在这样的傲视群雄的地步的？那所有的应用都是这样一个从 0 到有，从有到无所不有的这样一个过程。OK，那接下来希望同学们都可以在一个比较轻松愉悦的环境下面去聆听这一章节的故事会。好，同学们，我们正课环节再见。
