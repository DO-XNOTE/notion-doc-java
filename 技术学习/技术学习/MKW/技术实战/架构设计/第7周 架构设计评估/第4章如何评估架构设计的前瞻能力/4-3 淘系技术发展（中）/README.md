那么当年 sea shop 并没有走这种开源的道路，所以尽管最后 sea shop 到 night 也无奈选择了开源，但是什么为时已晚，已经在和 Java 的这个争夺战当中彻底的败下阵了。那所以从这个故事当中我们就可以看到民主开放，拥抱开源和闭关锁国的闭源这两个不同的方向，它所带来的殊途同归的最终结果。那并不是说 c sharp 的技术不如Java，或者 Java 技术比 c sharp 好，有时候真的无关技术好坏，而是选择，所以说选择大于努力。
