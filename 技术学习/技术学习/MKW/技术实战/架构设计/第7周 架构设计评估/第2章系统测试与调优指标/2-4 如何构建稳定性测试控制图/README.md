第一个条件，稳定性测试这里有几个要求，第一个要求是针对它的负载能力，那稳定性测试的测试条件需要在一定的负载压力下面进行，并且还要执行一个较长的时间，它不像压力测试或者是性能测试点到即止，它这里是要执行相当长的一段时间来确保系统可以稳定的运行。


那么因此这里就需要去我们人为的定义一些稳定性的指标，那什么样的指标叫正常，什么样的指标需要我们花精力分析来排查问题的？OK，那这是它的三个条件，那么稳定性测试的目的是什么呢？其实简单来说就两点，找出不稳定的因素，并且分析它的性能瓶颈。那因为稳定性这里咱前面提到的这两个条件，在一定负载下执行较长时间，因此非常容易发现某些资源的瓶颈，比如说你的资源没有得到释放，经过一段时间的运行，把 GVM 吃光了，或者缓存的滥用等等各种因素，它只能在一些长时间的执行之后才能暴露出来的一些问题。


OK，那看完了条件和目的，这里我们跟同学们稍微过一下稳定性测试的几个准备的步骤。第一个它的测试范围，这里我们要给定的，那测试范围里面通常你要测试哪些的链路，哪些主链路上的关键任务，或者是一些比较长时间执行的任务，这都是稳定性测试的常客。OK，那第二点我们要定义你的测试场景，你的负载条件，这里要通过哪些测试场景把这个负载给打出来，那约定好了范围之后，我们就可以着手去构建你的稳定性的模型，那么这里有一个基线和控制线这两点就是待一会儿我们要上的小学五年级、六年级的代数课程当中的内容了，牵扯到方差、标准差等等这些知识。


OK，那么构建这些基线目的是什么呢？其实就是衡量你的系统的一个KPI，看它在长时间运行过程当中，你的系统的响应时间，或者是系统资源，比方说是内存、 CPU 等等，是否是在一个正常的范围之内，这就是给他打 361 KPI 审核的。OK，那么最后一点，我们通过执行稳定性测试，然后把所有的测试结果汇总分析，找出你在测试过程当中的那些异常的点，超越了控制线的点。OK，那控制线这里同学们可以理解了，就是红线，你的测试结果你的 KPI 不能超过你的控制线这条红线。好嘞，那我们这堂语文课上完了，哎，接下来切换到数学老师状态，

OK，那我们这里画出一个二维的坐标系，它的横轴是你的系统在稳定性测试过程当中，它的执行的实践。那么纵轴是一些性能的指标，比如说我们这里以 r t response time 作为你的纵种。好，那接下来我这里划定一个中心线。OK，那么这个中心线其实是一个均线，那是经过你的稳定性测试之后所有的测量点它的平均值，那体现在这幅图里，就是你接口在稳定性测试过程当中平均返回时间。OK，我们叫它是数学期望值。那除了中心线以外，我们还有两条控制线。控制线就是前面咱说到的红线，你不能越界。那这几条线可不是我们拍脑袋想出来的，它是经过测试的结果分析统计之后才得出来的。那因此我们要先执行测试，拿到你的统计结果。那比如说我们这里随着时间推移，去定点地采集了每一个时间段你的系统的响应时间，那这每一个点其实都是一个输入值，那通过计算这些点的平均数，我们可以拿到谁均线，对吧？均线就算出来了，那么我们以这条线平均响应时间为基线，如何来计算这上下两条控制线？这里就要记出方差、标准差计算公式了。


好，那首先我们基于这个中心线计算一个总体的方差，OK，那方差公式大家应该小学的时候都比较熟悉，这是小学五六年级的数学课的内容。那怎么来计算呢？哎，我们这里看到这个 XX 是什么呢？ x 就是你每一个点它的对应的数值，那么它减去的这个数是谁？这个数就是咱的中心线平均线的均值。那我们用你每一个点的数值减去平均值，然后得出来的这个结果，把它做一个平方。那前面这个符号是什么呀？前面这个符号的意思就是一个累加，那我们这里要把每一个点获取到的数值都用这个公式把它计算出来这个平方数，然后再把这些数全部加起来之后除以一个n。OK，那这就获得了一个总体的方差，那方差是概率论统计的一个范畴，那它用来分析我们的一组数据，它的离散程度来衡量。离散程度怎么样来衡量，那它是把每一个随机变量，这每一个点上的值和它的数学期望值，就是咱这幅图当中的这个均线结合起来分析这两者之间的偏离程度，那计算出了总体方差之后，我们这里接下来要获取它的标准差。


同学们还记得标准差怎么计算吗？它是基于方差结果，然后给它开了一个根号，非常的简单，把前面这个数据开根号就可以计算出标准差，那它的定义就是方差的算术平方根，那在概率统计当中，我们经常使用它来作为统计分布程度上的测量的依据。OK，那么得到了标准差之后，我们就可以去计算控制线了。怎么来计算？在实践当中，我们一般来说是使用这个公式来计算标准差的数学期望，也就是我的中心线平均线正负标准差乘以3，那么上面的这条控制线是数学期望加上 3 倍的标准差，下面的这个控制线数据期望减去 3 倍的标准差，OK，那么对于超过了这些红线上的点，那我们就要去把它给揪出来。然后分析系统当前在这个点上的一些运行状态，找到有哪些资源瓶颈，或者是一些系统层面的问题，导致了咱的这个测量点偏离中心线太远太远。
