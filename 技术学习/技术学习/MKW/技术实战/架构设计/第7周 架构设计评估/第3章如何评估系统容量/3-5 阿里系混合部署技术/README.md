就是老师前面说的像搜索、下单这些需要立即返回的时效、延迟性、容忍度比较低的业务，我们也把它叫做在线业务，那在集团的应用当中是使用 Sigma 做调度的，那么除了在线任务以外，咱还有一些离线的后台业务，比如说我的统计分析、报表、交易对账等等，这都是后台的批处理。


那么是通过伏羲来做调度的，那我们将不同种类的业务混合部署到同一份的物理资源上面，那这样的话我的一份物理资源可以同时供给在线任务和离线任务，谁需要资源我供给谁，谁不需要，我就把这份资源让渡给真正需要的业务方，用这种方式提高你的资源利用率。okay，那从这个角度上来说，其实简单的一句话，混合部署它就是一个资源整合的一个过程，那把原本分别部署在不同服务器上的资源，我把它组合起来混合部署在同一个物理资源上面，那么这里必然牵扯到几个必须要解决的问题。比如说我们这里既然部署到同一个资源上，那一定牵扯到资源共享这个话题，那一个硬件资源，哎，有哪些好跟你共享的嘞？那你既然要脚踏两只船，同时支持在线离线，那我的硬件资源这里我们想一下有哪些需要共享出来？比如说 CPU OK，那么 CPU 资源其实它在内核的这个机制里面，我们以时间片轮询的方式分配给不同的进程来进行使用，那这是进程间的一个调度。


但是我们 CPO 的分配资源的力度其实是在什么呀？在和上 CPO 的核心数，那我这里把 CPO 的一个和我如果同时分配给多个任务，肯定需要确保这之间的优先级，对吧？共享不是同时使用，而是说这里面有个先后的顺序。那我们的总体方针是说在线的实施业务享有相对较高的优先级，因为他们对食盐的容忍性比较低，那么当在线任务比较闲的时候，我们可以把这部分 CPU 资源让渡给后台的离线任务，那同时如果在线任务忙了起来，那么你离线的任务就要把这个 CPU 资源给它交出去，那这是一点我们想还有什么硬件资源需要做共享的吗？还有一个非常重要的资源内存，那么内存这里其实它的让度和我们的 CPO 比较相似，也是实时业务优先，但是在内存分配这里可能有一个很多人，包括在阿里工作的同学可能都不知道的这样的一个小机关。


那我们内存假设有一个 g 的内存，那么我们分配给 Sigma 和伏羲之间，这两者可能分配到的内存会大于一个g。这就很像那个超卖的现象，包括像阿里云这些云服务商，他们在卖计算资源的时候，多多少少都会存在一些超卖现象。那我们在资源共享这里为什么内存会超卖？一个很简单的原因，我们后台服务整个阿里系集群大部分都是基于 Java 语言来写的，那么 Java 语言它的内存分配这里，那么具体到我们的容器级别，这里一部分内存自然是用给 GVM 那堆内存的一个开销，那么还有一部分其实是作为一个缓存，一个 catch 来使用的。


因此在 GVM 之外，咱在这个容器里，容器的内存往往会存在那么一定量的空闲内存，所以我们可以做超卖，超卖的是哪一部分？不是说克扣你 GVM 的内存，是把你容器之内的这部分空闲内存给它分配给在线离线业务来使用，那么除了这两个之外，还有一个 DISK 存储空间硬盘，那么这部分资源相对来说并不是紧俏，资源硬盘很便宜的，随便用。


那所以我们的资源共享基本上是属于 CPU 和内存级别，这里要做好控制，那既然咱共享这里也说到了有一些优先级，那么这就不得不牵扯到一个资源竞争，那么资源竞争的过程，哎，是先来后到妈，同学们想一想，是的吗？肯定不是。那么这里叫来得早不如来得巧，我们这里是有一个优先级的这样一个概念，那么一般来说，我们在这个混合部署的过程当中，会把你的资源分为上中下三个优先级，那优先级依次从高到低，最高的一层优先级是 CPU 的核心独占，那我直接把你的 CPU 的这个核给它绑定起来，那么这是最高优先级调度。


那么在这之下，我们还有一个优先级叫高优先级的业务，那高优先级业务一般来说都是集中在实事业务上，但是不可否认后台业务也有一些高优先级业务，那我们高优先级的业务可以驱赶低优先级的业务去抢占 CPU 资源，那么我们假设当前的实时业务和离线业务同时都有一个高优先级的业务，想去抢占 CPU 资源，那么这时候怎么说？这个时候其实就是一个先来后到了，那如果 CPU 已经被离线的高优先级业务抢占了，那这时候在线业务过来，他不会把我的离线业务踢下线，那么这是第二个优先级。


第三个优先级呢？我把它叫做绿皮车，很多 90 后小朋友都没坐过绿皮车的，那像老师上大学读研的时候，我印象中还没有高铁，那时候都是坐的绿皮车。绿皮车不光慢，它还有一个非常重要的特点，跟我们这个优先级非常匹配，叫见谁让谁，只要是动车、高铁任何车过来了，绿皮车都要停下来，把这个铁道给它出让出去。那所以在这个优先级的业务大部分是谁是离线业务，一旦有高优先级的业务来抢占我们这个坐着绿皮车的这些离线业务，就要把这个座位让出去。


那咱这里了解了混合部署的资源共享竞争优先级的内容之后，我们再来了解一下如何来划分你的实时任务，还有后台任务。

除此以外还有大部分的前台业务，我们也叫做 customer facing，就是直面客户的这部分业务。还有一个很重要的一点，中间件，那中间件可是非常重要的一环，尽管它不是直面客户的，但是保障你整个集群稳定性的一个非常重要的组件。那么我们把它划在了实施业务，使它拥有更高的优先级。

除了这两点以外，那还有一个很重要的点，尤其体现在双 11 大促这样的环节，那就是它的脉冲峰值非常高，可以在瞬间打出非常高的伤害，但是它续航可能并不厉害，它打出一波输出之后，慢慢这个流量慢慢就下来了，这是实时业务的一个特点。那么后台业务我们怎么来划分呢？比如说我们这里有很多的离线计算类的业务，那打个比方，那这里的报表业务统计， GMV PV 运营数据的统计，还有后台的交易对账，那么这类业务一般什么时候执行？在当天三更半夜零点之后，夜深人静的时候，后台的这些批处理，这些自动的 job 开始去跑起来。
