一种方法就是模块儿去做自我保护，这种的话就特别适合于内部的模块儿，比如说像刚才的服务。c，说我不希望参加压测，但是你又不愿意说，因为这个点来改动你的整个的一个链路调用这里我可以做好我的自我保护，我就可以在这个流量入口中去判断请求，以及你通过发过来的消息，它是否还有这个压测标记。如果有我就已经抛弃掉了，或者返回一个固定值，让你们的业务能往下走，但是我本身来说不响应你的压测流量，或者说是我本身的业务不受你的压测流量所影响。
