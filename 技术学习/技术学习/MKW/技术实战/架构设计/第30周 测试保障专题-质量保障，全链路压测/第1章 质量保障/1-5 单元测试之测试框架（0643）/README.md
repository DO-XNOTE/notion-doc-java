那 spring j unit 就是 j unit，加上 spring test 就等于这个 j spring j unit，它是在绝对的之上将它与 spring 做深度结合，从而能够使用 spring 的配置对整体进行管理。因为 spring 的配置大家知道很强大吗？那它也跟很多，比如像数据库，像一些RPC，像一些中间件都可以非常好地结合起来，那它也可以赋能给 g unit 去完成相关的一些事情。

那它有什么样的好处？第一个它的参数却可以是配置化的，那很多时候这个参数我们就可以把它单独拿出来了，通过一个外部的一个文件，以 JSON 的方式也好，以岔面的方式也好，通过外部的方式做配置，这样的话参数可以单独管理，甚至可以复用。然后在执行的过程中，我们也可以更多地去创造更多的丰富的这样的一个参数 case 来去做好相关的覆盖。


其次，它能够支持依赖性的case。虽然说单测尽量建议我们不要产生依赖，但是有的场景里你必须要产生依赖，那在这种情况下， test NG 可以帮助我们去管理这类case，这样子的话，我们就可以在面对这种复杂的节奏测试过程中去更好的完成相关的 case 的书写。


最后，它支持一些异常测试和超时测试，因为传统的 unit 它并不支持这一类。那通过猜测NG，它可以针对其中的某些异常进行触发，然，然后去验证其异常是否是我们期望的异常，然后包括一些超时情况下来完成相关的一些场景的一些验证，这样子的话它可以帮助我们更好的来完成相关的过程中，整体上一些复杂的场景，下一些复杂的需要去跟其他方需要完成更复杂的集成测试的。
